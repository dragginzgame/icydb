- `docs/audits/reports/2026-03-02/0.40-planner-executor-semantic-ownership.md`

consolidation, and structural guard coverage before larger model changes.

## In Scope (`0.40.x`)

- planner vs executor semantic ownership hardening
- load orchestration multiplicity reduction behind one staged execution spine
- continuation boundary centralization and cursor-policy ownership tightening
- architectural guard-test migration to structural/signature/type contracts
- release and tracking artifacts for audit slices

## Out of Scope (`0.40.x`)

- query-surface expansion
- compatibility shim work
- speculative refactors without an audit finding
- cost-based planner or new execution strategies

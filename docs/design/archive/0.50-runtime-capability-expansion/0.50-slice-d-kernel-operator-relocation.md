# 0.50 Slice D: Kernel/Operator Relocation

Status: Completed  
Owner area: `kernel` and `pipeline/operators`

## Intent

Slice D reduces kernel/operator overlap by tightening `ExecutionKernel` to an
orchestration boundary and pushing reusable operator mechanics into
pipeline-owned operator modules.

## Invariants (must remain unchanged)

- plan semantics: identical plan -> identical execution behavior
- continuation semantics: resume tokens remain stable and monotonic
- explain semantics: explain node correlation remains deterministic

## Work plan

### D1. Kernel/operator responsibility audit (completed)

- map `ExecutionKernel` functions that are pure operator mechanics
- map `pipeline/operators` functions that still depend on kernel-only wiring

Result:

- kernel orchestration-owned:
  - `materialize_with_optional_residual_retry(...)`
  - `resolve_execution_key_stream(...)`
- operator-owned mechanics:
  - key-stream DISTINCT decoration in `pipeline/operators/distinct.rs`
  - post-access operator semantics in `pipeline/operators/post_access/*`
- overlap identified:
  - kernel had a pass-through wrapper for operator-owned key-stream decoration
    (`decorate_key_stream_for_plan`) with no kernel-specific logic

### D2. Relocate operator mechanics (completed)

- move purely operator-level helpers out of kernel boundary where possible
- keep kernel focused on orchestration sequencing and fallback coordination

Progress:

- completed first cut:
  - removed kernel pass-through wrapper `ExecutionKernel::decorate_key_stream_for_plan`
  - rewired aggregate fast-path decoration to call pipeline operator-owned
    `decorate_key_stream_for_plan(...)` directly
- shared-kernel boundary note:
  - `resolve_execution_key_stream(...)` remains kernel-visible within
    `crate::db::executor` because aggregate execution modules still consume it;
    this is a valid shared orchestration boundary and not operator duplication

Files:

- `crates/icydb-core/src/db/executor/kernel/mod.rs`
- `crates/icydb-core/src/db/executor/aggregate/fast_path.rs`

### D3. Contract boundary lock (completed)

- ensure callers consume operator helpers through pipeline operator exports
- preserve existing invariant and route/load matrix test behavior

Result:

- no remaining kernel pass-through wrapper for operator-owned stream decoration
- aggregate fast-path now consumes operator decoration directly via
  `pipeline::operators` export surface
- kernel surface remains orchestration-focused (`resolve_execution_key_stream`,
  `materialize_with_optional_residual_retry`)

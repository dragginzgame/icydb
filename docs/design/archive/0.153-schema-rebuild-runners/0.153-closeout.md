# 0.153 Schema Rebuild Runners Closeout

## Status

0.153 closes as the field-path rebuild runner foundation for accepted schema
mutation work. It does not make SQL DDL live and does not wire production
reconciliation into real field-path rebuild execution; that is the first 0.154
developer-stability target.

The completed 0.153 surface is:

- schema-owned mutation execution plans and runner capability preflight
- checked `SchemaMutationRunnerInput` from accepted before/after snapshots
- runtime schema epoch and publication identity contracts
- non-unique field-path rebuild targets derived from accepted index snapshots
- deterministic staged field-path row/key materialization
- staged store buffers and write batches with rollback snapshots
- isolated `IndexStore` writer and validation contracts
- rollback reports for staged physical validation failures
- runtime invalidation and accepted snapshot publication sink boundaries
- final physical store publication phase for validated field-path stores
- publication readiness that stays false until the physical store is ready
- source-boundary guards for runner preflight, physical-store publication, and
  field-path runner accepted-schema authority
- 0.154 developer-stability design

## Deferred

The following remain deliberately outside 0.153:

- SQL DDL syntax
- production reconciliation invocation of the field-path runner
- production row-scan adapter from live stored rows into rebuild rows
- recovery handling for staged/building runner stores after restart
- deterministic expression-index rebuild execution
- secondary-index drop cleanup execution
- unique-index rebuilds and uniqueness conflict reporting
- online or concurrent index rebuild

## Closeout Invariants

- Accepted persisted schema snapshots remain the runtime/catalog authority.
- Generated model/index metadata remains limited to proposal, reconciliation,
  model-only, and test lanes.
- Field-path runner modules consume schema-owned runner input, execution plans,
  and accepted rebuild targets.
- Snapshot publication handoff is not enough to make generic runner diagnostics
  publishable; the physical store publish phase is required.
- Unsupported or incomplete mutation work still fails closed before publication.

0.154 should start from these contracts and wire only the supported non-unique
field-path path into developer-testable reconciliation. Expression rebuild and
secondary-index cleanup should stay blocked until their own validation,
rollback, publication, and recovery contracts exist.

# IcyDB 0.151 - Accepted Expression Index Contracts Status

## Current State

0.150 closed accepted field-path index authority and narrowed expression indexes to
named generated compatibility wrappers. 0.151 starts from that point and removes the
remaining generated-runtime expression-index authority.

## Target

Runtime expression index planning, explain, fingerprints, writes, uniqueness
validation, and recovery are backed only by accepted expression index contracts after
schema acceptance.

## Work Queue

- Define accepted expression index contract types. Done in `0.151.1`.
- Derive expression contracts during schema proposal and acceptance. Started in
  `0.151.1` for generated proposal projection and initial accepted snapshots.
- Persist expression contracts in accepted schema snapshots. Done in `0.151.1`.
- Expose expression contracts through `SchemaInfo` and `VisibleIndexes`.
  Opt-in `SchemaInfo` projection landed in `0.151.1`; accepted runtime
  `VisibleIndexes` routing started in `0.151.2`; the generated expression
  candidate lane was removed from accepted runtime visible indexes.
- Move planner, explain, cache identity, and reranking to accepted expression
  contracts. Reduced semantic planner/access-choice contracts now consume
  accepted field-path and accepted expression planner indexes on the accepted
  runtime lane. Accepted commit/cache schema fingerprints now hash the accepted
  persisted schema snapshot directly instead of appending generated index
  contracts.
- Move expression write keys, filtered membership, uniqueness validation, and recovery
  to accepted expression contracts. Commit preflight key construction,
  membership predicates, and unique validation have moved to accepted contracts;
  the generated expression write fallback has been removed. Startup index rebuild
  now has a guard proving it reaches accepted commit preflight instead of
  rebuilding generated expression keys directly.
- Delete production generated expression candidate wrappers. Done.
- Add guards for generated expression index runtime authority. Done for accepted
  runtime planning, writes, uniqueness validation, startup rebuild, and accepted
  commit/cache fingerprint identity.
- Rename the remaining standalone generated visible-index constructors as
  model-only surfaces. Done in `0.151.6`.
- Move the generated index access-contract adapter into `access::model_only`,
  rename it to `SemanticIndexAccessContract::model_only_from_generated_index`,
  and guard it so it remains on generated/model-only planner projection and
  tests, not accepted session/write/recovery runtime. Done in `0.151.6`.
- Move primary-key compare and primary-key range planner recognition to
  `SchemaInfo::primary_key_name` instead of direct generated `EntityModel`
  primary-key field identity. Done in `0.151.6`.

## Deferred

- New expression operations.
- SQL DDL for expression indexes.
- Arbitrary scalar-expression index keys.
- Backward compatibility fallbacks for older internal accepted-schema snapshots.

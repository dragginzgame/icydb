- `AUDIT-FOR-TOMORROW.md`

runtime/planner features before `EXPLAIN` is finalized in `0.42`.

- execution-shape changes that would otherwise churn `EXPLAIN` node output

- planner/access shape additions that introduce new stable execution nodes

- [x] allow index-backed COUNT fast path when strict index predicate compile succeeds
- [x] feed strict `IndexPredicateExecution` into existing-row COUNT stream descriptor
- [x] lock parity and scan-reduction behavior with strict-vs-uncertain matrix tests

### B) Field-target MAX bounded probe parity

- [x] keep non-tie-free MAX(field) shapes conservative (no bounded single-step probe)
- [x] enable bounded single-step probe for tie-free MAX(field) shapes:
      PK target or unique single-field leading index target
- [x] add route-level max-by(pk) hint coverage
- [x] add aggregate execution matrix proving one-key MAX(field) scan on unique index

### C) Index-range ORDER satisfaction extension

- [x] extend route/access-order contract for eligible index-range ORDER satisfaction
- [x] remove avoidable materialized sort in eligible index-range ordered windows
- [x] add route + parity tests for ORDER-satisfied index-range shapes

### D) Secondary-index IN multi-key pushdown

- [x] introduce dedicated multi-key secondary access node (replace union-expanded shape)
- [x] wire planner lowering to new access node
- [x] add executor traversal support and parity coverage

- [x] lock route + pagination behavior with focused tests

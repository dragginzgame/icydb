# 0.40 — Full-System Audit Tracker

## Status

In Progress
Target: 0.40
Scope: Workspace-wide architecture and correctness audit after `0.38` stabilization
Current patch context: `0.40.12` (as of 2026-03-03)

---

## 1. Objective

Run one explicit, end-to-end audit pass across `icydb-core` and facade boundaries
to eliminate semantic drift, boundary leaks, and classification ambiguity before
the next major modeling changes.

This tracker is execution-oriented: each workstream has concrete checks,
required artifacts, and clear exit criteria.

---

## 2. Non-Goals

- No feature expansion by default.
- No query-surface growth unless required to close a verified correctness gap.
- No compatibility shim work; prefer direct model cleanup.
- No speculative refactors without an attached audit finding.

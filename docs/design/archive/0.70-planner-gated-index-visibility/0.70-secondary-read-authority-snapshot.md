## Title

**Removed — Secondary Read Authority Snapshot**

---

## Status

Removed

---

## Reason

This note described a transitional execution-time correctness seam for
secondary reads.

That design is no longer the target `0.70` model.

The revised rule is:

* planner visibility owns index correctness
* execution trusts planned visible indexes

So secondary reads do not need a dedicated runtime correctness snapshot in the
intended architecture.

---

## Revised Rule

Secondary index-backed reads now follow the same visibility contract as the
rest of the design:

* invisible index -> planner must ignore it
* visible index -> planner may use it and execution assumes it is correct

Execution-time correctness resolution is therefore not part of the target
design.

---

This file remains only as a record that the earlier snapshot seam was

# 0.152 DDL Roadmap

## Position

DDL should not lead schema evolution. It should parse user intent and lower into
catalog-native `SchemaMutation` operations whose compatibility, rebuild, and
snapshot-publication semantics are owned by the schema subsystem.

This keeps SQL syntax from becoming a second migration authority beside the
accepted catalog snapshot.

## Layering

1. Parser frontend: recognizes syntax and produces unresolved user intent.
2. Binder: resolves entity, field, index, and expression references against the
   accepted catalog snapshot.
3. Mutation planner: builds a `MutationPlan` and classifies compatibility plus
   rebuild requirements.
4. Rebuild executor: performs required physical work before publication.
5. Snapshot publisher: writes the accepted snapshot and invalidates affected
   planner/cache state.

Only the mutation planner, fail-closed publication gate, and runner preflight
shell are in 0.152 scope.

## 0.152 Closeout

0.152 intentionally does not add SQL DDL syntax and does not execute physical
index rebuild or cleanup work. DDL remains a future frontend over the mutation
engine; rebuild execution remains a future backend behind runner preflight,
validation, rollback, and runtime invalidation.

## Initial DDL Surface Candidates

These are candidates for later parser work once mutation planning exists:

```sql
ALTER TABLE entity ADD COLUMN field TYPE NULL;
CREATE INDEX name ON entity (field);
CREATE INDEX name ON entity (LOWER(field));
DROP INDEX name;
```

The syntax is intentionally illustrative. 0.152 should not commit to parser
grammar before the internal mutation contracts are proven.

## Deferred Syntax

- `ALTER COLUMN TYPE`
- primary-key changes
- generated or stored computed columns
- foreign-key constraints
- concurrent index build controls
- arbitrary SQL expression indexes
- generalized migration scripts

## Acceptance Rule

Every DDL statement must lower to a mutation plan. If no mutation plan can be
constructed from accepted catalog metadata, the statement fails before any
runtime schema or store mutation occurs.

# 0.154 Audit Readiness

## Supported Developer Surface

0.154 supports one runtime schema mutation path:

- startup reconciliation may add one non-unique field-path secondary index for
  one entity when accepted-before and accepted-after snapshots differ only by
  that index
- the path is accepted-snapshot owned; generated `EntityModel` and `IndexModel`
  metadata remains proposal, reconciliation, model-only, or test-only
- SQL DDL remains out of scope

## Required Evidence For Audits

Constructive audits should verify this path through these boundaries:

- admission: `SchemaMutationSupportedExecutionPath::field_path_index_addition`
  admits only the single non-unique field-path index-add shape
- execution: startup reconciliation scans accepted rows, builds
  `SchemaMutationRunnerInput`, and invokes `SchemaFieldPathIndexRunner`
- publication: `StartupFieldPathRebuildGate` and
  `StartupFieldPathPublicationDecision` gate physical work and accepted-snapshot
  publication, including a final target-index physical-store revalidation before
  schema-store insertion
- visibility: runner reports are publishable only after target-scoped physical
  validation, runtime invalidation, physical-store publication, and accepted
  snapshot handoff

## Still Blocked

- expression-index rebuild
- secondary-index drop cleanup
- unique-index rebuild
- multiple index batches
- full row rewrites
- SQL DDL
- online or concurrent rebuild
- generalized recovery or resume

## Audit Target

Audits should now evaluate whether this one supported path remains narrow,
accepted-schema-owned, deterministic, and fail-closed at each boundary instead
of treating every rebuild-required publication path as absent.

## 0.154 Closeout

0.154 closes on the supported startup field-path index-add path only. The
release is intentionally ready for developer database testing without SQL DDL
or broader physical mutation support.

Required closeout evidence:

- `supported_developer_physical_path_admits_only_single_field_path_index_add`
  keeps the accepted execution surface limited to one non-unique field-path
  secondary index add and keeps expression rebuilds, secondary-index cleanup,
  metadata-only plans, rewrites, and unsupported plans outside the supported
  path.
- `reconcile_runtime_schemas_executes_supported_field_path_index_addition`
  proves runtime startup reconciliation can rebuild and publish that supported
  accepted-schema path end to end.
- `field_path_startup_rebuild_gate_*` and
  `field_path_startup_publication_decision_*` prove row/schema drift and final
  target-index physical-store drift fail closed before accepted-after schema
  publication.
- `schema_mutation_publication_boundary_uses_runner_preflight` guards the
  source boundary so accepted-after snapshot insertion remains behind runner
  input construction, accepted row contracts, rebuild-gate revalidation, final
  physical-store revalidation, and the startup publication decision.

Deferred after 0.154:

- SQL DDL as a frontend over accepted mutation requests
- expression-index rebuild execution
- secondary-index drop cleanup execution
- unique-index rebuilds and conflict reporting
- online or concurrent rebuild
- generalized recovery or resumable rebuild

# 0.199 Partial-Window Public-Doc Audit

Status: code-reading-backed and docs-search-backed 0.199.0 evidence note.
This slice changes public documentation examples only; it does not change
runtime behavior, admission policy, query APIs, cursor formats, persisted
formats, or compatibility behavior.

## Question

After 0.198 removed public load `.limit(...)` and introduced
`partial_window(...)` as the deliberate partial row-window primitive, are active
public docs teaching `partial_window(...)` as the new casual list/read cap?

## Read-Admission Contract Cleanup

The `READ_ADMISSION.md` examples now separate three concepts:

- rejected public shapes may still show `partial_window(...)` when the point is
  that a row cap does not prove a safe route;
- public list fixes use `page(limit)` instead of a row-window cap;
- trusted/admin examples may use `partial_window(...)` only after explicit
  trusted-lane opt-in.

## Result

The active public docs no longer teach `partial_window(...)` as the general
replacement for removed `.limit(...)`. It remains documented as a low-level,
deliberate partial row-window endpoint contract.

Residual risk is low. The next 0.199 diagnostic-vocabulary work should avoid
reintroducing brittle phrase checks that force exact prose in these guides.

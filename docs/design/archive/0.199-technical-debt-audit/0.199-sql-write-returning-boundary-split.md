# 0.199 SQL Write Returning Boundary Split

Status: structural cleanup with focused validation. This slice preserves SQL
write mutation behavior, `RETURNING` result semantics, row/response bounds,
cache behavior, cursor formats, persisted formats, and public API shape.

## Question

Can SQL write `RETURNING` stop mixing projection selection, public statement
result shaping, row-count caps, and encoded response-size probes in one module?

## Evidence

Before this slice,
`crates/icydb-core/src/db/session/sql/execute/write_returning.rs` owned:

- accepted-descriptor `RETURNING *` label derivation;
- field-list `RETURNING` selection and duplicate-field rejection;
- typed mutation result projection;
- materialized delete/update result projection;
- row-count budget enforcement;
- encoded response-size budget probing.

Those responsibilities are now split while preserving the same exported
SQL-execute boundary functions:

- `write_returning/projection.rs` owns field selection, projection labels, and
  statement-result shaping.
- `write_returning/bounds.rs` owns row-count and encoded response-size budget
  enforcement.
- `write_returning.rs` is the narrow boundary index that re-exports the
  existing functions used by SQL write execution.

## Non-Goals

- No SQL write mutation execution change.
- No `RETURNING` output shape change.
- No read/write admission or policy change.
- No public API, cache, cursor, storage, or persisted-format change.

## Complexity Delta

- Public API complexity: neutral.
- Runtime complexity: neutral.
- Review complexity: lower. Projection ownership and bounds enforcement can now
  be reviewed independently.

## Validation

- `cargo fmt --all`
- `cargo check -p icydb-core --all-features`
- `cargo clippy -p icydb-core --all-features --all-targets -- -D warnings`
- `cargo test -p icydb-core --lib --all-features sql_write -- --nocapture`
- `cargo test -p icydb-core --lib --all-features sql_grouped -- --nocapture`

## Residual Risk

Low. This is a module ownership split. Remaining risk is feature/test-target
visibility drift, covered by the all-target core clippy pass and focused SQL
write/grouped tests.

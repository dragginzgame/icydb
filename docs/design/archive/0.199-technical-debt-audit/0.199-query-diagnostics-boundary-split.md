# 0.199 Query Diagnostics Boundary Split

Status: structural cleanup with focused validation. This slice preserves
diagnostics DTO field shapes, measured execution behavior, read admission,
query-plan cache identity, cursor formats, persisted formats, and public API
shape.

## Question

Can session query diagnostics stop mixing public attribution DTO definitions
with hidden measured execution without changing attribution output?

## Evidence

Before this slice,
`crates/icydb-core/src/db/session/query/diagnostics.rs` owned:

- public Candid attribution DTOs for direct row, kernel row, grouped, scalar
  aggregate, fluent terminal, and full query execution counters;
- conversion helpers from executor phase counters into those DTOs;
- shared compile/cache/store attribution assembly;
- hidden `DbSession::execute_query_result_with_attribution`;
- conversion from prepared execution outcomes into diagnostics phase bundles.

Those responsibilities are now split while preserving the same re-exported
types and hidden execution method:

- `diagnostics/model.rs` owns attribution DTOs and phase-counter projection.
- `diagnostics/execution.rs` owns the hidden measured execution wrapper and
  prepared-outcome conversion.
- `diagnostics.rs` is the narrow module index that re-exports the existing
  public diagnostics DTOs and query-internal common attribution helper.

## Non-Goals

- No diagnostics field rename or DTO shape change.
- No fluent, SQL, planner, admission, cache, cursor, storage, or executor
  behavior change.
- No new measurement buckets.
- No public facade visibility change.

## Complexity Delta

- Public API complexity: neutral.
- Runtime complexity: neutral.
- Review complexity: lower. Attribution wire shape and measured execution can
  now be reviewed independently.
- File shape:
  - `diagnostics.rs`: 678 lines before, 14-line module index after.
  - `diagnostics/model.rs`: 570 lines.
  - `diagnostics/execution.rs`: 130 lines.

The total line count grows slightly from module headers/import separation, but
the behavioral owner boundaries are clearer.

## Validation

- `cargo check -p icydb-core --all-features`

## Residual Risk

Low. This is a module ownership split. Remaining risk is feature-target
visibility drift for diagnostics-only helpers; all-target clippy and focused
fluent attribution tests should be run before release.

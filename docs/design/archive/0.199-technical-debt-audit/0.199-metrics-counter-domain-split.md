# 0.199 Metrics Counter-Domain Split

Status: structural cleanup with focused validation. This slice preserves the
metrics event taxonomy, global/entity counter mirroring, counter arithmetic,
sink dispatch, metrics endpoint payloads, cursor formats, persisted formats,
and public facade shape.

## Question

Can metrics counter mutation helpers stop sharing one broad file while keeping
the concrete sink as the only event-to-state dispatch owner?

## Evidence

After the metrics sink split, `metrics/sink/counters.rs` still contained 37
event-domain helpers across five independent counter domains, plus one shared
gauge replacement helper:

- executor starts and outcomes;
- schema reconciliation and transition outcomes;
- SQL compile and write outcomes;
- query-plan and compiled-SQL cache activity;
- selected route and execution-shape attribution.

The helpers all mutate stable global or per-entity counter DTOs, but they do
not otherwise share policy or execution decisions. Keeping them together made
unrelated counter taxonomies part of the same review surface.

## Ownership

- `metrics/sink/counters.rs` owns shared gauge replacement and private domain
  re-exports for concrete dispatch.
- `metrics/sink/counters/execution.rs` owns executor start/outcome counters.
- `metrics/sink/counters/schema.rs` owns reconciliation/transition counters.
- `metrics/sink/counters/sql.rs` owns SQL compile/write counters.
- `metrics/sink/counters/cache.rs` owns cache outcome, size, and miss-reason
  counters.
- `metrics/sink/counters/planning.rs` owns route, choice-reason, grouped-mode,
  and prepared-shape counters.
- `metrics/sink/dispatch.rs` remains the only concrete `MetricsEvent` dispatch
  owner.

## Non-Goals

- No metrics event or counter field change.
- No counter arithmetic or global/entity mirroring change.
- No metrics report, compact code, or Candid payload change.
- No instrumentation call-site or sink behavior change.
- No attempt to deduplicate intentionally explicit global/entity mutations.
- No wasm or runtime performance claim.

## Complexity Delta

- Public API complexity: neutral.
- Runtime behavior: neutral.
- Review complexity: lower because counter mutation is grouped by event domain.
- File shape:
  - `counters.rs`: 1,188 lines before, 22-line parent after.
  - `counters/execution.rs`: 130 lines.
  - `counters/schema.rs`: 198 lines.
  - `counters/sql.rs`: 221 lines.
  - `counters/cache.rs`: 341 lines.
  - `counters/planning.rs`: 337 lines.

The total line count grows from module documentation, imports, and re-exports;
the mutation statements and event-domain behavior remain unchanged.

## Validation

- `cargo check -p icydb-core --all-features`
- `cargo test -p icydb-core --lib --all-features metrics -- --nocapture`
- `cargo clippy -p icydb-core --all-features --all-targets -- -D warnings`
- `cargo check -p icydb --no-default-features --features sql`
- `cargo fmt --all --check`
- `git diff --check`

## Residual Risk

Low. This is a private module ownership split. Remaining risk is import or
visibility drift between the counter-domain modules and concrete sink dispatch;
focused metrics tests plus all-target Clippy cover that surface.

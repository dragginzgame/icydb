# 0.199 SQL Select Read-Budget Test Boundary

Status: test-boundary cleanup with focused validation. This slice preserves
production SQL SELECT execution, read admission, SQL cache behavior, executor
strategy, cursor formats, persisted formats, and public API shape.

## Question

Can SQL SELECT production execution stop carrying test-only response-size Candid
probe DTOs and read-budget helper implementation directly in `select.rs`?

## Evidence

Before this slice, `crates/icydb-core/src/db/session/sql/execute/select.rs`
owned both production SELECT execution and test-only helpers for:

- applying custom read-admission policy caps in tests;
- projecting grouped execution hard limits from a test policy;
- estimating encoded projection/grouped response sizes for admission tests;
- Candid response-size probe DTOs.

Those helpers are only compiled for tests and are also reused by
`global_aggregate` tests. They now live in
`select/read_budget.rs`, while `select.rs` re-exports only the two test helpers
that sibling SQL execution test paths already used.

## Non-Goals

- No production read-admission policy change.
- No SQL SELECT execution or result shaping change.
- No test assertion semantic change.
- No public API, cache, cursor, or persisted-format change.

## Complexity Delta

- Public API complexity: neutral.
- Runtime complexity: neutral.
- Production SELECT review complexity: lower because test-only Candid response
  probes no longer sit in the execution module.

## Validation

- `cargo fmt --all`
- `cargo check -p icydb-core --all-features`
- `cargo clippy -p icydb-core --all-features --all-targets -- -D warnings`
- `cargo test -p icydb-core --lib --all-features read_admission -- --nocapture`
- `cargo test -p icydb-core --lib --all-features sql_grouped -- --nocapture`

## Residual Risk

Low. This is a test-only module ownership split. Remaining risk is all-target
test visibility drift, covered by the all-target core clippy pass and focused
read-admission/grouped SQL tests.

# 0.199 Complete-Small-Set Terminal Boundary

Status: 0.199.3 cleanup slice. This changes only private fluent terminal
wiring. It does not change public API names, read-admission policy, cursor
formats, persisted formats, generated SQL behavior, or complete-read semantics.

## Question

Can `collect_complete()` and its diagnostics attribution variant share the same
terminal-owned admission handoff?

## Evidence

`collect_complete()` owns complete-small-set intent by applying an internal
lookahead limit and a public-read policy before execution. Its diagnostics
variant used the same limit, policy, and admission setup before executing with
attribution.

That duplication was correct but drift-prone: a future change to complete-read
caps or admission handoff could update one path and miss the other.

## Change

Added one private helper:

- `with_complete_small_query(...)`

The helper owns:

- rejecting prior partial row windows;
- verifying non-paged mode;
- applying `COMPLETE_SMALL_EXECUTION_LIMIT`;
- constructing the public-read policy with the complete-read row and response
  byte caps;
- running read admission before execution.

Both `collect_complete()` and `collect_complete_with_attribution()` now execute
through that helper.

## Result

Complete-small-set execution and attribution now share one owner-local
admission handoff. The result cap and lookahead behavior remain unchanged.

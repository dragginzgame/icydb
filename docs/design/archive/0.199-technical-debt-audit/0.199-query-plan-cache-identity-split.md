# 0.199 Query Plan Cache Identity Split

Status: structural cleanup with focused validation. This slice preserves shared
query-plan cache keys, cache attribution, compile-phase attribution, cache
lookup/insert behavior, prepared-plan reuse, SQL/fluent planning behavior,
read admission, cursor tokens, persisted formats, and executor strategy.

## Question

Can shared query-plan cache identity and attribution DTOs move out of the
session cache behavior module without changing cache semantics?

## Evidence

`crates/icydb-core/src/db/session/query/cache.rs` mixed:

- shared query-plan cache key identity;
- accepted-schema cache identity projection;
- query-plan visibility DTOs;
- cache hit/miss attribution DTOs;
- compile-phase attribution recording;
- shared cache map ownership;
- cache miss reason classification;
- prepared-plan lookup, insert, and build behavior;
- typed and SQL planner handoff methods.

The key, identity, and attribution types define cache dimensions and reporting
buckets. They do not own cache storage, store visibility recovery, planner
selection, query execution, or cache insertion behavior.

## Change

Move query-plan cache identity and attribution ownership into
`crates/icydb-core/src/db/session/query/cache/identity.rs`:

- `QueryPlanVisibility`;
- `QueryPlanCacheKey`;
- `SchemaCacheIdentity`;
- `QueryPlanAcceptedSchema`;
- `QueryPlanCacheAttribution`;
- `QueryPlanCompilePhaseAttribution`;
- `QueryPlanCompilePhase`;
- `QueryPlanCompilePhaseRecorder`;
- shared query-plan cache method version;
- cache-key constructors and accessors.

The parent `cache.rs` keeps:

- the shared cache map;
- cache miss reason classification;
- cache lookup and insert recording;
- store visibility recovery;
- planner-visible index projection;
- typed and SQL prepared-plan handoff methods.

## Result

`crates/icydb-core/src/db/session/query/cache.rs` drops from about 1,276 lines
to about 932 lines. The new `session/query/cache/identity.rs` owner is about
403 lines and contains the cache identity and attribution model consumed by
session query cache behavior and SQL diagnostics.

## Validation

- `cargo fmt --all`
- `cargo check -p icydb-core --all-features`

## Non-Goals

- No cache-key semantic change.
- No query-plan cache method-version bump.
- No read-admission change.
- No planner or executor strategy change.
- No SQL/fluent behavior change.
- No cursor or persisted-format change.
- No performance claim.

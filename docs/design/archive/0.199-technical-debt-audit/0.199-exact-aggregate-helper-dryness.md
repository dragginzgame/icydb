# 0.199 Exact-Aggregate Helper DRY Slice

Status: small code cleanup with focused validation. This slice preserves public
API names, read-admission behavior, diagnostic DTOs, cursor formats, persisted
formats, and aggregate semantics.

## Question

Did 0.198's exact aggregate helper set introduce repeated wiring that can drift
as new helpers are added?

## Evidence

`crates/icydb-core/src/db/query/fluent/load/terminals.rs` already has a good
descriptor-driver boundary:

- each aggregate terminal has a concrete descriptor;
- `TerminalStrategyDriver` maps each descriptor to the session execution and
  explain entrypoints;
- semantic terminals reject prior row-window caps before execution or EXPLAIN.

The repeated parts were exact-aggregate terminal wiring:

- every exact aggregate execution helper repeated the same raw row-window
  rejection before terminal execution;
- every field-based exact aggregate repeated the same raw row-window rejection
  before field-slot resolution;
- every exact aggregate explain helper hand-mapped the returned plan to
  `ReadIntentKind::ExactAggregate`;
- count attribution had its own exact/partial intent handoff.

## Change

Added private owner-local helpers:

- `execute_exact_aggregate_terminal(...)`
- `execute_exact_aggregate_by_slot_terminal(...)`
- `explain_checked_exact_aggregate_terminal(...)`
- `explain_exact_aggregate_by_slot_terminal(...)`
- `explain_exact_aggregate_terminal(...)`

These delegate to the existing terminal descriptor driver and keep exact
aggregate read-intent attribution in one place. Field-based exact aggregates
still reject a prior row-window cap before field-slot resolution, preserving the
0.198 error-order contract.

## Result

This is a narrow DRY cleanup. It makes exact aggregate execution, field-slot
resolution, and explain attribution less fragile without changing the terminal
descriptor model or public API.

## Non-Goals

- No public aggregate API rename.
- No admission-policy change.
- No aggregate execution strategy change.
- No cursor-token or persisted-format change.
- No broad terminal dispatch enum.

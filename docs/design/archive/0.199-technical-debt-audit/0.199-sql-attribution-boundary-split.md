# 0.199 SQL Attribution Boundary Split

Status: structural cleanup with focused validation. This slice preserves SQL
diagnostic DTO names, attribution counter semantics, SQL execution behavior,
cache behavior, generated/controller SQL behavior, cursor formats, persisted
formats, and public API shape.

## Question

Can SQL attribution stop mixing compile DTOs, execution DTOs, covering/output
blob attribution, cache attribution, raw phase counters, and top-level query
assembly in one file?

## Evidence

Before this slice, `crates/icydb-core/src/db/session/sql/attribution.rs` was a
591-line diagnostics file that owned:

- compile attribution DTOs;
- execution attribution DTOs;
- pure and hybrid covering attribution DTOs;
- SQL query cache attribution DTOs;
- output blob attribution and recursive response-size accounting;
- raw SQL execute phase counters;
- top-level SQL query execution attribution assembly.

Those responsibilities are now split while preserving the same parent
re-export surface:

- `attribution/compile.rs` owns `SqlCompileAttribution`.
- `attribution/execution.rs` owns `SqlExecutionAttribution`.
- `attribution/covering.rs` owns `SqlPureCoveringAttribution` and
  `SqlHybridCoveringAttribution`.
- `attribution/cache.rs` owns `SqlQueryCacheAttribution`.
- `attribution/output_blob.rs` owns `SqlOutputBlobAttribution` and recursive
  output blob accounting.
- `attribution/phase.rs` owns `SqlExecutePhaseAttribution`.
- `attribution/query.rs` owns `SqlQueryExecutionAttribution`, its input bundle,
  and top-level assembly from phase counters.
- `attribution.rs` is now the diagnostics facade and re-export boundary.

## Non-Goals

- No SQL attribution field rename.
- No diagnostic/Candid payload shape change.
- No metric or counter semantic change.
- No SQL execution, cache lookup, compile, or response-shaping behavior change.
- No generated/controller SQL behavior change.
- No cursor, persisted-format, storage, or public API change.

## Complexity Delta

- Public API complexity: neutral.
- Runtime behavior: neutral.
- Review complexity: lower. Compile, execution, covering, output-size, cache,
  phase, and query-level attribution can now be reviewed independently.
- File shape:
  - `attribution.rs`: 591 lines before, 38-line module facade after.
  - `attribution/cache.rs`: 37 lines.
  - `attribution/compile.rs`: 49 lines.
  - `attribution/covering.rs`: 67 lines.
  - `attribution/execution.rs`: 50 lines.
  - `attribution/output_blob.rs`: 106 lines.
  - `attribution/phase.rs`: 166 lines.
  - `attribution/query.rs`: 126 lines.

The total line count grows modestly because each owner now has its own module
header and imports.

## Residual Risk

Low. This is a module ownership split. Remaining risk is visibility or lint
drift in feature combinations, covered by SQL-focused tests, core clippy, and
the `icydb` SQL facade check before release.

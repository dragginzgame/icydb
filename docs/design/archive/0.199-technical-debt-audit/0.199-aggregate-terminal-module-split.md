# 0.199 Aggregate Terminal Module Split

Status: structural cleanup with focused validation. This slice preserves public
fluent aggregate APIs, read-admission behavior, diagnostic DTOs, cursor tokens,
persisted formats, generated SQL authority, and executor strategy.

## Question

Can the remaining aggregate-heavy part of fluent load `terminals.rs` be split
without changing aggregate terminal behavior?

## Evidence

After the first terminal module split, `terminals.rs` still owned a large
aggregate method family:

- count and exact count;
- min/max id extrema;
- field min/max extrema;
- nth and median selection;
- sum and average terminals;
- count-distinct and min/max pair terminals;
- aggregate attribution and EXPLAIN wrappers.

That made the parent terminal file smaller than before, but it still mixed
aggregate API wrappers with projection, bytes, materialization, and response
cardinality terminals.

## Change

Move aggregate public terminal methods into
`terminals/aggregate.rs`.

The aggregate module delegates to the existing private support helpers for:

- exact aggregate limit ownership checks;
- field-slot resolution;
- descriptor execution;
- aggregate EXPLAIN;
- count attribution read-intent metadata.

The parent `terminals.rs` keeps execution, EXPLAIN execution, bytes,
first/last id, and response cardinality terminals after the projection terminal
split that followed in the same 0.199.12 batch.

## Result

`terminals.rs` is reduced from just under 950 lines to about 190 lines after
the aggregate and projection method splits. The aggregate method family now has
an explicit owner without adding a terminal enum, broad dispatch object,
compatibility layer, or execution-route rewrite.

## Validation

- `cargo check -p icydb-core --all-features`
- `cargo test -p icydb-core --all-features aggregate_terminals`
- `cargo test -p icydb-core --all-features read_admission`
- `cargo test -p icydb-core --all-features aggregate_explain`
- focused projection/ranking terminal tests
- `cargo clippy -p icydb-core --all-features --all-targets -- -D warnings`

## Non-Goals

- No public aggregate API changes.
- No terminal rename.
- No read-admission change.
- No EXPLAIN or diagnostic DTO change.
- No query planner or executor strategy change.
- No cursor or persisted-format change.

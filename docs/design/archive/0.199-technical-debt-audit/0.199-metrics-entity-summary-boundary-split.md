# 0.199 Metrics Entity Summary Boundary Split

Status: structural cleanup with focused validation. This slice preserves
per-entity rich metrics DTO names, Candid payload shape, rich metrics report
ordering, metrics sink behavior, compact metrics report behavior, cursor
formats, persisted formats, and metrics endpoint payload shape.

## Question

Can the per-entity rich metrics summary DTO and raw-counter projection stop
sharing one broad mutable-state file with raw metrics state?

## Evidence

After the rich report split, `crates/icydb-core/src/metrics/state.rs` still
owned:

- mutable runtime metrics state;
- raw operation and perf counters;
- per-entity rich summary DTOs;
- per-entity summary projection from mutable counters;
- rich summary derived-ratio helpers;
- summary activity scoring used by rich report ordering;
- reset/window helper functions.

Those responsibilities are now split while preserving the same parent
re-export surface:

- `metrics/state.rs` retains mutable state, raw perf/entity counters, reset
  helpers, and parent re-exports.
- A companion event-operation split moves global operation counters,
  `MetricRatio`, and global derived-ratio helpers into
  `metrics/state/ops.rs`.
- `metrics/state/summary.rs` owns `EntitySummary`, rich per-entity summary
  projection from `EntityCounters`, derived-ratio helpers, and activity scoring
  for rich report ordering.
- `metrics/state/report.rs` remains the top-level rich report/window owner.
- `metrics/state/compact.rs` remains the compact report owner.

## Non-Goals

- No per-entity rich metrics Candid payload shape change.
- No rich report sorting behavior change.
- No compact metrics payload or code change.
- No instrumentation, metrics sink, cache counter, or executor behavior change.
- No wasm/performance claim.

## Complexity Delta

- Public API complexity: neutral.
- Runtime behavior: neutral.
- Review complexity: lower. Per-entity rich summary payload shape and projection
  can now be reviewed without parsing mutable metrics state.
- File shape:
  - `state.rs`: 2384 lines before, 284 lines after the companion operation
    counter split.
  - `state/summary.rs`: 1183 lines.
  - `state/ops.rs`: 941 lines.

The total line count grows slightly from module headers/import separation, but
the large per-entity summary owner is now localized.

## Validation

- `cargo check -p icydb-core --all-features`
- `cargo test -p icydb-core --lib --all-features metrics -- --nocapture`
- `cargo clippy -p icydb-core --all-features --all-targets -- -D warnings`

## Residual Risk

Low. This is a module ownership split. Remaining risk is feature-target
visibility drift around the summary projection and activity-scoring helpers;
all-target clippy plus the entity summary Candid-shape, derived-ratio, and
report-ordering tests cover the current surface.

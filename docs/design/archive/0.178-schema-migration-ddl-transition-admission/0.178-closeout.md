# 0.178 Schema Migration / DDL Transition Admission Closeout

## Status

Closeout-ready as of 2026-06-04.

0.178 closes single-statement DDL transition admission for existing accepted
entities. SQL DDL now carries explicit expected and next schema-version intent,
binds against accepted schema authority, lowers through schema-owned mutation
admission, rejects unsupported transition classes before publication, and
publishes only through schema-owned accepted-snapshot publication with a final
accepted-identity recheck.

The line intentionally does not ship transition drafts, relation DDL, entity
birth/death/rename, multi-entity catalog transitions, broad online row
rewrites, new index-backfill capability beyond the existing schema-owned runner
path, generated-field adoption, managed-field DDL, version-gap imports, public
migration script files, or rollback tooling beyond fail-closed admission and
publication semantics.

## Completed Surface

- Mutating SQL DDL binds explicit `EXPECT SCHEMA VERSION` and `SET SCHEMA
  VERSION` clauses through parser/DDL DTOs before schema mutation admission.
- Schema-owned DDL admission enforces the version/fingerprint matrix before
  mutation compatibility and runner admission.
- True no-op DDL still requires `EXPECT SCHEMA VERSION`, does not require
  `SET SCHEMA VERSION`, publishes nothing, and rejects empty version bumps.
- Publication rechecks the binding-time accepted identity against the live
  accepted identity before replacing the accepted snapshot.
- Supported DDL field and secondary-index transitions derive accepted-after
  snapshots through schema-owned helpers rather than SQL DDL constructing
  catalog facts directly.
- Unsupported generated/managed ownership, unsupported physical work, and
  unsupported transition classes fail closed before accepted-after publication.
- Schema mutation execution, runner preflight, runtime publication identity,
  and mutation fingerprints are split into dedicated schema-owned modules
  rather than SQL DDL or hot query paths owning the semantics.
- Stable machine-readable admission/publication diagnostics are exposed through
  `SchemaDdlAdmissionError` and query-facing error detail mapping.

## Source Audit Notes

The closeout audit keeps the authority split established by 0.176 and 0.177:
accepted schema snapshots remain runtime authority, SQL DDL remains an
authoring frontend, and schema-owned mutation code owns candidate derivation,
fingerprint comparison, compatibility admission, runner preflight, publication,
and invalidation semantics.

The remaining schema mutation and SQL DDL hub pressure was reduced through the
late 0.178 structure slices. DDL preparation reporting, version-contract
binding, schema-owned DDL admission, accepted-snapshot delta classification,
execution/preflight contracts, mutation fingerprints, and runtime publication
identity now live in dedicated modules with source guards preventing regression
into the hubs.

No new persistence format compatibility fallback was introduced. Fingerprint
method mismatches remain fail-closed unless schema-owned code provides an
explicit same-domain normalization path in a later design.

## Deferred Boundary

The next migration line should start from this closed 0.178 boundary:

- transition drafts must remain unpublished authoring artifacts until a commit
  command evaluates the version/fingerprint matrix once;
- entity birth/death/rename and multi-entity transitions need a catalog-level
  version/fingerprint design before public DDL admission;
- relation DDL and broad online row rewrites need validation and rollback
  semantics before publication;
- new physical migration runners must continue to publish only after validated
  physical work, accepted-identity recheck, and accepted-snapshot invalidation
  can occur as one schema-owned publication boundary.

# 0.209 Opening Implementation Audit

Status: Slice 1 complete; no production format or semantic change

Baseline: `v0.208.1` at `dfd5f5a71`

## Outcome

The live 0.208.1 implementation confirms the temporal conflation described by
the design. One accepted `SchemaFieldDefault` controls both future insertion
omission and physical absence in older short rows. `SchemaRowLayout` uses the
complete `SchemaVersion` as its identity, and the canonical row envelope has no
layout stamp. The current code is internally consistent, but it cannot prove
whether an absent slot is legitimate history or corruption.

Slice 1 changes no production behavior. It records every authority that the
atomic format cut must move together and adds a test-only, compile-checked
specification of the new owner shapes. Those types are deliberately unavailable
to production callers until schema metadata, row bytes, decoders, and emitters
can switch in one landing unit.

## Production Propagation Inventory

### Default Metadata

The single default type reaches snapshot construction, schema codec,
publication validation, runtime field descriptors, SQL DDL binding, field
allocation/default mutation, reconciliation, schema description, generated
proposal lowering, and schema-aware query fixtures. The format cut must update
the owner and all consumers together; retaining `SchemaFieldDefault` as an alias
would preserve the ambiguity.

Primary production locations:

- `db/schema/snapshot.rs` owns the accepted field fact;
- `db/schema/codec.rs` owns its durable wire;
- `db/schema/runtime.rs` derives missing-field policy;
- `db/schema/mutation/field_default_encoding.rs` canonicalizes DDL payloads;
- `db/schema/mutation/{field,field_allocation}.rs` constructs candidates;
- `db/schema/reconcile/sql_ddl/field_metadata.rs` applies field metadata and
  row rewrites; and
- `db/sql/ddl/field.rs` binds SQL DDL defaults.

### Missing-Slot Materialization

`StructuralRowContract::missing_slot_value`, `missing_slot_payload`, and their
private materialization helper are the sole semantic owner today. They are
consumed by:

- dense and sparse structural validation;
- direct required-slot reads;
- retained-slot and indexed sparse readers;
- typed/entity and serialized structural patch completion; and
- SQL DDL field-metadata row rewriting.

This is one implementation, not duplicate code, but it owns two different
times. Slice 2 must split its callers by operation rather than wrap the helper
with renamed aliases.

### Row Envelope And Emitters

`db::codec` owns the only row envelope. Its fixed header is currently seven
bytes: two magic bytes, one format byte, and a four-byte payload length.
`data::persisted_row::contract::emit_raw_row_from_slot_payloads` is the shared
dense emitter, while `data::persisted_row::patch` adapts typed entities,
structural patches, old rows, and DDL payloads into that emitter.

The current slot container costs `2 + 8 * slot_count + payload_bytes`; including
the envelope, a row costs `9 + 8 * slot_count + payload_bytes`. The planned
layout stamp therefore has a known four-byte raw per-row floor before any
secondary stable-memory or journal framing effects.

Typed and structural save lanes both normalize before/after images to canonical
rows before marker construction. Recovery admits marker row bytes through
`RawRow` and accepted contracts; there is no separate recovery default
evaluator to preserve.

The SQL performance canister was rebuilt after the Slice 1 changes and remained
byte-identical at both raw comparison points: 3,938,804 built bytes and
3,662,158 shrunk bytes (zero-byte deltas). This is expected because the private
contract module is compiled only for tests.

The native timing is an opening comparison point, not a universal threshold.
Instruction evidence for actual typed, SQL, indexed, relation-bearing, and
journaled default-heavy writes must be captured against the new accepted-first
flow in later slices. The existing SQL performance harness has general heap and
journaled write matrices but no dedicated default-heavy entity, so Slice 1 does
not pretend that those measurements prove default-path cost.

## Compile-Checked Private Contract

The test-only `db::schema::temporal` module freezes these shapes without making
them runtime authority:

- `RowLayoutVersion` is non-zero and exhausts instead of wrapping;
- `SchemaInsertDefault` and `SchemaHistoricalFill` are different types even
  when their payload bytes initially match;
- historical `Reject`, `Null`, and payload states are explicit; and
- authored, omitted-insert, explicit-insert-default,
  explicit-update-default, and preserve-existing intents cannot collapse into
  an `Option` or boolean sentinel.

Slice 2 must move these meanings into production directly and delete the old
ambiguous owners. It must not retain the specification module as a parallel
implementation.

## Atomic Hard-Cut Boundary

The next slice must update, in one coherent landing unit:

1. accepted snapshot field/layout types and validation;
2. schema wire codec, bundle fingerprint, and current format identities;
3. row envelope encode/decode and exact slot-count validation;
4. every typed, structural, SQL, DDL, fixture, marker, and recovery row emitter;
5. dense, sparse, indexed, retained-slot, and direct readers; and
6. corruption/error tests for old, unstamped, short, long, zero-version, and
   future-version rows.

No production import/restore surface needs compatibility handling because none
exists. Any later logical import must use the ordinary current emitter. A
future physical restore lane is new authenticated functionality, not a reason
to keep decoding unstamped rows.

## Slice 1 Verdict

Slice 1 is complete. The repository still reads and writes exactly the 0.208.1
schema and row forms, the mixed-format risk is absent, and the next safe step is
the atomic Slice 2 hard cut.

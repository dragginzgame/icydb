# 0.209 Closeout Design And Status Addendum

Status: focused closeout complete on 2026-07-21; full-suite push validation is human-owned

Baseline: `v0.208.1` at `dfd5f5a71`

This addendum is the authoritative correction ledger for the 0.209 closeout
audit. It refines the implementation details in the
[0.209 design](0.209-design.md) without introducing a second temporal-default
architecture. The original design remains authoritative except where this file
records a narrower hard-cut decision.

## Closeout Outcome

The central 0.209 model is retained:

1. Rust defaults construct authored application values.
2. Accepted insertion policy resolves future write omission and explicit
   database-default requests.
3. Historical fill resolves only physical absence proved by an admitted older
   row-layout stamp.

The audit confirmed one current row envelope, one accepted mutation resolver,
and no compatibility decoder or generated-model runtime fallback. The bounded
closeout corrections below resolve the incomplete propagation found around
replacement identity, generated schema reconciliation, typed failures, and
test/documentation evidence.

## Corrected Design Decisions

### Replacement Identity

Replacement construction may use insertion policy for absent non-identity
fields, but it may not regenerate the row selected for replacement. The
mutation target owns row identity independently of field authorship:

- a keyless insert resolves a generated primary key from the accepted insert
  policy;
- a typed or keyed replacement carries its selected primary key as protected
  database-owned provenance into the complete after-image;
- the resolved after-image must match that target before commit preparation;
  and
- single, atomic-batch, and non-atomic-batch lanes use the same rule.

### Generated Default Reconciliation

For an existing generated-owned field, a generated default change is a
metadata-only accepted-schema candidate. Reconciliation changes only
`insert_default`. It preserves the field ID, slot, introduction layout,
historical fill, row-layout current/floor versions, origin, write policy, kind,
codec, indexes, and relations. It performs no row scan or derived-state
rebuild solely for the default change.

### Generated Fields After SQL DDL Fields

The current generated codec binds Rust fields to declaration-order slots, while
SQL DDL appends accepted-only dense slots. A later generated field would claim
an already allocated DDL identity. 0.209 therefore hard-cuts this combination:

- accepted DDL fields may remain absent from the generated model;
- later generated field addition after any trailing DDL-owned field rejects
  with a typed schema-transition cause; and
- reconciliation never remaps, aliases, or guesses fields by name.

Supporting interleaved generated and DDL-owned field evolution requires a
separate accepted-slot mapping design and is not implied by 0.209.

### Typed Failures

Every physical-shape transition that cannot allocate the next
`RowLayoutVersion` reports `RowLayoutVersionExhausted`. It must not collapse to
`PhysicalRunnerMissing`, generic unsupported execution, or an untyped unit
error.

Write omission and persisted-row corruption remain different classes.
Accepted write admission owns missing-required-field diagnostics. Persisted-row
validation owns layout-range and slot-count corruption diagnostics.

## Second-Pass Hard-Cut Sweep

The post-correction sweep found no production occurrence of the removed shared
default type, serialized structural-patch replay DTO, old generated-candidate
name, unstamped canonical row emitter, or generated-model runtime decode for the
accepted-layout regression fixture.

The remaining `EntityCreateFieldInput` exports are confined to hidden
derive-macro support. The ordinary `icydb::traits` facade no longer exposes the
transport type. Direct `RawRow` construction outside the stamped canonical
emitter is test-only malformed/corpus fixture construction. Remaining Rust
`Default` calls belong to explicit Rust-default generation, parser/metrics
state, or tests; none resolves accepted write omission or historical absence.

## Validation Policy

Focused package and named-test selections are agent-owned. Full workspace and
repository suites remain the human-owned push check required by `AGENTS.md`.
Closeout evidence records failures honestly; no tracker item becomes complete
until its production path and focused regression evidence both pass.

# 0.63 Status Tracker

Current state: in progress. `0.63` is landing as a bounded consolidation line
across query/session boundaries, projection/materialization ownership, and
mutation/query-plan authority cleanup.

## Scope

This status tracks the `0.63` mega-audit line defined in:

- `docs/design/0.63-mega-audit/0.63-design.md`

The goal remains consolidation, not feature expansion:

- collapse same-layer duplication
- keep one owner per authority boundary
- preserve the current wasm/runtime baseline while reducing drift

---

## Landed Audit Slices

- [x] Session query-execution error wrappers are centralized under the
  `db::session` boundary instead of being rebuilt in local query/SQL paths.
- [x] Reverse-index membership deltas and commit-window counter increments now
  flow through one `PreparedIndexDeltaKind` authority.
- [x] Commit-window metrics emission now shares one `PreparedRowOpDelta`
  translation path rather than rebuilding event payloads in parallel save/delete
  wrappers.
- [x] Projection/materialization now shares one prepared projection-plan owner
  across typed projection, slot-row validation, and structural SQL projection.
- [x] Compiled scalar projection evaluation now has one shared core instead of
  split structural-vs-typed recursive evaluators.
- [x] Query-origin unsupported execution construction is now owned by

- [x] Aggregate target field resolution now has one planner/query validation

- [x] Grouped aggregate target-field validation now has one validation helper

  checks.
- [x] Cursor invariant reasons now stay raw inside `CursorPlanError`, with the

  conversion boundaries; session paged-query routing also consumes the same
  cursor-owned ordering-contract message.
- [x] SQL query-lane dispatch now uses shared session-owned helpers for
  query-lane unsupported routing, prepared-statement lowering/binding,

  scalar-surface intent rejection instead of repeating local lane/intention
  mappings.

## Authority Goal

0.213 establishes this one-way boundary:

```text
icydb-model declarations
        |
        v
host-only application model
        |
        v
bounded icydb-schema fragments/proposal
        |
        v
catalog-native reconciliation
        |
        v
accepted schema
        |
        v
runtime planning, admission, persistence, recovery, and integrity
```

Generated application metadata remains proposal and adapter input. It never
becomes accepted or runtime database authority.

## Frozen Public Vocabulary

The layer suffixes are fixed before public proposal types land:

| Layer | Entity concept | Record concept |
| --- | --- | --- |
| model authoring graph | `icydb_model::node::Entity` | `icydb_model::node::Record` |
| generated model inspection | `EntityDescriptor` | `RecordDescriptor` |
| proposal contract | `EntityFragment` / `EntitySourceKey` | `RecordTypeFragment` / `TypeSourceKey` |
| accepted catalog | `AcceptedEntity` or `EntityCatalogEntry` | `AcceptedRecordType` |
| public runtime | `EntityName`, `EntityBinding`, or `TypedEntityBinding` | `InputValue::Record` / `OutputValue::Record` |

Proposal types use `Fragment`, `Proposal`, or `SourceKey`. Accepted/runtime
types use `Accepted`, `Catalog`, `Binding`, `Input`, or `Output`. The old
concrete `EntityModel` name is retired rather than transferred.

# 0.144 Structural Ownership Audit Status

## Current Status

Proposed. No structural patches have started under this design yet.

## Audit States

Use these states consistently:

* `Not started`: no ownership review yet
* `Auditing`: reading imports, exports, file sizes, and caller paths
* `Patch ready`: safe structural move identified
* `Patched`: structural cleanup applied and validated
* `Deferred`: issue found but not safe for this slice
* `Leave alone`: current structure is intentional

Area: `db/executor`
Owner: Runtime execution boundaries for validated plans.
Allowed imports: `db::executor` root re-exports and explicit artifacts from `db::query::{plan, explain}`.
Internal-only modules: executor route planning, pipeline operators, aggregate reducers, projection materializers, terminal row decoding, runtime context helpers, and explain descriptor fragments.
Finding: executor explain descriptor code owned an explain-only residual predicate projection that normalized query expression trees and stripped `COALESCE(bool_expr, FALSE)` guards before deriving an `ExplainPredicate`. That is display-only query expression interpretation, not executor descriptor assembly.
Classification: safe cleanup
Minimal patch: move the projection helper to `query::explain::predicate` and re-export it as a named query-owned explain artifact; executor descriptor assembly calls that boundary function. Future executor explain needs must extend this query-owned artifact instead of importing `Expr` and filling gaps by re-walking expression trees.
Validation: `cargo fmt --all --check`; `cargo check -p icydb-core --tests --features sql`; `git diff --check`.

Area: `db/executor`

Internal-only modules: executor explain descriptor fragments and route diagnostic assembly.

Classification: safe cleanup

## Patch Log

* Updated `query::builder::aggregate` module contract wording to include the
  concrete fluent terminal descriptors it now owns.
* Moved explain-only residual predicate expression projection from executor
  descriptor assembly into `query::explain`.
* Moved explain-only projection field label derivation from executor descriptor
  assembly into `query::explain`.
* Removed a no-op `Expr` variant match from grouped page finalization so
  executor route code no longer appears to branch on expression semantics.
* Deleted the executor route `semantics` forwarding module; aggregate meaning
  helpers now resolve directly through query-owned `AggregateKind` methods.
* Removed duplicate grouped execution-mode recomputation from route
  observability; explain/runtime diagnostics now project the stored route
  artifact.
* Moved SQL global aggregate executor terminal construction out of SQL
  lowering; lowering now returns executor-neutral aggregate plan fragments and
  session SQL execution performs the final executor DTO adaptation.
* Updated non-SQL session execution so shared prepared execution dispatch reads
  grouped/load/delete shape from the prepared plan instead of the original
  query intent.
* Split fluent ranked terminal session helpers into explicit top/bottom
  methods, removing the last direction flag branch from non-SQL fluent session
  adapters.

  audit. The SQL lowering audit intentionally stops at syntax-to-query

IcyDB 0.51 Design

Status: Active
Target: 0.51 minor line

Theme

0.51 engine contract stabilization.

Intent

0.51 focuses on stabilizing the core execution contracts of the engine prior to the 1.0 release.

Following the structural simplification work in 0.50, this line freezes the behavioral guarantees between planner, executor, continuation engine, numeric core, and storage.

The goal is to ensure:

deterministic query behavior

stable pagination semantics

consistent numeric evaluation

stable storage invariants

clear semantic authority ownership

This line introduces no new major query features.

Core Contracts to Stabilize

The following contracts must be explicitly defined and locked.

Planner → Executor Contract

Defines the execution surface between planning and runtime.

Guarantees:

identical logical query → identical executable plan

executor does not re-derive planning decisions

execution shape variants are deterministic

pushdown capabilities are explicitly signaled

Result:

Executable plans become stable artifacts.

Deterministic Ordering Contract

Defines total ordering behavior for queries that involve ordering or pagination.

Guarantees:

queries using ORDER produce deterministic row ordering

queries using LIMIT/OFFSET or continuation operate on a total order

non-unique ORDER keys receive a deterministic tie-breaker

Result:

Pagination and explain output remain stable.

Continuation Envelope Contract

Defines pagination and resume semantics.

Guarantees:

identical query + identical cursor → identical next page

continuation tokens remain monotonic

continuation advancement rules are deterministic

cursor envelopes remain stable across compatible versions

Result:

Pagination behavior becomes client-safe.

Projection Contract

Defines projection behavior across all execution paths.

Guarantees:

projection does not alter execution semantics

covering projections avoid row materialization

row projections remain consistent across access paths

Result:

Projection behavior is deterministic and predictable.

Aggregate Strategy Contract

Defines aggregation execution strategies.

Guarantees:

identical query → identical aggregation strategy

grouped vs scalar execution decisions remain deterministic

aggregate fast paths are explicitly defined

Result:

Aggregation behavior becomes stable and predictable.

Numeric Core Contract

Defines numeric evaluation rules.

Guarantees:

numeric comparison rules are unified

numeric ordering is consistent across planner and executor

aggregation arithmetic uses a single numeric authority

Result:

Numeric behavior becomes consistent across the engine.

Storage Invariant Contract

Defines persistence guarantees.

Guarantees:

index ordering is stable

key encoding rules are fixed

fingerprint and hashing behavior remains deterministic

row identity invariants remain stable

Result:

Storage behavior becomes stable across releases.

Semantic Authority Contract

Defines ownership of semantic rules.

Guarantee:

Each rule has exactly one authoritative owner.

Examples:

Concern	Authority
predicate legality	planner
aggregation legality	planner
continuation semantics	continuation engine
execution routing	planner
operator execution	executor

Result:

No duplicated policy logic across layers.

Slices
Slice A — Planner/Executor Contract Freeze

Tasks:

define executable plan invariants

document execution shapes

remove executor-side planning logic

add plan shape regression tests

Exit criteria:

identical query → identical executable plan

explain output deterministic

Slice B — Deterministic Ordering Contract

Tasks:

define total ordering rules

enforce deterministic tie-breakers where required

validate ORDER + LIMIT behavior

Exit criteria:

ordering deterministic across executions

pagination behavior stable

Slice C — Continuation Envelope Stability

Tasks:

document cursor envelope format

define continuation advancement rules

enforce monotonic pagination semantics

Exit criteria:

continuation tokens deterministic

pagination tests stable

Slice D — Projection Completion

Tasks:

unify projection paths

finalize covering index projection

eliminate partial projection logic

Exit criteria:

projection semantics stable across access paths

Slice E — Aggregate Strategy Stabilization

Tasks:

define aggregate execution strategies

lock grouped vs scalar aggregation behavior

finalize aggregate fast paths

Exit criteria:

aggregation strategy deterministic

Slice F — Numeric Core Unification

Tasks:

unify numeric comparison rules

consolidate numeric evaluation logic

eliminate duplicated numeric semantics

Exit criteria:

numeric behavior consistent across planner and executor

Slice G — Storage Invariant Freeze

Tasks:

document index ordering guarantees

lock key encoding rules

Exit criteria:

Tasks:

Exit criteria:

Non-Goals

0.51 does not include:

new query language features

join support

window functions

complex SQL parsing

storage format changes

These are postponed until after 1.0 stabilization.

Success Criteria

0.51 is complete when:

planner/executor contracts are stable

pagination semantics are deterministic

projection behavior is unified

aggregation strategies are deterministic

numeric evaluation is consistent

storage invariants are fixed

semantic authority duplication is eliminated

Expected Outcome

After 0.51:

execution behavior becomes deterministic

pagination becomes client-safe

explain output becomes stable

future refactors become safe

the engine is structurally ready for 1.0

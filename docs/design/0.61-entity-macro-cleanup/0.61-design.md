IcyDB 0.60 Design — Entity Macro Cleanup

Status: Proposed (v1)
Theme: Entity surface compression + runtime consolidation + macro output minimisation

1. Problem Statement

The current entity macro expansion produces:

A. Structural duplication

You generate three parallel representations:

Schema layer

Entity::new(...)

FieldList, Field, Value, Item, etc.

Model layer

FieldModel constants

EntityModel

static arrays

Runtime type

struct ComplexEntity

trait impls

visitor plumbing

sanitize/validate

view system

These are logically the same data repeated 3 times, with different shapes.

B. Monomorphisation explosion

Even after your 0.58/0.59 work:

CoreAsView<T>

CoreUpdateView<T>

Visitor traversal

perform_visit<T>

patch merge logic

list/option/enum handling

All still instantiate per field type, per entity.

With 37 fields:
→ You are generating hundreds of monomorphized instantiations.

C. View system duplication

You generate:

Entity

EntityView

EntityCreate

EntityUpdate

Each:

repeats the full field graph

triggers separate generic instantiations

multiplies WASM size significantly

D. Visitor pattern cost
perform_visit(visitor, &self.field, "field")

This:

is generic per field type

creates dispatch bloat

repeats string literals

expands linearly with field count

E. Const model duplication

You emit:

const __MODEL_FIELD_X: FieldModel
const __MODEL_FIELDS: [FieldModel; N]
const __ENTITY_MODEL: EntityModel

But you also already emit:

Entity::new(... FieldList ...)

→ This is duplicated semantic authority.

2. Design Goals
Primary

Reduce WASM size (major goal)

Eliminate redundant representations

Minimise macro output

Push logic into shared runtime kernels

Secondary

Preserve:

type safety at API boundary

schema expressiveness

Improve:

compile times

readability of expansion

3. Core Strategy
3.1 Collapse to Single Source of Truth

Keep ONLY one structural representation:

👉 EntityModel (or equivalent structural schema)

Remove:

FieldModel duplication

redundant Entity::new(...) schema tree

New rule:

Macro emits ONE structural model → everything derives from it.

3.2 Runtime-driven instead of macro-driven

Instead of generating:

visitors

merge logic

view conversions

👉 Move to:

shared runtime kernels operating on structural metadata

4. Key Architectural Changes
4.1 Field Table → Runtime Field Descriptor

Replace:

const __MODEL_FIELDS: [FieldModel; 37]

with:

static ENTITY_DESCRIPTOR: EntityDescriptor = EntityDescriptor {
    fields: &'static [FieldDescriptor],
};

Where FieldDescriptor contains:

name

kind (enum)

cardinality

sanitizer refs

offsets (optional, see below)

4.2 Offset-based field access (critical)

Instead of per-field generic code:

👉 generate field offsets once

FieldDescriptor {
    name: "string_test",
    offset: offset_of!(ComplexEntity, string_test),
    kind: FieldKind::Text,
}

Then runtime:

fn get_field_ptr(base: *const u8, field: &FieldDescriptor) -> *const u8

This enables:

generic traversal

generic sanitization

generic serialization

no per-field monomorphisation

4.3 Kill the Visitor Explosion

Replace:

perform_visit(visitor, &self.field, "field")

with:

for field in ENTITY_DESCRIPTOR.fields {
    runtime_visit(field, self_ptr, visitor);
}

→ single monomorphic loop

4.4 Remove CoreAsView / CoreUpdateView monomorphisation
Replace with:

Value-level conversion layer

Instead of:

CoreAsView<T>

Use:

fn to_value(ptr, FieldKind) -> Value
fn from_value(ptr, FieldKind, Value)

Then:

views become Value-backed DTOs

not type-specialised structs

Result

You eliminate:

per-field generic impls

per-type conversion chains

duplicate struct generation

4.5 Collapse View Types

Current:

Entity

EntityView

EntityCreate

EntityUpdate

Replace with:
struct EntityRow(ValueMap)
struct EntityPatch(ValuePatch)

Optional:

typed façade at API boundary only

Key rule:

Views are data, not types

4.6 Merge logic → structural engine

Replace:

CoreUpdateView::merge(&mut next.field, patch.field)

with:

apply_patch(entity_ptr, descriptor, patch)

Single monomorphic engine.

4.7 Sanitization

Already close to correct:

TypeSanitizer::new(PATH)
Extend:

Move all sanitization into:

run_sanitizers(entity_ptr, descriptor)

No per-field codegen.

4.8 Schema registration

Keep:

schema_write().insert_node(...)

But input should be:

👉 the same EntityDescriptor

5. Macro Output (Target State)
Current: ~1500–3000 LOC per entity
Target: ~50–150 LOC per entity

Example:

pub struct ComplexEntity {
    // fields
}

impl EntityKind for ComplexEntity {
    const ENTITY_TAG: EntityTag = ...
}

static DESCRIPTOR: EntityDescriptor = build_descriptor!(
    ComplexEntity {
        id: Ulid,
        string_test: Text,
        ...
    }
);

Everything else → runtime.

6. Expected Wins
WASM size
Area	Reduction
Visitor code	~80–90%
View system	~70–90%
Merge logic	~80%
Field duplication	~50%
TOTAL	30–60% overall entity-related reduction
Compile time

drastically reduced monomorphisation

fewer trait instantiations

Runtime

slightly more dynamic (pointer + descriptor)

but:

better cache locality

fewer branches

often faster in WASM

7. Migration Plan (Practical)
Phase 1 — Non-breaking

introduce EntityDescriptor

implement runtime traversal + sanitize + patch

keep old macro output

Phase 2 — Switch internals

visitor → descriptor loop

sanitize → descriptor loop

patch → descriptor loop

Phase 3 — Kill duplication

remove:

FieldModel

generated visitor impls

per-field merge logic

Phase 4 — View collapse

replace view structs with Value-based

optional typed façade layer

8. Risks
1. Loss of static typing in views

Mitigation:

keep typed API layer only

runtime uses Value

2. Unsafe pointer access (offsets)

Mitigation:

macro generates offsets → safe

no user exposure

3. Debuggability

Mitigation:

descriptor includes names + metadata

better introspection

9. Guiding Principle (0.60)

Entities should describe data, not generate behavior.

Right now:

macro generates behavior explosion

Target:

macro generates data description

runtime executes behavior

10. One-line Summary

0.60 replaces per-entity code generation with a single shared runtime engine driven by structural descriptors, eliminating monomorphisation and collapsing schema/model/view duplication.

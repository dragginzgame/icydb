# IcyDB 0.151 - Accepted Expression Index Contracts Status

## Current State

0.150 closed accepted field-path index authority and narrowed expression indexes to
named generated compatibility wrappers. 0.151 starts from that point and removes the
remaining generated-runtime expression-index authority.

## Target

Runtime expression index planning, explain, fingerprints, writes, uniqueness
validation, and recovery are backed only by accepted expression index contracts after
schema acceptance.

## Work Queue

- Define accepted expression index contract types.
- Derive expression contracts during schema proposal and acceptance.
- Persist expression contracts in accepted schema snapshots.
- Expose expression contracts through `SchemaInfo` and `VisibleIndexes`.
- Move planner, explain, cache identity, and reranking to accepted expression
  contracts.
- Move expression write keys, filtered membership, uniqueness validation, and recovery
  to accepted expression contracts.
- Delete production generated expression candidate wrappers.
- Add guards for generated expression index runtime authority.

## Deferred

- New expression operations.
- SQL DDL for expression indexes.
- Arbitrary scalar-expression index keys.
- Backward compatibility fallbacks for older internal accepted-schema snapshots.

# 0.147 Live Schema Authority Status

## Status

In progress. The 0.147 line is being closed as an exact-match and
generated-compatible runtime-guard phase, not as the first additive schema
transition phase.

The current repository has accepted schema metadata authority plus
generated-compatible accepted row-layout guards for prepared execution, typed
saves, and SQL `INSERT` / `UPDATE`. Runtime field decode and final write slot
selection are still generated-layout based, so non-exact schema transitions
remain rejected.

## Already Complete Before 0.147

0.146 completed the accepted-schema metadata foundation:

* persisted schema snapshots are stored in the schema store
* startup reconciliation writes first snapshots and rejects unsupported drift
* accepted schema snapshots are fallibly constructed and integrity checked
* accepted schema feeds SQL metadata, SQL admission, planning field types,
  cache fingerprints, and prepared static execution metadata
* nested leaf metadata is persisted and used for field-path inference/admission
* schema-store and accepted-schema footprint metrics are reported
* schema transition rejection diagnostics are concise enough for local upgrade
  testing

## 0.147.0 Closeout Already Shipped

The 0.147.0 closeout patch shipped:

* `scripts/dev/sql.sh --upgrade` for stable-memory-preserving local upgrades
* startup schema reconciliation before row replay/index rebuild when recovery
  is required
* concise row-layout transition diagnostics with field metadata for the first
  changed or extra field-slot fact

These changes support 0.147 work but do not complete live schema authority.

## 0.147.1 through 0.147.5 Progress

Later 0.147 patches added:

* transition-policy metrics separate from reconciliation metrics
* accepted row-layout runtime descriptors for generated-compatible layouts
* prepared execution handoff through descriptor-derived authority
* typed save and SQL `INSERT` / `UPDATE` guards before generated write codecs
  run
* trait-owned persisted-row field codecs, with removed `#[icydb(...)]` row
  hints rejected at compile time
* runtime-only `Value` boundaries documented separately from typed persistence
* text and blob length metadata carried through schema snapshots, with
  intentionally unbounded text/blob schema fields required to say `unbounded`
* accepted-schema `DESCRIBE` / `SHOW COLUMNS` field rendering through one
  persisted-field format, including explicit text/blob length contracts
* descriptor-owned SQL write slot and field-kind lookup before structural
  patches hand slots to generated-compatible codecs
* descriptor-owned generated-compatible row-layout primary-key slot lookup
* schema-runtime-owned generated-compatible layout validation before executor
  row decode consumes accepted descriptors
* accepted primary-key name and physical slot carried directly by the runtime
  descriptor before generated-compatible row decode consumes the layout
* SQL compiled-command cache miss compilation through the descriptor-derived
  authority paired with accepted `SchemaInfo`
* session-owned structural patch construction through accepted row-layout
  descriptors for callers that can build patches with a `DbSession`
* session-level structural write tests migrated to the session-owned patch
  constructor, leaving generated-model patch helpers only in low-level
  executor/data compatibility coverage

## Remaining Correctness Boundary

The main unresolved boundary is:

```text
accepted persisted schema
  -> transition decision
  -> live row-layout runtime descriptor
  -> row decode/write behavior
```

Until that is complete:

* generated row layout remains the runtime decode/write authority
* accepted schema remains metadata, planning, and compatibility-check authority
* schema evolution remains rejected except exact match
* field-level database defaults are not applied to stored rows

## Deferred Beyond 0.147 Unless Explicitly Pulled In

* broad migration scripting
* multi-version row decoding
* per-row schema versions
* online chunked row rewrites
* field removal with physical slot compaction
* relation-target migration

## Validation To Maintain

Minimum local validation for 0.147 schema patches:

```bash
cargo fmt --all
cargo test -p icydb-core db::schema --features sql -- --nocapture
cargo test -p icydb-core db::commit --features sql -- --nocapture
cargo test -p icydb-core db::session --features sql -- --nocapture
cargo check -p icydb-core --tests --features sql
cargo clippy -p icydb-core --all-targets --features sql -- -D warnings
make check-invariants
git diff --check
```

Use `scripts/dev/sql.sh --upgrade` for manual canister upgrade tests that must
preserve stable memory.

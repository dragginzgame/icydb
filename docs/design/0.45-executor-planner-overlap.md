# 0.45 - Executor Planner Overlap Cleanup

## Status

Draft  
Target: `0.45`  
Dependency: `0.45` feature cleanup plan  
Last updated: 2026-03-07

---

## 1. Objective

Reduce planner/executor overlap so semantics are planner-owned and executor
behavior is runtime-only.

This document is a focused companion to `0.45-feature-cleanup.md`.

---

## 2. Problem Statement

Overlap risk appears when planner and executor both:

- decide semantic eligibility
- derive equivalent shape policy independently
- encode comparable error meaning with different boundaries
- duplicate normalization and fallback gates

This creates drift risk, unclear ownership, and harder regression debugging.

---

## 3. Authority Boundary

Required ownership model:

- planner owns semantic decisions and admissibility
- route/access own shape + capability projection
- executor consumes canonical outputs and enforces runtime invariants

Executor must not re-plan semantics.

---

## 4. Overlap Cleanup Tracks (`0.45`)

| Track | Layer Boundary | Cleanup Action | Status |
|---|---|---|---|
| Semantic gate deduplication | planner -> executor | remove executor-side semantic re-derivation where planner authority exists | open |
| Shape contract consolidation | route/access -> executor | consume one canonical route-shape descriptor in executor hot paths | open |
| Error boundary normalization | planner/executor/storage | enforce canonical error mappings per layer authority | open |
| Explain/trace parity for overlap fixes | executor/diagnostics | keep explain + trace outputs aligned after overlap removals | open |
| Structural anti-regression guards | tests | block reintroduction of planner-like semantics in executor modules | open |

---

## 5. Implementation Notes

- preserve behavior; this is a cleanup pass, not feature expansion
- remove compatibility shims instead of layering new wrappers
- keep EXPLAIN DTO construction immutable (`ExplainPlan::new(...)`)
- document any intentional deferred overlap with explicit owner+rationale

---

## 6. Validation

Minimum validation for each touched overlap area:

- semantic parity tests (before/after behavior equivalence)
- boundary tests confirming planner ownership and executor consumption
- explain/trace parity tests for changed paths
- clippy + workspace tests remain warning-clean

---

## 7. Exit Criteria

Complete when:

- no known planner-semantic re-derivation remains in executor runtime paths for touched tracks
- overlap tracker rows are `complete` or explicitly `deferred` with rationale
- structural tests prevent overlap regressions from silently returning

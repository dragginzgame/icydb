# 0.154 Audit Readiness

## Supported Developer Surface

0.154 supports one runtime schema mutation path:

- startup reconciliation may add one non-unique field-path secondary index for
  one entity when accepted-before and accepted-after snapshots differ only by
  that index
- the path is accepted-snapshot owned; generated `EntityModel` and `IndexModel`
  metadata remains proposal, reconciliation, model-only, or test-only
- SQL DDL remains out of scope

## Required Evidence For Audits

Constructive audits should verify this path through these boundaries:

- admission: `SchemaMutationSupportedExecutionPath::field_path_index_addition`
  admits only the single non-unique field-path index-add shape
- execution: startup reconciliation scans accepted rows, builds
  `SchemaMutationRunnerInput`, and invokes `SchemaFieldPathIndexRunner`
- publication: `StartupFieldPathRebuildGate` and
  `StartupFieldPathPublicationDecision` gate physical work and accepted-snapshot
  publication
- visibility: runner reports are publishable only after target-scoped physical
  validation, runtime invalidation, physical-store publication, and accepted
  snapshot handoff

## Still Blocked

- expression-index rebuild
- secondary-index drop cleanup
- unique-index rebuild
- multiple index batches
- full row rewrites
- SQL DDL
- online or concurrent rebuild
- generalized recovery or resume

## Audit Target

Audits should now evaluate whether this one supported path remains narrow,
accepted-schema-owned, deterministic, and fail-closed at each boundary instead
of treating every rebuild-required publication path as absent.

# 0.49 Status (as of 2026-03-11, active 0.49.3 follow-through)

## 0.49 Scope

This status tracks `0.49` execution and observability stabilization work defined
in:

- `docs/design/0.49-execution-observability-hardening/0.49-design.md`

`0.49` focuses on architecture hardening and observability consistency:

- executor layer boundary stabilization (`scan -> pipeline -> aggregate -> terminal`)
- continuation compatibility fail-closed behavior
- additive EXPLAIN execution metadata
- EXPLAIN subsystem internal ownership stabilization
- additive executor row-flow metrics
- contract ownership cleanup under owner-named executor modules
- execution diagnostics-node correlation groundwork
- pipeline stage-descriptor cleanup to reduce orchestration branching
- EXPLAIN execution schema/parity hardening follow-through (`0.49.3`)

No new query-surface language features are in scope.

# 0.208 Opening Audit: Exact Composite Contracts

Status: Superseded by implementation evidence in the 0.208 design

## Opening Finding

Before 0.208, IcyDB represented custom records, tuples, and newtypes as a
generic `Structured { queryable }` field kind. The persisted contract did not
contain their nominal identity or complete shape, so accepted schema could not
independently prove what generated codecs were allowed to decode.

The current direction is a hard cut to exact nominal composites. No open-ended
persisted field kind is in scope.

## Authority Conflict Map

| Concern | Opening owner | Competing owner | Required owner |
| --- | --- | --- | --- |
| Composite identity | Generated Rust type | Generic accepted kind | Accepted composite catalog |
| Record members | Generated codec | Runtime map shape | Accepted composite definition |
| Tuple arity | Generated codec | Runtime list length | Accepted composite definition |
| Newtype payload | Generated codec | Inner runtime value | Accepted composite definition |
| Queryability | Boolean on generic kind | Planner capability checks | Derived accepted-schema capability |
| Persisted decode | Generated kind reconstruction | Accepted row layout | Accepted field plus composite catalog |
| Transition equality | Coarse field-kind equality | Generated proposal comparison | Accepted nominal codec and exact shape |

## Legacy Shapes To Remove

- `FieldKind::Structured { queryable }`;
- `AcceptedFieldKind::Structured { queryable }`;
- the accepted-catalog structured wire tag;
- field-type and capability branches driven by the queryability boolean;
- generated-model reconstruction after accepted schema exists;
- tests that accept arbitrary record/list payloads under the generic kind; and
- documentation describing structured persistence without an exact contract.

## Retained Complexity

The internal runtime value union and structural codecs remain necessary. They
carry values through expression, row, enum, collection, and composite
boundaries. They are not persisted schema authority and do not justify a loose
field kind.

Likewise, `RuntimeValueKind::Structured { queryable }` may remain as a narrow
runtime conversion classification. It must not be serialized as accepted
schema, used to reconstruct a composite definition, or exposed as the public
field contract.

## Opening Proof Requirements

Implementation must prove:

1. accepted schema can describe every supported record, tuple, and newtype
   without consulting generated models at runtime;
2. nominal identity and complete shape survive persistence and recovery;
3. malformed shape, unknown ID, cycles, and excessive depth fail closed;
4. schema reconciliation compares exact composite facts;
5. SQL and fluent capability decisions consume accepted schema;
6. old structured metadata and decode paths are absent; and
7. the hard cut has no alias, compatibility decoder, or legacy fallback.

## Opening Verdict

The generic structured field contract did not provide sufficient schema
authority. 0.208 should replace it outright with one exact composite catalog
and one ID-backed accepted field representation.

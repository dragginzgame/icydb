# 0.196 Optional SQLite Comparison Harnesses

Date: 2026-07-05

## Purpose

These harnesses turn the SQLite-comparison audit finding for differential
evidence into opt-in test artifacts. They compare overlapping SQL surfaces
between IcyDB and SQLite STRICT tables without making SQLite a production
dependency or a headline benchmark.

The preferred 0.196 reference path is the integration SQL perf-audit comparison:
it seeds the same main `PerfAudit*` table data into both the `sql_perf` canister
and a local SQLite database, then compares deterministic result signatures and
records IcyDB route facts beside SQLite `EXPLAIN QUERY PLAN` output. Its
default mode is a correctness report: it records common successes, IcyDB
planner failures, and result-signature mismatches without failing the ignored
test unless strict mode is explicitly enabled.

The smaller core harness remains useful as a fast local smoke test for the
native session SQL surface.

## Scope

- Both harnesses run only when ignored tests are explicitly requested.
- Both use `ICYDB_SQLITE3` to override the SQLite CLI path.
- Both write JSON and Markdown artifacts under `/tmp/icydb-sqlite-comparison`
  by default.
- The core harness covers SQLite default mode, WAL `synchronous=NORMAL`, WAL
  `synchronous=FULL`, and `synchronous=OFF` as an unsafe speed reference.
- The integration SQL perf-audit harness mirrors the main `PerfAuditUser`,
  `PerfAuditAccount`, `PerfAuditBlob`, and `PerfAuditToken` rows and indexes
  where SQLite can express the same shape.
- The integration harness records route family, route outcome, limit-stop
  facts, IcyDB instruction counters, store/index counters, result signatures,
  exact SQLite `EXPLAIN QUERY PLAN` rows, coarse SQLite plan summaries, plan
  alignment labels, IcyDB failure diagnostics, and optional SQLite CLI timing
  samples for the selected matrix scenarios.
- The integration harness default scenario set is the broad SQLite-compatible
  deterministic matrix subset for the mirrored audit tables. It excludes
  storage-mirror tables, metadata commands, raw blob payload projections,
  `OCTET_LENGTH`, selected boolean columns with incompatible display semantics,
  and full-entity `*` token projections.
- `ICYDB_SQL_PERF_SQLITE_KEYS` switches the integration harness back to an
  exact focused scenario-key list.
- `ICYDB_SQL_PERF_SQLITE_STRICT=1` makes the integration harness fail if any
  selected scenario has an IcyDB failure or common-success signature mismatch.
- `ICYDB_SQL_PERF_SQLITE_TIMING_SAMPLES=N` enables exploratory repeated SQLite
  CLI timing samples. The default is `0` so correctness runs are not dominated
  by CLI timing work.

## Non-Goals

- No production route or executor behavior changes.
- No cursor-token or persisted-format changes.
- No public API changes.
- No claim that native SQLite CLI timings are equivalent to IC stable-memory
  costs.
- No claim that SQLite and IcyDB plans should be textually identical.
  SQLite `EXPLAIN QUERY PLAN` is used as a planner-shape review signal and
  correctness cross-check companion, not as a required execution blueprint.
- No use of unsafe SQLite settings as the main performance baseline.

## Manual Command

Install the SQLite CLI with `make install-dev` or install `sqlite3` manually.
`make update-dev` reports when the CLI is missing but does not use sudo.

Core native-session smoke harness:

```bash
cargo test -p icydb-core --all-features sqlite_comparison_harness_runs_when_sqlite3_available -- --ignored --nocapture
```

Optional artifact path override:

```bash
ICYDB_SQLITE_COMPARISON_OUTPUT_STEM=/tmp/icydb-sqlite-comparison/custom cargo test -p icydb-core --all-features sqlite_comparison_harness_runs_when_sqlite3_available -- --ignored --nocapture
```

Integration SQL perf-audit mirror harness:

```bash
cargo test -p icydb-testing-integration --test sql_perf_matrix_audit sql_perf_generated_matrix_compares_sqlite_reference_fixture -- --ignored --nocapture
```

Strict correctness gate for a known-good focused subset:

```bash
ICYDB_SQL_PERF_SQLITE_STRICT=1 \
ICYDB_SQL_PERF_SQLITE_KEYS=user.select.pk.all.pk_asc.limit1,token.collection_stage_id.prefixed_stage_range.page_only.limit50 \
cargo test -p icydb-testing-integration --test sql_perf_matrix_audit sql_perf_generated_matrix_compares_sqlite_reference_fixture -- --ignored --nocapture
```

Optional artifact path and scenario-key overrides:

```bash
ICYDB_SQL_PERF_SQLITE_OUTPUT_STEM=/tmp/icydb-sqlite-comparison/sql_perf_custom \
ICYDB_SQL_PERF_SQLITE_KEYS=user.select.pk.all.pk_asc.limit1,token.collection_stage_id.branch_set.page_only.limit50 \
cargo test -p icydb-testing-integration --test sql_perf_matrix_audit sql_perf_generated_matrix_compares_sqlite_reference_fixture -- --ignored --nocapture
```

Default integration artifacts:

- `/tmp/icydb-sqlite-comparison/sql_perf_audit_sqlite_comparison.json`
- `/tmp/icydb-sqlite-comparison/sql_perf_audit_sqlite_comparison.md`

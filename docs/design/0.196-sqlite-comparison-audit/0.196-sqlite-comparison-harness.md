# 0.196 Optional SQLite Comparison Harness

Date: 2026-07-05

## Purpose

This harness turns the SQLite-comparison audit finding for differential
evidence into a small, opt-in test artifact. It compares a narrow overlapping
SQL surface between IcyDB's native session fixture and SQLite STRICT tables.

The harness is intentionally not a headline benchmark. SQLite runs through the
local `sqlite3` CLI and is not using Internet Computer stable memory, while
IcyDB runs against the in-process native fixture. The output is useful for
catching result drift and for exploratory timing context before a deeper
canister/stable-memory benchmark.

## Scope

- Runs only when the ignored test is explicitly requested.
- Uses `ICYDB_SQLITE3` to override the SQLite CLI path.
- Writes JSON and Markdown artifacts under `/tmp/icydb-sqlite-comparison` by
  default.
- Covers SQLite default mode, WAL `synchronous=NORMAL`, WAL
  `synchronous=FULL`, and `synchronous=OFF` as an unsafe speed reference.
- Compares deterministic result signatures for primary-order, filtered-order,
  range-order, count, and missing-primary-key scenarios.

## Non-Goals

- No production route or executor behavior changes.
- No cursor-token or persisted-format changes.
- No public API changes.
- No claim that native SQLite CLI timings are equivalent to IC stable-memory
  costs.
- No use of unsafe SQLite settings as the main performance baseline.

## Manual Command

Install the SQLite CLI with `make install-dev` or install `sqlite3` manually.
`make update-dev` reports when the CLI is missing but does not use sudo.

```bash
cargo test -p icydb-core --all-features sqlite_comparison_harness_runs_when_sqlite3_available -- --ignored --nocapture
```

Optional artifact path override:

```bash
ICYDB_SQLITE_COMPARISON_OUTPUT_STEM=/tmp/icydb-sqlite-comparison/custom cargo test -p icydb-core --all-features sqlite_comparison_harness_runs_when_sqlite3_available -- --ignored --nocapture
```

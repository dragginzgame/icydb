# 0.138.8 Aggregate Target Sealing Addendum

## Status

Implemented during the SQL aggregate lowering cleanup follow-up.

## Motivation

The `0.138` aggregate lowering split separates raw SQL aggregate terminal data
from prepared runtime strategy:

```text
SqlGlobalAggregateTerminal
  -> PreparedSqlScalarAggregateStrategy
  -> StructuralAggregateTerminal
```

That boundary is now guarded by tests, and runtime executor mapping is owned by
the strategy layer. The motivating weak point was that the prepared strategy
stored aggregate target identity as two independent optional fields.

Old shape:

```rust
PreparedSqlScalarAggregateStrategy {
    target_slot: Option<FieldSlot>,
    input_expr: Option<Expr>,
    filter_expr: Option<Expr>,
    distinct_input: bool,
    descriptor_shape: PreparedSqlScalarAggregateDescriptorShape,
}
```

The constructors preserved this invariant:

```text
Rows:  target_slot = None, input_expr = None
Field: target_slot = Some(_), input_expr = None
Expr:  target_slot = None, input_expr = Some(_)
```

That was behavior-safe because construction was centralized in `strategy.rs`,
but the representation could still express invalid states internally. The
cleanup replaced that illegal-state surface with a private sealed target enum.

## Goal

Make the aggregate target shape structurally unrepresentable instead of
constructor-enforced.

Implemented shape:

```rust
enum AggregateTarget {
    Rows,
    Field(FieldSlot),
    Expr(Expr),
}

PreparedSqlScalarAggregateStrategy {
    target: AggregateTarget,
    filter_expr: Option<Expr>,
    distinct_input: bool,
    descriptor_shape: PreparedSqlScalarAggregateDescriptorShape,
}
```

## Non-Goals

Do not change:

* SQL aggregate behavior
* public response shapes
* aggregate output labels
* `HAVING` semantics
* `DISTINCT` semantics
* executor terminal kinds
* grouped aggregate lowering
* persisted formats

This is an internal representation cleanup only.

## Ownership Rules

### `terminal.rs`

Keeps owning normalized SQL aggregate terminal facts:

* aggregate kind
* row, field, or expression input
* filter expression
* distinct flag

It must not map directly to executor terminal kinds.

### `strategy.rs`

Owns prepared runtime meaning:

* target resolution
* target shape sealing
* descriptor family
* executor terminal construction

This remains the only place allowed to convert aggregate target and descriptor
state into `StructuralAggregateTerminal`.

### `command/`, `projection/`, `lowering/`, `grouped/`

May carry prepared strategies or terminal lists as lowering artifacts, but must
not infer executor terminal behavior from raw terminal kind.

## Implemented Patch Shape

1. Added private `AggregateTarget` to `strategy.rs`.
2. Replaced `target_slot: Option<FieldSlot>` and `input_expr: Option<Expr>` with
   `target: AggregateTarget`.
3. Updated constructors:
   * `for_rows(...)`
   * `for_field_target(...)`
   * `for_expression_input(...)`
4. Preserved existing accessor API:
   * `target_slot() -> Option<&FieldSlot>`
   * `input_expr() -> Option<&Expr>`
   * `projected_field() -> Option<&str>`
5. Updated `into_executor_terminal(...)` to destructure `AggregateTarget`.
6. Extended existing lowering tests so each admitted row, field, and expression
   aggregate shape still projects the same visible strategy facts.

```bash
git diff --check
```

## Expected Outcome

After this cleanup, the prepared SQL scalar aggregate strategy cannot represent:

* both field and expression targets at once
* missing target data for field/expression aggregates
* expression payloads on row-count aggregates

The strategy layer remains the single runtime authority, and the terminal layer
remains a raw normalized SQL lowering artifact.

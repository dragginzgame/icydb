# 0.65 Design: Canonical Row Invariant (Hard-Cut)

## Preamble

* scope: persisted row format invariants and commit-time validation
* objective: eliminate structural ambiguity in persisted rows
* policy: **hard-cut — no backward compatibility layer**
* method: enforce canonical row shape at all ingress points

---

# 1) Core Invariant

## 1.1 Row Shape

For every persisted row:

```text
slot_count == model.fields().len()
```

* every declared field has exactly one slot
* no missing spans
* no partial rows

---

## 1.2 Slot Validity

For every slot:

```text
slot payload is present AND decodable
```

Valid slot states:

```text
Some(Value::X)
Some(Value::Null)
```

Invalid:

```text
None (absent)
malformed payload
non-decodable bytes
```

---

## 1.3 Canonical Encoding

All scalar slots must be encoded using:

```text
SCALAR_SLOT_PREFIX + canonical slot envelope
```

No acceptance of:

* raw CBOR scalar payloads
* pre-envelope formats
* implicit representations

---

# 2) Removal of Compatibility Paths

## 2.1 Deleted Behavior

The following are explicitly removed:

* read-time normalization of scalar slot payloads
* acceptance of raw CBOR scalar bytes
* tolerance for missing slot spans
* structural “best effort” decoding

---

## 2.2 Deleted Functions

Remove:

```text
normalize_legacy_scalar_slot_payload_bytes(...)
normalize_legacy_scalar_fields(...)
```

Remove all call sites.

---

## 2.3 Reader Behavior

`StructuralSlotReader` must:

* assume canonical format
* fail immediately on:

  * missing slot
  * malformed payload
  * incompatible format

No fallback logic.

---

# 3) Commit Boundary Enforcement

## 3.1 Validation Rule

Extend commit validation:

```rust
for slot in 0..model.fields().len() {
    match slots.get_value(slot)? {
        Some(_) => {}        // valid
        None => {
            return Err(
                InternalError::store_corruption(
                    format!("missing slot {slot} in persisted row")
                )
            );
        }
    }
}
```

---

## 3.2 Validation Phases

Commit validation becomes:

```text
Phase 1: structural decode (StructuralSlotReader)
Phase 2: primary key validation
Phase 3: full slot presence check (new)
Phase 4: per-slot decode via field contract
```

Failure at any phase aborts commit.

---

## 3.3 Mutation Path Guarantee

All mutation paths must produce:

```text
fully populated slot array
```

Specifically:

* insert
* update
* replace
* structural mutation input

must all emit canonical rows.

---

# 4) Schema Requirements

## 4.1 Field Completeness

Every field must resolve to a value at commit:

* explicit value
* or explicit `Value::Null`

---

## 4.2 Option Fields

```text
Option<T>::None → Value::Null
```

Never represented as missing slot.

---

## 4.3 Defaulted Fields

If defaults exist:

* applied before commit
* encoded as concrete value or null

---

## 4.4 Schema Evolution Constraint

When adding a field:

* it must be:

  * optional (nullable), or
  * have a default

Otherwise:

> existing rows become invalid and must be rewritten before use

---

# 5) Storage Policy

## 5.1 No Legacy Tolerance

Rows that violate canonical format:

* are rejected at commit
* are rejected at read

No automatic repair.

---

## 5.2 Data Migration

Out-of-band only:

* manual rewrite
* migration tool
* state reset

Not handled by runtime.

---

# 6) Runtime Simplification

## 6.1 StructuralSlotReader

Can assume:

```text
- slot exists
- payload exists
- payload decodes
```

Remove:

* Option<Option<...>> semantics
* absent-slot branching

---

## 6.2 Executor / Planner

May assume:

```text
all fields are structurally valid
```

No defensive decode checks required beyond contract decode.

---

# 7) Error Model

## 7.1 Failure Classes

Introduce or reuse:

* `store_corruption` → missing slot / malformed row
* `serialize_incompatible_persisted_format` → wrong encoding
* `serialize_corruption` → decode failure

---

## 7.2 No Silent Coercion

Disallowed:

```text
None → Value::Null
legacy → canonical
```

All violations must error.

---

# 8) Testing Requirements

## 8.1 Positive Tests

* full row with all slots → accepted
* null values → accepted
* all mutation modes produce full slots

---

## 8.2 Negative Tests

* missing slot → rejected at commit
* raw CBOR scalar payload → rejected
* malformed scalar → rejected
* truncated span table → rejected

---

## 8.3 Regression Tests

* previously accepted malformed non-indexed scalar → rejected
* legacy-format row → rejected (no normalization)

---

# 9) Expected Outcomes

## 9.1 Guarantees

After 0.65:

* no invalid rows can enter storage
* all rows are fully shaped
* all fields are decodable

---

## 9.2 Simplifications

* remove normalization logic
* reduce branching in readers
* remove compatibility code paths

---

## 9.3 Tradeoff

* breaking change for existing data
* requires migration or reset

---

# 10) Summary

0.65 introduces a **strict canonical row invariant**:

```text
no missing slots
no legacy formats
no fallback decoding
```

This is a **hard boundary**:

* simplifies runtime
* strengthens correctness
* eliminates structural ambiguity

at the cost of:

* requiring clean data going forward

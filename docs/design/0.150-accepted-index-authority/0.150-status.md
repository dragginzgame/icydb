# IcyDB 0.150 - Status

## Current State

Started.

0.150 starts from the 0.149 closeout: accepted row contracts now govern runtime
row decode, row emission, save validation, predicate/projection planning, and
session/SQL execution boundaries. Generated helpers that remain around row
materialization are named as generated/model-only or generated/test-only.

The next larger authority gap is index metadata. Accepted schema snapshots do
now persist field-path index contract metadata, but runtime planning,
fingerprints, and index key construction still use generated `EntityModel` and
`IndexModel` proposal data. Expression indexes also remain generated-runtime
authority until their stable accepted expression contract is designed.

## Completed

- Added persisted accepted field-path index contracts to schema snapshots.
- Added schema proposal projection from generated field-path indexes to
  accepted `FieldId`, accepted row slot, field path, persisted field kind,
  nullability, uniqueness, ordinal, store path, and predicate SQL display
  metadata.
- Kept expression indexes out of accepted index proposals until canonical
  expression metadata lands.
- Hard-cut the schema snapshot codec to version `3` with accepted field-path
  index contract encoding.
- Added accepted snapshot integrity validation for duplicate index identities,
  empty index keys, and index key field slots that diverge from accepted row
  layout.
- Routed accepted `SchemaInfo` field index membership through persisted index
  contracts instead of generated `EntityModel` index metadata.
- Added a `SchemaInfo` field-path index metadata surface backed by accepted
  persisted index contracts, while keeping generated index projection isolated
  to generated/model-only schema views.

## Initial Targets

- Route visible-index lookup through accepted metadata.
- Route planner access choice and query cache fingerprints through accepted
  index contracts.
- Route field-path index key construction through accepted `FieldId` plus
  validated accepted row-layout slots.
- Design expression index contracts separately around canonical expression
  metadata, deterministic derivation, planner eligibility, recovery parity, and
  EXPLAIN visibility.
- Route forward/reverse index key construction through accepted index metadata
  plus accepted row contracts once each contract family is represented.
- Decide whether standalone fluent planning and structural explain stay
  explicitly model-only or gain accepted-schema entrypoints.

## Out Of Scope Until Designed

- SQL DDL.
- General index migration/backfill policy.
- Field rename/remap through index contracts.
- Production fallback from missing accepted index metadata to generated
  `E::MODEL.indexes()`.

# IcyDB 0.58 Slice B Pre-Flight

Status: **Execution checklist for DataKey + index/cursor semantics**.

Scope:

- `DataKey` identity rewrite
- index/range/prefix semantics
- cursor/continuation safety

## 0. Non-Negotiable Invariant

All ordering, partitioning, and continuation behavior must be correct purely
from byte ordering of encoded keys.

If this is false, everything above storage framing is unreliable.

## 1. EntityTag Encoding (Highest Risk Area)

Required:

- [ ] `EntityTag` is encoded as fixed-width 8 bytes
- [ ] encoding is big-endian
- [ ] no varint/compact encoding is used
- [ ] no platform-dependent representation is used

Golden rule:

```text
encoded_key = [entity_tag_be_u64][rest_of_key...]
```

## 2. Entity Partition Isolation

Required:

- [ ] all keys for one entity are contiguous in keyspace
- [ ] no key from entity A can interleave with entity B
- [ ] prefix scan on `[EntityTag]` returns only that entity

Required test:

- insert interleaved logical keys across entities
- full scan per entity
- assert zero cross-entity leakage

## 3. Range Scan Correctness

Required:

- [ ] range bounds include `EntityTag`
- [ ] lower/upper bounds never omit tag
- [ ] no implicit "same entity" assumption exists

Forbidden pattern:

```text
range = (key_start, key_end)
```

Required pattern:

```text
range = ([tag][key_start], [tag][key_end])
```

## 4. Prefix Scan Semantics

Required:

- [ ] prefix is `[EntityTag] + prefix_bytes`
- [ ] prefix scan never crosses entity boundary

Edge case:

- empty-key prefix (`prefix = [tag]`) returns all rows for that entity only

## 5. Continuation / Cursor Safety

Required:

- [ ] continuation anchor includes `EntityTag`
- [ ] resume comparisons include `EntityTag`
- [ ] cross-entity resume is impossible

Failure mode to prevent:

```text
anchor = anchor_key
```

instead of:

```text
anchor = [tag][anchor_key]
```

Required test:

- interleave entities in storage
- scan one entity with pagination
- resume repeatedly
- assert no rows from other entities appear

## 6. Ordering Consistency

Required:

- [ ] ordering is `(EntityTag, Key...)`
- [ ] no hidden secondary ordering by name or type
- [ ] runtime comparator uses raw bytes only

Audit:

- [ ] no comparator logic inspects typed fields in runtime paths
- [ ] no comparator branches on entity schema/type in runtime paths

## 7. DataKey Construction

Required:

- [ ] all runtime constructors use `DataKey::new(entity_tag, raw_key)`
- [ ] no `<E: EntityKind>` `DataKey` constructors remain
- [ ] no hidden generic wrappers remain

Must be removed:

```text
DataKey::from_key::<E>(...)
```

## 8. Index Encoding Symmetry

Required:

- [ ] encode/decode are symmetric
- [ ] no implicit decode assumptions
- [ ] tag is read first and unconditionally

Required test:

- encode -> decode -> encode roundtrip
- assert byte-for-byte equality

## 9. Composite Keys

Required:

- [ ] `EntityTag` prefix is always first
- [ ] field ordering after tag prefix remains unchanged
- [ ] no encoding-boundary shift bugs are introduced

## 10. Empty / Minimal Keys

Required:

- [ ] smallest valid key is `[tag]`
- [ ] no zero-length ambiguity
- [ ] no ad-hoc special-case logic

## 11. Debug Visibility

Required:

- [ ] debug rendering shows key framing: `[EntityTag=<n>][raw bytes...]`
- [ ] `entity_tag_name(tag)` resolves all registered tags

## 12. Migration Boundary (`0.57` -> `0.58`)

Required:

- [ ] no code attempts to decode `EntityName`
- [ ] no fallback parsing survives
- [ ] no mixed-format identity logic survives

## 13. Must-Have Tests (Minimal Set)

1. Cross-entity isolation.
2. Range boundary isolation.
3. Prefix scan isolation.
4. Continuation resume isolation.
5. Ordering/grouping stability by tag.

## Common Failure Patterns

- tag omitted from upper bound -> range spills into next entity
- little-endian encoding -> lexicographic ordering breaks
- continuation anchor missing tag -> resume jumps partitions
- prefix scan using raw key only -> cross-entity contamination
- helper function reintroduces `<E>` -> hidden monomorphization returns

## Final Mental Model

After Slice B:

Runtime storage/index/cursor code is entity-type agnostic and keyed only by
byte-ordered `EntityTag` partitions plus raw key payload bytes.

If runtime logic "knows" entities beyond `EntityTag`, abstraction has leaked.

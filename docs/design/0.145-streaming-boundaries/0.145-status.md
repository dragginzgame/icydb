# IcyDB 0.145 — Streaming Boundaries Status

## Status

In progress.

## Current Decision

0.145 owns streaming-boundary cleanup and materialization reduction.

The deferred schema identity and slot-layout migration design has moved to
0.146 so streaming work can land first.

## Initial Audit Result

The current execution spine is structurally clean, but several hot runtime
paths still collect data earlier than necessary:

* grouped bounded count windows
* grouped page candidate finalization
* SQL `RETURNING` field projection
* aggregate helper row staging
* kernel page scan/post-access staging

## First Patch Candidate

Grouped count bounded windows were the first patch candidate.

Reason:

* local change
* no public API impact
* no query semantic change
* removes a clear `Vec` staging layer before an existing bounded heap

## Patch Log

* Removed the extra `qualifying` vector from bounded grouped `COUNT(*)`
  window selection. HAVING/resume-qualified candidates now stream directly
  into the retained bounded heap, and the retained heap is sorted only at the
  page-window boundary.
* Removed full grouped page candidate pre-collection from bounded grouped
  finalization. Bounded grouped pages now finalize candidates from the grouped
  bundle directly into the retained heap, while unbounded grouped output keeps
  its materialized candidate set for global ordering.
* Reduced SQL `RETURNING` field-list ownership for INSERT/UPDATE write
  results. Field-list returns now project directly from typed mutation outputs
  instead of building full rows first, and the shared returning shaper moves
  selected values out of owned rows instead of cloning them.
* Removed the second projected-row vector from materialized `min_max(field)`
  aggregate helper reduction. The helper now decodes each projected field
  value and folds it into min/max candidates directly.
* Streamed cursorless direct `DataRow` page offsets during collection for
  route-satisfied scalar loads. The scan now retains only post-offset rows
  while separately tracking matched rows for existing post-access diagnostics.
* Applied the same scan-time offset skip to the cursorless retained-slot short
  path, with the resolved post-scan tail told when the page window has already
  been applied during collection.
* Moved non-`DISTINCT`, planner-proven pure covering projection offset handling
  ahead of projected-row assembly for index/primary-key ordered covering paths.
  Missing-row-checking, reordered, and `DISTINCT` covering paths keep their
  existing materialized page-window boundaries.
* Removed the same discarded-key staging from primary-store pure covering
  projection when primary-key row existence is already planner-proven.
* Removed unnecessary `DataKey` staging from materialized aggregate field
  projection terminals that return only values. `values_with_ids(field)` keeps
  the keyed path because ids are part of its public output.
* Removed the temporary finalized-output vector from scalar aggregate terminal
  finalization. Reducer outputs now fill terminal-order slots directly as each
  reducer finalizes.
* Removed the projected `(id, value)` staging vector from materialized
  `nth(field)` and `median(field)` aggregate helpers. Decoded field values now
  enter the ordered candidate vector directly, preserving the required ordered
  materialization boundary.
* Removed count-only DISTINCT output vectors from scalar aggregate projection
  terminals. `COUNT(DISTINCT field)` now counts accepted canonical values
  directly, while `distinct_values(field)` keeps retaining accepted values
  because it returns them.
* Split value-only ranked field terminals onto a ranked-value materialization
  helper. `top_k_by_values(field, k)` and `bottom_k_by_values(field, k)` now
  retain primary-key tie-break data plus values, without row indices that only
  row-returning and values-with-ids terminals need.
* Stream planner-proven primary-store covering projections directly from the
  ordered key stream into final projected rows. This removes the intermediate
  key vector for primary-key/constant-only projections while leaving hybrid,
  DISTINCT, reordered, and cursor-sensitive covering paths unchanged.
* Removed the remaining count-only DISTINCT staging from scalar projection
  boundaries. Materialized `COUNT(DISTINCT field)` now counts decoded
  admissions directly, and covering-index count-distinct projection counts from
  already-windowed structural pairs instead of allocating a value-only vector.
* Removed the same value-only vector staging from covering-index first/last
  scalar value terminals. The selected value now moves out of the prepared
  structural pair vector after ordering and windowing are already resolved.
* Moved the singleton global DISTINCT scalar aggregate value out of the owned
  grouped output row during decode instead of cloning it after invariant
  checks.
* Removed the admitted-value clone from scalar aggregate terminal DISTINCT
  ingestion. Accepted values are inspected by the reducer by reference, then
  moved into the retained DISTINCT admission set.
* Switched scalar aggregate terminal field and expression reducers to borrowed
  ingestion. Source values stay owned by the row/cache unless DISTINCT
  admission or extrema selection requires retained ownership.
* Narrowed shared SQL `RETURNING field, ...` owned-row projection to an
  output-width selected-value buffer instead of a full-column optional slot
  vector per row.
* Started the deferred `INSERT ... SELECT` staging cleanup. SQL INSERT now
  builds structural mutation patches directly from each source shape: `VALUES`
  rows stay borrowed from the parsed statement, and selected source rows are
  consumed into patches after width validation instead of passing through one
  shared cloned/staged source-row layer.
* Moved index-ordered, non-`DISTINCT` hybrid covering projection OFFSET
  handling into the sparse-row assembly loop. Row-presence filtering still
  happens before the skip, but skipped OFFSET candidates no longer decode
  projected index components or build projected rows.
* Removed the raw `DataRow` staging layer from DELETE row resolution. The
  resolver now maps present key-stream rows directly into typed delete rows or
  structural kernel rows before the existing post-access selection boundary.
* Added a route-ordered covering `bytes_by(field)` page-window fold. The path
  keeps row-presence checks over covering candidates, but skips component decode
  and byte-length folding outside the effective `OFFSET/LIMIT` window.
* Applied the same route-ordered covering page-window fold to scalar aggregate
  projection terminals. `values_by`, `values_by_with_ids`, `first/last_value_by`,
  and DISTINCT/count-DISTINCT projection branches now avoid decoding values
  outside the effective window when index order already satisfies output order.
* Dropped offset-discarded raw covering component pairs before one- and
  multi-component decode in pure SQL covering projections when planner-proven
  index order already satisfies the requested page order.
* Moved row-returning ranked terminal winners out of the materialized response
  window after ranking instead of cloning full `DataRow` payloads.
* Moved ranked values-with-ids terminal winners out of the materialized response
  window after ranking instead of cloning selected `DataKey`s.
* Tightened ranked winner extraction so selected rows are moved with
  descending-index `swap_remove`, avoiding one temporary `Option<DataRow>`
  wrapper per materialized source row.
* Narrowed materialized `min_max(field)` helper value cloning to the one case
  where the same projected value must be retained for both extrema states.
* Replaced scalar DISTINCT reducer NULL admission cloning with a direct
  retained `Value::Null` marker.
* Added a consuming scalar value-reducer finalization path so scalar aggregate
  terminal extrema move out of reducer state at finalization.
* Moved pre-output SQL DISTINCT projection rows directly into canonical key
  storage for OFFSET windows, avoiding the accepted-key row clone when the row
  is discarded before output.
* Changed one-component pure covering projection row assembly to consume its
  decoded index value and move that value into the final matching projected
  cell, while repeated references still clone until the last owned output use.
* Added the same final-use move boundary for multi-component pure covering
  projection rows, so decoded component vectors are consumed into projected
  rows where possible.
* Changed hybrid covering projection row assembly to consume each row's decoded
  covering-component and sparse row-field maps, moving values into the final
  matching projected cell while preserving clones for repeated output columns.
* Added an owned-key structural row read helper for key-stream loops and moved
  DELETE candidate materialization plus streaming field-extrema folds onto it,
  avoiding the `DataKey` clone after storage row reads when the caller already
  owns the key.
* Extended the owned-key row read boundary to existing-row aggregate stream
  folds and terminal-value projection lookups that already construct or receive
  an owned `DataKey`.
* Removed the stale borrowed-key `DataRow` wrapper helper after all structural
  `DataRow` materialization callers moved to the owned-key boundary.

## Deferred

Schema identity, slot-layout authority, and schema migration work now belong to
0.146.

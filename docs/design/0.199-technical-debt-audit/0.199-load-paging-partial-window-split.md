# 0.199 Load Paging/Partial-Window Facade Split

Status: structural cleanup with focused validation. This slice preserves public
fluent load method names, read-intent behavior, read admission, cursor tokens,
diagnostic DTOs, persisted formats, and executor strategy.

## Question

Can public fluent load paging and deliberate partial-window wrapper ownership
move out of the parent load facade without changing behavior?

## Evidence

After the session catalog/write split, `crates/icydb/src/db/session/load.rs`
remained the largest public facade owner. It mixed:

- base fluent load query construction and shaping;
- grouped query shaping;
- public cursor page and continuation methods;
- trusted/admin batch paging;
- paged response conversion;
- deliberate partial-window wrapper type and materialization methods;
- semantic complete/existence/exact aggregate terminals;
- EXPLAIN and tracing helpers.

The paging methods and `PartialWindowLoadQuery` wrapper are separate endpoint
promise families. Paging converts core paged execution into public
`PagedResponse`; partial-window exposes only the explicit partial row-window
materialization path and deliberately omits semantic page/complete/exact
aggregate terminals.

## Change

Move cursor-page facade ownership into `crates/icydb/src/db/session/load/paging.rs`:

- `page(limit)`;
- `next_page(limit, cursor)`;
- `admin_batch(request)`;
- conversion from core paged execution into `PagedResponse`.

Move deliberate partial-window facade ownership into
`crates/icydb/src/db/session/load/partial_window.rs`:

- `PartialWindowLoadQuery`;
- partial-window execution and row materialization;
- partial-window trace/plan/explain helpers;
- partial-window diagnostics attribution.

`crates/icydb/src/db/session/load.rs` keeps `FluentLoadQuery`, base load query
shaping, grouped shaping, semantic terminals, exact aggregates, and the public
re-export of `PartialWindowLoadQuery`.

## Result

`crates/icydb/src/db/session/load.rs` drops from about 626 lines to about 460
lines. Public page and partial-window promises now have explicit facade owner
modules, making it harder for future cleanup to mix cursor paging, deliberate
partial materialization, and exact semantic terminals.

## Validation

- `cargo fmt --all`
- `cargo check -p icydb --all-features`

## Non-Goals

- No public API changes.
- No read-intent change.
- No read-admission change.
- No cursor-token change.
- No diagnostics DTO change.
- No persisted-format change.
- No executor strategy change.

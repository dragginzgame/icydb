# 0.199 Metrics Rich Report Boundary Split

Status: structural cleanup with focused validation. This slice preserves rich
metrics DTO names, Candid payload shape, metrics sink behavior, compact metrics
report behavior, cursor formats, persisted formats, and public API shape.

## Question

Can the rich metrics endpoint payload and window filter stop sharing one broad
mutable-state file with raw counters and compact report construction?

## Evidence

After the compact report split, `crates/icydb-core/src/metrics/state.rs` still
owned:

- mutable runtime metrics state;
- raw operation and perf counters;
- rich top-level metrics report DTOs;
- per-entity rich summary DTOs and projection;
- rich report window filtering and entity sorting;
- reset/window helper functions.

Those responsibilities are now split while preserving the same parent
re-export surface:

- `metrics/state.rs` retains mutable state, raw counter DTOs, per-entity rich
  summary DTO/projection, reset helpers, and parent re-exports.
- `metrics/state/report.rs` owns `EventReport`, `EventCounters`, rich report
  window filtering, rich entity ordering, and top-level rich report assembly.
- `metrics/state/compact.rs` remains the compact report owner.

## Non-Goals

- No rich metrics Candid payload shape change.
- No compact metrics payload or code change.
- No instrumentation, metrics sink, cache counter, or executor behavior change.
- No wasm/performance claim.

## Complexity Delta

- Public API complexity: neutral.
- Runtime behavior: neutral.
- Review complexity: lower. Rich endpoint payload construction can now be
  reviewed without parsing mutable metrics state and compact report code.
- File shape:
  - `state.rs`: 2558 lines before, 2384 lines after.
  - `state/report.rs`: 183 lines.

The total line count grows slightly from module headers/import separation, but
rich report ownership is now localized.

## Validation

- `cargo check -p icydb-core --all-features`
- `cargo test -p icydb-core --lib --all-features metrics -- --nocapture`
- `cargo clippy -p icydb-core --all-features --all-targets -- -D warnings`

## Residual Risk

Low. This is a module ownership split. Remaining risk is feature-target
visibility drift around the rich report builder re-export; all-target clippy and
the rich metrics Candid-shape tests cover the current surface.

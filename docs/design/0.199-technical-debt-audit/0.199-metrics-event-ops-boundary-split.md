# 0.199 Metrics Event Ops Boundary Split

Status: structural cleanup with focused validation. This slice preserves global
operation metrics DTO names, Candid payload shape, derived-ratio behavior,
metrics sink behavior, rich and compact report behavior, cursor formats,
persisted formats, and metrics endpoint payload shape.

## Question

Can global operation metrics counters and ratio helpers stop sharing one broad
mutable-state file with raw metrics state and report construction?

## Evidence

After the entity summary split, `crates/icydb-core/src/metrics/state.rs` still
owned:

- mutable runtime metrics state;
- global operation metrics DTOs;
- `MetricRatio` and global derived-ratio helpers;
- raw perf and per-entity counters;
- reset/window helper functions.

Those responsibilities are now split while preserving the same parent
re-export surface:

- `metrics/state.rs` retains mutable state, raw perf/entity counters, reset
  helpers, and parent re-exports.
- `metrics/state/ops.rs` owns `MetricRatio`, the ratio helper, `EventOps`,
  global operation counter accessors, and global derived-ratio helpers.
- `metrics/state/summary.rs` owns per-entity rich summary DTOs and projection.
- `metrics/state/report.rs` remains the top-level rich report/window owner.
- `metrics/state/compact.rs` remains the compact report owner.

`MetricRatio` is now explicitly re-exported through the public metrics facade
because public `EventOps` and `EntitySummary` accessors already return it.

## Non-Goals

- No global operation metrics Candid payload shape change.
- No rich or compact metrics payload change.
- No instrumentation, metrics sink, cache counter, or executor behavior change.
- No wasm/performance claim.

## Complexity Delta

- Public API complexity: neutral. `MetricRatio` becomes explicitly reachable
  through the metrics facade instead of being an implicit return type.
- Runtime behavior: neutral.
- Review complexity: lower. Global operation counter payload shape and derived
  ratios can now be reviewed without parsing mutable metrics state.
- File shape:
  - `state.rs`: 1216 lines before this split, 284 lines after.
  - `state/ops.rs`: 941 lines.

The total line count grows slightly from module headers/import separation, but
global operation metrics ownership is now localized.

## Validation

- `cargo check -p icydb-core --all-features`
- `cargo test -p icydb-core --lib --all-features metrics -- --nocapture`
- `cargo clippy -p icydb-core --all-features --all-targets -- -D warnings`
- `cargo check -p icydb --no-default-features --features sql`
- `cargo fmt --all --check`
- `git diff --check`

## Residual Risk

Low. This is a module ownership split. Remaining risk is feature-target
visibility drift around the operation metrics DTO and ratio helper; all-target
clippy plus the operation metrics Candid-shape and derived-ratio tests cover
the current surface.

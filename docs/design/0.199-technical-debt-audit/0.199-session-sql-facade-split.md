# 0.199 Session SQL Facade Split

Status: structural cleanup with focused validation. This slice preserves public
`DbSession` SQL method names, SQL attribution DTO paths, generated SQL
authority, read-admission behavior, diagnostic DTOs, cursor tokens, persisted
formats, and executor strategy.

## Question

Can public `DbSession` SQL facade wiring move out of the central session module
without changing SQL facade behavior?

## Evidence

After the read-intent and terminal cleanup, `crates/icydb/src/db/session/mod.rs`
was the next concentrated facade pressure point. It mixed:

- session construction and fluent load/delete entrypoints;
- generated/controller SQL facade methods;
- SQL perf attribution DTOs;
- SQL response decode measurement and final attribution totalization;
- row projection helpers for returning writes;
- catalog/show helpers;
- high-level write helpers.

The SQL facade methods already delegated to core SQL execution and only owned
public response conversion plus public attribution envelopes, so the code had a
clear owner boundary to split.

## Change

Move SQL facade ownership into `crates/icydb/src/db/session/sql.rs`:

- `SqlQueryPerfAttribution`;
- `SqlExecutionPerfAttribution`;
- `SqlPureCoveringPerfAttribution`;
- SQL response decode measurement;
- SQL public attribution finalization;
- trusted/admin SQL query/update/DDL facade methods;
- generated/public validated SQL update/delete facade helpers.

`db::session` re-exports the public attribution DTOs so existing public paths
remain stable.

## Result

`crates/icydb/src/db/session/mod.rs` drops from about 1,025 lines to about 650
lines. SQL facade response conversion and attribution accounting now have one
owner without changing SQL lowering, SQL execution, generated controller gates,
public read admission, or public method names.

## Validation

- `cargo check -p icydb --all-features`
- `cargo test -p icydb --lib --all-features public_sql_perf_attribution_total_stays_exhaustive_after_decode_finalize`
- `cargo test -p icydb --lib --all-features`
- `cargo clippy -p icydb --all-features --all-targets -- -D warnings`
- `cargo check -p icydb --no-default-features`
- `cargo check -p icydb --features sql`

## Non-Goals

- No public SQL API changes.
- No generated SQL authority change.
- No caller-controlled public SQL surface.
- No read-admission change.
- No diagnostic DTO change.
- No query planner or executor strategy change.
- No cursor or persisted-format change.

## 0.199.0 Scope

0.199.0 starts the technical-debt audit line by anchoring the read/query
boundary matrix after the 0.197 and 0.198 hard cuts. It does not reopen
read-intent API design, exact-key canonicalization, public read admission,
cursor formats, persisted formats, or executor route behavior.

The first implementation decision after this artifact should be based on
evidence from the matrix below, not on a broad cleanup impulse.

note before patching.

1. Exact-key proof ownership trace.
   - Owner boundary: planner/intent/admission/cache/executor.
   - Allowed: docs/tests/helpers that clarify the existing owner boundary.
   - Forbidden: new optimizer behavior, secondary unique-key canonicalization,
     persisted/cursor format changes.
   - Validation: focused exact-key planner/admission/cache tests.
   - Status: traced and first planner-local projection cleanup landed in
     [0.199-exact-key-proof-ownership.md](0.199-exact-key-proof-ownership.md).

2. Direct-query facade isolation review.
   - Owner boundary: public `icydb` facade vs generated/internal tooling.
   - Allowed: docs, visibility tightening if purely internal, small naming/doc

   - Forbidden: restoring legacy read caps, weakening admission, breaking
     generated APIs without a hard-cut design.
   - Validation: facade compile tests and generated canister checks.
   - Status: traced and doc-hidden cleanup landed in
     [0.199-direct-query-boundary.md](0.199-direct-query-boundary.md).

3. Partial-window public-doc audit.
   - Owner boundary: public guides, contracts, examples.
   - Allowed: migrate examples toward `page`, `collect_complete`, exact
     aggregate, or exact-key terminals where the endpoint is not deliberately
     partial.
   - Forbidden: phrase-level invariant churn or anti-resurrection tests.
   - Validation: docs invariants only where they check structured/current

   - Status: active public docs audited and read-admission examples cleaned in
     [0.199-partial-window-public-doc-audit.md](0.199-partial-window-public-doc-audit.md).

   - Owner boundary: diagnostic codes, CLI rendering, EXPLAIN/read-intent docs.

     [0.199-exact-aggregate-helper-dryness.md](0.199-exact-aggregate-helper-dryness.md).

## Do Not Do Now

- Do not start a cost-based optimizer.
- Do not redesign public read intent terminals.
- Do not change cursor-token or persisted formats.
- Do not reintroduce public load `.limit(...)`, `.offset(...)`, `.one()`, or
  `.all()`.
- Do not add compatibility aliases or anti-resurrection tests for removed
  legacy surfaces.
- Do not make SQL a public caller-controlled read surface.
- Do not rewrite executor route selection unless a narrow finding proves that
  incremental clarification is worse.

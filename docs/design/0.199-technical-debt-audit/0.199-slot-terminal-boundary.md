# 0.199 Slot Terminal Boundary Cleanup

Status: small code cleanup with focused validation. This slice preserves public
API names, read-admission behavior, diagnostic DTOs, cursor formats, persisted
formats, and field terminal semantics.

## Question

Do field-slot terminals repeat slot-resolution and non-paged session handoff
enough to create future drift?

## Evidence

`crates/icydb-core/src/db/query/fluent/load/terminals.rs` already has a
descriptor-driver boundary:

- each terminal descriptor maps to one session execution and explain endpoint;
- `execute_terminal(...)` owns admitted non-paged execution;
- `explain_terminal(...)` owns non-paged EXPLAIN routing;
- exact aggregate field terminals have a separate stricter helper because they
  must reject prior row-window caps before field-slot resolution.

The first repeated part was ordinary descriptor-backed field terminals. Many
methods manually performed:

1. `resolve_non_paged_slot(field)`;
2. construct a slot terminal descriptor;
3. execute or explain the descriptor.

The follow-up repeated part was custom field-slot session handoff. Bytes,
projection, and top/bottom-k helpers all had to perform slot resolution before
calling a session method through the same non-paged admission or EXPLAIN gate.

## Change

Added private owner-local helpers:

- `execute_slot_terminal(...)`
- `explain_slot_terminal(...)`
- `with_admitted_non_paged_slot(...)`
- `with_non_paged_slot(...)`

These helpers resolve the field slot once and then delegate to the existing
terminal descriptor driver boundary or the existing non-paged session handoff.
Descriptor-backed field terminals now use the descriptor helpers. Custom
bytes-by-field, shared projection, and top/bottom-k helpers now use the
session-handoff helpers.

## Remaining Separate Path

Exact aggregate field terminals still use separate helpers because they must
reject prior partial-window caps before field-slot resolution. That stricter
error-ordering is part of the exact aggregate contract, not ordinary field-slot
handoff.

## Result

This is a narrow DRY cleanup. It reduces repeated field-slot terminal wiring
without changing terminal descriptors, route selection, read-admission behavior,
public API, or result semantics.

## Non-Goals

- No public field terminal API rename.
- No admission-policy change.
- No projection or ranking execution strategy change.
- No cursor-token or persisted-format change.
- No broad terminal dispatch enum.

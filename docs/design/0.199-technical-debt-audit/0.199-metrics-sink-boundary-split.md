# 0.199 Metrics Sink Boundary Split

Status: structural cleanup with focused validation. This slice preserves the
metrics event taxonomy, metrics sink behavior, scoped override behavior, span
finish behavior, metrics endpoint payloads, cursor formats, persisted formats,
and public facade shape.

## Question

Can the metrics sink facade stop owning concrete event-to-state dispatch and
typed instrumentation span helpers in the same file?

## Evidence

After the metrics state/report splits, `crates/icydb-core/src/metrics/sink.rs`
still owned:

- the `MetricsSink` trait;
- scoped sink override routing and panic restoration;
- global concrete sink event dispatch into mutable metrics state;
- typed executor span lifetimes and performance-counter capture;
- typed `record_*_for_path` instrumentation helpers;
- metrics report/reset facade functions.

Those responsibilities are now split while preserving the same parent
`metrics::sink` surface:

- `metrics/sink.rs` owns the sink facade, scoped override stack, report/reset
  functions, and parent re-exports.
- `metrics/sink/dispatch.rs` owns `GlobalMetricsSink` and the concrete
  `MetricsEvent` application into mutable metrics state.
- `metrics/sink/instrumentation.rs` owns `Span`, `PathSpan`, performance-counter
  capture, and typed instrumentation helper wrappers.
- `metrics/sink/events.rs` remains the stable event taxonomy owner.
- `metrics/sink/counters.rs` remains the counter-bucket mutation parent; the
  later counter-domain split gives execution, schema, SQL, cache, and planning
  helpers separate child owners.

## Non-Goals

- No metrics event enum change.
- No metrics sink override behavior change.
- No span drop/finalization behavior change.
- No metrics counter, Candid payload, or report shape change.
- No instrumentation call-site change.
- No wasm/performance claim.

## Complexity Delta

- Public API complexity: neutral.
- Runtime behavior: neutral.
- Review complexity: lower. Sink override routing, event dispatch, and typed
  instrumentation spans now have separate owner files.
- File shape:
  - `sink.rs`: 864 lines before, 135 lines after.
  - `sink/dispatch.rs`: 530 lines.
  - `sink/instrumentation.rs`: 232 lines.

The total line count grows slightly from module headers/import separation, but
the sink facade no longer mixes override routing, concrete state mutation, and
span helper lifetimes.

## Validation

- `cargo check -p icydb-core --all-features`
- `cargo test -p icydb-core --lib --all-features metrics::sink -- --nocapture`
- `cargo clippy -p icydb-core --all-features --all-targets -- -D warnings`
- `cargo check -p icydb --no-default-features --features sql`
- `cargo fmt --all --check`
- `git diff --check`

## Residual Risk

Low. This is a module ownership split. Remaining risk is visibility drift around
internal re-exports from `metrics::sink`; focused sink and metrics tests plus
all-target clippy cover that surface.

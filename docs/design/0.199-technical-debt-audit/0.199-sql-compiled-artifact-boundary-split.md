# 0.199 SQL Compiled Artifact Boundary Split

Status: structural cleanup with focused validation. This slice preserves SQL
compile/cache behavior, execution dispatch, generated/controller SQL behavior,
cursor formats, persisted formats, and public API shape.

## Question

Can the session-owned compiled SQL artifact owner stop mixing compiled-command
variants, prepared-plan cache entries, schema fingerprinting, and immediate
execution context handoff in one file?

## Evidence

Before this slice, `crates/icydb-core/src/db/session/sql/compiled.rs` owned:

- compiled schema fingerprint construction and comparison;
- select/global-aggregate prepared-plan cache entries;
- direct-count prefix-spec cache entries;
- the generic-free `CompiledSqlCommand` enum used by SQL execution dispatch;
- `CompiledSqlInsertCommand` source-query handoff state;
- the accepted-schema execution context carried from compile to immediate
  execution.

Those responsibilities are now split while preserving the same parent
re-export surface:

- `compiled/cache.rs` owns schema fingerprints and prepared-plan cache entry
  DTOs.
- `compiled/command.rs` owns `CompiledSqlCommand`,
  `CompiledSqlInsertCommand`, command-shape predicates, and compiled command
  cache accessors.
- `compiled/context.rs` owns `SqlCompiledCommandExecutionContext`, the
  accepted-schema/catalog handoff from compile to execution.
- `compiled.rs` is now a narrow module index.

## Non-Goals

- No SQL compile behavior change.
- No SQL cache key or cache invalidation behavior change.
- No SQL execution dispatch change.
- No generated/controller SQL behavior change.
- No cursor, persisted-format, storage, or public API change.

## Complexity Delta

- Public API complexity: neutral.
- Runtime behavior: neutral.
- Review complexity: lower. Cache-entry DTOs, command variants, and
  accepted-schema execution context handoff can be reviewed independently.
- File shape:
  - `compiled.rs`: about 432 lines before, 14-line module index after.
  - `compiled/cache.rs`: 119 lines.
  - `compiled/command.rs`: 250 lines.
  - `compiled/context.rs`: 81 lines.

The total line count is approximately neutral because module headers move with
the new owners.

## Validation

- `cargo check -p icydb-core --all-features`
- `cargo test -p icydb-core --lib --all-features sql_surface -- --nocapture`
- `cargo clippy -p icydb-core --all-features --all-targets -- -D warnings`
- `cargo check -p icydb --no-default-features --features sql`

## Residual Risk

Low. This is a module ownership split. Remaining risk is all-target visibility
or lint drift, covered by core clippy and SQL-focused tests before release.

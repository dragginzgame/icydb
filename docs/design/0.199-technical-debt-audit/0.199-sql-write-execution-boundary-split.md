# 0.199 SQL Write Execution Boundary Split

Status: structural cleanup with focused validation. This slice preserves SQL
INSERT/UPDATE/DELETE behavior, generated/public write behavior, returning
behavior, write admission, SQL cache behavior, cursor formats, persisted
formats, and public API shape.

## Question

Can SQL write execution stop mixing accepted-schema authority helpers,
candidate-row resource accounting, compiled write dispatch, mutation metrics,
and mutation batch execution in one parent module?

## Evidence

Before this slice, `crates/icydb-core/src/db/session/sql/execute/write/mod.rs`
was a 996-line parent module that owned:

- compiled SQL write dispatch and default cache attribution;
- SQL write error and row-count metrics;
- primary-key literal decoding and composite key construction;
- accepted runtime descriptor validation;
- accepted field-slot lookup and literal normalization;
- explicit generated/managed field rejection;
- accepted save-contract projection;
- candidate row bounds, diagnostics, and staged-row buffers;
- mutation batch handoff and returning result shaping;
- candidate accounting tests.

Those responsibilities are now split while preserving the same internal
execution surface:

- `write/authority.rs` owns accepted-schema descriptor validation, key decoding,
  field normalization, generated/managed field guards, accepted save-contract
  projection, and accepted authority/schema-info lookup.
- `write/candidate.rs` owns candidate row bounds, diagnostics, accounting,
  staged mutation buffers, and focused candidate accounting tests.
- `write/mod.rs` remains the write execution coordinator for compiled write
  dispatch, write metrics, candidate collection from structural queries, and
  mutation batch execution.
- `write/insert.rs`, `write/update.rs`, and `write/delete.rs` keep their
  command-specific execution ownership.

## Non-Goals

- No SQL write behavior change.
- No generated/public SQL write policy change.
- No returning projection, response-byte, or row-bound behavior change.
- No SQL cache, compile, or execution dispatch semantic change.
- No cursor, persisted-format, storage, or public API change.

## Complexity Delta

- Public API complexity: neutral.
- Runtime behavior: neutral.
- Review complexity: lower. Accepted-schema write authority and candidate
  resource policy can now be reviewed without scanning compiled dispatch or
  command-specific execution.
- File shape:
  - `write/mod.rs`: 996 lines before, 323-line coordinator after.
  - `write/authority.rs`: 278 lines.
  - `write/candidate.rs`: 453 lines.

The total line count grows modestly because moved responsibilities now have
module headers and local imports.

## Residual Risk

Low. This is a module ownership split. Remaining risk is target-specific
visibility or lint drift around sibling `insert`, `update`, and `delete`
modules, covered by SQL write-focused tests and core all-target clippy.

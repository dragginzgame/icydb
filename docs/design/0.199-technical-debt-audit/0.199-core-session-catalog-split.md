# 0.199 Core Session Catalog And Accepted-Schema Split

Status: structural cleanup with focused validation. This slice preserves core
`DbSession` catalog, schema-description, storage-report, integrity-report,
accepted-schema cache, and save-contract method behavior. It does not change
public read intent, read admission, SQL lowering, generated SQL authority,
cursor tokens, persisted formats, accepted schema reconciliation policy, or
executor strategy.

## Question

Can core session catalog/observability facade ownership and accepted-schema
runtime authority move out of the central core session module without changing
behavior?

## Evidence

After the public facade split, `crates/icydb-core/src/db/session/mod.rs` still
mixed:

- session construction and fluent load/delete entrypoints;
- metrics and low-level executor construction;
- accepted schema query cache authority;
- typed save helper wiring;
- catalog/show/schema-description methods;
- storage-report and integrity-report methods.

The catalog/show/storage-report methods are session-owned DTO and observability
surfaces. Accepted-schema query cache and save-contract projection are
session-owned authority adapters. They consume schema reconciliation output, but
they do not own schema reconciliation policy, query planning, read admission,
mutation staging, or executor runtime semantics.

## Change

Move core catalog and observability facade ownership into
`crates/icydb-core/src/db/session/catalog.rs`:

- `show_indexes`;
- `show_indexes_for_model`;
- `try_show_indexes`;
- store/model and store/accepted-schema index formatting helpers;
- `show_columns`;
- `show_columns_for_model`;
- `try_show_columns`;
- `show_entities`;
- `try_show_entities`;
- `show_stores`;
- `show_memory`;
- `visible_indexes_for_store_accepted_schema`;
- `describe_entity`;
- `describe_entity_model`;
- `try_describe_entity`;
- `storage_report`;
- `storage_report_default`;
- `integrity_report`.

Move accepted-schema runtime authority into
`crates/icydb-core/src/db/session/accepted_schema.rs`:

- `AcceptedSchemaCatalogContext`;
- accepted-schema query cache keying and invalidation;
- runtime-hook accepted schema context loading;
- query accepted-schema context loading;
- accepted catalog snapshot selection reuse;
- accepted schema-info projection helpers;
- accepted entity authority projection helpers;
- accepted save-contract projection helpers;
- accepted catalog runtime counter test helpers.

The central core session module keeps session construction, fluent load/delete
entrypoints, metrics wiring, save wrapper response shaping, and low-level
executor construction.

## Result

`crates/icydb-core/src/db/session/mod.rs` drops from about 1,106 lines to about
314 lines. The new `session/catalog.rs` owner is about 291 lines and contains
the catalog/schema-description/storage-report surfaces that SQL metadata and
typed session APIs already treat as session-owned facade behavior. The new
`session/accepted_schema.rs` owner is about 552 lines and contains the
accepted-schema runtime authority and save-contract projection helpers that
query, SQL, catalog, and write adapters already consume as session-owned
authority.

## Validation

- `cargo fmt --all`
- `cargo check -p icydb-core --all-features`

## Non-Goals

- No public API changes.
- No read-intent change.
- No read-admission change.
- No SQL lowering or generated SQL authority change.
- No accepted schema reconciliation policy change.
- No query cache identity change.
- No cursor or persisted-format change.
- No executor strategy change.

# 0.199 Metrics Compact Report Boundary Split

Status: structural cleanup with focused validation. This slice preserves compact
metrics DTO names, Candid payload shape, numeric compact metric codes, metrics
sink behavior, rich metrics report behavior, cursor formats, persisted formats,
and public API shape.

## Question

Can the compact metrics endpoint payload stop sharing one broad mutable-state
file with rich report DTOs and metrics state updates?

## Evidence

Before this slice, `crates/icydb-core/src/metrics/state.rs` owned:

- mutable runtime metrics state;
- rich event/entity metrics report DTOs;
- compact numeric metric codes;
- compact metrics Candid DTOs;
- sparse compact report construction;
- rich report construction;
- reset/window helper functions.

Those responsibilities are now split while preserving the same parent
re-export surface:

- `metrics/state.rs` retains mutable state, raw counter DTOs, per-entity rich
  summary projection, reset helpers, and the existing compact public
  re-exports.
- `metrics/state/compact.rs` owns compact metric numeric codes, compact DTOs,
  sparse compact metrics projection, compact entity sorting, and compact window
  filtering.
- A later rich report split moves top-level rich report DTOs and rich window
  filtering into `metrics/state/report.rs`.

## Non-Goals

- No compact metric code rename or renumbering.
- No compact metrics Candid payload shape change.
- No rich metrics report DTO or behavior change.
- No instrumentation, metrics sink, cache counter, or executor behavior change.
- No wasm/performance claim.

## Complexity Delta

- Public API complexity: neutral.
- Runtime behavior: neutral.
- Review complexity: lower. Compact endpoint payload shape and sparse compact
  projection can now be reviewed without parsing mutable metrics state and rich
  report construction.
- File shape:
  - `state.rs`: 2976 lines before, 2558 lines after.
  - `state/compact.rs`: 435 lines.

The total line count grows slightly from module headers/import separation, but
compact report ownership is now localized.

## Validation

- `cargo check -p icydb-core --all-features`
- `cargo test -p icydb-core --lib --all-features metrics -- --nocapture`
- `cargo clippy -p icydb-core --all-features --all-targets -- -D warnings`

## Residual Risk

Low. This is a module ownership split. Remaining risk is feature-target
visibility drift around the compact builder re-export; all-target clippy and
the compact Candid-shape tests cover the current surface.

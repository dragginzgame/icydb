# 0.199 Projection Terminal Module Split

Status: structural cleanup with focused validation. This slice preserves public
fluent projection APIs, read-admission behavior, diagnostic DTOs, cursor tokens,
persisted formats, generated SQL authority, and executor strategy.

## Question

Can fluent load projected-value and ranked-row terminal methods move out of the
parent `terminals.rs` file without changing terminal behavior?

## Evidence

After the aggregate terminal extraction, `terminals.rs` still owned the
projection-heavy field terminal family:

- `values_by(...)` and `values_by_with_ids(...)`;
- shared projection helpers;
- distinct projected values;
- top/bottom row and value helpers;
- first/last projected values;
- projection EXPLAIN wrappers.

These methods already delegated to shared slot/admission helpers and output
conversion helpers, so their remaining coupling to `terminals.rs` was file
ownership rather than runtime authority.

## Change

Move projected-value and ranked-row public terminal methods into
`terminals/projection.rs`.

The projection module delegates to the existing private support helpers for:

- non-paged admission;
- field-slot resolution;
- projection/ranking session handoffs;
- descriptor-backed projection EXPLAIN.

It delegates runtime value conversion to `terminals/output.rs`.

## Result

`terminals.rs` is reduced to about 190 lines and now owns only core execution,
execution EXPLAIN, bytes, first/last id, and response cardinality terminals.
Projection terminal method ownership is explicit without adding a terminal
enum, broad dispatch object, compatibility layer, or execution-route rewrite.

## Validation

- `cargo check -p icydb-core --all-features`
- focused projection/ranking terminal tests
- focused read-admission tests
- focused aggregate/explain tests
- `cargo clippy -p icydb-core --all-features --all-targets -- -D warnings`

## Non-Goals

- No public projection API changes.
- No terminal rename.
- No read-admission change.
- No EXPLAIN or diagnostic DTO change.
- No query planner or executor strategy change.
- No cursor or persisted-format change.

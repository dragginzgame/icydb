# 0.199 SQL Session Surface And DDL Split

Status: structural cleanup with focused validation. This slice preserves SQL
surface helper names, generated/controller SQL routing, DDL publication
behavior, SQL execution behavior, cursor formats, persisted formats, and public
API shape.

## Question

Can `db::session::sql` stop owning generated endpoint routing, query/update
surface gating, DDL prepare/publish logic, and SQL query/update execution in one
parent file?

## Evidence

Before this slice, `crates/icydb-core/src/db/session/sql/mod.rs` was a 689-line
subsystem parent that owned:

- public re-exports for SQL policy, result, compile, execution, and attribution
  types;
- generated/controller SQL statement route helpers;
- query/update surface mismatch gating;
- public SQL query/update execution methods;
- SQL DDL preparation;
- SQL DDL physical publication and accepted-schema cache invalidation.

Those responsibilities are now split while preserving the same outward names:

- `sql/surface.rs` owns generated/controller SQL route helpers and query/update
  surface gating.
- `sql/ddl.rs` owns DDL preparation, physical publication, accepted-schema cache
  invalidation, and DDL execution metrics.
- `sql/mod.rs` remains the subsystem facade and keeps the public query/update
  execution methods plus shared compile-stage measurement helper.

## Non-Goals

- No generated/controller SQL routing change.
- No SQL query/update surface mismatch behavior change.
- No SQL DDL admission, preparation, publication, or metric behavior change.
- No SQL compile/cache/execution behavior change.
- No cursor, persisted-format, storage, or public API change.

## Complexity Delta

- Public API complexity: neutral.
- Runtime behavior: neutral.
- Review complexity: lower. Generated endpoint routing and DDL publication can
  now be reviewed without scanning query/update execution attribution code.
- File shape:
  - `sql/mod.rs`: 689 lines before, 217-line facade after.
  - `sql/surface.rs`: 276 lines.
  - `sql/ddl.rs`: 233 lines.

The total line count grows modestly because moved responsibilities now have
module headers and local imports.

## Validation

- `cargo check -p icydb-core --all-features`
- `cargo test -p icydb-core --lib --all-features sql_surface -- --nocapture`
- `cargo clippy -p icydb-core --all-features --all-targets -- -D warnings`
- `cargo check -p icydb --no-default-features --features sql`
- `cargo fmt --all --check`
- `git diff --check`

## Residual Risk

Low. This is a module ownership split. Remaining risk is test-target import or
visibility drift around the hidden generated/controller SQL helpers, covered by
the SQL surface test filter and core all-target clippy.

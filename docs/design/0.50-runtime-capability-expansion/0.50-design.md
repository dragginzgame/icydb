# IcyDB 0.50 Design

Status: Active  
Target: 0.50 minor line

## Theme

0.50 executor structural simplification.

## Intent

0.50 focuses on reducing executor complexity before the engine grows
further with broader language and explain surfaces.

Primary goals:

- consolidate continuation authority
- simplify pipeline/orchestration layers
- eliminate grouped runtime duplication
- reduce executor module and LOC surface while preserving behavior

## Slice 0: Preflight Guardrails

Before structural slices A-E, complete a preflight pass that lowers
refactor risk and locks behavioral contracts.

- define executor invariants and module boundaries that must not change
- lock core invariants explicitly:
  - plan semantics: identical plan -> identical execution behavior
  - continuation semantics: resume tokens remain stable and monotonic
  - explain semantics: explain node correlation remains deterministic
- build a capability and ownership matrix per executor module
- add characterization tests for continuation, grouped execution, and terminal behavior
- add plan/explain snapshot coverage for regression detection
- capture executor structure baseline:
  - module count
  - LOC per module
  - cross-module dependency graph
  - continuation call surface
- capture baseline query/plan benchmarks
- define one authority per cross-cutting concern (especially continuation and orchestration)

## Scope

Candidate areas:

- slice 0 preflight guardrails
- slice A continuation authority consolidation
- slice B pipeline/orchestrator simplification
- slice C grouped runtime duplication elimination
- slice D kernel/operator relocation
- slice E executor LOC reduction sweep (mechanical only)

## Slice E Guardrail

Slice E is strictly mechanical cleanup.

Allowed:

- module merging
- dead code removal
- helper consolidation
- comment/doc cleanup

Not allowed:

- semantic logic changes
- algorithm changes
- planner contract edits

## Non-Goals

- major language features
- breaking schema or storage changes

## Success Criteria

- preflight guardrails completed before structural edits
- executor boundaries become simpler and more explicit
- continuation and plan semantics remain stable under refactor
- explain/plan regression coverage catches unintended structural drift
- executor structure baseline shows measurable simplification by end of line
- no architectural regressions

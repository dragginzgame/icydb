# 0.50 Slice A: Continuation Authority Consolidation

Status: Ready to start after Slice 0 sign-off  
Owner area: `crate::db::executor::continuation`

## Intent

Slice A consolidates continuation ownership so cursor decode, continuation mode
resolution, and token emission remain under one executor authority boundary.

## Invariants (must remain unchanged)

- plan semantics: identical plan -> identical execution behavior
- continuation semantics: resume tokens remain stable and monotonic
- explain semantics: explain node correlation remains deterministic

## Scope

- reduce continuation policy leakage outside `executor::continuation`
- centralize continuation mode/projection helpers currently split across
  route/pipeline call sites
- keep terminal and post-access cursor emission behavior unchanged

## Non-Goals

- no new query-language features
- no planner contract redesign
- no algorithm changes to scan/ranking logic

## Work plan

### A1. Inventory and classification

- classify continuation call sites as:
  - authority APIs (keep in continuation)
  - orchestration consumers (pipeline entry/orchestrator)
  - policy derivation call sites (candidate to move)
- produce per-call-site migration decisions in this file.

- keep route/planner as policy consumers, not continuation-mode calculators
- keep the `route -> continuation` dependency one-directional.

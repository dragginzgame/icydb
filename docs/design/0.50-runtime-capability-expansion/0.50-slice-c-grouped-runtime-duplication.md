# 0.50 Slice C: Grouped Runtime Duplication Elimination

Status: In progress  
Owner area: grouped runtime boundaries (`pipeline/grouped_runtime`, `aggregate/runtime`, `route/grouped_runtime`)

## Intent

Slice C removes duplicated grouped-runtime layering and clarifies ownership
between route projection, pipeline grouped runtime assembly, and aggregate
runtime execution.

## Invariants (must remain unchanged)

- plan semantics: identical plan -> identical execution behavior
- continuation semantics: resume tokens remain stable and monotonic
- explain semantics: explain node correlation remains deterministic

## Work plan

### C1. Flatten grouped runtime projection wrapper (completed)

- removed `GroupedRuntimeProjection` indirection
- flattened projection fields into `GroupedExecutionContext`
- updated grouped route-stage assembly to construct flattened context directly

Files:

- `crates/icydb-core/src/db/executor/pipeline/grouped_runtime/runtime.rs`
- `crates/icydb-core/src/db/executor/pipeline/grouped_runtime/route_stage.rs`
- `crates/icydb-core/src/db/executor/pipeline/grouped_runtime/mod.rs`

### C2. Grouped ownership boundary audit (completed)

- verify grouped route projection helpers stay route-owned
- verify grouped fold/output mechanics stay aggregate-runtime-owned
- reduce cross-module grouped re-export surface where unnecessary

Result:

- grouped route projection remains route-owned (`route/grouped_runtime.rs`)
- grouped fold/output remains aggregate-runtime-owned (`aggregate/runtime/*`)
- grouped pagination/capability imports were migrated away from
  `pipeline::grouped_runtime` re-export paths where they crossed ownership boundaries
  (for example, `executable_plan`, grouped fold candidate/page-finalize modules,
  grouped route-stage projection contracts)

### C3. Grouped continuation surface tightening (completed)

- remove or narrow grouped continuation symbols exposed beyond required consumers
- preserve grouped route/pagination behavior contracts

Result:

- removed grouped continuation/pagination re-exports from
  `pipeline/grouped_runtime/mod.rs`
- non-pipeline grouped continuation consumers now import grouped types from
  executor continuation-owned boundaries instead of pipeline re-export shortcuts

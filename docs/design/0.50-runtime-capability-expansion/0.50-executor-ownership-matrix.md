# 0.50 Executor Ownership Matrix

Status: In progress  
Scope: Slice 0 baseline mapping

## Purpose

This matrix captures module ownership and cross-module coupling so refactors
can remove overlap without changing semantics.

## Orchestration Authority

- declared orchestration authority owner: `crate::db::executor::pipeline::orchestrator`
- rationale: load entrypoints should converge on one staged orchestration boundary so mode/trace/surface contracts cannot drift across wrappers.

## Continuation Authority

- declared continuation authority owner: `crate::db::executor::continuation`
- rationale: continuation decode/resolve/bind contracts must remain in one module to preserve monotonic token and replay semantics.

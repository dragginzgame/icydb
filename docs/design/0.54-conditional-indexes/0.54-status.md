## Slice A - Completed

Goal:

- Add conditional-index predicate metadata to schema and runtime index models
  without changing index storage format or commit/recovery semantics.

Evidence:

- `schema::node::Index` now carries optional predicate metadata.
- `model::index::IndexModel` now carries optional predicate metadata.
- Schema-derive index metadata can now emit optional predicate literals.
- Module-local tests lock display/accessor behavior for predicate and non-predicate
  index forms.

## Slice B - Completed

Goal:

- Validate conditional-index predicate metadata at schema construction time with
  explicit syntax-vs-schema-invalid boundaries.

Evidence:

- Added predicate-only SQL parsing helper:
  `db::sql::parser::parse_sql_predicate(...)`.
- `SchemaInfo::from_entity_model(...)` now validates optional index predicates:
  - parse SQL predicate text
  - validate predicate against entity schema contracts.
- Added explicit typed schema validation outcomes:
  - `ValidateError::InvalidIndexPredicateSyntax`
  - `ValidateError::InvalidIndexPredicateSchema`
- Added schema validation regression coverage:
  - accepts schema-valid index predicates
  - rejects invalid predicate syntax
  - rejects schema-invalid predicate field references.

## Slice C - Completed

Goal:

- Ensure mutation planning/enforcement treats conditional index predicates as
  index-membership gates for insert/update/remove and unique checks.

Evidence:

- `db::index::plan::plan_index_mutation_for_entity(...)` now compiles optional
  index predicates and gates old/new index key generation through predicate
  evaluation before membership prevalidation, unique checks, and commit-op synthesis.
- `db::commit::prepare::prepare_row_commit_for_entity_impl(...)` now uses the same
  membership gate when classifying per-key delta kinds, keeping conditional
  insert/remove annotations aligned with planned mutations.
- Added mutation-path regression coverage in commit tests:
  - false -> true -> false transitions for conditional index membership
  - conditional-unique behavior where predicate-false rows bypass unique checks
    but predicate-true duplicates fail closed.

## Slice D - Completed

Goal:

- Enforce filtered-index planner eligibility via deterministic query-predicate
  implication checks before index-route selection.

Evidence:

- `db::query::plan::planner::index_select::sorted_indexes(...)` now filters index
  candidates using index-predicate implication against the full query predicate.
- Planner compare/prefix/range candidate builders now consume that filtered index
  candidate set, so filtered indexes are excluded before route-shape selection.
- Added planner regression coverage:
  - filtered index rejected when query does not imply index predicate
  - filtered index accepted when implication holds (`active=true` conjunction)
  - filtered numeric range index accepted/rejected based on stronger/weaker
    lower-bound implication.

## Slice E - Completed

Goal:

- Lock end-to-end conditional-index behavior across live apply vs replay paths,
  conditional-unique conflict classification parity, and startup recovery rebuilds.

Evidence:

- Added conditional membership transition matrix parity test:
  - `conditional_index_forward_apply_and_replay_preserve_identical_store_state_for_membership_matrix`
    proves forward apply and replay converge on identical row/index snapshots.
- Added conditional-unique conflict classification parity test:
  - `conditional_unique_conflict_classification_parity_holds_between_live_update_and_replay`
    locks the same conflict class/message surface between live update and replay.
- Added startup rebuild conditional-index rebuild coverage:
  - `recovery_startup_gate_rebuilds_conditional_indexes_from_authoritative_rows`
    locks predicate-false/stale entry purge and predicate-true membership restore
    during startup recovery rebuild.

## Post-0.54 Boundary Note (0.55 Foundation)

- Current filtered-index implementation still has string-authoritative predicate
  side paths.
- Canonical predicate lowering should become the sole semantic authority for
  filtered-index predicates.
- Predicate semantics must be included in schema fingerprinting.
- This boundary hardening is a prerequisite/foundation item for 0.55
  expression-index work, not a full 0.54 redesign.

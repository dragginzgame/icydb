## Title

**Removed — Aggregate Existing-Rows Authority Snapshot**

---

## Status

Removed

---

## Reason

This design was removed in favor of a stricter index visibility invariant.

Aggregate existing-row shortcuts such as `COUNT` and `EXISTS` do not require a
dedicated runtime correctness snapshot.

If an index is visible to the planner, it is assumed to be:

* fully built
* fully populated
* fully correct

Execution therefore does not perform a separate correctness decision for these
aggregate shortcuts.

---

## Revised Rule

Aggregate existing-row shortcuts:

* rely on planner visibility
* do not require execution-time correctness resolution
* do not use a dedicated aggregate snapshot or resolver

This simplifies the execution model and keeps correctness enforcement at one
boundary:

* planner visibility

# 0.201 Trait Contract Hardening

Status: implemented, re-audited, and closed. Slices A through H and every
smaller vocabulary-trait candidate have final dispositions; no 0.201 work
remains open.

0.201 reduces overlapping trait authority, encodes invariants that are
currently documentary, removes obsolete post-0.200 persistence adapters, and
moves retained contracts toward their owning domains. It does not redesign the
query or executor architecture.

This is a separate design line rather than a 0.200 supplement. The obsolete
persisted-row adapters are post-0.200 cleanup, but visitor traversal, index read
ports, entity invariants, key conversion, and trait-module ownership are
independent concerns with separate owners and validation. Adding all of them to
the already normative schema-native-enum design would obscure both designs.

The slices in this document are landing units, not patch releases. Their letter
labels do not predict or require future package versions.

## Audit Snapshot

The audit was performed on 2026-07-12 against:

- branch `main`;
- commit `fa126b7765e3489055a32c8fa68923d64b290697`;
- 101 trait declarations under `crates/icydb-core/src`, including test-local
  declarations;
- 44 public traits concentrated under `crates/icydb-core/src/traits`;
- a 1,700-line `crates/icydb-core/src/traits/mod.rs` containing contracts,
  blanket implementations, helper functions, macros, and dependency
  reexports.

The initially clean worktree gained concurrent persisted-row, commit-test,
schema-mutation, and SQL-test changes during the audit. Those changes belong to
the accepted-row persistence work and are not part of this design. Persistence
findings were re-read against that moving state, but Slice B and Slice C remain
gated on a fresh reachability check after the overlapping work settles.

Evidence classifications used here are:

- **direct**: the trait contract and its call sites contradict each other;
- **reachability**: references show an adapter has no non-test consumer;
- **structural**: two trait families express the same port or invariant;
- **risk**: the current shape permits invalid implementations even when the
  generated implementation is correct.

## Problem Statement

The trait surface has aged in several different ways:

1. Some traits still describe a superseded generated or persisted-row path.
2. Some trait hierarchies duplicate the same capability through typed and
   structural adapters.
3. Some marker traits assert invariants that implementors can violate.
4. Some blanket implementations grant a narrow capability to a much wider set
   of types than the trait documentation permits.
5. Some default container implementations perform work already owned by the
   traversal driver.
6. The central public trait module has become both an API facade and an
   implementation owner.

The desired end state is not the smallest possible trait count. It is a trait
surface where every retained trait has a clear capability, more than cosmetic
value over an inherent method, one owning domain, and bounds that encode its
documented laws.

## Decisions

### Retain capability splits that express real one-way use

The following separations remain intentional:

- `RuntimeValueEncode` and `RuntimeValueDecode`;
- primary-key encoding and primary-key decoding;
- `AuthoredFieldProjection` and `FieldProjection`;
- immutable and mutable visitor cores;
- direct scalar, structural, and schema-selected persisted codecs;
- planner, row-reader, ordered-stream, projection, explain, and metrics ports
  that have multiple meaningful consumers or provide object-safe erasure.

One-way consumers should not acquire the opposite capability merely to make a
single `Codec` trait look symmetrical.

`PrimaryKeyEncode` is the primary-key encoding contract. The former
`PrimaryKeyCodec` name suggested a two-way contract even though decoding is
separate. Slice H applied this pre-1.0 hard rename with no alias.

### Delete traits that preserve only retired wiring

A public or generated-facing trait is not justified by:

- export chains with no production call site;
- test fixtures that implement the trait only because older generated code
  once required it;
- UI tests whose sole purpose is proving that a removed compatibility path is
  absent;
- stale derive documentation.

Before deletion, each candidate receives one fresh production reachability
scan. If a current generated or runtime consumer is found, the slice stops and
the owner is documented instead of silently translating that consumer through
a fallback.

### Collapse adapter-only duplicate ports

When one trait exists only to adapt into another trait with the same operation,
the consumer-facing port is canonical. Scoped visibility is sufficient where
outside crates cannot implement the trait; a same-scope empty sealing trait
does not add a boundary.

### Encode laws in bounds or constructors

Generated code following a convention is not proof that all implementations
follow it. Placement equality, singleton identity, key codec eligibility, and
buffer sizing must be enforced by bounds, sealed capabilities, or fallible
APIs.

### Keep one public facade and one generated facade

Application-facing contracts remain available from `icydb::traits`.
Generated-only contracts and dependency paths live under `icydb::__macro` or
`icydb::__reexports`. Core implementation modules own the definitions they
execute. The final organization must not retain parallel old and new paths.

## Normative Contracts

### Visitor traversal owns recursion

`Visitable::drive` and `Visitable::drive_mut` describe structure. Sanitization
and validation hooks operate on the current node only.

For `Option<T>` and `Vec<T>`:

- the wrapper hook implementations are no-ops;
- `drive` visits each contained node exactly once;
- `Option` preserves the current transparent/empty path segment;
- `Vec` reports child issues at the indexed path;
- automatic and custom hooks each run once per visited node.

`Box<T>` is intentionally different in the current traversal model. Its drive
implementation is transparent and delegates directly to the inner node's
structure, so its hook forwarding supplies the inner node's own hook call.
Slice A must preserve that one-call behavior or deliberately make `Box`
non-transparent in the same patch. It must not mechanically remove all
container forwarding.

### Accepted schema remains persistence authority

The persistence cleanup must preserve the 0.200 authority rule:

- accepted row contracts select persistence semantics;
- arbitrary runtime `Value` conversion is not a persistence codec;
- generated models remain proposal, reconciliation, and test conveniences;
- no runtime path reconstructs accepted schema from generated models;
- current bytes either decode under the current accepted contract or return a
  typed error.

`PersistedFieldMetaCodec` previously blanketed types through
`RuntimeValueEncode + RuntimeValueDecode`, contradicting those runtime traits'
explicit storage prohibition. Its helpers had export and test call sites but
no production caller, so Slice C removed the complete lane.

`PersistedFieldSlotCodec` was described by `icydb-derive` as the generated row
storage contract even though derive output asserted `RuntimeValueDecode` and
materialized from `SlotReader`. Its implementations, `SlotWriter`, and the
complete-patch writer were test-only remnants of the retired generated encode
direction and were removed in Slice B.

The cleanup does **not** merge every remaining persisted codec:

- `PersistedScalar` owns direct scalar leaf encoding;
- `PersistedStructuralValueCodec` owns recursive typed payload encoding;
- `PersistedByKindCodec` owns schema-selected encoding where the accepted kind
  is an explicit input.

Their definitions and implementations live beside the persisted-row codec that
uses them. Slice H hard-renamed the former `PersistedStructuredFieldCodec` to
`PersistedStructuralValueCodec` because scalar leaves and recursive containers
both implement it; no old-name alias is retained.

### Structural index read ports are canonical

The canonical internal contracts are:

- one primary-row read port;
- one index-entry/range read port;
- `IndexPlanReadView` where planning intentionally needs a combined view.

Before Slice D, the typed `PrimaryRowReader<E>` and `IndexEntryReader<E>`
traits fed only adapters or duplicate implementations of their structural
equivalents. Slice D removed them with all four `Sealed*Reader` traits.

After removal, the retained structural methods may take the unsuffixed names
`read_primary_row`, `read_index_entry`, and `read_index_keys_in_raw_range`.
That rename is part of the hard cut; `_structural` aliases are forbidden.

### Entity contracts prove placement and cardinality

`EntityPlacement` must enforce:

```rust
type Store: StoreKind<Canister = Self::Canister>;
type Canister: CanisterKind;
```

Removing the separate `Canister` associated type may be considered only if it
produces a smaller focused patch. Keeping it with an equality constraint is
the default because many current bounds name `E::Canister` directly.

`SingletonEntity` is not freely implementable. A sealed singleton-key
capability is implemented only for the maintained `()` and `Unit` key
representations, and `SingletonEntity` is derived from that proof. Generated
explicit singleton marker implementations and fixture wiring are removed.

Before Slice E, `EntityKeyBytes::write_bytes` panicked on a wrong buffer length,
and generated composite implementations repeated that assumption. It now
returns `Result<(), EntityKeyBytesError>`, distinguishing caller buffer-length
errors from fixed-width value overflow. `Id::project` and `Id::subaccount`
propagate that error. Maintained scalar and generated composite implementations
validate before writing, preserve the byte format, and add no per-key
allocation to the identity path.

### Key conversion is an explicit capability

`KeyValueCodec` is a key-only runtime projection. It must not be a blanket
consequence of general runtime conversion.

The maintained implementation uses:

- explicit implementations for supported scalar key types;
- an explicit generated implementation for composite key structs;
- one explicit identity implementation for the canonical internal `Value`
  carrier used after planner keys have already been lowered;
- one shared scalar eligibility list for relation-key and key-codec markers,
  so those capabilities cannot drift independently;
- round-trip tests for every maintained key family.

General `RuntimeValueEncode + RuntimeValueDecode` support does not grant key
capability. The internal `Value` carrier is not an authored scalar-key type and
is kept separate from scalar eligibility.

`RuntimeValueDecode for &str` is removed because it can never return a borrowed
value from an owned runtime `Value`. Borrowed string encoding may remain as an
encode-only convenience.

### Runtime collection decoding preserves one invariant set

Context-free and enum-context decoding of the same collection type must agree
on malformed input:

- sets reject duplicate decoded items;
- maps reject duplicate decoded keys;
- maps retain the canonical ordering requirements of the maintained runtime
  representation;
- contextual enum resolution does not weaken those checks.

Contextual implementations reuse shared insertion/canonicality helpers rather
than collecting directly.

### Trait hierarchy edges are non-redundant

Slice G applies the following target relationships where compile evidence
confirms the current semantic contract:

- `Visitable` continues to imply `Sanitize + Validate`;
- `TypeKind` does not repeat `Sanitize + Validate` alongside `Visitable`;
- `EntityKind` does not repeat `Kind` alongside `TypeKind`;
- `EntityKind` remains the model/placement contract because SQL lowering has
  intentional model-only implementors that are not entity values;
- `PersistedRow` combines `EntityKind + EntityValue` and therefore implies the
  value/projection capabilities needed by stored runtime callers;
- the facade `Entity` trait does not repeat `EntityValue` or
  `AuthoredFieldProjection` when those are already implied.

This normalization was compiler-driven. The current model-only SQL-lowering
types intentionally implement `EntityKind` without being entity values, so the
complete runtime proof belongs on `PersistedRow`; no second blanket entity
vocabulary trait is introduced.

### Trait definitions live with their owners

After earlier slices settle the surface, Slice H targets this ownership map:

| Contract family | Owning area |
| --- | --- |
| Entity keys, byte projection, key/runtime conversion, primary-key encode/decode, relation-key proof | `db::key_taxonomy` |
| Persisted scalar, structural, and by-kind codecs | `db::data::persisted_row::codec` |
| Runtime value conversion and collection helpers | `value` |
| Entity declaration, placement, value, and create materialization | `entity` |
| Visitable, sanitize, validate, and visitor contexts | `visitor` |
| Typed numeric round-trip conversion | `types::numeric_value`, exported from `types` |
| Database coercion, arithmetic, comparison, and ordering | `db::numeric` |
| Application-facing stable names | `icydb::traits` |
| Generated-only paths and third-party reexports | `icydb::__macro` / `icydb::__reexports` |

`EntityDeclaration` replaces `EntitySchema` because its generated `MODEL` is a
proposal for reconciliation and model-only tooling, not runtime schema
authority. Accepted schema snapshots remain authoritative for storage,
planning, and execution. The old name and core-traits path are not retained as
aliases.

The reorganization is a hard cut. It does not add old-path compatibility
modules, duplicate exports, or aliases.

## Implementation Slices

### Slice A: Visitor One-Call Correctness

Owner: `visitor`.

Allowed changes:

- make `Option` and `Vec` sanitize/validate hooks node-local no-ops;
- preserve or deliberately normalize `Box` transparency;
- add focused call-count and issue-path tests for automatic and custom hooks;
- document the container traversal law beside the implementations.

Forbidden changes:

- redesigning visitor object safety;
- merging mutable and immutable visitor cores;
- unrelated validation-rule changes.

Closeout evidence:

- one hook call per logical child;
- indexed `Vec` issue paths;
- unchanged top-level entrypoint behavior.

Expected shape: fewer recursive bodies, behaviorally corrective, wasm-neutral
or smaller.

Implemented on 2026-07-13. `Option` and `Vec` hooks are node-local no-ops,
structural traversal invokes each child's automatic and custom hooks once, and
indexed issue paths remain attached to the visited child. `Box` stays
transparent and forwards one logical hook call. Focused core and generated
visitor suites pass, production removes eight recursive hook bodies, and the
standard raw and canonically shrunk wasm artifacts are unchanged.

### Slice B: Retired Generated Slot Codec Cut

Owner: persisted-row generated bridge.

Sequencing gate: land or otherwise stabilize the overlapping accepted-row
work, then prove that current derive output and non-test runtime code do not
consume `PersistedFieldSlotCodec` or `SlotWriter`.

Allowed changes after that proof:

- remove `PersistedFieldSlotCodec` and `codec/slot.rs`;
- remove `SlotWriter` and its test-only complete-image writer;
- remove stale trait implementations from fixtures;
- update derive documentation to the current materialization contract;
- delete UI tests that exist only to keep the retired trait absent from
  `Value`;
- remove old export paths in the same slice.

Forbidden changes:

- replacing the lane with a compatibility wrapper;
- routing persistence through arbitrary runtime `Value` conversion;
- reconstructing accepted row authority from generated models.

Closeout evidence:

- current derive snapshots and downstream generated builds compile;
- typed and structural accepted-row save/materialization tests pass;
- no production or generated token names the removed contracts.

Expected shape: net deletion, simpler, wasm-neutral or smaller.

Implemented on 2026-07-13. A fresh reachability scan found no production or
generated-token consumer of `PersistedFieldSlotCodec` or `SlotWriter`; all
remaining implementations and writer calls were test-only scaffolding for the
retired encode lane. The trait, codec module, writer, exports, fixture impls,
and anti-resurrection UI tests are removed. Model-proposal test row creation
now routes through an accepted field contract, while generated materialization
continues to decode accepted runtime values from `SlotReader`. Focused
persisted-row, generated schema, and UI suites pass with all-feature Clippy.
The implementation removes 822 net lines across 22 code and test files. The
standard raw and canonically shrunk wasm artifacts are unchanged.

### Slice C: Runtime-Value Persistence Leak Cut

Owner: persisted-row codec selection.

Sequencing gate: the same fresh reachability proof as Slice B.

Allowed changes:

- remove `PersistedFieldMetaCodec`, `codec/meta.rs`, and their helper/export
  chain if they remain test/export-only;
- route maintained tests through accepted-row, scalar, structural, or by-kind
  owners as appropriate;
- remove tests whose only purpose is the retired lane.

Forbidden changes:

- preserving the helper names as forwarding shims;
- teaching `RuntimeValueEncode` or `RuntimeValueDecode` to own persisted bytes;
- merging the three maintained persistence lanes merely to reduce trait count.

Closeout evidence:

- no persisted encoder or decoder depends on runtime-only conversion traits;
- accepted-schema persistence tests and malformed-byte tests pass;
- generated package verification passes.

Expected shape: net deletion and clearer authority; measure raw wasm because
codec helpers may be wasm-reachable through existing exports.

Implemented on 2026-07-13. A fresh reachability scan found only the trait,
helper/export chain, and two direct test calls; no production or generated
consumer remained. `PersistedFieldMetaCodec`, `codec/meta.rs`, its public and
hidden wrappers, and the now-unreachable runtime-value strategy helpers are
removed. The two maintained typed-field tests now encode through accepted
field contracts. Focused persisted-row, mutation-save, generated schema, and
UI suites pass with all-feature Clippy. Slices B and C together remove 1,226
net Rust lines across 25 files. The standard raw and canonically shrunk wasm
artifacts remain unchanged.

### Slice D: Structural Index Reader Consolidation

Owner: `db::index::readers`.

Allowed changes:

- remove the typed reader pair and four sealing markers;
- keep one primary-row port and one index-entry/range port;
- update `Context`, `PreflightStoreOverlay`, and `StoreHandle` implementations;
- simplify method names without compatibility aliases;
- retain `IndexPlanReadView`.

Forbidden changes:

- changing commit preflight ordering or overlay semantics;
- widening reader visibility;
- coupling the planner to executor context internals.

Closeout evidence:

- commit/relation preflight tests pass against real, context, and overlay
  readers;
- index range membership behavior is unchanged;
- only the two canonical ports remain.

Expected shape: net deletion, behavior-neutral, wasm-neutral or smaller.

Implemented on 2026-07-13. A fresh reachability scan found no consumer of the
typed reader pair beyond adapters and duplicate implementations; the four
same-scope sealing markers enforced no additional boundary. `Context`,
`PreflightStoreOverlay`, and `StoreHandle` now implement the two structural
ports directly, and their methods use the unsuffixed hard-cut names. Focused
mutation-save, preflight, relation, and index suites pass with all-feature
Clippy. The implementation removes 250 net Rust lines across six files. The
standard raw and canonically shrunk wasm artifacts are each 8 bytes smaller
than the published 0.201.1 baseline.

### Slice E: Entity And Key Invariant Proofs

Owner: entity and key contracts plus generated entity implementations.

Allowed changes:

- constrain store/canister equality;
- derive singleton capability from a sealed unit-key proof;
- make key-byte writes fallible or construction-safe without allocating on the
  hot path;
- update current generated entity/key implementations;
- add positive and compile-fail tests for the maintained contracts.

Forbidden changes:

- singleton aliases for non-unit legacy key shapes;
- panic-based production key writes;
- format or index-key byte changes.

Closeout evidence:

- inconsistent placement and false singleton implementations do not compile;
- scalar and composite key bytes remain byte-for-byte unchanged;
- identity projection and relation-key tests pass.

Expected shape: slightly stronger types with neutral runtime cost. Benchmark
identity projection if the selected length-proof shape changes the hot loop.

Implemented on 2026-07-13. `EntityPlacement::Store` now requires its canister
to equal `EntityPlacement::Canister`. `SingletonEntity` is sealed and derived
only from the maintained unit-key proof, so generated entities and test
fixtures no longer emit marker implementations. `EntityKeyBytes` returns the
typed `EntityKeyBytesError`; scalar and generated composite encoders validate
their exact buffer, and identity projection propagates failures without adding
an allocation. Existing scalar and composite bytes are unchanged, including
the zero-length principal representation. The full core library suite,
generated-schema suite, core and macro UI suites, and all-target Clippy pass.
The slice adds 222 net lines across 22 implementation and test files; 105 of
those lines are the two compile-fail cases and their expected diagnostics. The
standard raw wasm artifact is 8 bytes smaller and the canonically shrunk
artifact is 12 bytes smaller than after Slice D. No identity-projection
benchmark exists in the repository; the unchanged allocation pattern and wasm
measurement provide the available cost evidence.

### Slice F: Key Capability And Runtime Decode Alignment

Owner: key conversion and runtime value conversion.

Allowed changes:

- replace the blanket key codec with explicit eligible implementations;
- remove impossible borrowed-string decoding;
- share duplicate/canonicality-preserving set and map decode helpers across
  contextual and context-free paths;
- add malformed enum-containing collection tests.

Forbidden changes:

- narrowing general runtime encode/decode solely because a type is not a key;
- accepting duplicate or noncanonical persisted/runtime collection shapes;
- adding an allocation-heavy key adapter.

Closeout evidence:

- every maintained scalar and composite key round-trips;
- ordinary and contextual collection decoders reject the same malformed
  shapes;
- cursor, fingerprint, explain, SQL-key, and structural-save tests pass.

Expected shape: explicit capability surface, modest code movement, neutral hot
path. Measure key/cursor performance if explicit implementations stop
inlining equivalently.

Implemented on 2026-07-13. The blanket `KeyValueCodec` implementation is gone.
Maintained scalar keys are emitted from the same eligibility list as scalar
relation keys, generated composite keys retain their direct implementation,
and the planner's already-lowered `Value` carrier has a separate explicit
identity implementation. Impossible borrowed `&str` decoding is removed.
Context-free and enum-context list, set, and map decoding now share helpers;
sets reject decoded collisions, while maps reject decoded-key collisions and
noncanonical raw ordering without cloning entries for normalization. Positive
round-trip tests cover every maintained scalar family and generated composite
keys, malformed enum-containing collection tests cover the shared invariants,
and compile-fail tests lock the collection and borrowed-string boundaries.
Cursor, fingerprint, explain, SQL-write, structural-save, full core, derive,
generated-schema, core UI, all-target check, and all-target Clippy validation
pass. The slice adds 236 net lines across seven implementation and test files;
199 lines are tests and expected diagnostics, while production code adds 37
net lines and removes the blanket and duplicate decode flows. The standard raw
wasm remains 2,366,611 bytes and the canonically shrunk wasm remains 2,204,560
bytes, exactly unchanged from Slice E. Key encoding and cursor conversion are
unchanged, and the identical wasm output provides the available codegen cost
evidence.

### Slice G: Supertrait Normalization

Owner: entity and type trait hierarchy.

Allowed changes:

- remove redundant supertrait edges and repeated generic bounds;
- make `EntityKind` the complete core runtime entity contract when compile
  evidence permits;
- simplify the public `Entity` facade accordingly;
- seal `ValueProjectionExpr` if it remains a deliberately closed public
  expression set.

Forbidden changes:

- adding a second blanket `Entity` synonym in core;
- changing authored versus accepted projection semantics;
- changing generated endpoint surfaces.

Closeout evidence:

- derive and facade compile tests pass;
- public generic examples need fewer but not weaker bounds;
- no model-only/runtime authority boundary is erased.

Expected shape: primarily type-level simplification with no runtime delta.

Implemented on 2026-07-13. `TypeKind` now obtains sanitize and validate through
`Visitable`, and `EntityKind` obtains `Kind` through `TypeKind`. Compile
evidence confirmed that SQL lowering intentionally uses model-only
`EntityKind` implementations, so `EntityKind` remains the model/placement
contract while `PersistedRow: EntityKind + EntityValue` is the complete stored
runtime contract. The public `Entity` facade now requires only `PersistedRow`,
and `EntityFor<C>` constrains the inherited canister directly. Across runtime,
session, SQL, executor, and tests, more than 230 repeated `EntityValue`, authored
projection, `EntityKind`, and `Sized` clauses were removed from persisted-row
bounds with their unused imports. `ValueProjectionExpr` is explicitly sealed
to the maintained numeric, rounded, and text helper types. Model-only SQL
lowering, full core, derive, generated-schema, core UI, macro UI, all-target
check, and all-target Clippy validation pass. The implementation removes about
91 net Rust lines across 71 files; the broad file count is mechanical bound
and import cleanup rather than added runtime structure. The standard raw wasm
remains 2,366,611 bytes and the canonically shrunk wasm remains 2,204,560 bytes,
exactly unchanged from Slice F. No runtime algorithm, encoding, or hot loop
changed, so a separate performance probe is not applicable.

### Slice H: Owner-Local Trait Organization

Owner: the trait facade and each destination domain.

This slice may be split by owner if it would exceed the routine file-count or
two-domain guidance. Moving runtime-value contracts, persistence contracts,
and entity contracts together is not required.

Allowed changes:

- move retained definitions and their implementations to their owners;
- leave `icydb::traits` as the deliberate application facade;
- move generated dependency paths to hidden generated facades;
- hard-rename `EntitySchema` to `EntityDeclaration` and rename the structured
  persisted codec where the earlier decisions require clearer vocabulary;
- remove standard-library and third-party trait reexports from the semantic
  trait facade when they exist only for generated code.

Forbidden changes:

- old-path modules or aliases;
- behavior changes mixed into mechanical moves;
- opportunistic executor or planner refactors.

Closeout evidence:

- each retained trait has one definition and one documented owner;
- application and generated downstream builds compile using only their
  intended facades;
- `traits/mod.rs` is a small facade rather than an implementation hub;
- source moves are reviewable separately from semantic changes.

Expected shape: mostly neutral line movement with reduced central-module
pressure. Report file count and line delta separately from net LOC.

Visitor-owner portion implemented on 2026-07-13. `Visitable`, the sanitize and
validate trait families, `Sanitizer`, `Validator`, and their primitive impl
macro now live under `visitor`; the top-level sanitize and validate entrypoints
and concrete runtime value implementations import that owner directly.
`icydb_core::traits` no longer duplicates those exports. The public
`icydb::traits` application facade reexports them directly from the visitor
owner, while generated implementations use `icydb::visitor` and core fixture
generation uses `icydb_core::visitor`. This preserves the deliberate
application facade without retaining the retired core path as a compatibility
alias. Full core, derive, generated-schema, core UI, macro UI, all-target check,
and all-target Clippy validation pass. The move touches about 39 Rust files and
adds roughly 12 net lines of module/import wiring while retaining one
definition per contract, so the ownership shape is simpler despite the
near-neutral line count. The standard raw wasm remains 2,366,611 bytes and the
canonically shrunk wasm remains 2,204,560 bytes, exactly unchanged from Slice
G. Persistence, numeric, entity, and remaining generated-only ownership moves
are still open parts of Slice H.

Runtime-value-owner portion implemented on 2026-07-13. `RuntimeValueEncode`,
`RuntimeValueDecode`, `RuntimeValueMeta`, `RuntimeValueKind`, accepted enum
decode context, `Collection`, `MapCollection`, their standard-container
implementations, and all shared list/set/map conversion helpers now live in
`value::runtime` and are exported from the `value` owner. The primitive runtime
impl macro is owner-private instead of leaking from the crate root.
`icydb_core::traits` no longer duplicates the runtime family. Generated runtime
conversion continues through the hidden `icydb::__macro` facade, while
generated collection traversal uses `icydb::value`; application-facing
`icydb::traits::{Collection, MapCollection}` remains the deliberate public
facade and reexports directly from the owner. Runtime enum/catalog decoding,
canonical set/map rejection, full core, derive, generated-schema, core UI,
macro UI, all-target check, and all-target Clippy validation pass. The move
touches about 48 Rust files, mostly owner-import rewiring, and removes roughly
80 net Rust lines while reducing `traits/mod.rs` to 933 lines. The standard raw
wasm remains 2,366,611 bytes and the canonically shrunk wasm remains 2,204,560
bytes, exactly unchanged from the visitor-owner portion. No runtime algorithm
or persisted representation changed.

Key-owner portion implemented on 2026-07-13. `EntityKey`, fixed-width key-byte
projection, scalar relation-key proof, `KeyValueCodec`, `PrimaryKeyEncode`, and
`PrimaryKeyDecode` now live beside the canonical key vocabulary in
`db::key_taxonomy::contracts` and are exported from the `db` owner. The former
`PrimaryKeyCodec` name was hard-renamed to `PrimaryKeyEncode` without an alias
so the one-way capability is explicit.
`icydb_core::traits` no longer duplicates the key family. Generated composite

## Non-Goals

0.201 does not:

- merge all encode/decode traits;
- replace trait objects with enums across the executor;
- redesign query planning, admission, execution, or cursor semantics;
- change persisted bytes, primary-key bytes, or database format versions;
- introduce legacy aliases, compatibility modules, dual dispatch, or fallback
  decoding;
- make generated models a runtime schema fallback;
- remove traits solely to minimize a count;
- combine unrelated cleanup merely because it touches `traits/mod.rs`.

## Validation Plan

Every code slice must:

- run formatting after edits;
- run focused owner tests before broader workspace checks;
- verify both derive crates and maintained generated downstream surfaces when a
  generated contract changes;
- preserve public API documentation for retained public traits;
- prove that production executor code has no new panic path;
- report files touched, approximate line delta, and whether implementation
  complexity became simpler, stayed neutral, or became more complex.

Wasm-reaching slices report raw non-gzipped `.wasm` bytes as the primary size
metric. Gzip is secondary context. Performance probes are required when a slice
changes key encoding, cursor conversion, row codec dispatch, or another hot
loop; purely type-level and docs-only slices may record those checks as not
applicable.

Removal slices must validate absence through current positive surfaces rather
than adding anti-resurrection tests for retired trait names.

## Closeout Standard

0.201 is complete when:

1. `Option` and `Vec` children sanitize and validate exactly once at the
   correct path.
2. No persisted byte encoder or decoder depends on the retired slot/meta
   adapter traits or treats runtime-only `Value` conversion as its codec;
   generated typed materialization may still convert an already accepted and
   decoded runtime value.
3. Primary and index reads each have one canonical internal port.
4. Entity store/canister equality and singleton cardinality are encoded by the
   type system.
5. Key-byte projection has no reachable production assertion or undocumented
   panic.
6. `KeyValueCodec` is granted only to maintained typed key families, generated
   composite keys, and the canonical internal `Value` carrier; every typed key
   family round-trips.
7. Contextual and context-free set/map decoding enforce identical duplicate
   and canonicality rules.
8. Supertrait graphs and facade bounds contain no known redundant edges.
9. Accepted schema snapshots remain the only runtime schema authority.
10. Retained traits have one owner, application traits have one public facade,
    and generated plumbing has one hidden facade.
11. No compatibility aliases, fallbacks, or old-path shims remain.
12. Each implementation slice records validation, complexity, raw wasm impact
    where applicable, and performance impact where applicable.

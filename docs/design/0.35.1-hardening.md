# 0.35.1 - GROUP BY Streaming Hardening Addendum

## Status

Draft - Design Addendum  
Target: 0.35.1  
Prerequisite: `0.35` grouped surface is landed and behavior-stable

---

# 1. Objective

Land pre-`0.36` grouped hardening gates so streaming/grouped architecture is
stable before `HAVING`/`DISTINCT` expansion.

This addendum is a closeout and hardening pass over grouped execution
semantics, not a feature-expansion release.

---

# 2. Scope (`0.35.1`)

In scope:

* grouped streaming/order-preservation hardening
* grouped `LIMIT`/`ORDER BY` behavioral invariants
* merge/intersection monotonic-order guarantees
* grouped cursor boundary integrity
* deterministic grouped strategy observability
* explicit anti-regression gate against implicit `Vec` fallback paths

Out of scope:

* `HAVING`
* grouped `DISTINCT` aggregate terminals
* projection-contract expansion
* continuation payload shape expansion beyond hardening needs

---

# 3. Final Hardening Checklist

`0.35.1` is complete only when all are true:

1. predicate execution is streaming and order-preserving
2. `DISTINCT` invalidates ordered grouping unless adjacency-based eligibility is proven
3. `LIMIT` after grouping does not force full-result buffering
4. `ORDER BY` insertion disables ordered grouping unless prefix-aligned with grouped keys
5. merge/intersect paths preserve strict monotonic key ordering
6. cursor resume cannot split a single group across boundaries
7. plan metrics record grouping strategy deterministically
8. no executor path implicitly falls back to `Vec` buffering

If any item is false, `0.35.1` remains in progress.

---

# 4. Exit Criteria

`0.35.1` is complete when:

* all hardening checklist items are verified by code and tests
* grouped architecture is declared release-clean for `0.36` feature work
* roadmap/status/changelog references reflect the hardening closure

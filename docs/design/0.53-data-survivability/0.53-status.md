## Slice Tracker

### Slice A - Stable Persisted Row Format & Schema Evolution

Goal:

- Introduce explicit persisted-row format versioning and fail-closed
  compatibility-aware decoding.

Exit gates:

- [ ] persisted row format versioning contract is implemented and documented
- [ ] N-1 decode compatibility coverage exists for row payloads
- [ ] decode incompatibility emits classified compatibility errors
- [ ] structural corruption emits corruption-classified errors

### Slice B - Commit Marker Compatibility & Replay Semantics

Goal:

- Version commit markers and lock replay semantic compatibility boundaries.

Exit gates:

- [ ] commit-marker format versioning is implemented
- [ ] recovery decode path is compatibility-aware and fail-closed
- [ ] replay semantic invariants are documented and regression-tested

### Slice C - Explicit Migration Engine (No Data Loss)

Goal:

- Execute migrations as explicit row-op commit streams with resume-safe
  recovery behavior.

Exit gates:

- [ ] migration plan contract is implemented
- [ ] migration execution is deterministic and replay-compatible
- [ ] interrupted migrations are idempotent or explicitly resume-safe
- [ ] migration failure paths are classified and observable

### Slice D - Corruption Detection & Integrity Tooling

Goal:

- Expand integrity detection and observability while preserving taxonomy
  boundaries.

Exit gates:

- [ ] integrity checks cover data/index and forward/reverse consistency
- [ ] corruption vs compatibility vs misuse taxonomy is enforced
- [ ] operator-facing integrity scan/reporting surfaces are documented

### Slice E - Upgrade, Replay & Recovery Hardening

Goal:

- Lock upgrade/replay survivability guarantees under the compatibility
  contract.

Exit gates:

- [ ] upgrade replay retains readable persisted rows within contract bounds
- [ ] crash-mid-migration recovery scenarios are regression-tested
- [ ] no silent row orphaning/discard paths remain in upgrade/recovery flows

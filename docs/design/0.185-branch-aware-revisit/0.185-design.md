# 0.185 Branch-Aware Query Revisited

This line continues the branch-aware route work that 0.183 deliberately kept
narrow. The source design is
`docs/design/0.183-branch-aware-routing/0.183-design.md`; this document is the
revisit plan after the 0.184 query-engine audit cleanup.

## Baseline

0.183 shipped the first production-shaped branch route:

```text
fixed_prefix = X
AND branch_field IN (small exact set)
ORDER BY primary_key ASC
LIMIT N
```

against indexes shaped like:

```text
(fixed_prefix, branch_field, primary_key)
```

The executor opens one ordered stream per branch, merges branch heads by primary
key, suppresses duplicate keys, and stops at the page/lookahead boundary.

0.184 removed a large amount of duplicated SQL/fluent planning and execution
plumbing. That makes 0.185 the right place to revisit branch-aware routing
without carrying two parallel implementations.

## 185.0 Scope

Keep the first 185.0 slice as a convergence and representation slice, not a
new broad optimizer:

- Prove equivalent SQL and fluent branch-heavy queries select the same explicit
  branch route identity.
- Confirm branch-set runtime behavior consumes the branch route/spec contract
  instead of reconstructing branch semantics from loose prefix lengths.
- Document which branch-aware route facts are runtime authority: index identity,
  fixed prefix, branch slot, branch values, ordered suffix, and primary-key
  direction.
- Inventory remaining branch, `IN`, union, and multi-lookup flows that should
  become shared branch-tree machinery.
- Leave adaptive cost routing and cursor-format changes behind explicit design
  gates unless a narrow correctness fix requires them.

## Invariants

- `IndexBranchSet` must stay distinguishable from generic unordered union or
  multi-lookup routes.
- Global order is proven only when every branch stream shares the same
  primary-key ordered suffix and direction.
- A branch route must not require materializing and sorting all matching rows.
- Access-proven prefix predicates must not survive as residual row predicates.
- Page execution remains pull-lazy and stops after the effective page/lookahead
  window.
- SQL and fluent surfaces should share the same lowered access/execution
  contract for equivalent branch-heavy predicates.
- Diagnostics must expose the selected branch route and enough route identity
  to prevent cache or fingerprint misclassification.

## Work Plan

1. Baseline convergence: add focused tests proving SQL and fluent target shapes
   share the same `IndexBranchSet` access identity and result page.
2. Representation audit: inspect runtime branch execution for any remaining
   branch-slot reconstruction from vector lengths or ad hoc prefix math.
3. Branch-tree inventory: list special-case `IN`, branch-set, multi-lookup,
   union, and intersection flows that can share one ordered stream contract.
4. Continuation design: specify how branch cursors resume each branch stream
   without replaying already-emitted keys. Current notes live in
   `continuation.md`.
5. Adaptive routing design: define how small `IN`, large sparse `IN`, and dense
   over-cap `IN` choose between branch-set, child-prefix expansion, fallback
   scan/filter/sort, or future cost-based routing. Current notes live in
   `adaptive-routing.md`.
6. Perf matrix broadening: cover representative SQL and fluent branch-heavy
   page/count/exists shapes without making one downstream application query the
   only benchmark.

## 185.0 Acceptance Gate

185.0 is ready when the branch-aware revisit has:

- an explicit design/status baseline;
- SQL/fluent route convergence proof for the original target shape;
- no known regression in the focused branch-set test suite;
- a documented next-slice list for cursor/adaptive/branch-tree work.

## Deferred Beyond 185.0

- Broad cost-based optimization.
- Prefix cardinality metadata redesign.
- Arbitrary non-primary-key branch merge ordering.
- Full cursor format redesign unless the continuation design identifies a
  required hard-cut.
- Making SQL DDL or generated models runtime authority for branch decisions.

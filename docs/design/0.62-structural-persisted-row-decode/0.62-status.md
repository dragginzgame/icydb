# 0.62 Status Tracker

Current state: in progress after `0.62.2`; architecture phase is complete and
post-cut wasm tightening continues.

## Scope

This status tracks the `0.62` persisted-row structuralization line and the
follow-up wasm-surface tightening that remained after the runtime serde graph
was removed.

Design reference:

- `docs/design/0.62-structural-persisted-row-decode/0.62-design.md`

---

## Landed Through `0.62.2`

- [x] Persisted rows use one slot-based structural format only; legacy row
  versions were hard-cut.
- [x] `PersistedRow` derive/codegen is the canonical typed boundary for row
  write/materialization.
- [x] Commit/index planning no longer decodes `E` from row bytes.
- [x] Index key extraction, predicate evaluation, projection execution, and
  relation validation are structural/slot-driven in runtime paths.
- [x] Scalar leaf codec v1 is in place for the scalar field families that were
  moved off CBOR.
- [x] Scalar-only predicate compilation exists and dispatches once at the
  execution boundary instead of branching per node.
- [x] Scalar-only projection compilation exists for structural SQL projection
  execution.
- [x] Reverse-index and strong-relation runtime paths decode storage keys
  directly instead of round-tripping through `Value`.
- [x] Commit control envelopes and commit-marker envelopes are binary.
- [x] Commit-marker payload decode is binary and structural.
- [x] Runtime `serde_cbor::from_slice` is gone from persistence/runtime decode
  paths.

---

## Current Architecture Boundary

Persistence:

- row container: slot-based structural format
- commit frame: binary
- commit payload: binary

Execution:

- hot scalar paths: `SlotReader -> scalar/native execution`
- generic fallback: explicit and isolated

Decode:

- scalar leaves: direct decode
- complex leaves: explicit structural decode in
  `crates/icydb-core/src/db/data/structural_field.rs`

Typed boundaries:

- entity reconstruction
- response/materialization boundaries that truly need typed values

## Recent Kept Wins

- Shared CBOR array/map walkers replaced repeated container loops in
  `structural_field.rs`.
- Tagged variant parsing was unified for schema enums and `Value` payloads.
- `ValueEnum` payload decode stopped building temporary owned strings before
  constructing `ValueEnum`.
- Untyped enum/shallow fallback dispatch was flattened by branching on CBOR
  major type directly.
- Projection/commit/unique row-key validation now shares one slot-owned
  storage-key validator instead of duplicating `Value -> StorageKey` round
  trips.

Measured local wins that were kept:

- `decode_value_enum_payload_bytes`: `4207 -> 3677`
- `decode_untyped_enum_payload_bytes`: `1017 -> 630`
- `decode_untyped_shallow_bytes`: `658 -> 304`
- `project_data_rows_from_projection_structural`: `7018 -> 6684`

## Reverted / Non-Paying Directions

- Broad map-helper unification in `structural_field.rs` regressed retained wasm
  and was reverted.
- Direct-loop rewrites for `Value::Map` payload decode regressed retained wasm
  and were reverted.
- Small decode-surface cleanups that did not delete a real retained path were
  not kept.

Rule kept from these attempts:

- accept or reject by measured retained/runtime wasm movement, not by
  architectural neatness alone

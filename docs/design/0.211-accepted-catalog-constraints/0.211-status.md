## Slice 1 Result

The accepted entity snapshot now carries:

- a nonzero `ConstraintId` representation and persisted allocator high-water
  state, where zero means no identity has yet been reserved;
- a nonzero stable `SchemaIndexId` on every accepted secondary-index
  definition, separate from its dense physical ordinal; and
- a nonzero stable `RelationId` on every accepted relation definition.

Initial generated snapshots assign deterministic logical identities. Accepted
schema reconstruction preserves allocator state and relation identity. Index
drop preserves each surviving logical index ID while compacting only physical
ordinals. Generated index rename matching now associates contracts by logical
ID rather than physical ordinal.

Decode rejects zero or duplicate logical index and relation IDs. The accepted
schema wire has one current version-1 `ICYU` profile; no fallback, inference,
alias, old-version branch, or dual decoder remains. Admission and runtime/cache
fingerprints include allocator, index-ID, and relation-ID changes.

### Version hard-cut audit

A repository-wide scan found no production version-2 codec, decoder, enum, or
dual-format branch. Every active durable format uses version 1 and exact
current-form decoding. The current SELECT generator, Tier C distribution
artifact, and diagnostics-attribution evidence identities have also been reset
to version 1 rather than accumulating higher-numbered identities. Historical
release and investigation documents retain their recorded version numbers as
historical evidence; they are not accepted runtime formats.

`SchemaVersion`, accepted schema revisions, and entity-local
`RowLayoutVersion` values may exceed 1 because they identify application
history within the current version-1 codec. They are not compatibility format
versions.

### Footprint

The slice currently touches 42 tracked or new files at approximately
`+756/-130` lines. Complexity increases deliberately by three identity types
and one persisted allocator state, while ownership becomes simpler: dense
physical ordinals and names no longer double as logical identity, no parallel
physical execution path was added, and every current internal format and test
evidence identity remains version 1.

The matched SQL performance Wasm subject is 3,967,488 raw non-gzipped bytes,
up 3,098 bytes (+0.0782%) from the recorded final 0.210.2 baseline of 3,964,390
bytes. Query and mutation hot-path instruction performance was not remeasured:
Slice 1 changes schema metadata, decoding, reconciliation, and fingerprints but
does not add a row, index-key, relation-key, query, or write-enforcement path.

The repository-wide suite remains human-owned under `AGENTS.md`. Focused
command results are updated as each slice closes.

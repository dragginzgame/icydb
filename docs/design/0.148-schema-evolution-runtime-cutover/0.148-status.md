# 0.148 Schema Evolution Runtime Cutover Status

## Status

Not started.

0.148 should begin after 0.147 closes. The design combines the first supported
schema evolution policy with the generated-bridge retirement needed to make
that policy safe.

## Target Outcome

```text
append-only nullable field added in generated proposal
  -> startup accepts transition
  -> old rows read with NULL for missing field
  -> new rows write current accepted layout
  -> unsupported transitions still fail closed
```

## Checklist

### Transition Policy

* [ ] Add `AppendOnlyAdditiveFields` or equivalent accepted transition plan.
* [ ] Carry added-field facts on the schema-owned plan artifact.
* [ ] Accept nullable append-only fields with no database default.
* [ ] Reject required append-only fields without explicit database default.
* [ ] Keep removals, reorders, type changes, primary-key changes, and nested
      leaf changes rejected.

### Row Decode

* [ ] Separate physical row slot count from accepted live slot count.
* [ ] Allow old rows to have fewer slots only under an accepted additive plan.
* [ ] Materialize missing nullable added fields as `NULL`.
* [ ] Keep missing primary-key slots fail-closed.
* [ ] Update dense decode.
* [ ] Update sparse indexed decode.
* [ ] Update one-slot required decode.
* [ ] Update lazy `StructuralSlotReader`.
* [ ] Preserve malformed-byte corruption taxonomy.

### Writes

* [ ] Ensure inserts emit current accepted slot count.
* [ ] Ensure updates over old rows preserve existing data and produce valid
      current-layout rows.
* [ ] Ensure replaces follow current accepted layout semantics.
* [ ] Keep required added fields without database defaults rejected before
      persistence.

### Generated Bridge Retirement

* [ ] Remove generated slot-count authority from runtime row decode.
* [ ] Remove generated missing-slot decisions from runtime readers.
* [ ] Remove generated write target-slot decisions from SQL/session write
      paths.
* [ ] Keep generated proposal construction and typed materialization bridges
      explicitly named.

### SQL / Metadata

* [ ] `DESCRIBE <entity>` shows accepted added fields and slots.
* [ ] `SHOW COLUMNS FROM <entity>` shows accepted added fields.
* [ ] `SELECT added_field` returns `NULL` for old rows.
* [ ] Predicates over added nullable fields preserve current `NULL` semantics.

### Validation

* [ ] Schema transition tests.
* [ ] Row decode old-row tests.
* [ ] SQL select/write upgrade tests.
* [ ] Startup/index rebuild tests for old rows.
* [ ] Local `scripts/dev/sql.sh --upgrade` smoke coverage.

## Explicit Deferrals

* Field removal.
* Field rename.
* Slot remapping.
* Type changes.
* Nested shape evolution.
* SQL `ALTER TABLE`.
* Per-row schema versions.
* Full generated `E::MODEL` removal from typed API surfaces.

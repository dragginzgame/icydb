# 0.148 Schema Evolution Runtime Cutover Status

## Status

In progress.

0.148 has started with the runtime decode foundation and the first additive
startup policy. The current implementation accepts only append-only nullable
field additions, publishes the generated snapshot as the new accepted schema,
and lets accepted row contracts read older shorter physical rows.

## Target Outcome

```text
append-only nullable field added in generated proposal
  -> startup accepts transition
  -> old rows read with NULL for missing field
  -> new rows write current accepted layout
  -> unsupported transitions still fail closed
```

## Completed Additive Runtime Foundation

The first 0.148 implementation slice added absence-policy-aware row decode and
the startup transition policy needed to reach it safely:

* `StructuralRowContract` now validates physical slot count separately from
  accepted logical slot count.
* Accepted contracts can synthesize `Value::Null` for missing trailing nullable
  fields through `AcceptedFieldAbsencePolicy::NullIfMissing`.
* Primary-key validation and primary-key projection use accepted field decode
  contracts before falling back to generated field metadata.
* Dense direct, sparse direct, and lazy structural row readers now share the
  row-contract primary-key raw-byte validator instead of each choosing the
  accepted/generated validation branch locally.
* Structural save preflight validates old-row baselines against accepted
  persisted field kinds for query-visible shape, exact decimal scale, text
  bounds, and deterministic set/map encodings.
* Lazy scalar reads, eager all-slot validation, direct projection value-storage
  checks, and dense direct row decode now use accepted field contracts before
  falling back to generated field metadata.
* Default one-slot sparse row decode now opens rows without pre-resolving
  generated selected-slot and primary-key field contracts.
* Accepted structural patch replay now overlays sparse current-layout payloads
  through accepted field decode contracts before generated fallback.
* Persisted-row contract helpers now own accepted-first runtime value decode
  scalar decode, and non-scalar validation selection for callers that already
  carry a `StructuralRowContract`.
* Grouped single-slot row reads now call the same row-contract sparse
  required-slot decoder as other runtime one-slot reads, removing the
  grouped-only generated-field bridge.
* Direct scalar `OCTET_LENGTH(...)` admission can use accepted leaf-codec facts
  without reopening generated field metadata.
* Generated-only contracts still require exact physical slot counts.
* Dense, sparse indexed, one-slot required, and lazy `StructuralSlotReader`
  paths share the same missing-slot behavior.
* Focused tests open a two-slot old row through three-slot accepted contracts
  and verify cached, dense, sparse, compact, and required-slot reads all return
  `NULL` for the missing appended nullable field while a missing appended
  required field still fails closed.
* Schema transition policy accepts only append-only nullable generated fields.
* Reconciliation writes the generated snapshot as the accepted schema when that
  transition is accepted.
* Schema transition metrics now count accepted append-only nullable field
  transitions separately from exact matches and rejection buckets.
* SQL SELECT coverage installs an old accepted two-field schema, inserts a
  two-slot physical row, accepts the nullable additive transition, and returns
  `NULL` for the missing appended field.
* SQL predicate coverage over the appended nullable field verifies `IS NULL`
  matches the old missing-slot row while `IS NOT NULL` and `= NULL` do not.
* SQL metadata coverage verifies `DESCRIBE` and `SHOW COLUMNS` expose the
  accepted appended nullable field and accepted slot after reconciliation.
* Startup recovery coverage now seeds old shorter physical rows and verifies
  secondary index rebuild plus update-marker replay both decode old rows
  through accepted schema contracts.
* Malformed-row coverage verifies accepted nullable compatibility does not hide
  corruption in present physical slots.
* Write-layout coverage verifies fresh inserts after an accepted nullable
  transition physically emit the current accepted slot count.
* Transition-policy coverage explicitly verifies field removal, field reorder,
  primary-key field drift, field type drift, nested-leaf drift, and required
  additive fields remain rejected.
* Unsupported required additive-field diagnostics now name the missing
  required/default absence-policy support instead of implying nullable
  additive decode/write support is absent.
* Existing SQL write coverage verifies unsupported required-field additions
  reject before insert/update/delete mutation staging.
* CLI command-surface coverage verifies the current `icydb canister upgrade`
  and `icydb sql` commands expose the replacement flow for the removed
  `scripts/dev/sql.sh` wrapper.
* Local dfx smoke coverage upgraded `demo_rpg` without resetting stable memory,
  then verified `DESCRIBE character` and a deterministic `SELECT` through
  `icydb sql`.

Next gate: continue generated bridge retirement where accepted field contract
lifetimes can replace remaining generated-compatible write/materialization
bridges without broad typed API changes.

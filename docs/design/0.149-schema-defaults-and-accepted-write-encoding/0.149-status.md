# IcyDB 0.149 - Status

## Current State

In progress.

0.149 has started with accepted write encoding and the first schema-owned
default payload contract. Defaults are now representable as accepted schema
slot payloads, can materialize missing appended row slots, and have a narrow
macro authoring surface through supported literal `default = ...` values and
explicit `db_default = ...` overrides.
Accepted structural insert/replace staging now uses those accepted defaults
when sparse write input omits defaulted fields.
Schema description rows also expose encoded database-default metadata without
decoding default payloads into runtime values.
The default payload boundary is now persisted-slot-byte based end to end:
macro-authored scalar defaults emit scalar slot envelopes, transition policy
validates through accepted field decode contracts, and structural insert/replace
tests cover omitted defaulted fields.
Accepted old-row canonicalization now emits rows through the selected row
contract, so accepted sparse patch replay and accepted row re-emission encode
through accepted field contracts instead of the generated runtime-value
canonicalizer.
Sparse structural patch serialization is now split into accepted and
generated lanes. Accepted writes encode through accepted field contracts only;
generated field-model encoding remains isolated to generated-only compatibility
callers.
Dense runtime-value row emission now has the same split, so accepted old-row
overlays emit final canonical rows through accepted field contracts only.
Mutation write-bound validation also chooses accepted or generated field
authority once before iterating patch or row slots.
Persisted-row direct read helpers now do the same for selected-slot decode,
direct projection, and value-storage scalar reads.
The shared row-contract decode helpers now expose the same accepted/generated
lane split for runtime value decode, scalar decode, and structural validation.
Structural save staging now splits existing-row identity validation, update
after-image construction, and commit before-image construction into accepted
and generated helpers too.
Typed save commit before-image construction follows the same split.
Structural mutation inputs now use separate generated sparse, accepted sparse,
and accepted complete constructors instead of carrying optional schema authority
inside the shared payload constructor.
Existing-row save-rule resolution now also has accepted-contract and
generated-contract entrypoints, so typed and structural save staging select row
identity authority before loading before-images.
Structural patch write-bound validation now follows the same pattern with
accepted-contract and generated-contract entrypoints before scalar bound loops.
Structural update patch replay and typed/structural commit-marker before-image
construction now select accepted or generated helpers directly from the staging
boundary instead of using option-taking helper wrappers.
Accepted before-image canonicalization now lives at the persisted-row data
boundary. Typed and structural save staging pass raw bytes plus the accepted
row-decode contract to data-layer helpers instead of rebuilding structural row
contracts locally.
Delete and typed-response fallback canonicalization now flow through the frozen
`RowLayout` instead of extracting the underlying `StructuralRowContract` at
call sites.
Generated-compatible accepted-schema projection is now centralized in schema
runtime, and accepted snapshot to structural row contract projection is
centralized at the data row-contract boundary.
Session writes, SQL writes, accepted authority setup, commit preflight, and
relation validation consume those helpers instead of locally repeating accepted
descriptor and compatibility proof construction.
Accepted executor row-layout construction now consumes the generated-compatible
row-shape proof directly, so no production helper can attach accepted decode
facts to a runtime layout after silently discarding that proof.
Typed-create authorship validation now reads accepted schema write policy from
the accepted row-decode contract when available, leaving generated `FieldModel`
write policy as the generated-only fallback.
Typed entity save invariant validation now also uses accepted row-decode
contracts for primary-key slot/name resolution and field value checks on
accepted save lanes, leaving generated row-layout validation as the
generated-only fallback.
Missing-field indexed-error context now comes from `SchemaInfo`, keeping that
write-validation detail behind the schema boundary instead of a direct
`E::MODEL` lookup.
Structural after-image primary-key mismatch diagnostics now also resolve the
primary-key field name from `SchemaInfo`.
Typed, batch, and structural save relation-validation gates now read
strong-relation presence from `SchemaInfo` instead of reopening generated
entity metadata at each save entrypoint.
Accepted write/decode helper entrypoints now build accepted-only structural
row contracts from entity path plus `AcceptedRowDecodeContract`, so accepted
lanes no longer carry generated model fallbacks just to serialize patches,
replay accepted sparse updates, or normalize accepted before-images.
SQL write selector routing now reads the primary-key field name from
`SchemaInfo`, and SQL `INSERT ... SELECT` preparation can use the accepted
schema entity name because SQL preparation no longer requires a generated-model
static entity string.
SQL compiled-command preparation now also reads the expected entity name from
`SchemaInfo`, keeping generated model access for the later SQL binding stages
that still consume generated structural metadata.
SQL UPDATE selector lowering now reads the primary-key fallback field name from
`SchemaInfo` before appending the deterministic ORDER BY tie-break.

## Closeout Queue

0.149 is now mostly an authority cleanup line rather than a defaults line. The
remaining patch-sized work should keep generated metadata only where it is a
typed compatibility, schema proposal, describe, or explicit model-only surface.

Recommended next slices:

- Accepted index authority: `SchemaInfo` still imports index membership from
  generated `EntityModel` because accepted schema snapshots do not yet persist
  full index definitions. Either persist enough index contract metadata for
  accepted runtime planning or keep this as the named final generated
  compatibility bridge.
- Model-only fluent planning boundary: session execution paths now mostly pass
  accepted `SchemaInfo`, but standalone fluent `.plan()` and model-only
  structural planning still use generated schema caches. Decide whether those
  remain explicit offline/model-only APIs or add accepted-schema planning
  entrypoints for every production caller.
- Structural explain convenience boundary: direct structural/query explain
  helpers are now explicitly model-only. Remaining decision is whether to keep
  public standalone explain as a model-only API or add accepted-schema
  standalone explain entrypoints.
- Final generated-wrapper tightening: audit remaining production exports around
  `SchemaInfo::cached_for_entity_model` and generated access-choice finalizers.
  Convert remaining runtime callers to accepted variants or mark wrappers
  test-only/model-only.
- Guard coverage: keep broad guards around runtime session/write/read paths so
  `EntityAuthority::for_type::<E>()`, generated `SchemaInfo`, generated scalar
  compilers, and generated predicate compilers cannot re-enter accepted runtime
  paths.

Expected closeout size if accepted index persistence stays out of scope:
two to four focused cleanup slices. If accepted snapshots must own index
definitions in this minor, expect a larger multi-slice design and migration
pass.

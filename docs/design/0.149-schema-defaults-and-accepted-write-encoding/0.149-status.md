# IcyDB 0.149 - Status

## Current State

In progress.

0.149 has started with accepted write encoding and the first schema-owned
default payload contract. Defaults are now representable as accepted schema
slot payloads and can materialize missing appended row slots, but authored
default syntax and default validation are still pending.

## Completed

- Added accepted structural field encoding from `PersistedFieldKind`.
- Added accepted runtime-value field-contract encoding from
  `AcceptedFieldDecodeContract`.
- Routed accepted structural patch serialization through accepted field
  contracts when a row contract is attached.
- Preserved decimal scale normalization in the accepted write encoder.
- Added generated-compatible parity tests for accepted structural field
  encoding.
- Kept `Value` runtime-only; persisted rows remain slot-framed field-codec
  bytes.
- Added `SchemaFieldDefault::SlotPayload(...)` for encoded persisted defaults.
- Added `FieldDatabaseDefault::EncodedSlotPayload(...)` as the generated-model
  bridge for future explicit database defaults.
- Added `DefaultIfMissing` missing-slot policy for accepted row contracts.
- Materialized missing accepted slots from encoded default payloads through the
  accepted field decode contract.
- Accepted append-only generated fields with explicit persisted default payloads
  in transition policy.

## Remaining

- Validate defaults against accepted field kinds during schema reconciliation.
- Add macro/schema syntax for authoring explicit database defaults.
- Encode generated authored defaults into slot payload bytes.
- Encode omitted structural write fields from accepted defaults where the write
  surface permits omission.
- Show default metadata in `DESCRIBE` and `SHOW COLUMNS`.
- Keep unsupported defaults and unsupported transition shapes fail-closed.

## Validation

- `cargo test -p icydb-core structural_field --features sql -- --nocapture`
  passed for the initial accepted encoder slice.
- `cargo test -p icydb-core accepted_row --features sql -- --nocapture`
  passed after defaulted missing-slot materialization.
- `cargo test -p icydb-core schema_transition_policy --features sql -- --nocapture`
  passed after defaulted additive transition admission.

# IcyDB 0.149 - Status

## Current State

In progress.

0.149 has started with accepted write encoding and the first schema-owned
default payload contract. Defaults are now representable as accepted schema
slot payloads, can materialize missing appended row slots, and have a narrow
macro authoring surface through supported literal `default = ...` values and
explicit `db_default = ...` overrides.
Accepted structural insert/replace staging now uses those accepted defaults
when sparse write input omits defaulted fields.
Schema description rows also expose encoded database-default metadata without
decoding default payloads into runtime values.
The default payload boundary is now persisted-slot-byte based end to end:
macro-authored scalar defaults emit scalar slot envelopes, transition policy
validates through accepted field decode contracts, and structural insert/replace
tests cover omitted defaulted fields.
Accepted old-row canonicalization now emits rows through the selected row
contract, so accepted sparse patch replay and accepted row re-emission encode
through accepted field contracts instead of the generated runtime-value
canonicalizer.

## Completed

- Added accepted structural field encoding from `PersistedFieldKind`.
- Added accepted runtime-value field-contract encoding from
  `AcceptedFieldDecodeContract`.
- Routed accepted structural patch serialization through accepted field
  contracts when a row contract is attached.
- Preserved decimal scale normalization in the accepted write encoder.
- Added generated-compatible parity tests for accepted structural field
  encoding.
- Kept `Value` runtime-only; persisted rows remain slot-framed field-codec
  bytes.
- Added `SchemaFieldDefault::SlotPayload(...)` for encoded persisted defaults.
- Added `FieldDatabaseDefault::EncodedSlotPayload(...)` as the generated-model
  bridge for future explicit database defaults.
- Added `DefaultIfMissing` missing-slot policy for accepted row contracts.
- Materialized missing accepted slots from encoded default payloads through the
  accepted field decode contract.
- Accepted append-only generated fields with explicit persisted default payloads
  in transition policy.
- Added explicit `db_default = ...` field macro syntax for overriding database
  defaults separately from Rust construction defaults.
- Encoded supported required primitive default literals into generated
  `FieldDatabaseDefault::EncodedSlotPayload(...)` metadata.
- Made supported literal `default = ...` values emit database defaults by
  default, while function/path defaults such as `Ulid::generate` remain
  Rust-only.
- Added fail-closed derive validation for unsupported default targets,
  optional/repeated fields, text bounds, literal-kind mismatches, and numeric
  range overflow.
- Validated append-only explicit default payloads through the accepted field
  kind during transition policy and rejected defaults that decode as `NULL`.
- Routed accepted structural insert/replace serialization through a complete
  after-image serializer that fills omitted nullable/defaulted slots from the
  accepted row contract.
- Added generated and accepted schema description metadata for encoded database
  defaults using `default=slot_payload(bytes=N)`.
- Fixed macro-authored scalar database defaults to emit persisted scalar slot
  envelopes instead of Structural Binary field payloads.
- Moved transition-policy default validation onto
  `AcceptedFieldDecodeContract` so schema defaults are checked through the same
  accepted decode boundary runtime row materialization uses.
- Added structural insert/replace tests proving omitted defaulted fields are
  filled from accepted schema defaults and replace does not inherit old values.
- Added a transition-policy guard proving database-default changes on existing
  fields remain rejected.
- Added an old-row canonical re-emission guard proving defaulted appended fields
  are written into the current accepted row image.
- Routed accepted structural row re-emission and accepted sparse patch replay
  through a row-contract canonical row emitter.
- Removed the dead generated-model runtime-value canonicalizer helper.

## Remaining

- Keep unsupported defaults and unsupported transition shapes fail-closed.

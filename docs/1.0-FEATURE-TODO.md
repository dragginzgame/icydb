# IcyDB 1.0 — FEATURE TODO

This defines the minimum feature surface and architectural completeness required for IcyDB 1.0.

---

# 1. Query Surface (Language Completeness)

## 1.1 Core Query Surface Closure

Baseline single-entity query forms are already present. The remaining 1.0 work
here is completeness and hardening at the edges, not landing `SELECT`/`WHERE`/
`GROUP BY`/`HAVING`/`ORDER BY`/`LIMIT` for the first time.

* [ ] Cross-form interaction remains complete where semantically valid
* [ ] Stable continuation / cursor-based pagination remains fully hardened

---

## 1.2 Result Ordering Semantics (Required)

* [ ] Default result ordering is undefined unless `ORDER BY` is explicitly specified
* [ ] Without `ORDER BY`, engine-specific access-path order may be used but must not be relied upon
* [ ] For one execution plan on one snapshot, unordered results remain deterministic
* [ ] `ORDER BY` null ordering is explicitly defined (`NULL` sorts before non-null values)

---

## 1.3 Projection (Bounded, With Remaining Edge Hardening)

### Structural Projection

* [ ] Select explicit field subsets
* [ ] Deterministic output shape
* [ ] Typed return surface
* [ ] Field omission semantics defined
* [x] Bounded scalar computed projection family
* [ ] Remaining computed boolean projection hardening:
  * [ ] Keep the admitted forms explicitly bounded
  * [ ] Preserve the distinction between scalar projection expressions and
    broad boolean expression grammar
  * [ ] Keep unsupported grouped boolean computed projections fail-closed until
    a dedicated slice admits them

### Aggregate Projection

* [ ] Grouped fields + aggregate outputs
* [ ] Projection validation against GROUP BY
* [ ] HAVING fields not auto-projected
* [ ] Deterministic aggregate naming / aliasing

* [ ] Index pushdown compatibility checks

# 12. 1.0 Completion Criteria

IcyDB 1.0 is achieved when:

* Projection is complete for structural, aggregate, and admitted bounded scalar
  expression outputs.
* Aggregation is strategy-stable.
* Planner model is frozen.
* Continuation envelope is stable.
* Numeric core is unified.
* Storage invariants are formally documented.
* No known semantic authority drift exists.
* Non-goals are enforced in documentation and design.

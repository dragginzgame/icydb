# IcyDB 1.0 TODO

This is the operational checklist for work that must not be forgotten before
1.0. The high-level feature contract lives in `docs/1.0-FEATURES.md`.

---

## Query Surface

* [ ] Cross-form interaction remains complete where semantically valid.
* [ ] Stable continuation / cursor-based pagination remains fully hardened.
* [ ] Default result ordering is undefined unless `ORDER BY` is explicitly
  specified.
* [ ] Without `ORDER BY`, engine-specific access-path order may be used but
  must not be relied upon.
* [ ] For one execution plan on one snapshot, unordered results remain
  deterministic.
* [ ] `ORDER BY` null ordering is explicitly defined (`NULL` sorts before
  non-null values).

---

## Projection

* [ ] Select explicit field subsets.
* [ ] Deterministic output shape.
* [ ] Typed return surface.
* [ ] Field omission semantics defined.
* [x] Bounded scalar computed projection family.
* [ ] Remaining computed boolean projection hardening:
  * [ ] Keep the admitted forms explicitly bounded.
  * [ ] Preserve the distinction between scalar projection expressions and
    broad boolean expression grammar.
  * [ ] Keep unsupported grouped boolean computed projections fail-closed until
    a dedicated slice admits them.
* [ ] Grouped fields + aggregate outputs.
* [ ] Projection validation against `GROUP BY`.
* [ ] `HAVING` fields not auto-projected.
* [ ] Deterministic aggregate naming / aliasing.

---

## Predicate Engine

* [ ] Typed comparison operators.
* [ ] Range predicates.
* [ ] Logical `AND` / `OR`.
* [ ] Parenthesis grouping.
* [ ] Null semantics defined.
* [ ] Deterministic predicate normalization.
* [ ] Index pushdown compatibility checks.
* [ ] No expression evaluation inside storage layer.
* [ ] Storage pushdown eligibility rules are explicit:
  * [ ] Equality predicates (`field = literal`, `field IN literal-set`) may
    push down when index-compatible.
  * [ ] Range predicates (`<`, `<=`, `>`, `>=`, bounded ranges) may push down
    when index order compatibility is preserved.
  * [ ] Field-vs-field, computed, or non-canonical predicates must remain
    post-fetch unless proven equivalent to canonical field predicates.

# IcyDB 1.0 TODO

This is the operational checklist for work that must not be forgotten before
1.0. The high-level feature contract lives in `docs/1.0-FEATURES.md`.

---

## Query Surface

* [x] Public SQL surface proof matrix covers the supported and intentionally
  unsupported single-entity SQL families through `execute_sql_query::<E>(...)`,
  `execute_sql_update::<E>(...)`, and `execute_sql_ddl::<E>(...)`.
* [x] Public SQL cross-form interaction is proven for admitted scalar and
  grouped query forms where semantically valid.
* [ ] Stable continuation / cursor-based pagination remains fully hardened.
* [ ] Default result ordering is undefined unless `ORDER BY` is explicitly
  specified.
* [ ] Without `ORDER BY`, engine-specific access-path order may be used but
  must not be relied upon.
* [ ] For one execution plan on one snapshot, unordered results remain
  deterministic.
* [x] Public SQL `ORDER BY` null ordering is explicitly defined and proven:
  `ASC` sorts `NULL` before present values, while `DESC` reverses the nullable
  comparator.

* [x] Public SQL EXPLAIN proof shows index-compatible predicates using an index

* [ ] No expression evaluation inside storage layer.
* [ ] Storage pushdown eligibility rules are explicit:
  * [ ] Equality predicates (`field = literal`, `field IN literal-set`) may

  * [ ] Field-vs-field, computed, or non-canonical predicates must remain
    post-fetch unless proven equivalent to canonical field predicates.

## Explicit 1.0 Non-Goals

These are intentionally out of scope for 1.0; they are not implementation
checklist items.

* Relational joins.
* Cost-based optimizer.
* Window functions.
* Recursive queries.
* Implicit transactional inference.
* Heuristic plan instability.
* Unbounded projection expressions beyond the admitted bounded scalar
  expression family.
* Distributed query execution.

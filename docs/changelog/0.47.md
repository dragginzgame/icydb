# IcyDB 0.47

This document aggregates detailed release notes for the `0.47.x` minor line.

## 0.47.1 - 2026-03-09 - Audit Pass Structural Follow-Through

### Summary

`0.47.1` continues the `0.47` audit-pass hardening track by decomposing the
load execution hotspot and aligning invariant/guard coverage with the new
module layout.

### Changed

- Converted `db::executor::load::execute` from one file into a directory module:
  - `load/execute/mod.rs` now owns execution-input and finalization
    orchestration boundaries.
  - `load/execute/fast_path.rs` now owns fast-path/fallback key-stream
    resolution policy.
- Reduced load execute module concentration from one `657 LOC` file to two
  focused modules (`332 LOC` + `343 LOC`) to stay within the `0.47`
  structural-pressure decomposition policy.
- Updated index-range-spec invariant governance to follow the new module
  layout in `scripts/ci/check-index-range-spec-invariants.sh`.
- Updated runtime route-consumer guard coverage so execution-mode-field access
  checks scan `load/execute/mod.rs` and `load/execute/fast_path.rs`.
- Preserved layer-authority invariants after the split, including:
  - `AccessPath decision owners: 3`
  - `RouteShape decision owners: 3`

### Testing

- `cargo test -p icydb-core -q`

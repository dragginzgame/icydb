## 0.47.5 - 2026-03-09 - Route/Validate Branch Containment and DRY Governance Hardening

### Summary

`0.47.5` continues the `0.47` structural-pressure track by cutting additional
route/validation branch surfaces and by hardening DRY audit governance so
high-risk duplication seams always produce explicit owner-bound follow-ups.

### Changed

- Reduced route branch pressure in route-owned fast-path/capability/guard gates:
  - `db/executor/route/guard.rs` now enforces fast-path arity constraints
    through fail-closed combinator guards.
  - `db/executor/route/fast_path.rs` now uses fail-closed PK stream
    verification guards and simplified predicate-uncertainty routing checks.
  - `db/executor/route/capability.rs` now centralizes order-contract gate
    eligibility through one route-owned boolean contract.
  - Route subtree branch proxy now trends from `37 -> 18` in this pass
    (`89` baseline from the 2026-03-09 audit start).
- Continued route-planner/validate branch containment by flattening tuple policy
  gates and removing conditional ladders in:
  - `db/query/plan/validate/{cursor_policy.rs,fluent_policy.rs}`
  - `db/executor/route/planner/execution/{shape_aggregate_count.rs,shape_aggregate_non_count.rs}`
  - `db/executor/route/fast_path.rs`
  while preserving planner/runtime behavior and route authority invariants
  (`AccessPath decision owners: 3`, `RouteShape decision owners: 3`).
- Reduced grouped validation branch pressure by converting grouped cursor/policy
  and ORDER validation gates to fail-closed combinator contracts:
  - `db/query/plan/validate/grouped/{cursor.rs,policy/mod.rs,projection_expr.rs,structure.rs}`
  - `db/query/plan/validate/order.rs`
  - Validation subtree branch proxy now trends from `51 -> 9` in this pass
    (`74` baseline from the 2026-03-09 audit start).
- Decomposed load projection-eval operator concentration by converting
  `db/executor/load/projection/eval.rs` into:
  - `db/executor/load/projection/eval/mod.rs` for row-context field and grouped
  aggregate slot resolution, and
  - `db/executor/load/projection/eval/operators.rs` for unary/binary operator
    semantics.
  This keeps operator arithmetic/boolean/equality/ordering behavior isolated
  from row/slot lookup wiring and reduces single-file concentration in the load
  projection hub (`396 LOC -> 162 + 253`).
- Decomposed load terminal ranking concentration by converting
  `db/executor/load/terminal/ranking.rs` into:
  - `db/executor/load/terminal/ranking/mod.rs` for terminal API entrypoints and
    planner-slot resolution wiring, and
  - `db/executor/load/terminal/ranking/materialized.rs` for deterministic
    top-k/bottom-k ranking over materialized rows.
  This keeps terminal entrypoint wiring separated from ranking insertion/tie
  break mechanics (`399 LOC -> 176 + 249`) and narrows the `executor/load`
  gravity-well surface.
- Decomposed grouped validation runtime/test concentration by converting
  `db/query/plan/validate/grouped/projection_expr.rs` and
  `db/query/plan/validate/grouped/structure.rs` into directory modules with
  dedicated `tests.rs` files:
  - `grouped/projection_expr/{mod.rs,tests.rs}`
  - `grouped/structure/{mod.rs,tests.rs}`
  This keeps planner runtime validation helpers isolated from test matrices
  while preserving grouped validation behavior and branch metrics.
- Continued grouped validation decomposition by converting:
  - `db/query/plan/validate/grouped/cursor.rs -> grouped/cursor/{mod.rs,tests.rs}`
  - grouped policy tests from
    `db/query/plan/validate/grouped/policy/mod.rs` into
    `db/query/plan/validate/grouped/policy/tests.rs`
  so grouped runtime cursor/policy gate code stays focused on planner behavior
  while grouped validation test matrices move to dedicated test modules.
- Hardened recurring DRY governance so reports cannot leave high-risk divergence
  seams actionless:
  - `docs/audits/AUDIT-HOWTO.md` now requires follow-up actions for DRY reports
    whenever `High-risk divergence-prone patterns > 0`, even if DRY index `< 6`.
  - `docs/audits/recurring/crosscutting/crosscutting-dry-consolidation.md` now
    requires a mandatory high-risk divergence ledger with owner boundary and
    cycle-worthiness rationale.

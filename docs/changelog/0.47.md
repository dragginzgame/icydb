# IcyDB 0.47

This document aggregates detailed release notes for the `0.47.x` minor line.

## 0.47.1 - 2026-03-09 - Audit Pass Structural Follow-Through

### Summary

`0.47.1` continues the `0.47` audit-pass hardening track by decomposing the
load execution hotspot and aligning invariant/guard coverage with the new
module layout.

### Changed

- Converted `db::executor::load::execute` from one file into a directory module:
  - `load/execute/mod.rs` now owns execution-input and finalization
    orchestration boundaries.
  - `load/execute/fast_path.rs` now owns fast-path/fallback key-stream
    resolution policy.
- Reduced load execute module concentration from one `657 LOC` file to two
  focused modules (`332 LOC` + `343 LOC`) to stay within the `0.47`
  structural-pressure decomposition policy.
- Updated index-range-spec invariant governance to follow the new module
  layout in `scripts/ci/check-index-range-spec-invariants.sh`.
- Updated runtime route-consumer guard coverage so execution-mode-field access
  checks scan `load/execute/mod.rs` and `load/execute/fast_path.rs`.
- Preserved layer-authority invariants after the split, including:
  - `AccessPath decision owners: 3`
  - `RouteShape decision owners: 3`

### Testing

- `make check-invariants`
- `cargo check -p icydb-core`
- `cargo clippy -p icydb-core -- -D warnings`
- `cargo test -p icydb-core -q`
- `make test`

## 0.47.0 - 2026-03-09 - Audit Pass Kickoff and Governance Cleanup

### Summary

`0.47.0` starts the audit-pass line with governance prep, recurring-audit
coverage expansion, baseline risk readout capture, and documentation
re-organization for lower operational overhead.

### Changed

- Completed kickoff prep for recurring audit execution and storage:
  - standardized report-path convention to `YYYY-MM/YYYY-MM-DD`.
  - added month-level report summaries for roll-up visibility.
  - co-located versioned design and status tracking for active slices.
- Expanded recurring audit definitions with accepted feedback:
  - strengthened resource-model compliance checks beyond policy-only validation
    (runtime enforcement and budget lifecycle coverage).
  - upgraded state-machine integrity checks with explicit exclusivity,
    transition completeness, illegal-transition rejection, and replay authority
    proof requirements.
  - strengthened boundary-envelope semantics checks with explicit monotonic
    resume proof, logical-to-raw bound mapping, and empty-envelope handling.
- Recorded `0.47` baseline audit summary and follow-up targets:
  - structural risk emphasis remained on complexity/velocity coordination
    pressure rather than storage-correctness regressions.
  - decision-owner containment and branch-surface reduction were set as primary
    short-cycle outcomes for the line.
- Refactored documentation organization to reduce audit/design sprawl:
  - normalized recurring audit domain tree and naming conventions.
  - grouped report history by month/day to keep directories navigable.
  - aligned design/status conventions under versioned design subdirectories.

### Testing

- `make check-invariants`
- `cargo check -p icydb-core`

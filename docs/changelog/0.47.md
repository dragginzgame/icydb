## 0.47.2 - 2026-03-09 - Audit Pass Structural Containment Continuation

### Summary

`0.47.2` continues `0.47` structural containment by splitting the remaining
route/load grouped execution hubs into narrower contract modules while keeping
planner/runtime behavior and invariant outcomes stable.

### Changed

- Decomposed `db::executor::route::contracts::execution` into focused modules
  so route mode/shape/spec contracts, grouped route observability payloads, and
  execution-route plan behavior no longer co-reside in one file.
- Decomposed `db::executor::load::contracts::grouped` into route-stage contract
  and stream/fold stage modules so grouped payload ownership and stage payload
  ownership are isolated.
- Decomposed `db::executor::load::grouped_runtime` into continuation contracts
  and runtime projection/context modules so grouped cursor-window semantics are
  isolated from grouped runtime observability plumbing.
- Decomposed `db::executor::route::hints` from one file into
  `route/hints/{mod.rs,load.rs,aggregate.rs}` so load-window/index-range hint
  eligibility contracts are isolated from aggregate bounded-probe/seek hint
  policy.
- Updated route-hint guard scan coverage to read all new route hint module
  files so runtime guard tests continue enforcing route-window and
  budget-safety filter gate contracts after the split.
- Preserved route/access authority and guardrail invariants through the split
  sequence:
  - `AccessPath decision owners: 3`
  - `RouteShape decision owners: 3`
  - zero cross-layer policy re-derivations in invariant checks

### Testing

- `cargo check -p icydb-core`
- `cargo clippy -p icydb-core -- -D warnings`
- `cargo test -p icydb-core grouped_route_strategy_to_metrics_strategy_mapping_is_stable -- --nocapture`
- `cargo test -p icydb-core route_feature_budget_shape_kinds_stay_within_soft_delta -- --nocapture`
- `cargo test -p icydb-core route_plan_mutation_is_materialized_with_no_fast_paths_or_hints -- --nocapture`
- `cargo test -p icydb-core route_plan_mutation_rejects_non_delete_mode -- --nocapture`
- `cargo test -p icydb-core route_hints_use_route_window_and_budget_safety_filter_gates -- --nocapture`
- `cargo test -p icydb-core --test compile_fail -- --nocapture`
- `make check-invariants`

## 0.47.1 - 2026-03-09 - Audit Pass Structural Follow-Through

### Summary

`0.47.1` continues the `0.47` audit-pass hardening track by decomposing the
load and grouped-contract execution hotspots, tightening grouped DISTINCT
planner/runtime contracts, and aligning invariant/guard coverage with the new
module layout.

### Changed

- Converted `db::executor::load::execute` from one file into a directory module:
  - `load/execute/mod.rs` now owns execution-input and finalization
    orchestration boundaries.
  - `load/execute/fast_path.rs` now owns fast-path/fallback key-stream
    resolution policy.
- Reduced load execute module concentration from one `657 LOC` file to two
  focused modules (`332 LOC` + `343 LOC`) to stay within the `0.47`
  structural-pressure decomposition policy.
- Converted `db::executor::aggregate::contracts::grouped` from one file into a
  directory module:
  - `aggregate/contracts/grouped/context.rs` now owns grouped budget/config
    and DISTINCT-admission policy plumbing.
  - `aggregate/contracts/grouped/engine.rs` now owns grouped state slots,
    ingest, and deterministic grouped finalization.
- Reduced grouped aggregate contracts concentration from one `683 LOC` file to
  focused modules (`411 LOC` + `289 LOC`, with `mod.rs` at `13 LOC`) to keep
  the high-growth aggregate contract surface below soft-cap guidance.
- Updated index-range-spec invariant governance to follow the new module
  layout in `scripts/ci/check-index-range-spec-invariants.sh`.
- Updated runtime route-consumer guard coverage so execution-mode-field access
  checks scan `load/execute/mod.rs` and `load/execute/fast_path.rs`.
- Reduced grouped DISTINCT policy/runtime duplication by changing planner
  handoff lowering from one `GlobalDistinctFieldAggregate { kind, ... }` shape
  to explicit strategy variants:
  - `GlobalDistinctFieldCount { target_field }`
  - `GlobalDistinctFieldSum { target_field }`
- Removed runtime grouped DISTINCT aggregate-kind policy re-derivation in load
  execution by consuming the planner-lowered strategy directly.
- Centralized grouped DISTINCT policy-reason projection under
  `GroupDistinctPolicyReason` by moving planner grouped error mapping to
  `planner_group_plan_error(...)`, so planner validation and runtime invariant
  reporting share one reason authority.
- Trimmed route planner hub ownership by moving
  `ExecutionRoutePlan::{for_mutation, grouped_observability}` from planner
  wiring into `db::executor::route::contracts`, keeping route payload behavior
  under route-contract ownership.
- Decomposed route planner entrypoint orchestration by moving route-plan
  entrypoint methods into `db::executor::route::planner::entrypoints`,
  reducing `planner/mod.rs` concentration (`311 LOC -> 103 LOC`) while keeping
  staged route derivation behavior unchanged.
- Decomposed route planner feasibility derivation by converting
  `db::executor::route::planner::feasibility` from one file to a directory
  module:
  - `route/planner/feasibility/mod.rs` now owns feasibility-stage orchestration.
  - `route/planner/feasibility/gates.rs` now owns index-range/load-scan gate
    contexts and first-violation policy checks.
  - `route/planner/feasibility/grouped_strategy.rs` now owns grouped ordered
    runtime revalidation and strategy projection.
  This reduced feasibility root concentration (`405 LOC -> 236 LOC`) and kept
  grouped strategy/runtime-gate policy under planner feasibility ownership.
- Decomposed route contract ownership by converting
  `db::executor::route::contracts` from one file to a directory module:
  - `route/contracts/execution.rs` now owns route plan payload, grouped
    observability, and route execution-mode/shape contracts.
  - `route/contracts/continuation.rs` now owns continuation/access-window
    contracts and tests.
  - `route/contracts/shape.rs` now owns route intent/shape and fast-path-order
    contract constants.
  - `route/contracts/capabilities.rs` now owns route capability snapshots and
    capability-count guards.
  This reduced route contract root concentration from one `914 LOC` file to
  focused modules (`525`, `241`, `103`, `58`, and `mod.rs` at `38 LOC`) while
  preserving route surface symbols required by compile-fail privacy tests.
- Decomposed route execution contracts by converting
  `db::executor::route::contracts::execution` from one file to a directory
  module:
  - `route/contracts/execution/mod.rs` now owns route mode/shape/spec contract
    types and execution-mode guard accounting.
  - `route/contracts/execution/observability.rs` now owns grouped route
    observability payload/rejection taxonomy.
  - `route/contracts/execution/plan.rs` now owns execution-route payload and
    route fast-path/diagnostic behavior.
  This reduced execution-contract root concentration (`525 LOC -> 177 LOC`)
  while preserving route behavior and compile-fail boundary expectations.
- Decomposed load contracts by converting `db::executor::load::contracts` from
  one file to a directory module:
  - `load/contracts/mod.rs` now owns scalar pagination/executor contracts.
  - `load/contracts/grouped.rs` now owns grouped stage/projection contracts.
  This reduced root load-contract concentration and kept grouped contract
  ownership explicit.
- Decomposed grouped load contracts by converting
  `db::executor::load::contracts::grouped` from one file to a directory module:
  - `load/contracts/grouped/mod.rs` now owns grouped contract boundary exports.
  - `load/contracts/grouped/route_stage.rs` now owns grouped planner/route
    payload contracts and grouped route-stage projection interfaces.
  - `load/contracts/grouped/stages.rs` now owns grouped stream/fold stage
    payload contracts.
  This reduced grouped-contract root concentration (`477 LOC -> 13 LOC`) and
  made grouped stage-surface growth explicit under stable load ownership.
- Decomposed grouped runtime contracts by converting
  `db::executor::load::grouped_runtime` from one file to a directory module:
  - `load/grouped_runtime/mod.rs` now owns grouped runtime boundary exports.
  - `load/grouped_runtime/continuation.rs` now owns grouped continuation
    window/capability/context contracts and continuation tests.
  - `load/grouped_runtime/runtime.rs` now owns grouped runtime projection and
    grouped execution-context contracts.
  This reduced grouped-runtime root concentration (`366 LOC -> 14 LOC`) and
  separated continuation contracts from runtime observability context contracts.
- Decomposed load entrypoint pipeline orchestration by converting
  `db::executor::load::entrypoints::pipeline` from one file to a directory
  module:
  - `load/entrypoints/pipeline/mod.rs` now owns mode selection surface and
    staged execution contracts.
  - `load/entrypoints/pipeline/orchestrate.rs` now owns execution-context
    loading, stage-order orchestration, and fail-closed cursor guard flow.
  This reduced entrypoint hub concentration while preserving linear stage
  ordering and continuation gating behavior.
- Decomposed grouped planner-policy validation by converting
  `query::plan::validate::grouped::policy` from one file to a directory module:
  - `grouped/policy/mod.rs` now owns grouped-policy orchestration boundaries.
  - `grouped/policy/rules.rs` now owns grouped-policy context types and
    first-violation rule evaluation helpers.
  This reduced grouped-policy root concentration (`414 LOC -> 234 LOC`) and
  made rule-surface growth explicit without changing grouped DISTINCT/HAVING
  planner outcomes.
- Decomposed plan-validate root taxonomy by extracting error classes and
  conversion mappings from `query::plan::validate::mod` into
  `query::plan::validate::errors`:
  - `validate/mod.rs` now owns module wiring and validator exports.
  - `validate/errors.rs` now owns plan/user/policy/cursor/group/expr taxonomy
    and typed conversion policy.
  This reduced validate root concentration (`507 LOC -> 45 LOC`) and made
  semantic-validation orchestration independent from error taxonomy growth.
- Preserved layer-authority invariants after the split, including:
  - `AccessPath decision owners: 3`
  - `RouteShape decision owners: 3`

## 0.47.0 - 2026-03-09 - Audit Pass Kickoff and Governance Cleanup

### Summary

`0.47.0` starts the audit-pass line with governance prep, recurring-audit
coverage expansion, baseline risk readout capture, and documentation
re-organization for lower operational overhead.

### Changed

- Completed kickoff prep for recurring audit execution and storage:
  - standardized report-path convention to `YYYY-MM/YYYY-MM-DD`.
  - added month-level report summaries for roll-up visibility.
  - co-located versioned design and status tracking for active slices.
- Expanded recurring audit definitions with accepted feedback:
  - strengthened resource-model compliance checks beyond policy-only validation
    (runtime enforcement and budget lifecycle coverage).
  - upgraded state-machine integrity checks with explicit exclusivity,
    transition completeness, illegal-transition rejection, and replay authority
    proof requirements.
  - strengthened boundary-envelope semantics checks with explicit monotonic
    resume proof, logical-to-raw bound mapping, and empty-envelope handling.
- Recorded `0.47` baseline audit summary and follow-up targets:
  - structural risk emphasis remained on complexity/velocity coordination
    pressure rather than storage-correctness regressions.
  - decision-owner containment and branch-surface reduction were set as primary
    short-cycle outcomes for the line.
- Refactored documentation organization to reduce audit/design sprawl:
  - normalized recurring audit domain tree and naming conventions.
  - grouped report history by month/day to keep directories navigable.
  - aligned design/status conventions under versioned design subdirectories.

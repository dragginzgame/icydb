# 0.117 Changelog

## 0.117.0

- `0.117.0` centralizes one more semantic-owner seam around expression families instead of widening SQL. Planner typing in [/home/adam/projects/icydb/crates/icydb-core/src/db/query/plan/expr/type_inference/mod.rs](/home/adam/projects/icydb/crates/icydb-core/src/db/query/plan/expr/type_inference/mod.rs) now owns the result-family meaning for the lowerable searched-`CASE`, `COALESCE`, and `NULLIF` branches that prepared fallback can actually see before bind time, so prepared SQL no longer re-derives those coarse families with its own local branch scans and unifiers inside [/home/adam/projects/icydb/crates/icydb-core/src/db/sql/lowering/prepare.rs](/home/adam/projects/icydb/crates/icydb-core/src/db/sql/lowering/prepare.rs).

- That owner move is intentionally bounded. Planner typing does not claim full pre-bind evaluation of parameterized SQL subtrees. Instead it now owns the specific expression-family rules that were previously split across planner typing and prepared fallback:
  - searched-`CASE` result-branch coarse-family unification through `infer_case_result_exprs_coarse_family(...)`
  - dynamic `COALESCE` / `NULLIF` result-family inference over the visible lowerable arguments through `infer_dynamic_function_result_exprs_coarse_family(...)`
  - the follow-on argument-family propagation rule once one dynamic result family is known through `dynamic_function_arg_coarse_family(...)`

- Before this patch, prepared fallback in [/home/adam/projects/icydb/crates/icydb-core/src/db/sql/lowering/prepare.rs](/home/adam/projects/icydb/crates/icydb-core/src/db/sql/lowering/prepare.rs) was already correctly bounded, but it still owned too much semantic-adjacent meaning:
  - it scanned lowerable searched-`CASE` result branches and unified them locally
  - it scanned lowerable `COALESCE` / `NULLIF` arguments and unified them locally
  - it then separately re-applied the “resolved dynamic result family becomes the argument family” rule when freezing parameter contracts

- After `0.117.0`, prepared fallback still owns prepared-SQL-specific diagnostics and contract freezing, but not the expression-family meaning underneath them:
  - `infer_planner_case_result_family(...)` now lowers the visible non-parameter result branches and asks planner typing for the shared searched-`CASE` family
  - `infer_dynamic_fallback_function_result_family(...)` now lowers the visible non-parameter dynamic-function arguments and asks planner typing for the shared dynamic result family
  - prepared fallback then maps that planner-owned family onto prepared parameter contract families without re-deriving the semantic rule locally

- The same patch also closes one smaller fixed-family split beside that main owner move. Planner canonicalization in [/home/adam/projects/icydb/crates/icydb-core/src/db/query/plan/expr/canonicalize.rs](/home/adam/projects/icydb/crates/icydb-core/src/db/query/plan/expr/canonicalize.rs) no longer keeps its own parallel list of text/numeric compare-operand functions. It now reads that fixed family from planner typing through `function_is_compare_operand_coarse_family(...)`, so the compare-operand function table has one planner owner instead of adjacent copies in typing and canonicalization.

- The final cleanup in this line is structural rather than behavioral. [/home/adam/projects/icydb/crates/icydb-core/src/db/query/plan/expr/type_inference/mod.rs](/home/adam/projects/icydb/crates/icydb-core/src/db/query/plan/expr/type_inference/mod.rs) now uses one internal `infer_folded_exprs_coarse_family(...)` helper for the visible-expression fold path shared by searched-`CASE` result-family inference and dynamic-function result-family inference. That keeps the new planner-owned expression-family surface from immediately re-growing two near-parallel “infer each visible expression, unify, then project to coarse family” loops.

- This remains fail-closed and does not widen prepared SQL. Unsupported parameter placements still fail where they failed before. The patch only changes who owns the expression-family meaning for the already-admitted visible-branch surface.

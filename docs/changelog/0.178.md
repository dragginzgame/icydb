# IcyDB 0.178 Changelog

## 0.178.6

### 📝 Summary

- Moves SQL DDL field-default, field-nullability, and field-rename candidate
  ownership checks behind schema-owned mutation resolvers.
- Preserves existing no-op ordering and SQL bind diagnostics while narrowing
  the SQL DDL frontend to intent binding and diagnostic mapping.

### 🔧 Changed

- Adds schema-owned candidate resolvers for `ALTER COLUMN SET DEFAULT`,
  `ALTER COLUMN DROP DEFAULT`, `ALTER COLUMN SET NOT NULL`,
  `ALTER COLUMN DROP NOT NULL`, and `RENAME COLUMN`.
- Keeps matching metadata no-ops accepted before generated-field ownership
  rejection, preserving the existing behavior for same-name renames,
  matching nullability, and absent defaults.
- Moves rename admission target construction into schema mutation so SQL DDL
  lowering no longer synthesizes a renamed field snapshot for admission.

### 🧪 Testing

- Adds boundary guards proving SQL DDL field metadata changes do not classify
  generated-field ownership directly and do not synthesize rename admission
  snapshots.
- Keeps the existing SQL DDL bind, publication, no-op, and rejection coverage
  green across default, nullability, and rename paths.

## 0.178.5

### 📝 Summary

- Moves more SQL DDL candidate construction behind schema-owned mutation
  authority without changing user-visible DDL behavior.
- Keeps SQL DDL as an authoring frontend: it binds intent and maps typed
  schema-owned candidate failures back into the existing bind diagnostics.

### 🔧 Changed

- Adds schema-owned default payload encoding helpers for `ADD COLUMN DEFAULT`
  and `ALTER COLUMN SET DEFAULT`, so accepted field-codec selection,
  strict-literal canonicalization, and persisted default payload emission no
  longer live in the SQL DDL binder.
- Adds schema-owned `DROP COLUMN` candidate resolution that classifies unknown
  fields, primary-key fields, generated-owned fields, and index-dependent
  fields before SQL maps those typed outcomes to existing diagnostics.
- Adds schema-owned secondary-index candidate construction for SQL DDL
  `CREATE INDEX`, including durable ordinal allocation and SQL DDL origin
  assignment from the accepted snapshot.
- Removes the secondary-index ordinal allocator from `SchemaInfo`, keeping
  projection metadata out of DDL catalog identity allocation.

### 🧪 Testing

- Adds boundary guards proving SQL DDL no longer owns default payload encoding,
  field-drop candidate classification, or secondary-index identity allocation.
- Keeps the existing SQL DDL bind, preparation, publication, no-op, and
  rejection coverage green across default, drop-column, and create-index
  paths.

## 0.178.4

### 📝 Summary

- Moves SQL DDL `ADD COLUMN` field identity allocation back under
  schema-owned mutation authority.
- Closes remaining SQL DDL admission proof gaps for invalid next versions,
  rollback, and direct schema-store writes.

### 🔧 Changed

- Adds a schema-owned additive-field candidate builder that assigns DDL field
  ID, row slot, field origin, and write policy from accepted snapshot state.
- Updates the SQL DDL `ADD COLUMN` binder so it supplies author intent and
  default/type facts, then delegates durable catalog allocation to schema
  mutation code.
- Keeps DDL-owned additive field behavior unchanged while removing local field
  ID and slot allocation helpers from the SQL DDL frontend.

### 🧪 Testing

- Extends SQL DDL admission coverage for invalid `SET SCHEMA VERSION 0` and
  positive rollback attempts, proving both typed admission reasons and
  query-visible `SchemaDdlAdmissionError` variants.
- Adds boundary guards proving SQL DDL/session SQL do not write accepted schema
  storage directly and that `ADD COLUMN` allocation policy stays in schema
  mutation code.
- Keeps the retained-slot `ADD COLUMN` regression coverage green after moving
  field ID and slot allocation.

## 0.178.3

### 📝 Summary

- Closes the remaining SQL DDL diagnostic edge for publication races.
- Tightens DDL schema-version contract parsing without adding new transition
  classes.

### 🔧 Changed

- Maps schema-store publication race losses crossing the SQL DDL execution
  boundary into the query-visible DDL admission taxonomy as
  `PublicationRaceLost`, while preserving the lower store-origin race detail
  at the schema publication boundary.
- Applies the same SQL DDL execution-error mapping across field and index DDL
  publication arms so race-loss diagnostics remain consistent after
  preparation succeeds.
- Rejects duplicate `EXPECT SCHEMA VERSION` and `SET SCHEMA VERSION` clauses
  within the same schema-version contract group, not only when duplicates are
  split across prefix and suffix positions.
- Keeps DDL schema-version contract values parser-literal-only and rejects
  parameterized version values before catalog binding or mutation admission.

### 🧪 Testing

- Adds query-intent error mapping coverage for `PublicationRaceLost`.
- Extends SQL DDL publication-race coverage to prove the low-level store
  detail and query-visible admission detail both remain available at their
  respective boundaries.
- Extends parser coverage for flexible contract ordering, duplicate contract
  clauses, and non-literal schema-version values.

## 0.178.2

### 📝 Summary

- Tightens SQL DDL admission diagnostics without adding new DDL transition
  classes.
- Requires true no-op DDL statements to declare the accepted schema version
  they observed with `EXPECT SCHEMA VERSION`, while still rejecting empty
  `SET SCHEMA VERSION` bumps.

### 🔧 Changed

- Adds stable query-visible SQL DDL admission variants for preparation
  failures, including missing expected/next versions, stale expected versions,
  invalid version literals, empty bumps, version gaps, unsupported transition
  classes, physical-runner gaps, and validation failures.
- Preserves schema-owned version/fingerprint matrix reasons through DDL
  mutation admission and query error detail instead of relying on rendered
  diagnostic text.
- Maps DDL binding, lowering, and schema mutation admission failures into the
  same outer admission taxonomy while keeping existing unsupported runtime
  class behavior for canister callers.

### 🧪 Testing

- Extends SQL DDL admission coverage to assert structured admission variants
  at both preparation and query-execution boundaries.
- Updates no-op DDL coverage so `IF EXISTS` / `IF NOT EXISTS` no-ops use
  `EXPECT SCHEMA VERSION` only and prove no publication or version bump occurs.

## 0.178.1

### 📝 Summary

- Makes SQL DDL accepted-schema publication compare the live accepted identity
  against the identity captured during DDL binding before writing the
  accepted-after snapshot.
- Rejects stale prepared transitions when another schema publication wins the
  race, leaving the newer accepted schema unchanged.

### 🔧 Changed

- Adds a schema-store publication path that rechecks the latest accepted
  catalog identity inside schema publication instead of trusting the
  accepted-before snapshot captured earlier in statement execution.
- Threads the binding-time accepted identity through SQL DDL mutation
  execution, including metadata-only transitions and physical index
  publication adapters.
- Keeps startup reconciliation on the existing unconditional publication path
  while SQL DDL uses the conditional publication gate.

### 🧪 Testing

- Adds regression coverage for a stale single-statement DDL candidate losing a
  publication race to a competing schema update.
- Extends boundary guards around schema-store version-passive publication and
  runner-backed index publication.

## 0.178.0

### 📝 Summary

- Opens the schema migration DDL transition admission line on top of accepted
  catalog authority and explicit schema-version contracts.
- Requires mutating SQL DDL to declare the accepted schema version it expects
  and the next schema version it intends to publish before schema-owned
  mutation planning or publication can run.
- Keeps accepted schema snapshots as runtime authority: DDL authors transition
  intent, while schema-owned code derives candidates, admits or rejects,
  publishes accepted snapshots, and invalidates stale runtime identity.

### ➕ Added

- Adds explicit `EXPECT SCHEMA VERSION` and `SET SCHEMA VERSION` contracts for
  mutating SQL DDL, with declared next versions admitted through the
  schema-owned version/fingerprint gate before publication.
- Defines the first `0.178.1` implementation slice as existing-entity,
  single-statement DDL transition admission over already-supported
  schema-owned transition classes.
- Records the publication-time accepted-identity recheck required to reject
  stale DDL when another schema publication wins the race before publication.

### 🔧 Changed

- Preserves accepted DDL-owned fields, indexes, and retired slots across
  generated reconciliation after DDL publication while keeping plain
  version-only drift fail-closed.
- Makes the DDL version/fingerprint matrix total for missing `EXPECT`, missing
  `SET`, true no-ops, empty bumps, gaps, rollback, and fingerprint-method
  mismatch behavior.
- Narrows draft, relation, entity birth/death, managed/generated ownership,
  online rewrite, and new index-backfill work out of the first implementation
  slice unless a later patch explicitly scopes them.

### 📚 Documentation

- Tightens the active 0.178 design around exact `0.178.1` scope, grammar
  restrictions, stable machine-readable diagnostics, ownership default-reject
  behavior, cursor invalidation by embedded accepted identity, and closeout
  proof groups.

### 🧪 Testing

- Records the intended parser/binder, admission-matrix, publication-path,
  no-op, ownership, and unsupported-transition proof groups for the first
  implementation slice.

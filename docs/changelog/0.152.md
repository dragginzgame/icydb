# 0.152.x

## 0.152.8

`0.152.8` adds the design for the next schema evolution minor without starting
runtime rebuild execution yet. The `0.153` design defines physical schema
mutation runners as the backend that will consume the `0.152` preflight
boundary, execute accepted-schema-backed rebuild or cleanup work, validate the
result, invalidate runtime planning state, and only then allow accepted snapshot
publication.

### Design

- Added the `0.153` schema rebuild runner design document.
- Defined the runner boundary around accepted schema snapshots and
  `SchemaMutationExecutionPlan`, not generated model/index authority.
- Scoped initial runner work to non-unique field-path index rebuilds,
  deterministic expression-index rebuilds, and non-required secondary-index
  cleanup.
- Kept unique index constraints, full row rewrites, SQL DDL syntax, online
  concurrent rebuilds, and generalized migration scripts deferred.
- Made staged store visibility a first-class design invariant, so rebuilt or
  cleanup-affected index state must remain hidden until accepted snapshot
  publication.
- Added snapshot identity/runtime epoch requirements so accepted schema
  publication invalidates or distinguishes stale planner, visible-index,
  prepared-plan, row-layout, explain, and cursor state.
- Clarified validation semantics for full-scan live-row authority, duplicate
  handling, ordering checks, deleted/tombstoned rows, and deferred concurrency
  semantics.
- Recorded the required phase order:
  preflight, physical work, validation, invalidation, publication, diagnostics.

### Validation

- `git diff --check`

## 0.152.7

`0.152.7` adds schema-owned runner capability derivation, admission checks, a
runner preflight shell, and publication-boundary preflight checks for mutation
execution plans without wiring a physical runner or widening publication.
Execution plans now expose the exact coarse capabilities a future runner must
satisfy before accepted rebuild or cleanup steps can run.

### Runner Capabilities

- Added internal runner capability contracts for accepted field-path index
  rebuilds, accepted expression-index rebuilds, accepted secondary-index
  cleanup, physical work validation, runtime invalidation, and full row
  rewrites.
- Added execution-plan capability derivation from accepted execution steps,
  deduplicating repeated capabilities and ignoring unsupported actions so they
  cannot be mistaken for executable work.
- Added runner admission checks that classify execution plans as publishable
  now, runner-ready, missing required capabilities, or rejected.
- Added a runner contract/preflight shell that deduplicates advertised
  capabilities and maps execution plans to no-work, ready, missing-capability,
  or rejected before any future physical runner starts.
- Routed transition publication checks through runner preflight using an empty
  runner contract in startup reconciliation, keeping rebuild-required mutation
  publication fail-closed until physical execution exists.
- Added write-boundary guard coverage that pins startup reconciliation to the
  runner-preflight publication boundary instead of the raw publication-status
  check.
- Kept metadata-only plans capability-free and kept rewrite/unsupported plans
  outside the publishable rebuild path.
- Closed the 0.152 design docs around the intended endpoint: catalog-native
  mutation planning, compatibility classification, fail-closed publication, and
  runner preflight are in scope; SQL DDL and physical rebuild execution are
  deferred.
- Added focused mutation tests for metadata-only, field-path rebuild,
  expression-index rebuild, secondary-index cleanup, full rewrite, and
  unsupported capability derivation, including missing-capability admission and
  runner preflight/publication behavior.

## 0.152.6

`0.152.6` adds the first schema-owned mutation execution boundary without
making rebuild-required mutations publishable. Mutation plans can now derive an
execution-facing plan that distinguishes immediately publishable metadata-only
work from index rebuild/cleanup work that needs a future physical runner and
from unsupported rewrite-class mutations.

### Execution Boundary

- Added internal mutation execution readiness contracts for publishable-now,
  physical-runner-required, and unsupported mutation plans.
- Added execution steps for accepted field-path index rebuilds, accepted
  expression-index rebuilds, accepted secondary-index cleanup, physical work
  validation, runtime invalidation, full row rewrites, and unsupported actions.
- Added an execution gate that reduces an execution plan to ready-to-publish,
  awaiting-physical-work, or rejected status for future callers.
- Routed execution-plan derivation through the existing schema-owned rebuild
  plan so rebuild classification, physical work contracts, and publication
  policy stay separate.
- Kept startup reconciliation fail-closed for rebuild-required plans; this slice
  only names the future runner contract.
- Added focused mutation tests proving metadata-only plans have no execution
  steps, index cleanup schedules cleanup before validation/invalidation, and
  rewrite/unsupported plans remain non-executable.

### Release Workflow

- Split `make patch`, `make minor`, and `make major` into bump-only targets
  that run the release gate and mutate version/release-surface files without
  staging, committing, tagging, or pushing.
- Added explicit `release-stage`, `release-commit`, and `release-push` targets,
  plus one-shot `release-patch`, `release-minor`, and `release-major` targets
  for human-owned releases.
- Added an IcyDB release-surface sync script that updates the root README
  workspace version and git tag examples after `cargo set-version`.
- Updated release governance and security-check guidance to match the staged
  release flow.

## 0.152.5

`0.152.5` adds accepted secondary-index drop cleanup targets without making
index drops runtime-visible through schema mutation publication. Drop requests
now lower from accepted persisted index snapshots into schema-owned cleanup
targets that carry the persisted store identity needed by a future physical
cleanup runner.

### Accepted Drop Cleanup Targets

- Added internal secondary-index drop cleanup target contracts derived from
  accepted `PersistedIndexSnapshot` metadata.
- Changed non-required secondary-index drop planning to carry accepted index
  ordinal, name, store path, uniqueness flag, and predicate SQL metadata instead
  of only an index name.
- Kept secondary-index drops rebuild-gated and non-publishable until a physical
  cleanup runner owns store cleanup and planner/cache invalidation.
- Kept unique secondary-index drops fail-closed until constraint-removal policy
  is explicit.
- Removed the now-dead generic `BuildIndex { name, kind }` rebuild action after
  field-path and expression index builds moved to dedicated accepted targets.

snapshots into schema-owned targets that preserve mixed key-item order,

## 0.152.2

`0.152.2` adds schema-owned rebuild planning contracts and the internal mutation
request-lowering surface for catalog mutations without widening runtime
publication. Rebuild-required mutations now describe the physical work they
imply while remaining blocked until a future slice owns execution and
validation.

### Rebuild Planning Contracts

- Added internal `SchemaRebuildPlan`, `SchemaRebuildAction`, and
  `SchemaRebuildIndexKind` contracts derived from `MutationPlan`.
- Classified non-unique field-path index additions, accepted expression-index
  additions, secondary-index drops, full row rewrites, and unsupported
  mutations as explicit rebuild actions.
- Routed mutation publication status through the rebuild plan for metadata-safe
  plans, preserving fail-closed behavior for any plan that still requires
  physical work before accepted runtime visibility.
- Added mutation tests proving index add/drop plans derive concrete rebuild
  actions and unsupported/full-rewrite plans stay explicit.

### Mutation Request Lowering

- Added internal `SchemaMutationRequest` as the schema-owned request vocabulary
  for exact matches, append-only fields, index additions, expression-index
  additions, secondary-index drops, nullability changes, and incompatible
  mutations.
- Added `schema_mutation_request_for_snapshots` so accepted snapshot/proposal
  diffs lower into requests before becoming `MutationPlan` values.
- Routed transition policy through `SchemaMutationRequest::lower_to_plan`
  instead of constructing mutation plans directly in transition code.
- Added mutation tests proving snapshot diffs lower into append-only requests
  and that index/drop/unsupported request variants still derive the same
  rebuild-gated plans.

## 0.152.0

`0.152.0` starts catalog-native schema mutation groundwork after the accepted
schema/index authority migration. The initial slice introduces internal mutation
plan contracts, rebuild classifications, deterministic mutation-plan
fingerprints, and design docs that make SQL DDL a future frontend onto accepted
snapshot mutation semantics rather than the migration authority.

### Catalog-Native Mutation Groundwork

- Added internal `SchemaMutation`, `MutationCompatibility`,
  `RebuildRequirement`, and `MutationPlan` contracts under `db::schema`.
- Routed the existing append-only nullable/default-backed field reconciliation
  acceptance through a mutation plan while preserving current runtime behavior.
- Added a reconciliation publication guard so accepted-runtime schema snapshots
  can only publish mutation plans that are metadata-safe and require no rebuild,
  keeping index/full-rewrite mutation plans staged until physical rebuild
  orchestration exists.
- Moved snapshot-delta shape detection into the mutation layer with
  `SchemaMutationDelta` and `classify_schema_mutation_delta`, so transition
  policy consumes catalog-native mutation classification instead of detecting
  append-only field shapes locally.
- Added deterministic mutation-plan fingerprinting for audit identity.
- Classified future additive indexes, expression indexes, index drops,
  nullability changes, and incompatible rewrites into explicit rebuild or
  fail-closed buckets.
- Added schema-mutation and DDL-roadmap design docs for the 0.152 direction.

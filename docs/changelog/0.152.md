# 0.152.x

## 0.152.2

`0.152.2` adds schema-owned rebuild planning contracts and the internal mutation
request-lowering surface for catalog mutations without widening runtime
publication. Rebuild-required mutations now describe the physical work they
imply while remaining blocked until a future slice owns execution and
validation.

### Rebuild Planning Contracts

- Added internal `SchemaRebuildPlan`, `SchemaRebuildAction`, and
  `SchemaRebuildIndexKind` contracts derived from `MutationPlan`.
- Classified non-unique field-path index additions, accepted expression-index
  additions, secondary-index drops, full row rewrites, and unsupported
  mutations as explicit rebuild actions.
- Routed mutation publication status through the rebuild plan for metadata-safe
  plans, preserving fail-closed behavior for any plan that still requires
  physical work before accepted runtime visibility.
- Added mutation tests proving index add/drop plans derive concrete rebuild
  actions and unsupported/full-rewrite plans stay explicit.

### Mutation Request Lowering

- Added internal `SchemaMutationRequest` as the schema-owned request vocabulary
  for exact matches, append-only fields, index additions, expression-index
  additions, secondary-index drops, nullability changes, and incompatible
  mutations.
- Added `schema_mutation_request_for_snapshots` so accepted snapshot/proposal
  diffs lower into requests before becoming `MutationPlan` values.
- Routed transition policy through `SchemaMutationRequest::lower_to_plan`
  instead of constructing mutation plans directly in transition code.
- Added mutation tests proving snapshot diffs lower into append-only requests
  and that index/drop/unsupported request variants still derive the same
  rebuild-gated plans.

## 0.152.1

`0.152.1` tightens the rebuild-publication boundary for catalog-native schema
mutations and refreshes the audit baseline before the rebuild-orchestration
work starts.

### Mutation Publication Gate

- Added schema-owned `MutationPublicationStatus` and
  `MutationPublicationBlocker` contracts so `MutationPlan` decides whether a
  plan can be published as accepted runtime schema.
- Moved reconciliation onto the mutation publication gate instead of checking
  compatibility and rebuild enums locally, keeping rebuild-required and
  non-metadata-safe plans fail-closed through one schema-owned decision.
- Added mutation tests proving only metadata-safe/no-rebuild plans are
  publishable and that rebuild-required index plans stay blocked before
  accepted snapshot publication.

### Audit Coverage

- Refreshed the stale boundary/envelope audit and added a DESC
  empty-envelope regression for cursor continuation symmetry.
- Simplified the recurring index-integrity audit report scope to
  `index-integrity`, added accepted runtime index authority and catalog
  mutation readiness sections, and ran a new `Method V4` report.
- Simplified the recurring cursor-ordering audit report scope to
  `cursor-ordering`, added accepted runtime cursor authority checks, split
  scalar/grouped cursor matrices, and ran a new `Method V4` report.
- Simplified the recurring invariant-preservation audit report scope to
  `invariant-preservation`, added accepted runtime authority and catalog
  mutation publication invariants, and ran a new `Method V4` report.

## 0.152.0

`0.152.0` starts catalog-native schema mutation groundwork after the accepted
schema/index authority migration. The initial slice introduces internal mutation
plan contracts, rebuild classifications, deterministic mutation-plan
fingerprints, and design docs that make SQL DDL a future frontend onto accepted
snapshot mutation semantics rather than the migration authority.

### Catalog-Native Mutation Groundwork

- Added internal `SchemaMutation`, `MutationCompatibility`,
  `RebuildRequirement`, and `MutationPlan` contracts under `db::schema`.
- Routed the existing append-only nullable/default-backed field reconciliation
  acceptance through a mutation plan while preserving current runtime behavior.
- Added a reconciliation publication guard so accepted-runtime schema snapshots
  can only publish mutation plans that are metadata-safe and require no rebuild,
  keeping index/full-rewrite mutation plans staged until physical rebuild
  orchestration exists.
- Moved snapshot-delta shape detection into the mutation layer with
  `SchemaMutationDelta` and `classify_schema_mutation_delta`, so transition
  policy consumes catalog-native mutation classification instead of detecting
  append-only field shapes locally.
- Added deterministic mutation-plan fingerprinting for audit identity.
- Classified future additive indexes, expression indexes, index drops,
  nullability changes, and incompatible rewrites into explicit rebuild or
  fail-closed buckets.
- Added schema-mutation and DDL-roadmap design docs for the 0.152 direction.

# IcyDB 0.40

This document aggregates detailed release notes for the `0.40.x` minor line.

## 0.40.4 - 2026-03-02 - Load Pipeline Stage-State Hardening

### 📝 Summary

`0.40.4` hardens the unified load execution and grouped-route semantic
ownership boundaries after entrypoint unification.

### 🔧 Changed

- Refined `execute_load` stage handoff in
  `crates/icydb-core/src/db/executor/load/entrypoints.rs` from a
  single optional-slot `LoadPipelineState` to stage-typed artifacts:
  `LoadAccessState` and `LoadPayloadState`.
- Removed optional artifact extraction helpers from load orchestration state and
  made stage inputs required-by-construction at each pipeline boundary.
- Kept the same linear stage order (`build -> access -> grouping -> paging ->
  tracing -> surface`) while reducing coordination-state drift pressure.
- Updated load structure guard baseline in
  `crates/icydb-core/src/db/executor/tests/load_structure.rs` to enforce a
  zero optional-slot contract for pipeline-stage artifacts.
- Removed grouped HAVING semantic policy re-check from executor route
  feasibility and kept grouped strategy policy under planner hint authority.
- Removed remaining grouped aggregate/DISTINCT semantic re-checks from executor
  grouped-route feasibility; runtime feasibility now revalidates capability
  constraints only.
- Added grouped runtime revalidation-surface guard coverage via
  `grouped_ordered_runtime_revalidation_flag_count_guard` and route budget
  tests.
- Tightened grouped strategy-hint helper visibility and routed route/explain
  consumers through explicit plan-root wrappers:
  `grouped_plan_strategy_hint_for_route(...)` and
  `grouped_plan_strategy_hint_for_explain(...)`.

## 0.40.1 - 2026-03-02 - Executable Access Contract Hardening

### 📝 Summary

`0.40.1` makes executor access routing rely on one executable contract shape
instead of re-reading raw `AccessPath` variants at runtime.

### 🔧 Changed

- Removed runtime dependency on raw `AccessPath` variants inside executor stream
  lowering and route fast-path checks.
- Made `ExecutableAccessPath` carry execution payload directly, while index
  execution shape is derived from `ExecutionBounds`.
- Preserved cursor anchor validation behavior by carrying index-range semantic
  bounds in executable payload instead of borrowing planner path objects.
- Removed the direct access-path stream request route variant and kept routed
  stream resolution on the access-plan request contract.
- Added a structural guard test that fails if executor runtime modules start
  matching `AccessPath::...` variants again.

## 0.40.2 - 2026-03-02 - Continuation and Load Entrypoint Unification

### 📝 Summary

`0.40.2` centralizes continuation protocol orchestration and removes the
separate scalar/grouped load entrypoint flow split by routing both through one
internal load execution pipeline.

### 🔧 Changed

- Added `crates/icydb-core/src/db/executor/continuation/mod.rs` as the
  executor-owned continuation facade for cursor revalidation/runtime bindings,
  grouped paging window derivation, and grouped next-token construction.
- Updated `crates/icydb-core/src/db/executor/load/entrypoints.rs` to use one
  internal dispatch pipeline for scalar and grouped load execution while
  keeping public entrypoint APIs unchanged.
- Routed grouped pagination window derivation and grouped continuation token
  construction through the continuation facade instead of duplicating mechanics
  in grouped fold helpers.

---

## 0.40.3 - 2026-03-02 - Load Structure Guards and Semantic Ownership Tightening

### 📝 Summary

`0.40.3` locks the new load execution spine with structural guard tests and
continues planner-vs-executor ownership cleanup for grouped cursor policy.

### 🔧 Changed

- Added load-structure guard tests that lock one linear `execute_load` stage
  order (`build -> access -> grouping -> paging -> tracing -> surface`) and
  fail if optional stage artifacts in pipeline state grow beyond budget.
- Added test-only guard exports from load entrypoints so structural contracts
  are enforced by type/signature checks instead of source-text scans.
- Removed grouped global-distinct fold direct calls to grouped cursor-policy
  semantic helper and routed production grouped cursor-policy helper access
  through the continuation-facing plan wrapper.
- Tightened grouped cursor-policy helper visibility in planner semantics and
  kept a test-only shim for planner contract tests.

# IcyDB 0.40

This document aggregates detailed release notes for the `0.40.x` minor line.

## 0.40.10 - next

### 📝 Summary

`0.40.10` closes the error taxonomy and index-integrity audit slice by
tightening cursor classification boundaries and adding fail-closed recovery
coverage for unique constraints, reverse-relation index symmetry, and
prefix-lowered key bounds.

### 🔧 Changed

- Tightened cursor-plan runtime classification in
  `crates/icydb-core/src/error.rs`:
  - `InternalError::from_cursor_plan_error(...)` now preserves
    `ContinuationCursorInvariantViolation` as
    `ErrorClass::InvariantViolation` at `ErrorOrigin::Cursor`.
  - Cursor token/payload/signature/window/boundary/type mismatch variants map
    to `ErrorClass::Unsupported` at `ErrorOrigin::Cursor`.
  - Added `InternalError::cursor_unsupported(...)` as the canonical
    constructor for cursor-origin unsupported failures.
- Added classification integrity coverage in
  `crates/icydb-core/src/error.rs`, including
  `classification_integrity_cursor_conversion_matrix_is_restricted` with
  explicit variant-to-class expectations and restricted-class assertions.
- Aligned scalar PK cursor mismatch tests in
  `crates/icydb-core/src/db/executor/tests/pagination/cursor_pk.rs` to the
  new cursor taxonomy (`Unsupported` at cursor origin for malformed cursor
  boundary shapes/types).
- Added one unique replay fail-closed regression in
  `crates/icydb-core/src/db/commit/tests.rs`:
  `recovery_replay_interrupted_conflicting_unique_batch_fails_closed`.
  This simulates two conflicting unique writes in one persisted marker and
  asserts recovery remains conflict-classed, marker-persistent, and
  no-partial-index-state.
- Added reverse-index startup rebuild symmetry regressions in
  `crates/icydb-core/src/db/executor/tests/semantics.rs`:
  - `recovery_startup_rebuild_drops_orphan_reverse_relation_entries`
  - `recovery_startup_rebuild_restores_missing_reverse_relation_entry`
  These cover both stale-orphan and missing-reverse partial-drift directions.
- Added prefix-lowering edge equivalence property-style coverage in
  `crates/icydb-core/src/db/index/key/codec/tests.rs`:
  `index_key_prefix_lowering_matches_direct_prefix_membership_property`.
  The test asserts lowered raw bounds match direct prefix membership and
  include expected suffix min/max sentinel edges.
- Clarified uniqueness planning comments in
  `crates/icydb-core/src/db/index/plan/unique.rs` and
  `crates/icydb-core/src/db/index/plan/mod.rs` to reflect the active
  preflight reader model (committed state or overlay-staged logical view).

## 0.40.9 - 2026-03-03 - Cursor Ordering Audit and Dry Consolidation

### 📝 Summary

`0.40.9` closes the cursor ordering audit and the dry consolidation audit.
This patch adds cross-layer ordering/continuation property coverage and
continues boundary cleanup so grouped policy legality remains planner-owned and
error mapping remains stable across query/session/executor surfaces.

### 🔧 Changed

- Added cross-layer ordering-domain property coverage in
  `crates/icydb-core/src/db/index/envelope.rs`
  (`cross_layer_canonical_ordering_is_consistent`) to verify canonicalized
  key ordering remains consistent across envelope containment and continuation
  advancement comparisons.
- Added grouped continuation property coverage in
  `crates/icydb-core/src/db/executor/tests/paged_builder.rs`
  (`grouped_continuation_property_preserves_monotonicity_no_duplicates_and_full_union`)

  union parity across paged grouped cursor resume (ASC/DESC matrix).
- Added cursor error-mapping parity coverage in
  `crates/icydb-core/src/db/session.rs` so both conversion paths
  (`ExecutorPlanError::Cursor -> QueryError` and
  `CursorPlanError -> PlanError -> QueryError`) preserve identical cursor
  variant payloads for representative mismatch/decode cases.
- Continued dry consolidation of executable-plan boundaries by routing grouped
  handoff and scalar consistency reads through
  `ExecutablePlan::{grouped_handoff,consistency}`, removing runtime

## 0.40.8 - next

### 📝 Summary

`0.40.8` closes the complexity-accretion audit slice by collapsing repeated
route/load derivation logic into planner-projected contracts, narrowing runtime
access branching to one descriptor boundary, and reducing top-level plan error
surface drift.

### 🔧 Changed

- Added planner-projected route contracts in
  `crates/icydb-core/src/db/query/plan/model.rs`:
  `PlannerRouteProfile` (secondary pushdown applicability +
  continuation policy) and `ContinuationPolicy`
  (`requires_anchor`, `requires_strict_advance`, `is_grouped_safe`).
- Added planner projection entrypoint
  `AccessPlannedQuery::planner_route_profile(...)` in
  `crates/icydb-core/src/db/query/plan/semantics/logical.rs` and removed
  route-owned secondary pushdown applicability derivation.
- Threaded planner-projected continuation policy through route planning in
  `crates/icydb-core/src/db/executor/route/planner/{mod.rs,feasibility.rs}`,
  including grouped-strategy hint carriage from grouped handoff into route
  intent/feasibility.
- Extended route contracts in
  `crates/icydb-core/src/db/executor/route/contracts.rs` so
  `ExecutionRoutePlan` carries `continuation_policy`, and route hint gates in
  `crates/icydb-core/src/db/executor/route/hints.rs` now consume policy helpers
  instead of re-deriving continuation mode rules.
- Replaced structural runtime stream requests with execution descriptors:
  `RoutedKeyStreamRequest::AccessDescriptor(AccessExecutionDescriptor)` and
  `ordered_key_stream_from_access_descriptor(...)`, with callsites updated
  across load/aggregate fast paths
  (`load/{execute,fast_stream,pk_stream,secondary_index,index_range_limit}.rs`,
  `aggregate/fast_path.rs`).
- Reduced structural-peek surface by removing
  `AccessPath::as_index_prefix` and `AccessPlan::as_index_prefix_path`, and
  by switching continuation prepare/revalidate boundaries in
  `crates/icydb-core/src/db/executor/continuation/mod.rs` to executable access
  paths (`plan.to_executable().as_path().copied()`).
- Collapsed planner error taxonomy in
  `crates/icydb-core/src/db/query/plan/validate/mod.rs` from many top-level
  variants into:
  `PlanError::Semantic(Box<SemanticPlanError>)` and
  `PlanError::Cursor(Box<CursorPlanError>)`, with semantic domain variants moved
  under `SemanticPlanError` and mapping/tests updated accordingly.
- Added boundary drift guard
  `runtime_as_inner_calls_are_limited_to_boundary_adapters` in
  `crates/icydb-core/src/db/executor/stream/access/tests.rs` to keep runtime
  `.as_inner()` access local to explicit adapter boundaries.
- Reduced runtime `.as_inner()` fan-out further by routing grouped handoff and
  grouped-shape checks through executable-plan boundary helpers
  (`ExecutablePlan::grouped_handoff`, `ExecutablePlan::is_grouped`,
  `ExecutablePlan::consistency`) and by consuming scalar load entrypoint plans
  into one `logical_plan` spine before route/materialization.
- Removed the last runtime `.as_inner()` aggregate adapter call sites by
  deriving aggregate route/preparation state from a consumed `logical_plan`
  spine and rebuilding executable-plan contracts after descriptor derivation.
- Tightened the `.as_inner()` boundary guard allowlist to an empty runtime set
  after removing direct `.as_inner()` use from session, grouped route, load
  entrypoints, aggregate projection, and aggregate extrema/descriptor adapters.
- Boxed scalar cursor variants in
  `crates/icydb-core/src/db/executor/load/entrypoints.rs`
  (`LoadCursorInput::Scalar`, `LoadPreparedCursor::Scalar`) to satisfy
  large-enum variant constraints without widening grouped cursor payload shape.

## 0.40.7 - next

### 📝 Summary

`0.40.7` closes the boundary-semantics audit for cursor continuation, resume
windows, and anchor validation. This patch documents and hardens the
duplication/omission safety model around index-range paging.

### 🔧 Changed

- Audited and locked continuation bound rewrite semantics: resume rewrites
  exactly one edge to `Excluded(anchor)` (never both edges), which is the
  invariant that prevents replay duplicates without introducing omissions.
- Audited monotonicity guarantees for continuation advancement and verified
  strict forward progress is enforced on canonical index-order keys.
- Audited envelope-containment boundaries and confirmed fail-closed behavior
  for out-of-envelope, wrong-identity, and malformed anchors.
- Split anchor validation into explicit typestate phases in
  `crates/icydb-core/src/db/cursor/anchor.rs`:
  `ValidatedIdentityIndexRangeCursorAnchor` (identity checks) and
  `ValidatedInEnvelopeIndexRangeCursorAnchor` (envelope containment proof).
- Kept proof-carrying anchor construction sealed to validation paths so raw
  anchors are not propagated through executor state by accident.
- Added property coverage for continuation safety in
  `crates/icydb-core/src/db/executor/tests/pagination/range_edges_trace.rs`
  (`load_index_range_cursor_property_matrix_preserves_union_monotonicity_and_resume_suffix`)
  to assert union parity, strict monotonic anchors, and resume-suffix behavior.

  `grouped_ordered_runtime_revalidation_flag_count_guard` and route budget

## 0.40.1 - 2026-03-02 - Executable Access Contract Hardening

### 📝 Summary

`0.40.1` makes executor access routing rely on one executable contract shape
instead of re-reading raw `AccessPath` variants at runtime.

### 🔧 Changed

- Removed runtime dependency on raw `AccessPath` variants inside executor stream
  lowering and route fast-path checks.
- Made `ExecutableAccessPath` carry execution payload directly, while index
  execution shape is derived from `ExecutionBounds`.
- Preserved cursor anchor validation behavior by carrying index-range semantic
  bounds in executable payload instead of borrowing planner path objects.
- Removed the direct access-path stream request route variant and kept routed
  stream resolution on the access-plan request contract.
- Added a structural guard test that fails if executor runtime modules start
  matching `AccessPath::...` variants again.

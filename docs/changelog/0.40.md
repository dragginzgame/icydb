## 0.40.7 - next

### 📝 Summary

`0.40.7` closes the boundary-semantics audit for cursor continuation, resume
windows, and anchor validation. This patch documents and hardens the
duplication/omission safety model around index-range paging.

### 🔧 Changed

- Audited and locked continuation bound rewrite semantics: resume rewrites
  exactly one edge to `Excluded(anchor)` (never both edges), which is the
  invariant that prevents replay duplicates without introducing omissions.
- Audited monotonicity guarantees for continuation advancement and verified
  strict forward progress is enforced on canonical index-order keys.
- Audited envelope-containment boundaries and confirmed fail-closed behavior
  for out-of-envelope, wrong-identity, and malformed anchors.
- Split anchor validation into explicit typestate phases in
  `crates/icydb-core/src/db/cursor/anchor.rs`:
  `ValidatedIdentityIndexRangeCursorAnchor` (identity checks) and
  `ValidatedInEnvelopeIndexRangeCursorAnchor` (envelope containment proof).
- Kept proof-carrying anchor construction sealed to validation paths so raw
  anchors are not propagated through executor state by accident.
- Added property coverage for continuation safety in
  `crates/icydb-core/src/db/executor/tests/pagination/range_edges_trace.rs`
  (`load_index_range_cursor_property_matrix_preserves_union_monotonicity_and_resume_suffix`)
  to assert union parity, strict monotonic anchors, and resume-suffix behavior.

## 0.40.5 - 2026-03-03 - Grouped Stage Ownership and Contract Bundling

### 📝 Summary

`0.40.5` continues post-unification executor cleanup by making ownership and
request boundaries more explicit in grouped/load and access-stream internals.

### 🔧 Changed

- Replaced the multi-argument page materialization entry in
  `crates/icydb-core/src/db/executor/load/page.rs` with explicit
  request-contract bundling via `PageMaterializationRequest` to keep
  logical/physical/paging/continuation inputs on one typed boundary.
- Replaced the multi-argument physical key-stream resolver entry in
  `crates/icydb-core/src/db/executor/stream/access/physical.rs` with
  `PhysicalStreamRequest` so lowered-spec constraints and execution hints are
  passed as one explicit physical-boundary contract.
- Replaced the wide secondary aggregate fast-path fold helper argument list in
  `crates/icydb-core/src/db/executor/aggregate/fast_path.rs` so it now
  consumes `AggregateFastPathInputs` directly plus probe hint override,
  reducing duplicate context-shape drift risk across aggregate fast-path code.
- Replaced `finalize_execution` output-plumbing argument lists in
  `crates/icydb-core/src/db/executor/load/execute.rs` with
  `ExecutionOutcomeMetrics`, and split materialized attempts via
  `into_page_and_metrics()` to keep finalization reporting contracts explicit.
- Restructured grouped route-stage state in
  `crates/icydb-core/src/db/executor/load/mod.rs` from one flattened
  `GroupedRouteStage` payload into explicit nested ownership bundles:
  `GroupedPlannerPayload`, `GroupedRoutePayload`, `IndexSpecBundle`, and
  `GroupedExecutionContext`.
- Aligned grouped HAVING planner-bypass invariant test expectation with current
  executor invariant origin taxonomy (`ErrorOrigin::Query`) in
  `crates/icydb-core/src/db/executor/tests/aggregate/core_contract_matrices.rs`.
- Updated grouped global DISTINCT kind-guard unit assertions in
  `crates/icydb-core/src/db/executor/load/grouped_distinct.rs` to satisfy
  strict clippy bool-assert style (`assert!`/`assert!(!...)`).

## 0.40.4 - 2026-03-02 - Load Pipeline Stage-State Hardening

### 📝 Summary

`0.40.4` hardens the unified load execution and grouped-route semantic
ownership boundaries after entrypoint unification.

### 🔧 Changed

- Refined `execute_load` stage handoff in
  `crates/icydb-core/src/db/executor/load/entrypoints.rs` from a
  single optional-slot `LoadPipelineState` to stage-typed artifacts:
  `LoadAccessState` and `LoadPayloadState`.
- Removed optional artifact extraction helpers from load orchestration state and
  made stage inputs required-by-construction at each pipeline boundary.
- Kept the same linear stage order (`build -> access -> grouping -> paging ->
  tracing -> surface`) while reducing coordination-state drift pressure.
- Updated load structure guard baseline in
  `crates/icydb-core/src/db/executor/tests/load_structure.rs` to enforce a
  zero optional-slot contract for pipeline-stage artifacts.
- Removed grouped HAVING semantic policy re-check from executor route
  feasibility and kept grouped strategy policy under planner hint authority.
- Removed remaining grouped aggregate/DISTINCT semantic re-checks from executor
  grouped-route feasibility; runtime feasibility now revalidates capability
  constraints only.
- Added grouped runtime revalidation-surface guard coverage via
  `grouped_ordered_runtime_revalidation_flag_count_guard` and route budget
  tests.
- Tightened grouped strategy-hint helper visibility and routed route/explain
  consumers through explicit plan-root wrappers:
  `grouped_plan_strategy_hint_for_route(...)` and
  `grouped_plan_strategy_hint_for_explain(...)`.
- Hardened grouped global DISTINCT execution runtime boundary in
  `crates/icydb-core/src/db/executor/load/grouped_distinct.rs` with explicit
  fail-closed kind validation (`COUNT`/`SUM` only) before aggregate fold
  mechanics execute.
- Added planner-bypass invariant guard coverage for unsupported global DISTINCT
  grouped aggregate kinds in
  `crates/icydb-core/src/db/executor/tests/aggregate/core_contract_matrices.rs`
  (`grouped_global_distinct_unsupported_kind_is_executor_invariant_only_when_planner_is_bypassed`).
- Added focused unit coverage for global DISTINCT kind guard behavior in
  `crates/icydb-core/src/db/executor/load/grouped_distinct.rs`
  (`grouped_global_distinct_kind_guard_accepts_count_and_sum`,
  `grouped_global_distinct_kind_guard_rejects_non_count_sum`).
- Added executor boundary guard for grouped scalar DISTINCT policy violations in
  `crates/icydb-core/src/db/executor/load/grouped_route.rs`, routed through
  planner semantic helper
  `grouped_distinct_policy_violation_for_executor(...)`.
- Added planner-bypass invariant coverage for grouped scalar DISTINCT policy
  violations in
  `crates/icydb-core/src/db/executor/tests/aggregate/core_contract_matrices.rs`
  (`grouped_scalar_distinct_policy_violation_is_executor_invariant_only_when_planner_is_bypassed`).
- Added executable-plan boundary wrapper
  (`ExecutablePlan::grouped_distinct_policy_violation_for_executor`) and routed
  grouped route guards through it so grouped DISTINCT policy probing no longer
  reaches through `plan.as_inner()` in route code.
- Removed direct `LogicalPlan` enum matching from
  `crates/icydb-core/src/db/executor/delete/mod.rs` and made grouped delete
  rejection use executable-plan shape (`ExecutablePlan::is_grouped()`) at the
  executor boundary.

## 0.40.1 - 2026-03-02 - Executable Access Contract Hardening

### 📝 Summary

`0.40.1` makes executor access routing rely on one executable contract shape
instead of re-reading raw `AccessPath` variants at runtime.

### 🔧 Changed

- Removed runtime dependency on raw `AccessPath` variants inside executor stream
  lowering and route fast-path checks.
- Made `ExecutableAccessPath` carry execution payload directly, while index
  execution shape is derived from `ExecutionBounds`.
- Preserved cursor anchor validation behavior by carrying index-range semantic
  bounds in executable payload instead of borrowing planner path objects.
- Removed the direct access-path stream request route variant and kept routed
  stream resolution on the access-plan request contract.
- Added a structural guard test that fails if executor runtime modules start
  matching `AccessPath::...` variants again.

## 0.40.2 - 2026-03-02 - Continuation and Load Entrypoint Unification

### 📝 Summary

`0.40.2` centralizes continuation protocol orchestration and removes the
separate scalar/grouped load entrypoint flow split by routing both through one
internal load execution pipeline.

### 🔧 Changed

- Added `crates/icydb-core/src/db/executor/continuation/mod.rs` as the
  executor-owned continuation facade for cursor revalidation/runtime bindings,
  grouped paging window derivation, and grouped next-token construction.
- Updated `crates/icydb-core/src/db/executor/load/entrypoints.rs` to use one
  internal dispatch pipeline for scalar and grouped load execution while
  keeping public entrypoint APIs unchanged.
- Routed grouped pagination window derivation and grouped continuation token
  construction through the continuation facade instead of duplicating mechanics
  in grouped fold helpers.

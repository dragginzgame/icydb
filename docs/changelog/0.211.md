# IcyDB 0.211: Accepted Constraint Catalog

## 0.211.0

### ⚠️ Breaking

- Hard-cuts accepted entity snapshots to the sole current version-1 `ICYU`
  profile. Development stores carrying the preceding accepted-schema profile
  must be recreated; no compatibility reader or migration shim remains.

### ➕ Added

- Adds persisted non-reusing constraint-ID high-water state and stable logical
  identities for secondary indexes and relations.
- Separates logical index identity from the dense physical ordinal used by
  index-key storage, allowing physical compaction without changing a surviving
  index's catalog identity.

### 🔧 Changed

- Initial generated schema proposals assign deterministic logical identities,
  while accepted reconciliation and schema mutations preserve stored identity.
- Accepted admission and runtime/cache fingerprints now include constraint-ID
  allocator state plus logical index and relation identities.
- All active internal format and SQL correctness evidence identities remain at
  version 1 under the pre-1.0 hard-cut policy.

### 🧪 Testing

- Covers current schema round trips, malformed identity rejection, physical
  ordinal compaction, generated index rename association, and fingerprint
  sensitivity.
- Raw non-gzipped SQL performance Wasm measured 3,967,488 bytes, an increase of
  3,098 bytes (+0.0782%) over the final 0.210.2 baseline. No row, index-key,
  relation-key, cursor, or journal format changed.

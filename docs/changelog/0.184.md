## 0.184.43

### 📝 Summary

- Continues the query-engine and generated-surface audit across public
  generated endpoint names, COUNT/EXISTS cardinality preflight, and SQL
  EXPLAIN feature boundaries.
- Moves generated public canister method names from `__icydb_*` to `icydb_*`
  while keeping hidden generated Rust wrappers collision-safe.
- Splits SQL EXPLAIN into an opt-in `sql-explain` feature so ordinary SQL
  canisters can omit EXPLAIN parser, lowering, execution, and rendering
  payloads.

### ⚠️ Breaking

- Callers that invoked generated canister endpoints by name must switch from
  `__icydb_*` public method names to `icydb_*` public method names.
- Crates or canisters that need SQL EXPLAIN must enable `sql-explain`; enabling
  `sql` alone keeps ordinary SQL query/write support but rejects EXPLAIN.

### 🔧 Changed

- Generated SQL, fixtures, schema, snapshot, and metrics endpoints now publish
  `icydb_*` method names while generated Rust wrappers retain internal
  collision-avoidance names.
- COUNT and EXISTS prefix-cardinality preflight admission, ordered-plan
  eligibility, and output conversion now flow through one terminal-kind helper
  while preserving direct-count cache boundaries.
- SQL EXPLAIN parser/model, prepared lowering, compiled-command payloads,
  session execution, result conversion, and table rendering are gated behind
  `sql-explain`.
- SQL test/demo/perf canisters opt into `sql-explain` where EXPLAIN coverage or
  local debugging remains expected.

### 🧹 Cleanup

- Removed ordinary-SQL retention of EXPLAIN-only descriptor property strings,
  lowering envelopes, accepted-authority planning helpers, and projection
  debug labels.
- Kept diagnostics independent from SQL EXPLAIN so diagnostics builds do not
  imply the SQL frontend.
- Split parser, lowering, and session tests so plain `sql` coverage validates
  EXPLAIN rejection and `sql-explain` coverage validates EXPLAIN behavior.

### 📊 Audit

- Targeted `one_entity_sql_query` wasm-release measurement dropped from
  2,987,809 to 2,903,845 raw built `.wasm` bytes, and from 2,778,264 to
  2,702,001 shrunk `.wasm` bytes after moving EXPLAIN out of ordinary `sql`.
- COUNT/EXISTS prefix-cardinality behavior remains on the same cache and
  execution contracts; the slice only centralizes terminal-family branching.

### 🧪 Testing

- Revalidated plain `sql` and `sql-explain` public crate checks.
- Revalidated core plain `sql` and `sql-explain` test builds.
- Revalidated focused parser and lowering coverage for EXPLAIN rejection and
  EXPLAIN execution lowering.
- Rebuilt the targeted `one_entity_sql_query` wasm-size report.

### 🧭 Migration Notes

- No schema migration is required.
- No cursor format, persistence format, query result semantics, write
  semantics, or direct-count cache migration is required.
- Generated endpoint clients should update method-name calls to `icydb_*`.
- SQL EXPLAIN users should enable `sql-explain` explicitly in addition to, or
  instead of directly enabling, `sql`.

# 0.161.x

## 0.161.5

### 🧹 Cleanup

- Continued the modular hygiene pass across `icydb-derive` and
  `icydb-schema-derive`. The proc-macro entrypoints remain the public
  generated-code boundary, while derive expansion helpers, derive validation
  helpers, and generated-model lowering helpers are scoped to
  crate-private/private owners where in-tree consumers allow it.
- Updated the CSH audit method to `CSH-1.2`, adding deletion-pressure checks,
  consumer-legitimacy columns, and explicit dispositions such as `DELETE NOW`,
  `NARROW NOW`, `INLINE NOW`, `MOVE OWNER`, and `DEFER WITH TRIGGER`.
- Added runtime text validators and macro-test fixtures for camelCase, Sentence
  case, and UPPER-KEBAB-CASE so all currently retained `icydb-utils` case
  variants have runtime validation consumers.
- Centralized changelog governance around root-only `Unreleased` notes so
  detailed minor changelog files remain release-prep artifacts rather than a
  second live unreleased ledger.

### 🧪 Testing

- Added macro validation coverage for the new camelCase, Sentence case, and
  UPPER-KEBAB-CASE validators.

## 0.161.4

### 🧹 Cleanup

- Cleaned up `icydb-config-build` by moving integration-style tests out of the
  crate root, making current-directory discovery fallible instead of panicking,
  parsing `Cargo.toml` for workspace-root detection, and centralizing repeated
  canister-surface lookup logic.
- Refreshed `icydb-build` documentation for the current generated endpoint
  surface and split generated actor glue rendering into store, SQL, and schema
  modules while keeping codegen behavior unchanged.
- Audited the public `icydb` facade crate and refreshed stale internal-boundary
  wording. The hidden `model`, `metrics`, `__macro`, and `__reexports`
  surfaces remain intentional generated-code, CLI observability, and dependency
  bridge boundaries.
- Tightened the facade boundary so generated-only `EntityValue` and
  `PersistedRow` implementations, slot reader/writer signatures, and generated
  internal-error plumbing now target the hidden `__macro` lane instead of normal
  `icydb::traits` / `icydb::db` imports. The public facade keeps the semantic
  `Entity` and `CreateInput` traits for application and repository helper
  bounds.
- Cleaned downstream-style fluent query examples in the SQL perf audit canister
  and PocketIC perf scenarios to use the public `filter_*` helpers instead of
  raw `FieldRef::new(...)` construction.

## 0.161.3

### 🧹 Cleanup

- Split `icydb-cli` observability into focused snapshot, metrics, schema,
  schema-check, and shared render modules. Generic ICP transport remains owned
  by `icp::call`, while observability owns endpoint orchestration and rendering
  without depending on shell internals. The schema-check surface is further
  separated into command orchestration, drift analysis, and report rendering.

## 0.161.2

### 🧹 Cleanup

- Moved generic `icp canister call` query/update command construction and hex
  response decoding out of the SQL shell module into the `icp` transport
  boundary. The SQL shell keeps SQL routing, escaping, prompts, and recovery
  hints, while observability commands now share the same ICP call helpers
  without depending on shell internals.

## 0.161.1

### 🧹 Cleanup

- Narrowed `icydb-utils` to its live workspace surface by removing unused
  public hash digest helpers and broad RNG helper exports. Core runtime
  ownership now covers `Xxh3` value hashing and the seeded `next_u128` random
  draw used by generated ID/subaccount paths, leaving `icydb-utils` as a
  focused shared case-conversion crate for runtime validators and derives.

## 0.161.0

### 🔧 Changed

- Started the schema/derive hygiene line by reducing duplicated stable-memory
  policy surface. Derive-side memory validation now reuses the schema-owned
  app memory ID range and stable-key segment predicate, while tests guard the
  expected app ID range, reserved `255` rejection, stable-key segment grammar,
  full stable-key grammar, generated store key shape, and commit stable-key ABI
  parity between generated `CanisterKind` metadata and schema declarations.
- Removed the `canic-cdk` dependency from `icydb-schema` by replacing the lone
  schema timestamp call with a local schema-owned system-time helper.
- Narrowed clearly internal schema facade surface: schema validation is no
  longer exported as a public namespace, the concrete validating visitor is
  crate-private, and unused schema graph convenience helpers were removed or
  narrowed where in-tree consumers allowed it.

### 🧪 Testing

- Added focused schema and derive tests for memory ID policy, stable-key
  grammar, generated stable-key shape, and commit stable-key parity.

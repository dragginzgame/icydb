# 0.161.x

## 0.161.3

### 🧹 Cleanup

- Split `icydb-cli` observability into focused snapshot, metrics, schema,
  schema-check, and shared render modules. Generic ICP transport remains owned
  by `icp::call`, while observability owns endpoint orchestration and rendering
  without depending on shell internals.

## 0.161.2

### 🧹 Cleanup

- Moved generic `icp canister call` query/update command construction and hex
  response decoding out of the SQL shell module into the `icp` transport
  boundary. The SQL shell keeps SQL routing, escaping, prompts, and recovery
  hints, while observability commands now share the same ICP call helpers
  without depending on shell internals.

## 0.161.1

### 🧹 Cleanup

- Narrowed `icydb-utils` to its live workspace surface by removing unused
  public hash digest helpers and broad RNG helper exports. Core runtime
  ownership now covers `Xxh3` value hashing and the seeded `next_u128` random
  draw used by generated ID/subaccount paths, leaving `icydb-utils` as a
  focused shared case-conversion crate for runtime validators and derives.

## 0.161.0

### 🔧 Changed

- Started the schema/derive hygiene line by reducing duplicated stable-memory
  policy surface. Derive-side memory validation now reuses the schema-owned
  app memory ID range and stable-key segment predicate, while tests guard the
  expected app ID range, reserved `255` rejection, stable-key segment grammar,
  full stable-key grammar, generated store key shape, and commit stable-key ABI
  parity between generated `CanisterKind` metadata and schema declarations.
- Removed the `canic-cdk` dependency from `icydb-schema` by replacing the lone
  schema timestamp call with a local schema-owned system-time helper.
- Narrowed clearly internal schema facade surface: schema validation is no
  longer exported as a public namespace, the concrete validating visitor is
  crate-private, and unused schema graph convenience helpers were removed or
  narrowed where in-tree consumers allowed it.

### 🧪 Testing

- Added focused schema and derive tests for memory ID policy, stable-key
  grammar, generated stable-key shape, and commit stable-key parity.

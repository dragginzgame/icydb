# 0.163.x

## 0.163.2

Role-specific stable allocation metadata. Purpose: make data, index, and
schema storage diagnostics describe the accepted schema contract that actually
owns each allocation role, without changing durable allocation identity.

### 🔧 Changed

- Added role-specific schema metadata fingerprints for data, index, and schema
  storage snapshot rows.
- Data snapshots now fingerprint accepted row-layout and field contracts with
  accepted index contracts removed.
- Index snapshots now fingerprint accepted index catalog contracts.
- Schema snapshots continue to fingerprint the full accepted schema-store
  catalog payload.

### 🧪 Testing

- Added schema-store coverage proving index-catalog changes affect index and
  schema allocation metadata without changing data allocation metadata.
- Extended storage-report assertions so data, index, and schema rows must carry
  distinct role-specific fingerprints when accepted schema authority exists.

## 0.163.1

Stable allocation identity diagnostics. Purpose: carry generated memory
allocation identity into runtime diagnostics without making metadata part of
allocation identity.

### 🔧 Changed

- Added generated data/index/schema allocation identity descriptors to store
  registry wiring.
- Added memory IDs and stable keys to data-store, index-store, and schema-store
  storage snapshot rows.
- Updated CLI storage snapshot tables to render allocation identity next to
  entries, byte counts, index state, and schema metadata.

### 🩹 Fixed

- Fixed mixed-direction `ORDER BY` over composite primary-key row streams, such
  as `ORDER BY x ASC, y DESC`, by routing unfiltered unordered access paths
  through the direct materialized-order row lane.
- Added a fail-closed guard so non-route-satisfied kernel-row ordering requires
  materialized order slots instead of silently preserving raw key-stream order.

### 🧪 Testing

- Added storage-report assertions for generated allocation identity and updated
  Candid shape guards for the expanded snapshot DTOs.
- Rechecked focused storage-report, CLI snapshot rendering, and build-code
  compilation after wiring generated allocation identity through diagnostics.
- Added focused executor coverage for mixed-direction direct row ordering and
  for materialized-order lane selection without retained slots.

## 0.163.0

Stable allocation metadata foundation. Purpose: make the stable-memory
allocation boundary explicit before wiring richer allocation diagnostics.

### 🔧 Changed

- Split stable allocation descriptors into durable identity
  (`memory_id + stable_key`) and diagnostic schema metadata
  (`schema_version + schema_fingerprint`).
- Added explicit absent and accepted-schema metadata construction paths so
  generated-model metadata cannot become allocation metadata authority.
- Added schema-store catalog metadata to storage snapshots, derived from the
  latest persisted accepted schema snapshots rather than generated models.
- Updated CLI storage snapshot rendering to show schema-store catalog metadata
  separately from data and index store footprint rows.

### 🧪 Testing

- Added schema allocation identity tests proving metadata changes do not alter
  durable allocation identity.
- Added schema-store catalog metadata tests covering absent metadata,
  latest-snapshot selection, and insertion-order independence.
- Rechecked focused schema, storage-report, and CLI snapshot rendering tests,
  clippy for touched crates, workspace compilation, and diff hygiene.

# 0.120 Changelog

## 0.120.0

- `0.120.0` starts the pipeline-unification line by deleting the last hard route-planner bypasses instead of just thinning them:
  - mutation no longer shortcuts directly to a ready-made execution route plan
  - load no longer keeps separate initial-vs-resumed public route builders
  - route consumers no longer select planner paths by calling different public builder names

- The route planner now exposes one public staged builder surface in:
  - [/home/adam/projects/icydb/crates/icydb-core/src/db/executor/planning/route/planner/entrypoints.rs](/home/adam/projects/icydb/crates/icydb-core/src/db/executor/planning/route/planner/entrypoints.rs)
  - [/home/adam/projects/icydb/crates/icydb-core/src/db/executor/planning/route/planner/mod.rs](/home/adam/projects/icydb/crates/icydb-core/src/db/executor/planning/route/planner/mod.rs)
  - [/home/adam/projects/icydb/crates/icydb-core/src/db/executor/planning/route/mod.rs](/home/adam/projects/icydb/crates/icydb-core/src/db/executor/planning/route/mod.rs)

- That surface is `RoutePlanRequest` plus `build_execution_route_plan(...)`. The old public per-family route builders were removed:
  - `build_execution_route_plan_for_load(...)`
  - `build_execution_route_plan_for_mutation(...)`
  - `build_execution_route_plan_for_aggregate_spec(...)`
  - `build_execution_route_plan_for_grouped_plan(...)`
  - `build_initial_execution_route_plan_for_load(...)`

- Mutation routing now follows the same staged route path as the other execution families:
  - mutation intent is derived in [/home/adam/projects/icydb/crates/icydb-core/src/db/executor/planning/route/planner/intent.rs](/home/adam/projects/icydb/crates/icydb-core/src/db/executor/planning/route/planner/intent.rs)
  - mutation feasibility is derived in [/home/adam/projects/icydb/crates/icydb-core/src/db/executor/planning/route/planner/feasibility/mod.rs](/home/adam/projects/icydb/crates/icydb-core/src/db/executor/planning/route/planner/feasibility/mod.rs)
  - staged execution assembly handles the mutation shape in [/home/adam/projects/icydb/crates/icydb-core/src/db/executor/planning/route/planner/execution/mod.rs](/home/adam/projects/icydb/crates/icydb-core/src/db/executor/planning/route/planner/execution/mod.rs)
  - the old direct constructor path through `ExecutionRoutePlan::for_mutation(...)` was removed from [/home/adam/projects/icydb/crates/icydb-core/src/db/executor/planning/route/contracts/execution/plan.rs](/home/adam/projects/icydb/crates/icydb-core/src/db/executor/planning/route/contracts/execution/plan.rs)

- Load routing now has one public builder shape instead of a split initial/resumed surface:
  - the caller supplies continuation state in the request
  - runtime load entrypoints pass `Some(authority)` so PK fast-path cursor validation and load terminal fast-path derivation still happen at the same route boundary
  - EXPLAIN can still supply a precomputed load terminal fast-path override without reopening a second route-entry function

- All route consumers were moved onto the single staged builder surface:
  - scalar execution in [/home/adam/projects/icydb/crates/icydb-core/src/db/executor/pipeline/entrypoints/scalar/mod.rs](/home/adam/projects/icydb/crates/icydb-core/src/db/executor/pipeline/entrypoints/scalar/mod.rs)
  - grouped runtime route-stage assembly in [/home/adam/projects/icydb/crates/icydb-core/src/db/executor/pipeline/grouped_runtime/route_stage.rs](/home/adam/projects/icydb/crates/icydb-core/src/db/executor/pipeline/grouped_runtime/route_stage.rs)
  - mutation preflight in [/home/adam/projects/icydb/crates/icydb-core/src/db/executor/mutation/mod.rs](/home/adam/projects/icydb/crates/icydb-core/src/db/executor/mutation/mod.rs)
  - aggregate execution in [/home/adam/projects/icydb/crates/icydb-core/src/db/executor/aggregate/mod.rs](/home/adam/projects/icydb/crates/icydb-core/src/db/executor/aggregate/mod.rs)
  - load and aggregate EXPLAIN descriptors in [/home/adam/projects/icydb/crates/icydb-core/src/db/executor/explain/descriptor/load.rs](/home/adam/projects/icydb/crates/icydb-core/src/db/executor/explain/descriptor/load.rs) and [/home/adam/projects/icydb/crates/icydb-core/src/db/executor/explain/descriptor/aggregate.rs](/home/adam/projects/icydb/crates/icydb-core/src/db/executor/explain/descriptor/aggregate.rs)

- The access/explain side also shrank in this patch line:
  - EXPLAIN access projection was already moved to the planner-owned `AccessPlanProjection` walk in [/home/adam/projects/icydb/crates/icydb-core/src/db/query/explain/access_projection.rs](/home/adam/projects/icydb/crates/icydb-core/src/db/query/explain/access_projection.rs)
  - chosen-route classification and labels in `access_choice` were already moved off `ExplainAccessPath` transport and onto planner-owned access traversal
  - `AccessPlannedQuery` seeding now also stays on `AccessPlan` directly in [/home/adam/projects/icydb/crates/icydb-core/src/db/query/plan/access_plan.rs](/home/adam/projects/icydb/crates/icydb-core/src/db/query/plan/access_plan.rs) instead of converting access shapes into `ExplainAccessPath` just to seed non-index access-choice diagnostics
  - the non-index seed helper itself now lives on direct `AccessPlan` matching in [/home/adam/projects/icydb/crates/icydb-core/src/db/query/plan/access_choice/mod.rs](/home/adam/projects/icydb/crates/icydb-core/src/db/query/plan/access_choice/mod.rs)

- `0.120.0` also adds structural guardrails so this route-surface collapse stays hard:
  - [/home/adam/projects/icydb/crates/icydb-core/src/db/executor/planning/route/tests/structural_guards.rs](/home/adam/projects/icydb/crates/icydb-core/src/db/executor/planning/route/tests/structural_guards.rs) now pins:
    - one public route-builder surface
    - direct `ExecutionRoutePlan` construction staying inside staged assembly
  - [/home/adam/projects/icydb/crates/icydb-core/src/db/query/plan/tests/structural_guards.rs](/home/adam/projects/icydb/crates/icydb-core/src/db/query/plan/tests/structural_guards.rs) now also pins planner access-choice seeding against routing through `EXPLAIN` access transport

- This patch is intentionally subtractive and consolidating:
  - no widened execution semantics
  - no widened EXPLAIN feature surface
  - no new route family
  - fewer public route entrypoints
  - fewer planner-to-explain transport detours

  - `cargo test -p icydb-core route_planner_public_surface_has_single_route_builder -- --nocapture`
  - `cargo test -p icydb-core query::plan -- --nocapture`

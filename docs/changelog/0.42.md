# IcyDB 0.42

This document aggregates detailed release notes for the `0.42.x` minor line.

## 0.42.0 - 2026-03-06 - EXPLAIN Descriptor Foundation

### 📝 Summary

This slice starts the `0.42` EXPLAIN contract freeze by adding stable
execution-descriptor fields and moving aggregate terminal descriptor assembly
behind one centralized executor-owned boundary.

### 🔧 Changed

- Added `execution_mode` (`Streaming` / `Materialized`) and `node_properties`
  to scalar aggregate terminal execution descriptors so EXPLAIN can expose
  materialization behavior and additive node-local metadata.
- Added an executor-local descriptor assembly module
  (`db::executor::explain::descriptor`) and rewired aggregate terminal explain
  projection to consume that one canonical assembly path.
- Added seek-route descriptor metadata emission under `node_properties.fetch`
  for aggregate edge-seek shapes so EXPLAIN surfaces can report bounded seek
  fetch size without introducing new top-level descriptor fields.
- Added a first canonical execution-node descriptor surface
  (`ExplainExecutionNodeDescriptor` + `ExplainExecutionNodeType`) and wired
  aggregate-terminal explain payloads to project that node descriptor via
  `execution_node_descriptor()`.
- Added deterministic base text rendering for execution-node descriptors
  (`render_text_tree`) and locked aggregate-terminal node text output in
  session aggregate explain tests.
- Added scalar load execution descriptor-tree assembly over canonical load route
  planning, including descriptor children for predicate/order/distinct/pagination
  stages plus route-owned seek/pushdown metadata nodes.
- Added query/fluent explain-execution surfaces (`Query::explain_execution()`
  and `.explain_execution()`) that return the canonical execution-node
  descriptor tree without executing the query.
- Expanded deterministic text rendering to include core descriptor metadata
  (access summary, predicate metadata, ordering source, and pagination fields).
- Preserved recursive access topology in load execution descriptors so
  `Union`/`Intersection` roots carry explicit child access nodes.
- Centralized predicate-stage descriptor emission so load shapes report either
  `IndexPredicatePrefilter` (strict compiled prefilter) or
  `ResidualPredicateFilter` (post-access evaluation) from one assembly path.
- Added canonical JSON rendering for execution-node descriptors with fixed key
  order and recursive descriptor/access child serialization.

# IcyDB 0.42

This document aggregates detailed release notes for the `0.42.x` minor line.

## 0.42.1 - 2026-03-06 - EXPLAIN Follow-Up Completion

### 📝 Summary

`0.42.1` closes the first EXPLAIN follow-up slice by hardening descriptor
stability, adding direct text/JSON/verbose adapter surfaces, and expanding
verbose runtime route diagnostics so planner and executor decisions are easier
to inspect deterministically.

### 🔧 Changed

- Added first-class execution explain adapters on query and fluent load
  terminals: `explain_execution_text()`, `explain_execution_json()`, and
  `explain_execution_verbose()`.
- Added descriptor-level verbose text rendering
  (`render_text_tree_verbose()`) so verbose output can be consumed directly
  from the canonical explain descriptor tree.
- Added executor-owned verbose scalar load diagnostics assembly so verbose
  explain output carries explicit route diagnostics for pushdown eligibility,
  pushdown rejection reasons, predicate-stage routing, and bounded seek/hint
  projections.
- Froze the EXPLAIN node-type vocabulary contract with deterministic guard
  tests so descriptor node naming remains stable across follow-up refactors.
- Added verbose diagnostics matrix/snapshot coverage and deferred-node guard
  assertions for currently un-emitted scalar families to keep the EXPLAIN
  contract fail-closed as new node families are activated.

## 0.42.0 - 2026-03-06 - Initial EXPLAIN Execution Output

### 📝 Summary

`0.42.0` introduces execution EXPLAIN output for load queries so callers can
inspect runtime execution shape before running the query.

### ➕ Added

- Added canonical execution descriptor output for scalar load plans, including
  access-path choice, filter stage placement, ordering strategy, and limit
  handling.
- Added stable text and canonical JSON descriptor renderers as the baseline
  EXPLAIN surface for runtime execution inspection.

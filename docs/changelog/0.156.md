# 0.156.x

## Implementation

`0.156` now has the first host-only `icydb.toml` config generation slice.

### ➕ Added

- Added `icydb-config-build`, a build-helper crate that owns strict
  `icydb.toml` parsing, validation, config resolution, and generated Rust
  constant emission.
- Added support for absent config defaulting all optional surfaces off,
  per-canister `[canisters.<name>.sql] readonly = true` and `ddl = true`
  switches, canister-name validation against generated schema names, and
  normalized duplicate-name rejection.
- Added generated actor build options so build scripts can pass the validated
  SQL surface booleans into `icydb-build` without making the actor generator
  parse TOML.
- Wired the demo RPG, lightweight SQL test, and recurring audit canisters to
  config-aware build scripts. The demo canister uses the project-level
  `icydb.toml`, while the SQL test canister keeps a crate-local `icydb.toml`;
  their read-only SQL query plus DDL/fixture endpoints are now generated from
  config instead of the old manual macro.
- Config discovery now walks upward from the canister crate and uses the
  nearest `icydb.toml`, so grouped canister directories can own shared config.
- Added `icydb config show` and `icydb config check`. `show` displays the
  resolved project config without requiring an ICP environment, and adds sync
  markers when `--environment` is supplied; `check` returns nonzero when config
  and the selected ICP environment are out of sync.

### 🧹 Cleanup

- Removed the old default `icydb::build!`/`icydb-build::generate` actor
  generation path. Build scripts now pass explicit generated options through
  `icydb::build_with_options!()`.
- Removed the old manual `icydb::admin_sql_query!()` endpoint macro. SQL endpoint
  shape now comes only from generated config plus actor build options.
- Removed demo SQL default target config entirely. `icydb sql` now requires an
  explicit `--canister`, while `readonly` and `ddl` independently control
  generated DB surfaces.
- The README SQL snippet now documents the config-based opt-in instead
  of the removed manual macro.

## Design

`0.156` opens a tentative design for an `icydb.toml` project manifest.

### Design

- Added the `0.156` `icydb.toml` design document.
- Scoped the first config surface to per-canister DB SQL surfaces:
  `[canisters.<name>.sql] readonly = true` and `ddl = true`.
- Required TOML parsing to stay in a host-only build crate, with generated typed
  Rust constants or project metadata injected instead of runtime TOML loading.
- Made `icydb.toml` the sole endpoint-shape authority for DB SQL surfaces and
  called for removing or compile-erroring the old manual admin SQL macro with no
  shim.
- Captured validation expectations for unknown fields, malformed config,
  canister-name matching, build-script rerun behavior, wasm dependency graph
  checks, and raw non-gzipped wasm size checks.

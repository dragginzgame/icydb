# 0.156.x

## 0.156.4

`0.156.4` moves generated metrics and storage snapshot endpoints behind
`icydb.toml` and keeps generated observability endpoint names aligned with the
`__icydb_*` convention.

### ➕ Added

- Added `[canisters.<name>.metrics] enabled = true` and `reset = true`
  switches so metrics report/reset endpoints are generated only when config
  opts a canister in. The generated methods are `__icydb_metrics` and
  `__icydb_metrics_reset`.
- Added `[canisters.<name>.snapshot] enabled = true` so the generated storage
  inventory endpoint is emitted only when config opts a canister in. The
  generated method is `__icydb_snapshot`.
- Added `--metrics`, `--metrics-reset`, and `--snapshot` to
  `icydb config init`, including `--all` coverage for all three switches.
- Added direct observability commands: `icydb snapshot --canister <name>`,
  `icydb metrics --canister <name>`, and
  `icydb metrics --canister <name> --reset`.

### 🔧 Changed

- Extended generated config constants and actor build options with
  `ICYDB_METRICS_ENABLED`, `ICYDB_METRICS_RESET_ENABLED`, and
  `ICYDB_SNAPSHOT_ENABLED`.
- Updated build scripts to pass metrics and snapshot switches from generated
  config into actor code generation.
- Renamed the generated storage snapshot endpoint to `__icydb_snapshot` so
  generated exported endpoint functions consistently use the `__icydb_*`
  prefix.
- Updated `icydb config show` to display configured metrics and snapshot
  surfaces next to SQL surfaces.

### 📚 Documentation

- Documented that `__icydb_snapshot` reports store/entity/index inventory,
  estimated bytes, index state, and corruption counters over recovered stores.
- Updated README observability examples to use the new CLI commands instead of
  raw `icp canister call`.

## 0.156.3

`0.156.3` continues the config follow-through by adding config generation,
separating fixture lifecycle endpoints from DDL, and cleaning up the
user-facing SQL command docs.

### ➕ Added

- Added `icydb config init --canister <name>` to generate a default
  `icydb.toml`. It writes to an existing resolved config when `--force` is used,
  otherwise chooses the visible Cargo workspace root when one exists and falls
  back to the selected start directory.
- `config init` keeps the file DB-specific: it generates SQL switches and does
  not write ICP environments, principals, or canister ID mappings.

### 🔧 Changed

- Split fixture lifecycle endpoint generation into its own
  `[canisters.<name>.sql] fixtures = true` switch. `ddl = true` now controls the
  `__icydb_ddl` update endpoint only, while `fixtures = true` controls
  `__icydb_fixtures_reset` and `__icydb_fixtures_load`.
- Extended generated config constants and actor build options with a
  `sql_fixtures` lane so build scripts pass the three DB SQL surfaces
  independently: `readonly`, `ddl`, and `fixtures`.
- Updated `icydb config show` to display `fixtures` as a distinct configured
  SQL surface.
- Renamed the active generated endpoints to `__icydb_query`,
  `__icydb_ddl`, `__icydb_fixtures_reset`, and `__icydb_fixtures_load`;
  the fixture hook is `icydb_fixtures_load`.
- Documented the generated endpoint naming rule: generated Rust endpoint
  function names are exported verbatim with the `__icydb_*` prefix and should
  not rely on endpoint `name = ...` attribute overrides.

### 🧹 Cleanup

- Updated the local SQL demo README examples so every `icydb sql` invocation
  passes the required `--canister demo_rpg`.
- Clarified that `icydb sql` and canister lifecycle commands require an explicit
  canister target, while `--environment` still defaults to `demo`.
- Made `icydb config check` print an explicit note when it only validates the
  local config because no `--environment` was supplied for ICP sync checking.
- Split local setup, install, PocketIC test setup, wasm-report, and
  troubleshooting guidance out of the README into `INSTALLING.md`.
- Clarified `SECURITY.md` around `make install-canister-deps`, which installs
  Cargo wasm helper tools but does not run `sudo` or install OS packages.

## 0.156.1

`0.156.1` closes the developer-friendliness follow-up for the config and local
tooling cleanup line.

### 🧹 Cleanup

- Removed the unused workspace publish wrapper and `make publish` target.
  Publishing remains explicit maintainer work through `cargo publish`, with no
  repo script reading crates.io credentials.
- Removed `make install-env`; local Rust, OS package, ICP CLI, and optional
  DFINITY utility installation is now README-documented operator setup instead
  of a repo-owned bootstrap script.
- Clarified README onboarding for PocketIC-backed tests: use `POCKET_IC_BIN`
  for a local executable binary, or explicitly opt into the pinned GitHub
  download with `ICYDB_ALLOW_POCKET_IC_DOWNLOAD=1` and optional
  `POCKET_IC_SERVER_SHA256` verification.
- Added README examples for passing wasm report controls through
  `SIZE_REPORT_ARGS` and `AUDIT_REPORT_ARGS`.
- Updated the recurring wasm-footprint audit prompt to use
  `wasm-audit-report.sh` flags instead of the removed `WASM_AUDIT_*`,
  `WASM_CANISTER_NAME`, and `WASM_PROFILE` environment controls.

## Implementation

`0.156` now has the first host-only `icydb.toml` config generation slice.

### ➕ Added

- Added `icydb-config-build`, a build-helper crate that owns strict
  `icydb.toml` parsing, validation, config resolution, and generated Rust
  constant emission.
- Added `SECURITY.md` with the local development trust model for bootstrap,
  PocketIC downloads, publishing, tag deletion, canister refresh, and git hooks.
- Added support for absent config defaulting all optional surfaces off,
  per-canister `[canisters.<name>.sql] readonly = true`, `ddl = true`, and
  `fixtures = true` switches, canister-name validation against generated schema
  names, and normalized duplicate-name rejection.
- Added generated actor build options so build scripts can pass the validated
  SQL surface booleans into `icydb-build` without making the actor generator
  parse TOML.
- Wired the demo RPG, lightweight SQL test, and recurring audit canisters to
  config-aware build scripts. The demo canister uses the project-level
  `icydb.toml`, while the SQL test canister keeps a crate-local `icydb.toml`;
  their read-only SQL query plus DDL/fixture endpoints are now generated from
  config instead of the old manual macro.
- Config discovery now walks upward from the canister crate and uses the
  nearest `icydb.toml`, so grouped canister directories can own shared config.
- Added `icydb config show` and `icydb config check`. `show` displays the
  resolved project config without requiring an ICP environment, and adds sync
  markers when `--environment` is supplied; `check` returns nonzero when config
  and the selected ICP environment are out of sync.

### 🧹 Cleanup

- Removed the old default `icydb::build!`/`icydb-build::generate` actor
  generation path. Build scripts now pass explicit generated options through
  `icydb::build_with_options!()`.
- SQL endpoint shape now comes only from generated config plus actor build
  options.
- Removed demo SQL default target config entirely. `icydb sql` now requires an
  explicit `--canister`, while `readonly` and `ddl` independently control
  generated DB surfaces.
- The README SQL snippet now documents the config-based opt-in instead
  of the removed manual macro.
- Host-touching maintainer scripts now require clearer operator intent:
  PocketIC downloads need `ICYDB_ALLOW_POCKET_IC_DOWNLOAD=1`, and tag deletion
  requires an exact typed confirmation. The unused workspace publishing wrapper
  and local bootstrap target were removed; publishing is manual `cargo publish`
  work, and local setup is documented as operator-owned README work.
- Local development setup is now documented in the README instead of hidden in
  sudo-backed Make targets. `install-dev` was removed, and `update-dev` now
  only checks prerequisites, installs the pinned Wasm target, and runs a locked
  dependency fetch.
- Removed `SQLQ_CANISTER` and `SQLQ_HISTORY_FILE` environment-variable
  support from the CLI, made the demo SQL helper positional-argument based, and
  hardcoded publish registry polling intervals instead of exposing extra env
  knobs.
- Replaced the internal `ICYDB_CANISTER_WASM_PROFILE` and
  `ICYDB_CANISTER_SQL_MODE` bridge variables with explicit
  `build_fixture_canister --profile ... --sql-mode ...` flags owned by the wasm
  reporting scripts.
- Replaced `WASM_PROFILE`, `WASM_SQL_VARIANTS`, `WASM_CANISTER_NAME`, and
  `WASM_AUDIT_*` report controls with explicit `wasm-size-report.sh` and
  `wasm-audit-report.sh` flags. The Make targets now pass optional
  `SIZE_REPORT_ARGS` / `AUDIT_REPORT_ARGS` as make variables instead of
  requiring shell environment configuration.

## Design

`0.156` opens a tentative design for an `icydb.toml` project manifest.

### Design

- Added the `0.156` `icydb.toml` design document.
- Scoped the first config surface to per-canister DB SQL surfaces:
  `[canisters.<name>.sql] readonly = true`, `ddl = true`, and
  `fixtures = true`.
- Required TOML parsing to stay in a host-only build crate, with generated typed
  Rust constants or project metadata injected instead of runtime TOML loading.
- Made `icydb.toml` the sole endpoint-shape authority for DB SQL surfaces.
- Captured validation expectations for unknown fields, malformed config,
  canister-name matching, build-script rerun behavior, wasm dependency graph
  checks, and raw non-gzipped wasm size checks.

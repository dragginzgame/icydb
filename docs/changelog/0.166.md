# IcyDB 0.166 Changelog

## 0.166.0

### ➕ Added

- Started the SQL surface contract proof audit with a durable matrix that maps
  documented SQL claims to public query, update, and DDL entrypoints.

  ```text
  SQL_SUBSET.md claim -> public SQL entrypoint -> evidence row
  execute_sql_query::<E>(...) -> read/explain/introspection proof
  execute_sql_update::<E>(...) -> mutation proof
  execute_sql_ddl::<E>(...) -> accepted-catalog DDL proof
  ```

- Added the initial matrix rows for supported SELECT, mutation, DDL,
  introspection, blob, predicate, HAVING, alias, RETURNING, and unsupported SQL
  families.

- Mapped the first SQL surface matrix rows to existing public-entrypoint test
  evidence, marking proven rows for scalar SELECT, projection, aliases,
  aggregates, grouped output, EXPLAIN, core SQL writes, and DDL mutation
  families, then resolving the initial partial rows as direct public-surface
  proof was added.

  ```text
  execute_sql_query::<E>(...) tests -> SELECT/projection/aggregate proof
  execute_sql_update::<E>(...) tests -> INSERT/UPDATE/DELETE proof
  execute_sql_ddl::<E>(...) tests -> CREATE/DROP/ALTER proof
  ```

- Added direct public-query entrypoint coverage for metadata SQL so
  `DESCRIBE`, `SHOW COLUMNS`, `SHOW ENTITIES`, and `SHOW TABLES` are proven
  through `execute_sql_query::<E>(...)` rather than only helper surfaces.

- Added direct public-update entrypoint coverage for `INSERT`, `UPDATE`, and
  `DELETE ... RETURNING *` so the wildcard returning contract is proven through
  `execute_sql_update::<E>(...)`.

- Added public-update fail-closed coverage for malformed and oversized SQL blob
  literals so blob parsing errors are proven not to partially mutate rows.

- Split the broad `WHERE` predicate proof row into concrete predicate-family
  rows, with direct public-query evidence for comparisons, boolean trees,
  field paths, field-to-field predicates, membership, ranges, null tests,
  boolean truth tests, prefix text predicates, direct `STARTS_WITH`, and
  searched `CASE`.

  ```text
  WHERE predicates -> per-family proof rows
  field-to-field / BETWEEN / IN / NULL / truth predicates -> public rows
  prefix LIKE / ILIKE / STARTS_WITH -> bounded prefix proof
  ```

- Added public-query fail-closed coverage for unsupported SQL families,
  including joins, subqueries, CTEs, set operations, window functions,
  scalar SQL cursor-token syntax, transaction statements, byte-metric
  functions, simple `CASE`, non-prefix text patterns, and grouped row-field
  text predicates.

- Fixed SQL expression lowering so `LIKE` / `ILIKE` only admit the same bounded
  prefix pattern as predicate parsing: exactly one trailing `%`, with no
  internal `%` or `_` wildcard.

  ```text
  LIKE 'al%' -> admitted prefix predicate
  LIKE '%al%' -> rejected
  LIKE 'a_' -> rejected
  ```

- Added public-update unchanged-state coverage for unsupported subquery/CTE
  mutations and unsupported computed or aggregate `RETURNING` projections.

### 📚 Documentation

- Marked the 0.166 design as in progress and linked the concrete proof matrix
  artifact so subsequent slices can start from contract rows rather than
  implementation convenience.

## 0.166.1

### 🩹 Fixed

- Fixed scalar SQL ordering over composite primary-key components so explicit
  `ORDER BY` terms remain the leading sort semantics and only missing
  primary-key components are appended as deterministic tie-breakers.

  ```text
  ORDER BY y DESC -> y DESC, x ASC
  not -> x DESC, y DESC
  ```

### 📚 Documentation

- Reconciled the 1.0 TODO list with the public SQL proof matrix by adding
  explicit checked items for the proven SQL query, update, DDL, projection,
  predicate, and metadata surfaces without marking broader engine-level 1.0
  TODOs complete.

- Updated the SQL surface mapping note so implementation-facing SQL ownership
  points at the 0.166 proof matrix as the public-entrypoint evidence ledger.

- Advanced the 0.166 design status to a closeout candidate after the first
  public-surface proof pass, while keeping final closeout gated on
  release-slice validation and review of future SQL contract changes.

- Added explicit matrix rows for wrong-entrypoint SQL ownership so query,
  update, and DDL statement families stay separated by public API rather than
  only by implementation helpers.

- Corrected the SQL subset contract for grouped text-function projections:
  text transforms over grouped key values are part of the supported statement
  surface, while raw non-group field text transforms remain outside grouped key
  authority.

## 0.166.2

### 🧪 Testing

- Added explicit fail-closed proof for SQL casts through both public query and
  public update entrypoints, so `CAST(...)` remains outside the admitted
  bounded expression family instead of becoming accidental parser surface or
  mutating state.

### 📚 Documentation

- Made supported aggregate expression-input, `DISTINCT`, and `FILTER` shapes
  explicit in the SQL subset contract and proof matrix, matching the existing
  public aggregate coverage.

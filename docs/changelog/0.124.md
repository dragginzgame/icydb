# 0.124 Changelog

## 0.124.0

- `0.124.0` starts the scalar-function semantics formalization line by introducing one planner-owned function registry and then making SQL parser and lowering consume that registry-driven ownership surface instead of keeping phase-local function family tables.

- The new planner-owned registry lives in [/home/adam/projects/icydb/crates/icydb-core/src/db/query/plan/expr/function_semantics.rs](/home/adam/projects/icydb/crates/icydb-core/src/db/query/plan/expr/function_semantics.rs):
  - added:
    - `FunctionCategory`
    - `FunctionNullBehavior`
    - `FunctionDeterminism`
    - `FunctionTypeInferenceShape`
    - `FunctionSpec`
  - `Function` now owns:
    - `spec()`
    - `type_inference_shape()`
    - `is_compare_operand_coarse_family()`

- Planner typing no longer acts as the hidden scalar-function registry owner:
  - [/home/adam/projects/icydb/crates/icydb-core/src/db/query/plan/expr/type_inference/mod.rs](/home/adam/projects/icydb/crates/icydb-core/src/db/query/plan/expr/type_inference/mod.rs)
  - removed the local function-family typing ladder
  - compare-operand coarse-family checks now also read the shared function registry surface instead of staying local to type inference

- The canonical planner function taxonomy in [/home/adam/projects/icydb/crates/icydb-core/src/db/query/plan/expr/ast.rs](/home/adam/projects/icydb/crates/icydb-core/src/db/query/plan/expr/ast.rs) is now mechanically ordered and de-duplicated:
  - `Function` is `#[remain::sorted]`
  - duplicate `Ceil` versus `Ceiling` planner identities were collapsed to one canonical `Function::Ceiling`
  - parser-style `CEIL` input is still accepted, but now canonicalizes directly to `Function::Ceiling` instead of surviving as a second semantic function identity

- The SQL-side scalar-function bridge now consumes enum-owned ownership surfaces instead of carrying separate local dispatch ladders:
  - [/home/adam/projects/icydb/crates/icydb-core/src/db/sql/parser/model.rs](/home/adam/projects/icydb/crates/icydb-core/src/db/sql/parser/model.rs)
    - `SqlScalarFunction` now owns:
      - `planner_function()`
      - `uses_round_special_case()`
      - `is_casefold_transform()`
      - `non_where_call_shape()`
      - `where_call_shape()`
    - `ceil` and `ceiling` now both parse to one canonical `SqlScalarFunction::Ceiling`
  - [/home/adam/projects/icydb/crates/icydb-core/src/db/sql/parser/projection.rs](/home/adam/projects/icydb/crates/icydb-core/src/db/sql/parser/projection.rs)
    - parser surface admission now routes through enum-owned scalar-function support checks
    - scalar-function call dispatch now routes through `SqlScalarFunctionCallShape`
    - the old unary scalar parser wrapper fell out and was deleted
    - WHERE casefold admission/canonicalization no longer keeps raw `LOWER | UPPER` pair-matches
  - [/home/adam/projects/icydb/crates/icydb-core/src/db/sql/parser/clauses.rs](/home/adam/projects/icydb/crates/icydb-core/src/db/sql/parser/clauses.rs)
    - `ORDER BY` no longer keeps its own local `ROUND` branch separate from the shared scalar-function parse lane
  - [/home/adam/projects/icydb/crates/icydb-core/src/db/sql/lowering/expr.rs](/home/adam/projects/icydb/crates/icydb-core/src/db/sql/lowering/expr.rs)
    - lowering now reads `SqlScalarFunction::planner_function()` instead of a free SQL-to-planner mapping ladder

- The same canonical `Ceiling` identity now flows through the remaining scalar-function consumers that previously treated `Ceil` and `Ceiling` as distinct semantic functions:
  - [/home/adam/projects/icydb/crates/icydb-core/src/db/executor/projection/eval/scalar_function.rs](/home/adam/projects/icydb/crates/icydb-core/src/db/executor/projection/eval/scalar_function.rs)
  - [/home/adam/projects/icydb/crates/icydb-core/src/db/query/plan/expr/preview.rs](/home/adam/projects/icydb/crates/icydb-core/src/db/query/plan/expr/preview.rs)
  - [/home/adam/projects/icydb/crates/icydb-core/src/db/query/builder/aggregate.rs](/home/adam/projects/icydb/crates/icydb-core/src/db/query/builder/aggregate.rs)
  - [/home/adam/projects/icydb/crates/icydb-core/src/db/sql/lowering/aggregate.rs](/home/adam/projects/icydb/crates/icydb-core/src/db/sql/lowering/aggregate.rs)

- Surface admission, fixed-scale recovery, and boolean-function ownership also moved onto the shared function owner:
  - [/home/adam/projects/icydb/crates/icydb-core/src/db/query/plan/expr/function_semantics.rs](/home/adam/projects/icydb/crates/icydb-core/src/db/query/plan/expr/function_semantics.rs)
    - added:
      - `FunctionSurface`
      - `BooleanFunctionShape`
      - `NullTestFunctionKind`
      - `TextPredicateFunctionKind`
      - `FieldPredicateFunctionKind`
      - `AggregateInputConstantFoldShape`
    - `Function` now also owns:
      - `supports_surface(...)`
      - `fixed_decimal_scale(...)`
      - `boolean_function_shape()`
      - `boolean_null_test_kind()`
      - `boolean_text_predicate_kind()`
      - `boolean_field_predicate_kind()`
      - `aggregate_input_constant_fold_shape()`
      - `is_casefold_transform()`
  - [/home/adam/projects/icydb/crates/icydb-core/src/db/session/sql/projection/labels.rs](/home/adam/projects/icydb/crates/icydb-core/src/db/session/sql/projection/labels.rs)
    - fixed-scale recovery no longer open-codes `ROUND`
  - [/home/adam/projects/icydb/crates/icydb-core/src/db/query/plan/expr/canonicalize.rs](/home/adam/projects/icydb/crates/icydb-core/src/db/query/plan/expr/canonicalize.rs)
    - scalar/grouped boolean-function admission and normalization now read the shared boolean-function owner
  - [/home/adam/projects/icydb/crates/icydb-core/src/db/query/plan/expr/predicate_compile.rs](/home/adam/projects/icydb/crates/icydb-core/src/db/query/plan/expr/predicate_compile.rs)
    - boolean predicate compilation now reads both the shared boolean-function family and the finer owner-side subkinds instead of re-matching raw function variants locally

- Aggregate-input constant folding no longer keeps duplicated foldable-function family ladders in both fluent aggregate building and SQL aggregate lowering:
  - [/home/adam/projects/icydb/crates/icydb-core/src/db/query/builder/aggregate.rs](/home/adam/projects/icydb/crates/icydb-core/src/db/query/builder/aggregate.rs)
  - [/home/adam/projects/icydb/crates/icydb-core/src/db/sql/lowering/aggregate.rs](/home/adam/projects/icydb/crates/icydb-core/src/db/sql/lowering/aggregate.rs)
  - both now read `Function::aggregate_input_constant_fold_shape()` for the admitted `ROUND`, `COALESCE`, `NULLIF`, and unary numeric literal-fold surface

- Aggregate-input canonicalization itself is now planner-owned in one shared seam instead of being duplicated between fluent aggregate construction and SQL aggregate lowering:
  - [/home/adam/projects/icydb/crates/icydb-core/src/db/query/plan/expr/aggregate_input.rs](/home/adam/projects/icydb/crates/icydb-core/src/db/query/plan/expr/aggregate_input.rs)
    - now owns the recursive literal-only aggregate-input fold pass
    - now owns numeric literal normalization for canonical aggregate identity
    - now owns `canonicalize_aggregate_input_expr(...)` for both builder- and SQL-lowered expression inputs
  - [/home/adam/projects/icydb/crates/icydb-core/src/db/query/builder/aggregate.rs](/home/adam/projects/icydb/crates/icydb-core/src/db/query/builder/aggregate.rs)
    - no longer owns its own private aggregate-input constant-fold and numeric-normalization pipeline
  - [/home/adam/projects/icydb/crates/icydb-core/src/db/sql/lowering/aggregate.rs](/home/adam/projects/icydb/crates/icydb-core/src/db/sql/lowering/aggregate.rs)
    - no longer reruns a second aggregate-input constant-fold pass before calling the canonicalizer
    - now lowers aggregate expression inputs once and hands the resulting planner expression directly to the shared aggregate-input owner

- Casefold wrapper ownership is now explicit on the function enums instead of open-coded in parser/planner helper seams:
  - [/home/adam/projects/icydb/crates/icydb-core/src/db/sql/parser/model.rs](/home/adam/projects/icydb/crates/icydb-core/src/db/sql/parser/model.rs)
  - [/home/adam/projects/icydb/crates/icydb-core/src/db/sql/parser/projection.rs](/home/adam/projects/icydb/crates/icydb-core/src/db/sql/parser/projection.rs)
  - [/home/adam/projects/icydb/crates/icydb-core/src/db/query/plan/expr/ast.rs](/home/adam/projects/icydb/crates/icydb-core/src/db/query/plan/expr/ast.rs)
  - the remaining raw `LOWER | UPPER` pair-matches were removed from those query/parser surfaces in favor of enum-owned `is_casefold_transform()` checks

- This keeps the first `0.124.0` cut bounded:
  - one planner-owned scalar-function registry now exists
  - planner typing consumes it instead of owning a parallel hidden ladder
  - SQL parser and SQL lowering now consume enum-owned scalar-function identity, call-shape, surface-admission, casefold, and aggregate-input fold ownership
  - aggregate-input canonicalization no longer exists as two parallel recursive fold/normalize implementations in builder and SQL lowering
  - boolean predicate shaping now reads shared function-family and subkind ownership instead of repeating local function-variant ladders
  - duplicate `CEIL` versus `CEILING` semantic identities are gone from the live planner and parser taxonomies
  - evaluator redesign and broader scalar-function execution coverage remain separate follow-up work

# 0.57 Changelog

## 0.57.1 - 2026-03-16 - BYTES-by Covering Closure

### Summary

`0.57.1` expands bounded BYTES optimization coverage for `bytes_by(field)` by
reusing existing covering/index-only projection families where safe, and adds
`explain_bytes_by(field)` terminal metadata projection while keeping strict-mode
shapes fail-closed.

### Changed

- Expanded bounded BYTES optimization coverage for `bytes_by(field)` terminals:
  - `bytes_by_slot(...)` now reuses existing covering-index projection routing
    for ignore-mode no-residual shapes instead of always materializing rows.
  - prefix-bound constant-covering shapes now short-circuit to
    `serialized_len(constant) * aggregate_count(window)` without allocating
    repeated projected values.
  - test-only optimization diagnostics now use dedicated BYTES-by counters
    (`BytesByCoveringIndexFastPath`, `BytesByCoveringConstantFastPath`) so
    bytes-by routing is not conflated with generic field-projection counters.
  - strict-mode (`MissingRowPolicy::Error`) now explicitly bypasses both
    constant-covering and covering-index `bytes_by(...)` fast paths to preserve
    missing-row fail-closed behavior.
  - `ExecutablePlan` now owns one shared `bytes_by` projection-mode classifier
    (`field_materialized` / `field_covering_index` /
    `field_covering_constant`) consumed by terminal routing and explain
    projection metadata.
  - fluent load now exposes `explain_bytes_by(field)` to project bounded
    terminal metadata (`terminal`, `terminal_field`,
    `terminal_projection_mode`, `terminal_index_only`) without executing
    `bytes_by(...)`.
  - added bytes-by covering marker/parity regressions for:
    - index-covering eligible shapes
    - constant-covering prefix-bound shapes
    - residual-predicate constant-covering shapes
    - strict-mode bypass and strict missing-row corruption surfacing
  - added explain-bytes-by regression coverage for:
    - filtered load-shape metadata projection
    - strict-mode materialized fail-closed labeling
    - unknown-field fail-closed classification
- Tightened generated reduced-SQL dispatch control flow so lane callbacks consume
  one prepared SQL envelope instead of reparsing/repreparing per callback:
  - core/facade session now expose `prepare_sql_dispatch_parsed(...)` and
    `execute_sql_dispatch_prepared(...)` surfaces to split prepare and execute
    phases explicitly.
  - core/facade session now expose
    `execute_sql_dispatch_query_lane_prepared(...)` so generated lane callbacks
    bind only query/explain execution lanes.
  - core/facade session now expose schema-model introspection helpers
    (`describe_entity_model(...)`, `show_indexes_for_model(...)`,
    `show_columns_for_model(...)`) so descriptor-owned introspection lanes do
    not require per-entity SQL execution wrappers.
  - generated `sql_dispatch::query(...)` now parses once, resolves entity route,
    prepares once against that route entity, then executes the prepared envelope
    through per-entity callbacks.
  - descriptor lane callback signatures now accept `SqlPreparedStatement` and
    are limited to query/explain execution, while DESCRIBE/SHOW INDEXES/SHOW
    COLUMNS route through descriptor-model introspection helpers.
  - this preserves existing lane semantics and fail-closed behavior while making
    parse/prepare authority explicit before dynamic lane callback dispatch.
- Added a Slice A executor-entry shim in core session query execution:
  - `execute_query(...)` now delegates to one new `execute_query_dyn(mode, plan)`
    entrypoint after intent planning.
  - this keeps existing typed APIs stable while creating one central execution
    boundary for subsequent descriptor-driven executor migration slices.
- Added a Slice B1 load-orchestrator dynamic boundary in executor pipeline:
  - added `LoadExecutionDescriptor` as immutable stage-loop authority for load
    orchestration control flow.
  - `LoadExecutor::execute_load(...)` now delegates through
    `execute_load_dyn(descriptor, ...)` using the canonical release descriptor.
  - this preserves existing typed execution behavior while introducing one
    descriptor-backed hook for subsequent executor-internal migration slices.
- Reduced route-planner control-flow monomorphization in executor internals:
  - moved route intent normalization and route execution-stage derivation from
    `LoadExecutor<E>` impl methods into shared planner-owned free functions.
  - moved shape builders (`load` / `aggregate_count` /
    `aggregate_non_count` / `aggregate_grouped`) to shared planner-owned
    functions and exposed shared route-mode gates
    (`load_streaming_allowed`, `aggregate_non_count_streaming_allowed`) at the
    route module boundary.
  - `build_execution_route_plan::<E>(...)` now retains only entity-specific
    feasibility/profile derivation while reusing shared non-generic intent +
    execution-stage assembly flow.
  - added model-based route capability helpers
    (`derive_budget_safety_flags_for_model`, `stream_order_contract_safe_for_model`)
    and rewired typed capability wrappers to delegate through those model-based
    helpers.
  - promoted route order-contract checks to one model-based helper
    (`access_order_satisfied_by_route_contract_for_model`) and rewired
    post-access ordering gates and scalar page-kernel post-access ordering to
    use that helper directly instead of per-entity route-order wrappers.
- Reduced SQL-lowering monomorphization for entity-routed dispatch paths:
  - `lower_select_prepared::<E>(...)` now delegates SQL shape analysis to one
    new entity-agnostic lowering phase (`lower_select_shape(...)`) before typed
    `Query<E>` binding.
  - scalar/grouped projection and HAVING normalization contracts now resolve in
    shared non-generic helpers, while typed bind applies only query-surface
    operations (`group_by`, projection bind, HAVING bind, predicate/order/page).
  - this preserves fail-closed SQL behavior while shrinking per-entity lowered
    control-flow duplication in `sql_dispatch` query/explain callbacks.
- Reduced grouped-fold runtime monomorphization in executor internals:
  - moved grouped HAVING evaluation and grouped projection shaping into shared
    executor-owned free helpers (`group_matches_having`,
    `project_grouped_row(s)_from_projection`) so those loops no longer compile
    as `LoadExecutor<E>` methods per entity.
  - moved grouped candidate-row alignment/selection and grouped page
    materialization loops into shared non-entity runtime helpers, while keeping
    typed boundaries only where required (`AggregateEngine<E>` finalization and
    grouped next-cursor emission).
  - grouped fold now finalizes typed aggregate outputs into canonical
    `(group_key, aggregate_value)` iterators first, then runs shared HAVING,
    resume-boundary, selection-bound, and pagination control flow over
    non-generic value payloads.
  - moved grouped fold stream ingestion control flow into shared non-generic
    helpers (`ingest_grouped_rows_loop_dyn`, `ingest_grouped_fold_input_dyn`)
    so key-stream traversal and per-engine short-circuit loop ownership compile
    once, while typed wrappers inject row decode/predicate and
    `AggregateEngine<E>` fold actions.
  - quickstart debug `twiggy monos` now drops
    `LoadExecutor<E>::execute_group_fold` from top offenders after this ingest
    control-flow extraction; the remaining top executor hotspot shifts to
    grouping-stage projection application.
- Reduced scalar runtime scan/predicate monomorphization in executor internals:
  - moved ordered key-stream scan loops used by scalar materialization and
    payload-byte windows into shared non-generic helpers in
    `runtime_context`, with typed wrappers only injecting row-consistency read
    policy.
  - moved row-stream reducer key iteration into one shared key-driver helper so
    terminal row-collector execution no longer duplicates stream loop control
    flow per entity.
  - rewired `PredicateProgram` runtime evaluation to use one shared
    slot-reader callback evaluator (`eval_with_resolved_slots` over
    `FnMut(slot) -> Option<Value>`), leaving `eval::<E>(...)` as a thin typed
    adapter.
  - rewired scalar projection expression evaluation to use one shared
    slot-reader callback evaluator (`eval_expr_with_slot_reader`) and updated
    projection materialization to call that shared evaluator directly per row,
    leaving typed wrappers only at row slot-access boundaries.
  - introduced an erased kernel retry boundary
    (`DynMaterializedExecutionAttempt`) so scalar residual-retry control flow
    runs through one shared non-generic orchestrator while typed
    route-materialization/decode remains fail-closed and explicitly downcast at
    the typed wrapper boundary.
  - introduced a shared scalar page-kernel orchestration helper
    (`execute_scalar_page_kernel_dyn`) so scan-budget validation, ordered-stream
    scan loop ownership, and kernel-row collection control flow run through one
    non-generic boundary with callback-injected typed decode/slot extraction.
  - scalar page-kernel row buffering now uses a non-generic `KernelRow`
    envelope (`DataRow` + resolved slot values), and post-access
    filter/order/cursor/page/delete-limit phases now execute over that shared
    row shape before typed response decode at the boundary.
  - removed `Box<dyn Any>` page payload transport from scalar page-kernel
    orchestration; the dynamic kernel now returns concrete `Vec<KernelRow>`
    payloads and row-scan metrics directly.
  - introduced a shared post-access phase orchestration helper
    (`apply_post_access_kernel_dyn`) so filter/order/cursor/page/delete-limit
    control flow executes once with callback-injected typed row operators.
  - moved scalar page-kernel post-access filter/order/cursor/page/delete-limit
    control flow to one model-based non-generic helper
    (`apply_post_access_to_kernel_rows_dyn(model, ...)`) so ordering and
    continuation-boundary loops no longer compile as `LoadExecutor<E>` methods.
  - added route-level model-based order-contract helper
    (`access_order_satisfied_by_route_contract_for_model(model, plan)`) and
    rewired the existing typed helper to delegate, keeping route authority
    unchanged while reducing per-entity post-access route-check duplication.
  - introduced a shared scalar-page finalization helper
    (`finalize_scalar_page_control_flow_dyn`) so post-access completion,
    continuation derivation, and projection-alignment control flow execute
    through one non-generic boundary before typed response emission.
  - `LoadExecutor::<E>::materialize_key_stream_into_page(...)` now acts as a
    thin typed adapter that injects entity-owned row-read consistency and
    decode/materialize callbacks, then downcasts the dynamic page payload
    fail-closed at the typed boundary.

## 0.57.0 - 2026-03-16 - SHOW TABLES Alias + LOWER LIKE Prefix SQL

### Summary

`0.57.0` starts the `0.57` line with two bounded reduced-SQL expansions:
`SHOW TABLES` now aliases the existing `SHOW ENTITIES` lane, and
`LOWER(field) LIKE 'prefix%'` now lowers into the existing casefold
starts-with predicate path. Both additions preserve fail-closed behavior
outside explicitly supported shapes.

### Changed

- Expanded reduced SQL parser support for one bounded show-style alias:
  - added `TABLES` keyword recognition for `SHOW TABLES`.
  - `SHOW TABLES` now parses to the canonical `SqlStatement::ShowEntities`
    lane.
  - unsupported `SHOW ...` fail-closed labels remain deterministic and now
    enumerate `SHOW TABLES` among supported families.
- Kept lowering authority unchanged while extending alias coverage:
  - `compile_sql_command("SHOW TABLES", ...)` lowers to
    `SqlCommand::ShowEntities`.
  - no new execution lane or planner-emulation path was introduced.
- Extended core/facade session behavior and lane diagnostics:
  - `show_entities_sql(...)` now accepts `SHOW ENTITIES` and `SHOW TABLES`.
  - wrong-lane rejection wording for query/explain surfaces now explicitly
    references `SHOW ENTITIES/SHOW TABLES`.
  - dedicated route and lane matrix tests now include `SHOW TABLES` parity and
    fail-closed checks across non-entity-listing surfaces.
- Updated reduced SQL contract and design baseline docs:
  - `SQL_SUBSET` now documents `SHOW ENTITIES` and `SHOW TABLES` executable
    shapes under one dedicated introspection lane.
  - `0.57` design baseline and Slice A target wording now align with current
    shipped scope and this alias expansion.
- Expanded reduced SQL predicate lowering with one bounded casefold family:
  - parser now accepts `LOWER(<field>) LIKE '<prefix>%'` in `WHERE` clauses.
  - that shape lowers to one text-casefold `StartsWith` predicate intent over
    the referenced field.
  - adjacent `LIKE` families remain fail-closed with deterministic labels:
    - plain `<field> LIKE ...` remains unsupported.
    - non-prefix wildcard patterns for `LOWER(field) LIKE ...` remain
      unsupported.
- Extended core/facade SQL session parity coverage for the new predicate family:
  - core `query_from_sql(...)` parity now includes
    `LOWER(field) LIKE 'prefix%'` lowering checks.
  - facade `query_from_sql(...)` now includes a dedicated lowered-intent check
    for the same shape.
  - unsupported-feature detail-preservation matrices now include new `LIKE`
    fail-closed labels across core/facade SQL surfaces.

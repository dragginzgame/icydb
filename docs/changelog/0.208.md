# IcyDB 0.208 Changelog

## 0.208.1 - 2026-07-21

### 📝 Summary

- Completes the exact-composite closeout by making the accepted value catalog
  authoritative for publication validation, authored writes, SQL projection,
  and accepted-schema introspection.
- Removes the remaining local projection and session-catalog decisions that
  could disagree with accepted schema authority.

### ⚠️ Breaking

- Accepted `DESCRIBE` and `SHOW COLUMNS` output now renders a composite's
  nominal path, codec, complete record/tuple/newtype shape, nested kinds, and
  nullability instead of exposing an opaque internal type ID. This is a
  pre-1.0 hard cut with no legacy rendering alias.
- Accepted bundles with incomplete or inconsistent nested-leaf, primary-key,
  index, or relation metadata now fail closed instead of publishing a contract
  that disagrees with its value catalog.

### 🩹 Fixed

- Validates every persisted composite nested-leaf descriptor against the
  complete accepted catalog shape, including recursive member paths, kind,
  nullability, and leaf count.
- Rejects whole composites and collections in unsupported primary-key, index,
  and relation-key roles while retaining accepted nested scalar paths.
- Makes schema capability classification the sole owner of SQL result
  selectability, so explicit projection and `SELECT *` no longer pass through
  a second composite-specific filter and reconstruction flow.
- Preserves typed authored-value admission causes through entity saves and
  structural patches; only actual canonical encoding failures become
  persistence errors.
- Makes accepted introspection consume the same revision-scoped value-catalog
  handle as execution and fail closed when nominal catalog identity is missing.

### 🗑️ Removed

- Deletes the duplicate composite projection helpers, snapshot-only session
  catalog accessor, and opaque composite type-ID diagnostic rendering.
- Leaves no production `AnyValue`, `AnyType`, retired structured schema kind,
  compatibility decoder, or alternate exact-composite execution path.

### 🧪 Testing

- Focused publication tests cover valid complete contracts plus missing,
  mismatched, malformed, and unsupported-role rejection.
- Focused SQL and write tests cover exact accepted descriptions, unresolved
  catalog failure, `SELECT *`, explicit composite projection, and typed
  admission classification through entity and structural-patch writes.
- Strict Clippy, native and Wasm feature checks, formatting, diff hygiene, and
  repository architecture guards pass. Full repository suites remain the
  user-owned push validation boundary.

### 📊 Audit

- The corrected exact-only SQL performance Wasm is 3,933,708 raw non-gzipped
  bytes. It remains 60,286 bytes (1.51%) smaller than the discarded open-ended
  experiment and is 2,950 bytes (0.08%) above the initial 0.208 exact-only
  capture because accepted introspection now carries complete catalog truth.
- Focused 20,000-iteration native measurements report 299 ns/op to normalize a
  three-member exact record, 176 ns/op for a two-element exact tuple, and 275
  ns/op for a record containing that tuple. Borrowed validation measures 25,
  15, and 27 ns/op respectively and allocates no temporary collection.
- The correction batch changes approximately 30 core files, adding 1,166 lines
  and removing 346 before documentation. Most growth is fail-closed invariant
  coverage; production ownership is simpler at SQL projection and session
  catalog boundaries.

## 0.208.0 - 2026-07-21

### 📝 Summary

- Replaces generic persisted structured fields with exact nominal record,
  tuple, and newtype contracts.
- Makes accepted schema independently authoritative for every persisted custom
  value: nominal identity, codec, member names, tuple arity, nested kinds, and
  nullability now survive publication and recovery without generated-model
  reconstruction.
- Publishes enum definitions, composite definitions, entity snapshots,
  revision, and fingerprint as one store-local accepted authority.
- Removes the experimental open-ended `AnyValue`/`AnyType` direction. IcyDB
  has no schema-less or heterogeneous persisted field type.

### ➕ Added

- Adds generated composite metadata for records, tuples, and newtypes:
  `CompositeCodec`, `CompositeShapeModel`, `CompositeFieldModel`, and
  `CompositeElementModel`.
- Adds `FieldKind::Composite { path, codec, shape }` as the sole generated
  contract for a custom persisted Rust type. The path is nominal identity and
  the shape is exhaustive rather than a runtime hint.
- Adds one canonical `AcceptedCompositeCatalog` per accepted store revision.
  It owns deterministic non-zero type IDs and exact accepted definitions;
  accepted field kinds retain only `CompositeTypeId` references.
- Adds catalog discovery through entity fields, enum payloads, collections,
  and nested composites, with deterministic canonical ordering.
- Adds bounded, fallible composite-catalog encoding and decoding with checks
  for invalid IDs, missing references, duplicate or unsorted record members,
  cycles, excessive depth, unknown tags, and future versions.
- Adds accepted-schema descriptions for nominal composite type and shape so
  diagnostics observe the same catalog authority used by runtime execution.

### ⚠️ Breaking

- Removes public/generated `FieldKind::Structured { queryable }` and internal
  `AcceptedFieldKind::Structured { queryable }`. There is no alias, deprecated
  spelling, compatibility decoder, or dual dispatch.
- Replaces `FieldStorageDecode::Value` with
  `FieldStorageDecode::CatalogValue`. The new name describes a canonical
  recursive storage lane whose exact admissible shape still comes from the
  accepted catalog.
- Changes the accepted schema snapshot profile and accepted bundle contents to
  include exact composite authority. Earlier development schema/catalog forms
  intentionally fail closed and must not be migrated through fallback code.
- Whole composite values are not scalar SQL literals, comparison or ordering
  operands, grouping keys, aggregate inputs, relation keys, primary keys, or
  index leaves. Supported nested scalar paths retain capabilities derived from
  their accepted path contracts.
- Manual code that constructs `FieldKind` or `FieldStorageDecode` values must
  move directly to the new exact forms. Generated schemas acquire the new
  metadata by regeneration.

### 🔧 Changed - Generated Model

- Generated record, tuple, and newtype metadata now emits one nominal
  `FieldKind::Composite` with its codec, complete recursive shape, field names,
  tuple arity, and nested nullability.
- Record members retain stable persisted names; tuple elements retain position
  and arity; newtypes remain nominally distinct from their inner value. Empty
  records and empty tuples remain distinct shapes.
- Recursive enum payload metadata can resolve generated kinds without creating
  recursive associated-constant graphs, while accepted IDs remain the runtime
  authority after publication.
- Generated nested-field projections remain a convenience view only. They do
  not reconstruct missing accepted composite definitions.

### 🔧 Changed - Accepted Authority And Publication

- Accepted snapshots store `AcceptedFieldKind::Composite { type_id }`; complete
  definitions are stored once in the accepted composite catalog.
- Reconciliation constructs enum and composite candidates before entity
  publication, resolves cross-catalog references, and publishes both catalogs
  atomically with all entity snapshots.
- Reconciliation preserves existing nominal type IDs and rejects path, codec,
  shape, member, arity, nested-kind, or nullability drift. Deterministic new
  definitions may be appended only when surviving identities remain stable.
- Accepted bundle identity and schema fingerprints now include composite
  definitions. Map and composite field kinds use separate fingerprint domains.
- Renames the combined runtime catalog authority from the misleading enum-only
  `AcceptedEnumCatalogHandle` to `AcceptedValueCatalogHandle`; no alias remains.
- Query caches, prepared execution, row decoding, SQL projection, mutation
  preparation, diagnostics, and schema introspection carry the same verified
  revision-scoped value-catalog handle forward.

### 🔧 Changed - Admission And Persistence

- Authored record values must contain every accepted member exactly once and no
  unknown member. Canonical output follows accepted member order.
- Authored tuples must match accepted arity and element order exactly. Newtype
  payloads must match the accepted inner contract while retaining nominal type
  identity.
- Nested values are normalized recursively against accepted enum, composite,
  collection, and nullability contracts before row bytes are emitted.
- Canonical decode is bounded and fallible, then revalidates the decoded value
  against the same accepted contract before exposure. Malformed state cannot
  become an empty/default value or an alternate decode path.
- Contract construction, enum/composite catalog decoding, authored-value
  admission, and canonical persistence share one recursion-depth boundary.
- The accepted schema snapshot now carries the required `ICYX` contract profile.
  Enum, composite, bundle, and root envelopes expose only their current
  pre-1.0 identities and versions.
- Nested accepted leaves no longer persist redundant storage-decode and leaf-
  codec facts; those mechanics are derived from their authoritative accepted
  kinds.

### 🔧 Changed - Query And Write Semantics

- SQL and typed/fluent paths derive composite capability from accepted schema
  rather than a stored queryability boolean or the current runtime value.
- Unsupported whole-composite predicates, ordering, grouping, aggregation,
  indexing, relations, and SQL DDL kinds reject before execution.
- Explicitly accepted nested scalar paths continue through the ordinary shared
  planning and execution architecture; 0.208 does not add a second composite-
  specific query system.
- Omission remains a field-level accepted-schema decision. A required composite
  omitted from a write rejects; an explicit nullable value admits `NULL`; an
  accepted database default materializes its exact value. Rust `Default` does
  not grant database omission rights.
- Typed list, set, and map writes may contain exact composite values. Outer
  collection validation preserves collection shape while nominal leaf
  validation remains catalog-owned.

### 🩹 Fixed

- Gives map and composite field kinds separate fingerprint discriminators,
  preventing structurally crafted map and composite contracts from sharing a
  schema/index-allocation hash stream.
- Rejects recursive contracts at the same depth at which admission and
  canonical persistence reject their values, preventing accepted but
  uninhabitable schemas.
- Preserves typed saves for lists, sets, and maps containing exact composites
  instead of rejecting them at the coarse pre-catalog storage check.
- Makes accepted composite reconciliation directly prove stable IDs and exact
  shapes while permitting deterministic additions.
- Ensures accepted-schema publication, recovery selection, row decoding, and
  public projection all carry both enum and composite authority from the same
  verified revision.

### 🗑️ Removed

- Deletes all IcyDB-owned `AnyValue`, `AnyType`, generic persisted structured
  kinds, stored structured-queryability flags, compatibility readers, and old
  generated fixture terminology.
- Deletes the accepted structured wire discriminator and the old schema paths
  that admitted arbitrary list/map payloads under a coarse field kind.
- Deletes planner and executor decisions based on the obsolete structured
  queryability flag.
- Deletes duplicate nested-leaf codec state that could disagree with accepted
  field-kind authority.
- Retains `RuntimeValueKind::Structured` and structural codec names only for
  internal runtime/storage representation; neither is a schema contract.

### 🧭 Migration Notes

- Regenerate schema-derived code so custom records, tuples, and newtypes emit
  exact composite metadata.
- Replace manual `FieldKind::Structured` construction with an exact
  `FieldKind::Composite` proposal, or use the schema derive output.
- Replace manual `FieldStorageDecode::Value` references with
  `FieldStorageDecode::CatalogValue` only when the value has an exact accepted
  recursive contract.
- Recreate pre-0.208 development stores. This release intentionally does not
  decode or translate the previous accepted schema/catalog profile.
- Do not use `RuntimeValueKind::Structured` as a schema declaration. It remains
  an internal execution/storage classification only.
- Existing cursor encoding and the ordinary SQL/fluent query architecture are
  unchanged by this release.

### 🧪 Testing

- Focused coverage proves exact record success and missing, unknown, duplicate,
  wrong-kind, and nullability rejection.
- Tuple tests prove exact arity and element order; newtype tests prove nominal
  identity and inner-contract validation; empty record and empty tuple remain
  distinguishable.
- Nested tests cover enum, composite, list, set, map, nullable, recursive-depth,
  and byte-budget boundaries.
- Persisted-format tests cover round trips, frozen current vectors, malformed
  IDs and tags, unknown references, cycles, future versions, corruption, and
  shape drift after decode.
- Reconciliation and publication tests cover deterministic IDs, stable existing
  identities, exact-shape drift rejection, bundle atomicity, and recovery-owned
  catalog selection.
- Execution tests cover typed collection saves, SQL/fluent capability parity,
  projection, grouping rejection, defaults, and accepted row decoding.
- A dedicated regression proves map and composite fingerprint domains cannot
  collide even when a composite ID is constructed from the map child stream.
- All-target/all-feature `icydb-core` checking, strict Clippy, no-default-feature
  checking, formatting, diff hygiene, generated composite tests, and repository
  architecture guards passed during closeout.
- Full repository and workspace suites remain user-owned push validation under
  the repository agent rules.

### 📊 Audit

- The exact-only SQL performance Wasm is 3,930,758 raw non-gzipped bytes. The
  discarded open-ended experiment measured 3,993,994 bytes, so removing it cut
  63,236 raw bytes (1.58%). The exact-only result is within 348 bytes of the
  earlier exact-contract checkpoint.
- The closeout tree changes approximately 130 files relative to 0.207 and adds
  about 5,400 lines while removing about 2,770. Complexity increased because
  exact catalog metadata, admission, bounded codecs, and proof tests are real
  new invariants; ownership is nevertheless narrower because no loose type or
  compatibility path survives.
- No dependency-version, cursor-format, or package-version change is part of
  the 0.208 implementation slice.

# 0.62 Changelog

## 0.62.1 - 2026-03-21 - Slot-Format Hard Cut And Scalar Slot Readers

### Summary

`0.62.1` is the representation-phase-change follow-up to `0.62.0`.

The opening patch proved that structural persisted-row decode could remove a
meaningful amount of per-entity runtime cost, but it also clarified the next
constraint: keeping multiple persisted-row generations alive and continuing to
route important runtime paths through whole-row or generic-value decode would
flatten the wasm win quickly.

This patch therefore makes the hard cut that earlier compatibility work had
been deferring:

- legacy persisted-row formats are gone instead of being carried in parallel
- the slot-based row container is now the only live persisted-row shape
- commit, recovery, and index planning use slot readers as the canonical row
  authority
- common scalar leaf payloads stop riding through CBOR and use one dedicated
  binary leaf codec family instead
- runtime field-projection seams no longer fall back to `SlotReader::get_value`

The important architectural result is that persisted rows now have one live
container contract and one explicit split:

- row container: presence, slot order, field spans, version envelope
- leaf codec: scalar binary codec first, CBOR fallback only where still needed

### Measurement

No new wasm-size report was recorded inside `0.62.1`.

This patch is still size-motivated, but it is primarily a representation and
runtime-boundary cleanup step:

- it removes dead persisted-format compatibility ballast
- it keeps persisted-row hot paths on the slot seam
- it starts deleting CBOR from scalar leaf payloads
- it keeps the remaining `get_value` sites constrained to deliberate complex
  fallback surfaces rather than runtime field projection

### Next Step

The next credible wasm-facing cut is no longer another row-container rewrite.
That part is done.

The remaining slope work is now inside the leaf/fallback boundary:

- remove more complex hot-path fallback reads that still need `get_value`
- continue shrinking CBOR dependence for field kinds that still pay the generic
  value decode graph
- measure again once another real leaf-codec surface disappears rather than
  after small structural cleanup

## 0.62.0

`0.62.0` opens the structural persisted-row decode line. The goal is no longer
to flatten executor wrappers; it is to collapse non-boundary persisted-row
deserialization onto one shared structural ownership model so runtime paths stop
paying per-entity `serde_cbor` decode graphs unless they are at a true typed
boundary.

## 0.62.3 - 2026-03-22 - Structural Leaf Tightening And Audit Refresh

### Summary

`0.62.3` stays in the post-architecture phase opened by `0.62.2`: no new
representation changes, no new compatibility layers, and no return to generic
decode. The work here is measurement-first tightening of the explicit decoder
surface that now dominates the remaining wasm cost.

The main retained owners were:

- storage-key field decode inside
  [/home/adam/projects/icydb/crates/icydb-core/src/db/data/structural_field.rs](/home/adam/projects/icydb/crates/icydb-core/src/db/data/structural_field.rs)
- account payload parsing on that same structural path
- duplicate primary-key slot validation inside projection materialization
- the old date parser, which still retained much more parsing machinery than
  the fixed-width persisted shape actually needed

This patch tightens those specific surfaces, remeasures the canonical
`one_simple` vs `ten_simple` slope, and records the next incremental reduction
in per-entity wasm bloat.

## 0.62.2 - 2026-03-21 - Structural Runtime Serde Removal And Leaf Tightening

### Summary

`0.62.2` closes the architecture phase opened by `0.62.0` and hard-cut by
`0.62.1`.

After the slot row format, scalar slot codecs, and slot-reader planning work
landed, the remaining size pressure was no longer “generic row decode.” It was
the residual runtime decode and execution surfaces that still retained
`Value`-centric or `serde_cbor`-centric machinery:

- commit-marker payload decode still used one generic serde entrypoint
- scalar predicates, index expressions, and projections still retained generic
  `Value` fallback machinery in important execution paths
- `structural_field.rs` still owned the remaining explicit complex-leaf decode
  surface, with repeated enum/container parsing logic keeping more code alive
  than necessary

This patch removes the last runtime `serde_cbor::from_slice` path, pushes
predicate/index/projection execution onto shared scalar programs where the
query shape allows it, and then tightens the remaining structural field decode
surface with measurement-driven cuts only.

The resulting runtime split is now explicit:

- persisted rows: structural slot container only
- commit frames and payloads: structural binary decode only
- scalar execution: slot/scalar programs, no generic `Value` path
- complex leaf decode: explicit structural field decode only

## 0.62.1 - 2026-03-21 - Slot-Format Hard Cut And Scalar Slot Readers

### Summary

`0.62.1` is the representation-phase-change follow-up to `0.62.0`.

The opening patch proved that structural persisted-row decode could remove a
meaningful amount of per-entity runtime cost, but it also clarified the next
constraint: keeping multiple persisted-row generations alive and continuing to
route important runtime paths through whole-row or generic-value decode would
flatten the wasm win quickly.

This patch therefore makes the hard cut that earlier compatibility work had
been deferring:

- legacy persisted-row formats are gone instead of being carried in parallel
- the slot-based row container is now the only live persisted-row shape
- commit, recovery, and index planning use slot readers as the canonical row
  authority
- common scalar leaf payloads stop riding through CBOR and use one dedicated
  binary leaf codec family instead
- runtime field-projection seams no longer fall back to `SlotReader::get_value`

The important architectural result is that persisted rows now have one live
container contract and one explicit split:

- row container: presence, slot order, field spans, version envelope
- leaf codec: scalar binary codec first, CBOR fallback only where still needed

### Changed

- hard-cut persisted-row and commit-marker compatibility:
  - removed the previous-version row and marker constants and the legacy
    compatibility wording around the old multi-generation persisted formats
  - made the slot-framed row envelope the only live row format and reset the
    format constant surface to version `1`
  - made commit-marker decode fail closed on legacy raw control-slot and
    unversioned marker payloads instead of silently supporting old encodings
- introduced the persisted-row slot seam as the new canonical row boundary:
  - added `PersistedRow`, `SlotReader`, `SlotWriter`, and
    `StructuralSlotReader` ownership in
    [/home/adam/projects/icydb/crates/icydb-core/src/db/data/persisted_row.rs](/home/adam/projects/icydb/crates/icydb-core/src/db/data/persisted_row.rs)

# 0.1 Changelog

## 0.1.20 - 2025-12-24 - Metrics and Numeric/Temporal Hardening

### Summary

`0.1.20` tightens metrics accounting and hardens numeric/temporal edge handling at API and conversion boundaries.

### Changed

- Added metrics fields `rows_scanned`, `exists_calls`, and `plan_full_scan`.
- Counted scan rows during load, exists, and delete paths and reported average rows scanned per load.
- Rejected pre-epoch RFC3339 timestamps and made negative `Timestamp::from_i64` return `None`.
- Made `Date::new` return epoch for out-of-range years and removed the public `i32` field surface.
- Rejected negative `from_i64` and `from_f64` inputs for `Duration`, `E8s`, and `E18s`.
- Changed `E18s::to_decimal` to return `None` on overflow instead of wrapping, and surfaced `[overflow]` in display output.
- Converted numeric validator misconfiguration handling from panics to explicit errors and made invalid clamp configs a no-op.

### Validation

- Added coverage for timestamp/date edge cases, negative numeric inputs, E18s overflow, and metrics scan/exists counters.

---

## 0.1.19 - 2025-12-23 - Upsert and Unique-Index Correctness Pass

### Summary

`0.1.19` hardens upsert and unique-index execution behavior, while tightening key-planning and exists-window semantics.

### Changed

- Fixed upsert unique-index matching to resolve against sanitized inputs.
- Added upsert result helpers that expose insert-versus-update outcomes.
- Added upsert merge helpers so update composition stays executor-owned.
- Renamed `UniqueIndexSpec` to `UniqueIndexHandle` for API clarity.
- Moved `FromKey` to the core traits module (import-path change for callers).
- Added strict `DeleteExecutor::by_unique_index` with corruption checks and delete metrics emission.
- Ensured save paths sanitize entities before primary-key extraction to keep key/index derivation consistent.
- Updated primary-key `IN` planning to return empty results for empty lists and deduplicate keys.
- Enforced PK `IN` element convertibility to storage keys and accepted text-key identifiers (`Ulid`, `Principal`, `Account`).
- Sorted index candidates in index-backed loads to keep deterministic key ordering.
- Skipped non-indexable equality values in index planning to avoid false negatives.
- Made `LoadExecutor::exists` respect caller `offset`/`limit`, including `limit = 0` as `false`.
- Removed the `db!(debug)` macro arm; debug path is `db!().debug()`.

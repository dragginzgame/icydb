# IcyDB 0.182 Changelog

## 0.182.18

### 📝 Summary

- Hard-cuts facade SQL projection rows from eager display strings to typed
  `OutputValue` cells.
- Keeps blob projection payloads binary through the public SQL result boundary,
  rendering hex text only when callers explicitly ask for display rows.
- Removes the dead string-projection SQL statement result lane so scalar
  projection payloads have one semantic output contract.

### 🔧 Changed

- `RowProjectionOutput` and `ProjectionRows` now expose
  `Vec<Vec<OutputValue>>` rows instead of `Vec<Vec<String>>`.
- SQL result conversion preserves typed projection cells and applies display
  rendering only through table/shell rendering helpers or `rendered_rows()`.
- SQL write `RETURNING` response-size probes now measure the typed payload
  shape instead of first rendering projection cells to strings.
- Fixed-scale decimal display contracts remain explicit text cells where the
  SQL projection label requires a rendered scale that the raw decimal value
  cannot represent by itself.

### 🗑️ Removed

- Removes the unused `ProjectionText` statement-result variant and its facade
  conversion branch after production SQL projection paths converged on typed
  output rows.

### 📊 Audit

- Direct typed-boundary checks cover blob, text, decimal, signed/unsigned
  integer, float, and null projection values.
- A Candid payload-size regression check proves blob projections avoid the old
  hex-text expansion at the SQL facade boundary.
- The only remaining `Vec<Vec<String>>` row payload in the SQL facade is the
  grouped-output path, which remains separate from scalar/blob projections.

### 🧪 Testing

- Validates focused SQL facade conversion, SQL blob behavior, SQL write
  `RETURNING`, generated canister SQL projection/update-returning coverage,
  clippy, formatting, and whitespace checks.

## 0.182.17

### 📝 Summary

- Moves owned SQL output values through projection conversion without cloning
  blob payloads.
- Adds diagnostics-enabled blob payload attribution to SQL perf audits so
  returned blob bytes are visible separately from rendered hex-size estimates.
- Clarifies perf-audit row-store reporting as `data_store.get` so pure
  covering reads are not confused with index traversal cost.

### 🔧 Changed

- SQL projection conversion now consumes owned output values where possible,
  preserving large blob buffers instead of cloning them during facade
  packaging.
- SQL perf audit samples report blob value count, raw blob bytes, and the
  corresponding rendered-hex byte estimate.
- Perf matrix/report wording distinguishes row-store `data_store.get` access
  from index traversal and covering-read behavior.

### 📊 Audit

- Blob-returning audit scenarios can now separate inherent payload return cost
  from avoidable display-string expansion.
- Covering-read reports can show zero row-store reads without implying that no
  index traversal occurred.
- The `.17` audit established the remaining blob hotspot as payload movement
  itself, which led directly into the `.18` typed SQL projection boundary.

### 🧪 Testing

- Validates SQL perf audit/report shape, blob payload attribution, and focused
  SQL projection/blob checks.

## 0.182.16

### 📝 Summary

- Splits SQL perf-matrix hotspot reports so ordinary indexed fixtures,
  pure-covering reads, direct data-row scans, retained/kernel scans, and
  intentional heap/journaled storage mirror baselines are ranked separately.
- Keeps matching expression-index projections on pure covering reads when the
  projection expression is already present in the accepted index key.
- Makes expression-derived covering fields visible in explain/projection
  diagnostics without claiming that the raw source field is recoverable from
  the derived index component.

### 🔧 Changed

- SQL perf-matrix reports now include main-fixture-only hotspot tables and a
  dedicated heap-versus-journaled unindexed storage mirror comparison.
- Covering-read planning can project accepted/static expression index
  components such as `LOWER(field)`, `UPPER(field)`, `TRIM(field)`, and
  `LOWER(TRIM(field))` directly from index key payloads when the requested
  projection expression matches the index expression.
- Covering projection execution and explain descriptors distinguish raw index
  components from expression index components while preserving fail-closed
  behavior for raw field projections that are not actually stored in the
  expression index.

### 📊 Audit

- The full deterministic SQL perf matrix generated 1,680 scenarios, executed
  1,599 scenarios, and retained the same 81 expected query-plan failures as
  the previous deterministic baseline.
- Expression-index projection wins now show up as `store.get` dropping to
  zero for matching `LOWER(name)` and `LOWER(handle)` projection/order cases.
- Current remaining main-fixture hotspots are still dominated by blob payload
  return costs and `SHOW ENTITIES`, while the heap/journaled mirror rows remain
  intentional unindexed scan baselines.

### 🧪 Testing

- Validates expression-index covering planner coverage, end-to-end SQL
  `LOWER(...)` projection/order execution, SQL perf-matrix report rendering,
  the full deterministic PocketIC perf matrix, clippy, formatting, and
  whitespace checks.

  cost is dominated by key streaming, row reads, ordering, or page-window work.

## 0.182.11

  projection shapes that do not need nested field-path materialization.

  passthrough, and nested field-path projections where retained slots are not

## 0.182.7

### 📝 Summary

- Removes the remaining internal direct-stable runtime backend from core store
  wrappers, registry capabilities, scans, recovery, diagnostics, and durable
  test fixtures.
- Leaves durable runtime recovery on the journaled cached-stable path only,
  while heap remains explicitly live-only.
- Cleans stale direct-stable syntax affordances from user-facing README
  examples and the derive parser so `storage(stable(...))` no longer has a
  named legacy branch.

### 🗑️ Removed

- Removes direct stable backend enum variants and constructors from data,
  index, and schema stores.
- Removes the old stable capability constructors, stable runtime mode labels,
  direct stable allocation identity bundle, and stable snapshot storage label.
- Removes direct stable scan/read branches and converts remaining durable
  tests/fixtures to journaled cached-stable stores with journal-tail state.
- Removes the derive parser's special `storage(stable(...))` rejection branch;

### 🔧 Changed

- Marker-bound journal recovery now prevalidates journal batch records before
  fold/replay so corrupt marker-bound durable payloads fail closed without
  partial application.

- Mixed heap/durable relation fixtures now wire explicit runtime hooks for the
  entities they recover, so recovery and reverse-index rebuilds use the same
  entity metadata as write paths.
- README storage examples now show `storage(journaled(...))` as the durable
  store form and describe `heap()` as non-durable.

### 🧭 Migration Notes

- Direct stable runtime storage is fully retired. Durable stores must use
  `storage(journaled(...))`; volatile fixtures can use `storage(heap())`.
- Existing direct stable syntax remains unsupported, but it is no longer kept
  alive as a dedicated parser branch or documentation surface.

### 🧪 Testing

# 0.143.x

## 0.143.0

`0.143.0` opens a bugfix, refactor, optimization, and cleanup slice by fixing
one value-storage validation bug in generated enum payload handling. Fresh rows
whose enum variant payload stored a ULID could fail reload because the enum
payload splitter checked for `NULL` with the generic Structural Binary parser
instead of the value-storage skipper. The generic parser does not own local
value-storage tags such as the ULID tag, so valid fresh bytes were rejected
before the ULID payload decoder ran.

### 🩹 Fixed

- fixed generated enum payload splitting for variants carrying local-tagged
  value-storage payloads such as `Value::Ulid`
- fixed fresh-row reloads for simple enum-backed fields whose selected variant
  carries a ULID payload

### 🔧 Changed

- routed the value-storage null probe through the value-storage skipper so the
  validation path understands the same local tag set as value decode
- kept local value-storage tag ownership inside the value-storage boundary
  instead of widening the generic Structural Binary parser

### 🧭 Boundary notes

- persisted bytes do not change
- by-kind structural field decoding remains separate from value-storage decode
- `NULL` payload detection still validates that the nested payload contains
  exactly one bounded value

### 🧪 Validation

- ran `cargo fmt --all`
- ran
  `cargo test -p icydb-core binary_value_storage_enum_splitter_accepts_local_tagged_payload -- --nocapture`
- ran `cargo test -p icydb-core binary_value_storage -- --nocapture`

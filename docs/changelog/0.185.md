# IcyDB 0.185 Changelog

## 0.185.15

### 📝 Summary

- Continues branch-aware route cleanup by carrying ordered index leaf-stream
  preservation as an explicit executor policy instead of a raw boolean.
- Groups physical fetch hints and ordered leaf-stream preservation into one
  executor access-stream policy.
- Centralizes scalar page-window shape checks for index-prefix-set execution
  behind route capability helpers.
- Leaves route admission, cursor format, scan hint math, page windows, result
  payloads, and diagnostics semantics unchanged.

### 🧹 Cleanup

- Added an executor-internal `IndexLeafOrderPolicy` with canonical-key-order
  and preserve-physical-leaf-order variants.
- Added an executor-internal `AccessStreamExecutionPolicy` that pairs
  `physical_fetch_hint` with `IndexLeafOrderPolicy`.
- Fallback runtime now constructs and passes one access-stream policy instead
  of separate fetch-hint and leaf-order arguments.
- Verified fast-stream and aggregate direct traversal callers now pass
  canonical-key-order access-stream policies explicitly.
- Executable access bindings, traversal inputs, structural physical requests,
  and physical stream bindings carry the grouped policy across their handoff
  boundaries.
- Physical prefix-family streaming still converts the policy's leaf-order
  member to the local primary-key suffix resume policy at stream construction.
- Removed the obsolete `StreamExecutionHints` wrapper.
- Added route capability helpers for index-prefix-set page fetch hints and
  branch-set page keep caps.
- Scalar materialized fallback now classifies `IndexMultiLookup` and
  `IndexBranchSet` page-fetch eligibility through the shared shape helper.
- Terminal page materialization now classifies branch-set row-cap eligibility
  through the same route-owned shape boundary.
- Updated route assertions to check ordered leaf-stream preservation through
  the policy contract.

### 🧪 Testing

- Revalidated core SQL-feature builds, route-policy and fast-stream coverage,
  primary-order route suffix coverage, whitespace checks, invariant guards,
  and full Clippy pass.

### 🧭 Migration Notes

- No schema migration is required.
- No SQL syntax, fluent API, result payload, cursor format, persistence format,
  public endpoint behavior, route admission, page-window semantics, or route
  diagnostics are intended to change.
- Full branch-tree replacement, cursor-format design, and broader cost-based
  routing remain deferred follow-up work.

## 0.185.14

### 📝 Summary

- Continues branch-tree cleanup by sharing prefix-set execution-shape selection
  between physical key prefix streams and covering component prefix streams.
- Keeps scalar decoded-key streams and covering component-row streams
  payload-specific while moving the empty, single-prefix, materialized
  fallback, and ordered-merge shape split into one executor helper.
- Makes the shared prefix-set classifier consume an explicit merge-safety
  contract instead of a raw ordered-merge boolean.
- Routes covering child-prefix expansion through the same scalar lookahead
  fetch-window projection used by route planning.
- Moves the expanded child-prefix index shape into the shared metadata
  expansion result instead of reconstructing it in the physical stream path.
- Threads the primary-key suffix resume policy explicitly through shared
  prefix-family streaming instead of hiding it behind a generic leaf-order flag.
- Leaves route admission, cursor semantics, prefix pruning, store resolution,
  resume anchors, and payload decoding with their existing owners.

### 🧹 Cleanup

- Added a shared prefix-set shape classifier for already-active sibling
  prefixes.
- Physical merged prefix streams now use the shared classifier before building
  direct single-prefix streams or ordered sibling merges.

  direct bounded scan, materialized fallback, or ordered sibling merge.
- The shared prefix-set classifier now receives `OrderedMergeSafe` or
  `RequiresMaterialization` at each call site, keeping fallback semantics
  visible where physical and covering routes enter it.
- Covering child-prefix expansion no longer keeps a private page arithmetic
  copy for offset, limit, and continuation lookahead.
- Covering child-prefix expansion now enters the route helper through the
  route-owned access-window contract instead of an externally exposed raw
  fetch-limit helper.
- The shared child-prefix expansion helper now returns an explicit expanded
  prefix family containing both the adjusted index shape and lowered child
  prefix specs.
- Physical single-prefix, multi-lookup, and branch-set streaming now share one
  prefix-family stream helper for expected-prefix-count validation and merged
  stream request construction.
- The shared physical prefix-family helper now receives the exact merge-resume
  policy for each route family, keeping primary-key suffix continuation
  explicit at the stream-construction boundary.
- Sparse child-prefix expansion now runs the same lowered-prefix count
  validation before metadata enumeration that materialized and normal streamed
  multi-lookup execution already use.

### 🧪 Testing

  materialized-fallback prefix-set shapes.
- Revalidated the helper tests, covering window tests, core SQL-feature build,
  whitespace checks, and full Clippy pass.

### 🧭 Migration Notes

- No SQL syntax, fluent API, result payload, cursor format, persistence format,

  diagnostics are intended to change.

  routing remain deferred follow-up work.

## 0.185.13

### 📝 Summary

- Continues branch-aware route cleanup by sharing the flat ordered sibling
  merge used by scalar key streams and covering component streams.
- Keeps decoded-key streams and covering component-row streams payload-specific
  while moving the common ordered merge mechanics into one executor driver.
- Shares the zero, one, two, and many sibling-stream shape split so two-stream
  routes keep their existing pair merge and three-or-more stream routes use
  the flat driver.
- Hardens the SQL-on-canister runtime against the security-audit denial-of-service
  surfaces that were practical to fix locally in the engine.
- Bounds recursive SQL parsing/lowering, session cache growth, persisted value
  decoding, commit-marker decoding, write staging, decimal rendering, and
  `RETURNING` response probing.
- Tightens facade validators so text length, URL, MIME, and E.164 phone checks
  fail closed with clearer semantics.

### 🧹 Cleanup

- The shared flat merge driver now owns sibling head selection,
  duplicate-head clearing, and defensive duplicate-emission suppression.
- Scalar key streams retain their row-identity witness based on entity tag and
  primary key, while covering component streams keep exact decoded store-key
  identity.
- Pairwise intersections and two-way ordered merges remain separate because
  they have different contracts from the flat sibling merge.

### 🔧 Changed

- SQL expression handling now has explicit depth and input/token guardrails
  before parser or lowering recursion can consume the native stack.
- SQL compiled-command and query-plan caches now have bounded lifetime growth
  instead of retaining every distinct caller-supplied shape indefinitely.
- Structural value-storage and commit-marker decode now avoid untrusted
  up-front vector reservations and reject excessive nested decode depth.
- SQL write execution now validates public INSERT staging, DELETE staging, and
  write `RETURNING` response limits before commit-visible side effects.
- Executor panic surfaces from ordered cursor resume, grouped top-K finalize,
  and commit marker divergence now return typed errors, with a CI guard keeping
  production executor code free of `panic!`, `assert!`, `.unwrap()`, and
  `.expect()`.
- Aggregate lowering now deduplicates repeated aggregate leaves through
  indexed interners while preserving first-seen output order.
- Canonical value comparison no longer silently treats future same-rank
  mismatched variants as equal; debug builds assert the rank invariant and
  release builds retain deterministic tag ordering.

### 📊 Audit

- Closed the audit's critical/high engine-owned clusters for SQL recursion,
  unbounded session caches, decode-bomb allocations, decode recursion, and
  production trap surfaces.
- Addressed medium/low follow-through for multi-lookup fetch hints,
  mutation-batch reserve caps, decimal zero rendering, write response probing,
  text length semantics, and web/phone/MIME validator strictness.
- Deliberately left canister principal authorization as an embedder boundary:
  the engine still does not enforce caller identity or per-caller isolation.
- Deferred broader policy changes for LIMIT-less public result ceilings and
  recovery-rebuild streaming; those remain design/product decisions rather
  than local safety patches.

### 🧪 Testing

- Revalidated flat merge, key-stream, covering, branch-set, SQL+diagnostics
  Clippy, formatting, and whitespace checks.
- Revalidated focused SQL parser/lowering, cache, decode, write, aggregate,
  validator, and value-comparison regression coverage.
- Revalidated core/facade SQL builds, diagnostics-only core builds, formatting,
  whitespace checks, and invariant guard scripts.

### 🧭 Migration Notes

- No schema migration is required.
- No cursor format, row format, stable-memory protocol, SQL syntax, or fluent
  API change is intended.
- Some malformed inputs and oversized public write shapes now reject earlier
  and more consistently; facade text length checks now count Unicode
  characters for text while blob and collection length checks remain byte/item
  based.
- Full branch-tree replacement, cursor-format design, and broader cost-based
  routing remain deferred follow-up work.

# IcyDB 0.185 Changelog

## 0.185.0

### 📝 Summary

- Starts the branch-aware query revisit from the 0.183 production baseline.
- Establishes the 0.185 design/status baseline for route representation,
  SQL/fluent convergence, continuation safety, adaptive routing boundaries,
  and deferred cost-based work.
- Keeps this first slice focused on proof and consolidation rather than a
  broad optimizer rewrite.

### 🔧 Changed

- Branch-set EXPLAIN access DTOs now carry the branch field and ordered suffix
  identity explicitly, making the first-slice order-preservation proof visible
  instead of relying only on tuple shape.
- Branch-set, multi-lookup, and sparse child-prefix expansion paths now share
  merged-prefix stream construction for empty-prefix pruning, fair branch
  chunk sizing, primary-key suffix resume anchors, per-prefix stream setup, and
  duplicate-suppressing merge handoff.
- SQL and fluent target branch-set queries are guarded to share the same
  branch-route identity and return the same primary-key ordered page.
- Verbose EXPLAIN now surfaces sparse `IN` child-prefix expansion state,
  target fetch, and cap fetch diagnostics.
- DESC sparse child-prefix expansion remains explicitly rejected until reverse
  continuation and ordering semantics are designed.

### 📚 Documentation

- Added the 0.185 branch-aware revisit design baseline.
- Added a 0.185 status file recording the current runtime contract,
  representation audit, branch-tree inventory, and next planned cleanup.
- Added continuation notes documenting the current global primary-key boundary
  plus per-prefix raw resume-anchor model.
- Added adaptive-routing notes documenting current route families, the admitted
  sparse expansion boundary, and deferred cost/cardinality decisions.

### 🧹 Cleanup

- Removed another source of branch-route duplication by routing runtime
  branch-set, multi-lookup, and child-prefix stream setup through one shared
  helper.
- Kept planner admission free to compute candidate shapes, but kept downstream
  runtime behavior on spec/lowered-prefix data instead of reconstructing branch
  semantics from loose prefix lengths.

### 🧪 Testing

- Added focused route tests for DESC sparse child-prefix expansion rejection.
- Added focused SQL/fluent branch-route identity coverage.
- Added verbose EXPLAIN coverage for sparse `IN` child-prefix expansion
  diagnostics.
- Revalidated focused branch-set and route-planning tests, SQL+diagnostics
  Clippy, formatting, and whitespace checks.

### 🧭 Migration Notes

- No schema migration is required.
- No SQL syntax, fluent API, result payload, cursor format, persistence format,
  query semantics, public endpoint behavior, cache semantics, or EXPLAIN label
  removals are intended to change.
- Per-branch cursor hard-cut, cost-based routing, prefix-cardinality-driven
  route choice, and DESC sparse child-prefix expansion remain deferred follow-up
  work.

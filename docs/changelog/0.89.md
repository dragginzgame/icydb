# 0.89 Changelog

## 0.89.1

- finished the first aggregate-input follow-through slice by making grouped aggregate `ORDER BY <alias>` work for expression-backed aggregate terms such as `AVG(age + 1) AS avg_plus_one`, instead of only the older field-target aggregate forms.
- fixed grouped planner strategy selection so expression-backed grouped aggregate order terms reserve the bounded `top_k_group` lane instead of being rejected through the older ordered-streaming aggregate gate before grouped Top-K planning can run.
- widened grouped retained-slot layout projection so grouped fold keeps every row slot referenced by compiled aggregate-input scalar expressions, which lets bounded Top-K grouped finalize evaluate expression-backed aggregates from the retained row view instead of failing with missing-slot planner invariants.
- pinned the new alias parity across the public SQL surfaces with targeted lowering, grouped execution, `EXPLAIN`, and SQL compile/cache identity regressions so `ORDER BY avg_plus_one` and `ORDER BY AVG(age + 1)` stay on the same grouped Top-K contract.
- added planner-side constant folding for bounded aggregate-input expressions and canonicalized `SUM` / `AVG` numeric literal inputs onto one decimal accumulator shape, so terminals like `SUM(2 * 3)` and `SUM(6)` or `AVG(ROUND(2 * 3, 1))` and `AVG(6.0)` now deduplicate before execution instead of compiling duplicate aggregate work.

```sql
SELECT name, AVG(age + 1) AS avg_plus_one
FROM IndexedSessionSqlEntity
GROUP BY name
ORDER BY avg_plus_one DESC, name ASC
LIMIT 2;
```

## 0.89.0

- widened aggregate terminals from field-only inputs to planner-owned scalar input expressions across the shared aggregate model, so aggregate identity, type inference, projection labels, and lowering no longer fork on a separate field-versus-expression path.
- widened the SQL parser and lowering surface to preserve aggregate input expressions such as `COUNT(1)` and `AVG(age + 1)` on the shared expression spine instead of collapsing them back to one field target or rejecting them generically.
- enabled expression-backed aggregate execution on the global aggregate lane for `COUNT`, `SUM`, `AVG`, `MIN`, and `MAX` by reusing the existing scalar projection evaluator instead of introducing a separate aggregate-input runtime.
- enabled the same aggregate-input execution model on the grouped aggregate lane by making grouped aggregate specs and grouped execution specs expression-owned, compiling grouped aggregate inputs once, and evaluating them per row through the shared scalar projection evaluator during grouped fold.
- kept outward SQL behavior aligned with the widened aggregate-input shape by preserving expression-based labels, explicit grouped/global explain output, structural fingerprints, and shared query-cache identity for aggregate input expressions.

```sql
SELECT age, AVG(age + 1)
FROM SessionSqlEntity
GROUP BY age
ORDER BY age ASC
LIMIT 10;
```

# 0.63 Changelog

## 0.63.11 - 2026-03-25 - Public DTO Serialize Cleanup

### Summary

`0.63.11` continues the late `0.63` follow-through as an API/contract cleanup
patch rather than another runtime or wasm-reduction slice.

This patch removes `serde::Serialize` from public Candid-facing DTO families
where encoding was no longer a live runtime requirement. The affected surfaces
already use Candid as their external wire contract and only still retained
serde serialization because local tests were asserting CBOR shape. Those tests
now assert on Candid type shape directly, so the DTOs no longer keep serde
serialization machinery just to satisfy test-only fixtures.

- remove unused `Serialize` derives from public SQL, describe, query-expression,
  diagnostics, metrics, and error DTO families
- replace local serde/CBOR shape tests with Candid-shape contract tests
- keep the runtime behavior, SQL feature set, and measured raw wasm size
  effectively unchanged

### Changed

- removed `Serialize` from the public SQL query result surface:
  - `crates/icydb/src/db/sql/mod.rs`
  - `SqlQueryRowsOutput` and `SqlQueryResult` now keep only the traits needed
    for the actual canister/query contract
- removed `Serialize` from the describe-surface DTO family:
  - `crates/icydb-core/src/db/schema/describe.rs`
  - `EntitySchemaDescription`
  - `EntityFieldDescription`
  - `EntityIndexDescription`
  - `EntityRelationDescription`
  - `EntityRelationStrength`

- removed `Serialize` from the public query-expression DTO family:

- removed `Serialize` from diagnostics/storage report DTOs:

## 0.63.10 - 2026-03-25 - SQL Wasm Footprint Reduction

### Summary

`0.63.10` keeps the current SQL feature set intact but shifts the `0.63`
follow-through from audit/process work to direct SQL canister footprint
reduction.

The biggest cut comes from tightening the generated SQL query export to the
surface it already intends to represent: a query-only lane. That change stops
the `canister_query query` root from retaining the delete execution subtree.
The rest of the patch trims retained SQL-support cost rather than removing SQL
features: non-`sql` builds are clean again for side-by-side measurement, the
snapshot Candid payload is lighter, and the EXPLAIN descriptor surface uses
shorter stable property/reason labels that reduce `.rodata`.

- make generated SQL query dispatch truly select-only without removing EXPLAIN
  or other SQL features from the product surface
- restore clean `sql-off` builds so SQL-on vs SQL-off wasm comparisons are
  trustworthy again
- trim snapshot and EXPLAIN descriptor payload cost to keep the remaining SQL
  surface lighter

### Changed

- removed the accidental delete-retention bug from the generated SQL query

## 0.63.0 - 2026-03-23 - Post-De-Monomorphization Consolidation Audit

### Summary

`0.63.0` starts the post-`0.62` consolidation line.

This release is not a new runtime feature phase. The large wasm-driven
de-monomorphization work already landed across `0.58` through `0.62`; the
remaining problem was structural drift after that rewrite:

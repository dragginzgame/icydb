# 0.63 Changelog

## 0.63.1 - 2026-03-24 - Post-De-Monomorphization Consolidation Audit Follow-Through

### Summary

`0.63.1` continues the same mechanical `0.63` audit line after the initial
`0.63.0` ownership cleanup.

This patch does not expand the executable SQL or query surface. It keeps
draining the same post-`0.62` consolidation seams:

- grouped planner/runtime handoff invariants that were still rebuilt locally
- aggregate, stream, route, continuation, and post-access invariant wrappers
  that still routed through old free constructor paths
- the remaining shared `db/error` constructor layer that sat between DB code
  and the canonical `InternalError` taxonomy owner

The result is a cleaner end state for the audit standard itself: constructor
ownership now lives primarily on the owning error or contract type, and the
old DB-wide forwarding layer is gone.

### Changed

- continued grouped planner/runtime boundary cleanup:
  - `crates/icydb-core/src/db/query/plan/group.rs`
  - `crates/icydb-core/src/db/query/plan/grouped_layout.rs`
  - `crates/icydb-core/src/db/query/plan/semantics/group_distinct.rs`
  - `crates/icydb-core/src/db/query/plan/semantics/group_model.rs`
  - `crates/icydb-core/src/db/executor/pipeline/contracts/grouped/route_stage/projection.rs`

  - grouped layout, grouped DISTINCT, grouped HAVING, and grouped route-stage
    invariants now stay on planner/grouped owner types instead of being rebuilt

- drained the remaining executor-invariant wrapper drift across runtime helpers:
  - `crates/icydb-core/src/db/executor/aggregate/terminals.rs`

## 0.63.0 - 2026-03-23 - Post-De-Monomorphization Consolidation Audit

### Summary

`0.63.0` starts the post-`0.62` consolidation line.

This release is not a new runtime feature phase. The large wasm-driven
de-monomorphization work already landed across `0.58` through `0.62`; the
remaining problem was structural drift after that rewrite:

- repeated error and validation wrappers at query/session boundaries
- duplicate field-resolution helpers in planner and SQL entrypoints
- split projection/materialization authority after the structural projection
  refactor
- mutation and reverse-index delta bookkeeping spread across parallel paths
- cursor invariant handling that still mixed raw reasons with already-prefixed
  internal-error messages
- SQL session helpers that kept rebuilding the same lane, grouping, and
  prepared-lowering checks locally

`0.63.0` attacks those seams directly. The result is a cleaner internal
authority map with the current executable surface kept stable.

Design and tracking references:

- `docs/design/0.63-mega-audit/0.63-design.md`
- `docs/design/0.63-mega-audit/0.63-status.md`

### Changed

- consolidated session query error wrappers under one owner:
  - `crates/icydb-core/src/db/session/mod.rs`
  - `crates/icydb-core/src/db/session/query.rs`
  - `crates/icydb-core/src/db/session/sql.rs`
  - session-facing query invariant, unsupported-query, and continuation
    serialization wrappers now live in one session boundary instead of being
    rebuilt ad hoc in local query and SQL paths
- consolidated mutation and reverse-index delta authority:
  - `crates/icydb-core/src/db/commit/prepared_op.rs`
  - `crates/icydb-core/src/db/executor/mutation/commit_window.rs`
  - `crates/icydb-core/src/db/relation/reverse_index.rs`
  - reverse-index membership changes now flow through one
    `PreparedIndexDeltaKind` authority
  - commit-window counter increments and index metric emission now share one
    delta translation path
- collapsed projection/materialization authority:
  - `crates/icydb-core/src/db/executor/projection/materialize/mod.rs`
  - `crates/icydb-core/src/db/executor/projection/materialize/structural.rs`
  - `crates/icydb-core/src/db/executor/projection/eval/mod.rs`
  - `crates/icydb-core/src/db/executor/projection/eval/scalar.rs`
  - typed projection rows, slot-row validation, and structural SQL projection
    now use one prepared projection-plan owner
  - compiled scalar projection evaluation now shares one recursive core instead
    of separate typed and structural paths
- tightened query/session unsupported and intent construction:
  - `crates/icydb-core/src/error/mod.rs`
  - `crates/icydb-core/src/db/query/intent/errors/mod.rs`
  - query-origin unsupported execution construction now belongs to
    `InternalError` / `QueryError`
  - grouped-query intent rejection now uses query-error-owned constructors
    rather than local `QueryError::Intent(...)` assembly
- collapsed duplicate aggregate and grouped field-resolution helpers:
  - `crates/icydb-core/src/db/query/plan/validate/symbols.rs`
  - `crates/icydb-core/src/db/query/plan/validate/mod.rs`
  - `crates/icydb-core/src/db/query/fluent/load/validation.rs`
  - `crates/icydb-core/src/db/session/sql.rs`
  - `crates/icydb-core/src/db/query/plan/validate/grouped/structure/mod.rs`
  - `crates/icydb-core/src/db/query/plan/validate/grouped/policy/rules.rs`
  - aggregate target field resolution now has one planner/query validation
    owner used by fluent aggregate terminals and SQL aggregate dispatch
  - grouped aggregate target-field validation now uses one helper across
    grouped structure validation and grouped global-DISTINCT policy checks
- normalized cursor invariant handling:
  - `crates/icydb-core/src/db/cursor/error.rs`
  - `crates/icydb-core/src/db/cursor/validation.rs`
  - `crates/icydb-core/src/db/cursor/mod.rs`
  - `crates/icydb-core/src/db/error/cursor.rs`
  - `crates/icydb-core/src/db/query/plan/continuation.rs`
  - `crates/icydb-core/src/db/executor/executable_plan.rs`
  - `crates/icydb-core/src/db/session/query.rs`
  - cursor invariant reasons now stay raw inside `CursorPlanError`
  - the canonical executor-invariant prefix is now applied once at the
    cursor-to-internal error boundary
- simplified SQL session query-lane ownership:
  - `crates/icydb-core/src/db/session/sql.rs`
  - query-lane unsupported routing now has one helper
  - prepared SQL statement lowering and typed binding now have one shared owner
  - grouped vs scalar SQL query-intent gating now uses one session helper
  - lowered query/explain lane execution now routes through one shared query-lane
    dispatcher
- simplified planner expression type inference field lookup:
  - `crates/icydb-core/src/db/query/plan/expr/type_inference/mod.rs`
  - schema field-kind lookup for expression inference now uses one local helper
    instead of repeating the same `UnknownExprField` mapping across field and
    aggregate target paths

# IcyDB 0.191 Changelog

## 0.191.2

### 📝 Summary

- Adds a persisted-format inventory for the active durable byte and identity
  surfaces covered by the 0.191 format policy.
- Extends the durability documentation invariant guard so future changes cannot
  silently drop the inventory, representative durable surfaces, or the
  format-change review rule.

### 📚 Documentation

- Added a contract-level inventory for stable-memory allocation roles, commit
  control-slot bytes, commit markers, journal tails, fold watermarks, raw data
  keys, raw row envelopes, structural field and value-storage payloads,
  accepted schema snapshots, secondary-index keys and entries, and cursor
  tokens.
- Linked the inventory from the durability contract, persisted-format policy,
  operator guide, README, foundations, and roadmap docs.

### 🧪 Testing

- Extended the durability documentation invariant guard to require the
  persisted-format inventory, representative durable surface rows, and the
  review rule for deciding whether a touched owner module changes persisted
  bytes or identity.

### 🧭 Migration Notes

- No public API, SQL/fluent behavior, generated endpoint behavior, persisted
  format, response shape, cursor format, or cache identity changes are intended.
- The inventory and guard expansion are documentation and CI-governance only.

### 🧪 Testing

- The guard verifies the root 0.191 changelog link, detailed 0.191 changelog
  presence, durability contract, persisted-format policy, operator guide,
  evidence note, and streaming recovery follow-up.

### 🧭 Migration Notes

- No public API, SQL/fluent behavior, generated endpoint behavior, persisted
  format, response shape, cursor format, or cache identity changes are intended.
- This is a documentation and CI-governance hardening slice only.

## 0.191.0

### 📝 Summary

- Starts the durability productization and persisted-format policy line after
  the 0.190 IC reliability proof expansion.
- Turns the reliability evidence into operator-facing contracts for journaled
  durability, volatile heap storage, guarded recovery, backup/import scope,
  checksum posture, and persisted-format change classification.
- Raises recovery characterization coverage while keeping the production
  recovery-size boundary explicit.

### 📚 Documentation

- Added a durability contract that defines the current IC-native support
  boundary for journaled stores, heap stores, guarded recovery, mutation
  atomicity, backup/import scope, checksums, recovery-size limits, stable-memory
  partitioning, and non-IC host environments.
- Added a persisted-format policy that requires every durable byte-layout
  change to be classified as test-only, pre-1.0 hard cut,
  backward-compatible reader extension, or format-breaking migration.
- Added an operator durability guide covering storage choice, stable-memory
  memory IDs, write atomicity, async/reentry boundaries, guarded access,
  recovery guarantees, unsupported raw backup/import, checksum posture, and
  recovery-size limits.
- Linked the new durability and format-policy contracts from the README,
  foundations, roadmap, SQL subset, atomicity, and transaction-semantics docs.

### 🧪 Testing

- Raised the checked host startup secondary-index recovery characterization
  floor from 256 rows to 1,024 rows.
- Added a mixed ordinary, conditional, and expression index rebuild recovery
  floor over 128 rows per shape.
- Added a PocketIC same-WASM upgrade/reentry instruction probe over a 32-row
  journaled `sql_perf` fixture, with a checked regression budget and recorded
  representative instruction sample.

### 🧭 Migration Notes

- No persisted marker, journal, row, schema, index, cursor, or structural-value
  format change is introduced by 0.191.0.
- Raw stable-memory backup, restore, and import remain outside the supported
  product surface.
- The current line intentionally does not add persisted checksums.
- The new recovery-size evidence is a host and PocketIC regression floor, not a
  production IC recovery-size ceiling.
- Streaming index fold/rebuild remains tracked as future work before broad
  production-scale recovery guarantees.

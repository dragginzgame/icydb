# 0.154.x

## 0.154.3

`0.154.3` makes the supported startup schema-runner path more diagnosable
before widening physical index-store publication.

### Changed

- Added a startup index-store preflight for the supported non-unique field-path
  index-add path. The preflight decodes existing physical index keys and
  classifies store occupancy as target-index entries versus unrelated-index
  entries before the runner executes.
- Kept populated physical index stores fail-closed until scoped multi-index
  publication validation exists, but now reports target, unrelated, and total
  entry counts at the startup boundary.
- Removed the remaining active legacy Makefile setup references and refreshed
  the current wasm footprint audit instruction to use `icp-built` and
  `icp-shrunk` artifact names.

## 0.154.1

`0.154.1` completes the narrow database-default encoder slice needed by
generated schema metadata after literal construction defaults became persisted
database defaults, hard-cuts unsigned runtime/persisted metadata naming to the
`Nat*` vocabulary, and wires the first supported schema-runner startup
execution path.

### Fixed

- Every primitive field can now encode database-default payloads through the
  existing persisted field-codec lanes instead of relying on a derive-local
  allowlist with missing primitive coverage.
- Decimal fields now encode database-default payloads through the existing
  by-kind Decimal field codec instead of being rejected by derive-time
  validation.
- Int, Nat, Int128, and Nat128 fields now encode database-default integer
  literals through their existing by-kind integer codecs, including unsuffixed
  128-bit literals that exceed the 64-bit range.
- Date, Duration, and Timestamp fields now encode database-default literals
  through their existing scalar codecs; numeric literals use raw day or
  millisecond units, while string literals use the runtime type parsers.
- Account, Principal, Subaccount, Ulid, Blob, and Unit fields now support
  explicit database defaults through the same by-kind codecs used by normal
  persisted row values.
- Optional single-value fields now accept non-null literal database defaults,
  so missing accepted slots materialize through the existing
  `DefaultIfMissing` absence policy instead of failing derive validation.
- Invalid Decimal text defaults still fail during schema macro validation.

### Runtime Vocabulary

- Unsigned numeric runtime and persisted metadata names now use `Nat*`
  consistently across generated schema primitives, public runtime types, field
  kinds, persisted field kinds, value variants, scalar codecs, and wire labels.

### Startup Execution

- Wired runtime schema reconciliation to execute the supported non-unique
  field-path index-add path on startup by scanning accepted rows, decoding them
  through accepted row contracts, running the field-path index runner, checking
  publication readiness, and publishing the accepted snapshot only after the
  physical runner succeeds.
- Kept metadata-only schema-store callers fail-closed for physical mutation
  work, so only runtime reconciliation owns startup physical execution.
- Required the physical index store to be empty before startup execution can
  write the rebuilt index entries; existing multi-index physical stores stay
  rejected until scoped publication validation exists.

## 0.154.0

`0.154.0` starts the developer-supported accepted-schema mutation path by
adding a fail-closed admission guard for the only physical mutation shape in
scope: one non-unique field-path secondary index add.

### Schema Mutation Admission

- Added a supported execution-path contract that accepts only the ordered
  field-path rebuild shape: build one field-path index, validate physical work,
  then invalidate runtime state.
- Added a plan-level supported-path guard so mixed catalog changes cannot pass
  just because their physical execution steps look valid.
- Recognized exactly one added non-unique field-path index as an explicit
  accepted snapshot delta while keeping multiple index additions, unique
  indexes, expression indexes, and reordered/changed prior indexes
  incompatible.
- Routed reconciliation diagnostics through the supported-path admission guard
  so the future startup runner path is distinct from generic unsupported
  physical mutation work.
- Kept expression-index rebuilds, secondary-index drops, metadata-only plans,
  full rewrites, unsupported mutations, unique index targets, empty field-path
  targets, and malformed execution plans rejected before supported runner
  wiring can consume them.

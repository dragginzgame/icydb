# IcyDB 0.199 Changelog

## 0.199.29

### 📝 Summary

- Continues the 0.199 technical-debt audit line with per-entity rich metrics
  summary and global operation metrics owner-boundary splits.
- Splits `EntitySummary`, per-entity summary projection, derived-ratio helpers,
  and summary activity scoring out of the mutable metrics state file.
- Splits `EventOps`, `MetricRatio`, global operation counter accessors, and
  global derived-ratio helpers out of the mutable metrics state file.
- Keeps rich metrics payload shape, rich report ordering, compact metrics
  payload shape, metrics sink behavior, instrumentation behavior, cursor
  formats, and persisted formats unchanged.

### 🔧 Changed

- Moves `EntitySummary`, raw-counter projection into `EntitySummary`,
  per-entity derived-ratio helpers, and rich report activity scoring into
  `metrics/state/summary.rs`.
- Moves `EventOps`, `MetricRatio`, global operation counter accessors, and
  global derived-ratio helpers into `metrics/state/ops.rs`.
- Leaves `metrics/state.rs` as the owner of mutable metrics state, raw
  perf/entity counters, reset helpers, and parent re-exports.
- Re-exports `MetricRatio` through the metrics facade because public
  `EventOps` and `EntitySummary` accessors already return it.
- Keeps `metrics/state/report.rs` as the top-level rich report/window owner.
- Keeps `metrics/state/compact.rs` as the compact report owner.

### 📚 Documentation

- Adds 0.199 audit notes for the metrics entity summary and event operation
  boundary splits.
- Updates the 0.199 boundary matrix with the separated entity summary and
  global operation owners.
- Updates the rich report boundary note so it no longer describes per-entity
  summary DTOs as owned by `state.rs`.

### 🧪 Validation

- `cargo test -p icydb-core --lib --all-features metrics -- --nocapture`
- `cargo clippy -p icydb-core --all-features --all-targets -- -D warnings`
- `cargo check -p icydb --no-default-features --features sql`
- `cargo fmt --all --check`
- `git diff --check`

### 📊 Performance

- No runtime performance improvement is claimed.
- Full deterministic performance matrices were not rerun.
- The changes are owner-boundary cleanup only; they do not alter metrics
  counters, instrumentation call sites, metrics payloads, cursor formats,
  persisted formats, or executor strategy.

### 📊 Complexity Delta

- Public API complexity stays neutral.
- `MetricRatio` is now explicitly reachable through the metrics facade because
  it already appears in public metrics accessor return types.
- Runtime behavior stays neutral.
- Metrics review complexity drops because per-entity rich summary payload shape
  and global operation counter payload shape no longer live beside mutable
  metrics state.

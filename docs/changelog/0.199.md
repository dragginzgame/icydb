# IcyDB 0.199 Changelog

## 0.199.19

### 📝 Summary

- Continues the 0.199 technical-debt audit line with focused query diagnostics
  and SQL direct-count owner-boundary cleanup.
- Splits query diagnostics attribution DTOs from the hidden measured execution
  wrapper while preserving public diagnostics field shapes and attribution
  behavior.
- Moves SQL direct-count cardinality prefix-spec construction out of the shared
  query-plan cache module and into the SQL direct-count execution owner.

### 🔧 Changed

- Replaces the broad `db::session::query::diagnostics` implementation file
  with a module index plus `model` and `execution` owner modules.
- Keeps query diagnostics public DTO re-exports, Candid field names, fluent
  attribution payloads, and hidden measured execution semantics unchanged.
- Moves SQL-only direct `COUNT(*)` cardinality route helper ownership from
  `db::session::query::cache` to `db::session::sql::execute::direct_count`.
- Routes SQL EXPLAIN direct-count metadata eligibility through the same
  SQL-owned direct-count helper instead of reaching into the query-cache owner.
- Keeps public API, read admission, SQL semantics, query cache identity, cursor
  formats, persisted formats, and executor routing unchanged.

### 🩹 Fixed

- Removes SQL direct-count cardinality planning imports from the shared
  query-plan cache owner, keeping cache ownership focused on lookup/insert,
  visibility, and accepted-schema handoff.
- Separates diagnostics DTO model conversion from the hidden measured
  execution wrapper so future attribution field changes do not have to touch
  the execution path by default.

### 📚 Documentation

- Adds 0.199 audit notes for the query diagnostics boundary split and SQL
  direct-count cardinality boundary cleanup.
- Updates the 0.199 boundary matrix with the separated diagnostics and SQL
  direct-count owners.

### 🧪 Validation

- `cargo check -p icydb-core --all-features`
- `cargo clippy -p icydb-core --all-features --all-targets -- -D warnings`
- `cargo test -p icydb-core --lib --all-features branch_set -- --nocapture`
- `cargo test -p icydb-core --lib --all-features sql_aggregate -- --nocapture`
- `cargo test -p icydb-core --lib --all-features sql_explain -- --nocapture`
- `cargo test -p icydb-testing-integration --test fluent_perf_audit -- --nocapture`
- `cargo check -p icydb --no-default-features --features sql`
- `cargo fmt --all --check`
- `git diff --check`

### 📊 Performance

- No runtime performance improvement is claimed.
- Full deterministic performance matrices were not rerun.
- The changes are ownership moves only; they do not alter planner, admission,
  cache identity, cursor, storage, SQL semantics, diagnostics field shapes, or
  executor strategy.

### 📊 Complexity Delta

- Public API complexity stays neutral.
- Runtime behavior stays neutral.
- Query diagnostics review complexity drops because attribution DTO/model code
  and hidden measured execution now have separate owners.
- SQL direct-count review complexity drops because cardinality route helper
  construction now lives beside direct-count execution and SQL EXPLAIN

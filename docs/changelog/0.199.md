# IcyDB 0.199 Changelog

## 0.199.10

### 📝 Summary

- Continues the 0.199 technical-debt audit line with a focused direct-query
  facade cleanup.
- Removes unused hidden facade row execution for prebuilt `Query<E>` values.
- Removes hidden raw-query extraction methods from public facade load/delete
  wrappers.
- Preserves public fluent read/delete APIs, diagnostic query hooks, SQL
  authority, read-admission behavior, cursor tokens, persisted formats, and
  executor strategy.

### 🗑️ Removed

- Removes hidden `DbSession::execute_query(&Query<E>)` from the public facade
  after repository search found no non-doc runtime callers.
- Removes hidden `.query()` extraction methods from facade load,
  partial-window, delete, and delete-returning wrappers.
- Keeps lower-level `Query<E>` diagnostic/planning hooks hidden for trace and
  attribution use.

### 📚 Documentation

- Updates the public facade guide to describe direct query diagnostics rather
  than direct query execution.
- Updates the read-admission contract inventory so hidden prebuilt-query
  surfaces are diagnostic-only, not row-execution endpoints.
- Updates the direct-query boundary note and 0.199 boundary matrix to record
  the hard-cut removal of facade direct row execution and raw-query extraction.

### 🧪 Validation

- `cargo fmt --all`
- `cargo test -p icydb --lib --all-features`
- `cargo check -p canister_audit_sql_perf`
- `cargo clippy -p icydb --all-features --all-targets -- -D warnings`
- `cargo fmt --all --check`
- `git diff --check`

### 📊 Performance

- No runtime performance improvement is claimed.
- Full deterministic performance matrices were not rerun.
- The change removes unused hidden facade surfaces and does not alter planner
  or executor strategy.

### 📊 Complexity Delta

- Public API complexity decreases because the facade no longer exposes hidden
  raw-query extraction or hidden prebuilt row execution.
- Internal diagnostic complexity stays neutral because trace and attribution
  hooks remain available.
- Implementation complexity decreases with a net code deletion in the facade
  session wrappers.

- Adds a core `execute_delete_rows(...)` session helper that rejects non-delete

  matching the existing delete-count fail-closed mode guard.
- Removes the old broad core row materialization helper and updates internal
  tests to use the scalar rows-only helper where they need row materialization.
- Moves delete-returning field-slice selection behind
  `DeleteReturningSelection`, so the facade calls the shared row-projection
  renderer through one handoff.

### 📚 Documentation

- Updates the 0.199 boundary matrix to mark delete returning as a reduced
  drift risk between delete-count and row-returning terminals.

### 📊 Performance

## 0.199.0

### 📝 Summary

- Starts the 0.199 technical-debt audit line after the 0.197 optimizer and
  0.198 read-intent hard cuts.
- Adds a boundary matrix for representative read/query shapes so future cleanup
  work starts from evidence rather than broad refactor pressure.
- Normalizes the 0.198 and 0.199 design directories to descriptive names.
- Records focused evidence notes for exact-key proof ownership, direct-query
  facade boundaries, partial-window public docs, read-admission invariant
  checks, diagnostic vocabulary, and exact aggregate helper wiring.
- Keeps public query behavior, read admission, cursor tokens, persisted formats,
  and generated SQL boundaries unchanged.

### 🔧 Changed

- Direct fluent `.query()` extraction remains available for internal/generated
  tooling but is hidden from normal public API documentation.
- Planner-local selected primary-key proof logic now has a named projection
  helper, reducing duplication in the predicate-stripping path without changing
  selected access semantics.
- Read-admission contract examples now use `page(limit)` for public-list fixes
  instead of teaching `partial_window(...).execute_rows()` as a general
  replacement for removed public load `.limit(...)`.
- Read-admission invariant checks still guard section anchors, API tokens,
  diagnostic enum parity, budgets, and forbidden public policy surfaces, but no
  longer require exact explanatory guide sentences.
- Exact aggregate EXPLAIN helpers now share one private read-intent attribution
  path for `ReadIntentKind::ExactAggregate`.

### 📚 Documentation

- Adds 0.199 audit artifacts for:
  - the read/query boundary matrix;
  - exact-key proof ownership;
  - direct-query facade isolation;
  - partial-window public-doc cleanup;
  - read-admission invariant hardening;
  - diagnostic vocabulary ownership;
  - exact aggregate helper DRY cleanup.
- Updates the 0.198 closeout artifact links to the renamed 0.198 design
  directory.
- Keeps historical 0.198 design feedback intact under the renamed design
  directory.

### 📊 Complexity Delta

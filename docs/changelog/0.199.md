# IcyDB 0.199 Changelog

## 0.199.5

### 📝 Summary

- Continues the 0.199 technical-debt audit line with a focused existence
  terminal cleanup.
- Centralizes existence execution, diagnostics attribution, and EXPLAIN
  read-intent wiring in private fluent terminal helpers.
- Preserves public existence APIs, read-admission behavior, diagnostic DTOs,
  cursor tokens, persisted formats, generated SQL authority, and existence
  semantics.

### 🔧 Changed

- `exists()` now delegates to `execute_existence_terminal(...)`, which owns the
  semantic terminal guard and existing `ExistsRowsTerminal` execution handoff.
- `exists_with_attribution()` now delegates to
  `execute_existence_terminal_with_attribution(...)`, which owns the same
  guard plus `ReadIntentKind::ExistenceCheck` attribution.
- `explain_exists()` now delegates to `explain_existence_terminal(...)`, which
  owns the same guard plus `ReadIntentKind::ExistenceCheck` EXPLAIN metadata.
- `not_exists()` and `explain_not_exists()` intentionally continue to reuse
  the existence terminal behavior with negated result semantics.

### 📚 Documentation

- Adds an existence terminal boundary note for the 0.199 audit line.
- Extends the fluent attribution boundary note with the existence follow-up.
- Updates the 0.199 boundary matrix to mark existence execution, attribution,
  and EXPLAIN wiring as reduced drift risks.

### 🧪 Validation

- `cargo fmt --all`
- focused default fluent read-intent tests
- focused existence EXPLAIN route contract test
- focused existence attribution test
- `bash scripts/ci/check-read-admission-invariants.sh`
- `cargo clippy -p icydb-core --all-features --all-targets -- -D warnings`
- `cargo fmt --all --check`
- `git diff --check`

### 📊 Performance

- No runtime performance improvement is claimed.
- Full deterministic performance matrices were not rerun.
- The change affects private fluent terminal wiring and existence diagnostics
  attribution, not query execution strategy.

### 📊 Complexity Delta

- Public API complexity stays neutral.
- Existence terminal-boundary complexity decreases because execution,
  attribution, and EXPLAIN wiring now share owner-local helpers.
- Implementation complexity remains low and localized to fluent terminal
  wiring.

## 0.199.4

### 📝 Summary

- Continues the 0.199 technical-debt audit line with a focused exact-aggregate
  terminal cleanup.
- Centralizes exact-aggregate raw-window guard, field-slot resolution,
  execution, and EXPLAIN wiring in private fluent terminal helpers.
- Preserves public aggregate APIs, read-admission behavior, diagnostic DTOs,
  cursor tokens, persisted formats, generated SQL authority, and aggregate
  semantics.

### 🔧 Changed

- `count_exact()`, `min_id_exact()`, and `max_id_exact()` now share
  `execute_exact_aggregate_terminal(...)` for exact aggregate guard and
  terminal execution.
- Field-based exact aggregates now share
  `execute_exact_aggregate_by_slot_terminal(...)`, which rejects prior raw
  row-window caps before resolving the field slot and executing the aggregate.
- Field-based exact aggregate EXPLAIN helpers now share
  `explain_exact_aggregate_by_slot_terminal(...)`, preserving the same
  raw-window-before-field-resolution error ordering as execution.
- No-field exact aggregate EXPLAIN helpers now use
  `explain_checked_exact_aggregate_terminal(...)` for terminal-owned guard
  validation and exact-aggregate read-intent attribution.
- Removed the old one-helper-per-aggregate raw-window guard wrappers in favor
  of passing the exact `IntentError` to the shared exact-aggregate helpers.

### 📚 Documentation

- Updates the exact-aggregate helper DRY note with the broader helper boundary.
- Updates the 0.199 boundary matrix to mark exact aggregate execution,
  field-slot resolution, EXPLAIN attribution, and count attribution as reduced
  drift risks.

- `cargo clippy -p icydb-core --all-features --all-targets -- -D warnings`
- `cargo fmt --all --check`
- `git diff --check`

### 📊 Performance

- No runtime performance improvement is claimed.
- Full deterministic performance matrices were not rerun.

### 📊 Complexity Delta

- Implementation complexity remains low and localized to fluent terminal
  wiring.

  `count_exact_with_attribution()` now share
  `with_fluent_terminal_read_intent(...)` for applying diagnostic
  `ReadIntentKind` metadata to hidden fluent terminal attribution results.
- `count_with_attribution()` and `count_exact_with_attribution()` now share
  `execute_count_terminal_with_attribution(...)` for the hidden count terminal
  execution call.

## 0.199.0

### 📝 Summary

- Starts the 0.199 technical-debt audit line after the 0.197 optimizer and
  0.198 read-intent hard cuts.
- Adds a boundary matrix for representative read/query shapes so future cleanup
  work starts from evidence rather than broad refactor pressure.
- Normalizes the 0.198 and 0.199 design directories to descriptive names.
- Records focused evidence notes for exact-key proof ownership, direct-query
  facade boundaries, partial-window public docs, read-admission invariant
  checks, diagnostic vocabulary, and exact aggregate helper wiring.
- Keeps public query behavior, read admission, cursor tokens, persisted formats,
  and generated SQL boundaries unchanged.

### 🔧 Changed

- Direct fluent `.query()` extraction remains available for internal/generated
  tooling but is hidden from normal public API documentation.
- Planner-local selected primary-key proof logic now has a named projection
  helper, reducing duplication in the predicate-stripping path without changing
  selected access semantics.
- Read-admission contract examples now use `page(limit)` for public-list fixes
  instead of teaching `partial_window(...).execute_rows()` as a general
  replacement for removed public load `.limit(...)`.
- Read-admission invariant checks still guard section anchors, API tokens,
  diagnostic enum parity, budgets, and forbidden public policy surfaces, but no
  longer require exact explanatory guide sentences.
- Exact aggregate EXPLAIN helpers now share one private read-intent attribution
  path for `ReadIntentKind::ExactAggregate`.

### 📚 Documentation

- Adds 0.199 audit artifacts for:
  - the read/query boundary matrix;
  - exact-key proof ownership;
  - direct-query facade isolation;
  - partial-window public-doc cleanup;
  - read-admission invariant hardening;
  - diagnostic vocabulary ownership;
  - exact aggregate helper DRY cleanup.
- Updates the 0.198 closeout artifact links to the renamed 0.198 design
  directory.
- Keeps historical 0.198 design feedback intact under the renamed design
  directory.

### 📊 Performance

### 📊 Complexity Delta

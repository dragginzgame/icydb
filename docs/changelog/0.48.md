# IcyDB 0.48

This document aggregates detailed release notes for the `0.48.x` minor line.

## 0.48.1 - 2026-03-10 - Explain Small Features and Access-Choice Diagnostics

### Summary

`0.48.1` extends the post-boundary explain slice with additive execution-node
metadata and planner-derived access-choice diagnostics, while keeping runtime
query behavior unchanged.

### Changed

- Scalar load execution roots now expose additive index-route metadata:
  `prefix_len`, pushed `fetch` budgets, explicit `covering_scan=false`, and
  access-choice diagnostics (`access_choice_chosen`,
  `access_choice_chosen_reason`, `access_choice_alternatives`,
  `access_choice_rejections`).
- ORDER BY execution nodes now report explicit sort-elimination state through
  `order_satisfied_by_index=true|false`.
- Predicate stage descriptors now expose pushed predicate summaries for strict
  prefilter nodes and pushed-access vs residual split visibility for residual
  filter nodes.
- Access-choice alternatives now come from planner-candidate snapshots rather
  than model-only index hints.
- Access-choice reason construction was tightened with planner-owned typed
  reason enums, projected to stable explain reason-code strings.
- Access-choice projection now derives chosen-route shape through
  `ExplainAccessPath` projection so raw `AccessPath` variant references remain
  constrained to `db::access` boundary modules.

### Testing

- `cargo fmt --all -- --check`
- `cargo check -p icydb-core`
- `cargo clippy -p icydb-core --lib -- -D warnings`
- `cargo test -p icydb-core session_load_explain_execution_ -- --nocapture`
- `cargo test -p icydb-core query_explain_execution_ -- --nocapture`
- `cargo test -p icydb-core runtime_raw_access_path_references_stay_within_access_boundary -- --nocapture`

## 0.48.0 - 2026-03-10 - Explain Boundary and Plan Identity Hardening

### Summary

`0.48.0` finalizes the explain-boundary hardening slice by making planner-owned
contracts the authority for plan identity and continuation signatures, while
keeping `EXPLAIN` as a deterministic presentation surface.

### Changed

- Plan fingerprint and continuation-signature derivation now hash directly from
  planner-owned query/access/grouping contracts instead of requiring explain DTO
  projection in production paths.
- Plan-hash call sites now consume planned-query fingerprint authority directly
  (`Query`, planned-query, compiled-query, and session trace surfaces).
- Cursor continuation guard ownership is consolidated under cursor contracts:
  envelope validation, resume-bound derivation, and directional advancement are
  now exposed through cursor-owned APIs consumed by index scan execution.
- Cursor-anchor range-bound reason mapping moved to cursor ownership; access
  lowering no longer owns cursor-anchor bound-mapping variants.
- `EXPLAIN` behavior remains stable while production identity paths no longer
  depend on explain-only helpers; targeted explain/session parity suites are
  rerun and green.
- Structural ownership language was aligned across docs/tests so plan-hash
  semantics explicitly reference planner contracts.

# IcyDB 0.48

This document aggregates detailed release notes for the `0.48.x` minor line.

## 0.48.0 - 2026-03-10 - Explain Boundary and Plan Identity Hardening

### Summary

`0.48.0` finalizes the explain-boundary hardening slice by making planner-owned
contracts the authority for plan identity and continuation signatures, while
keeping `EXPLAIN` as a deterministic presentation surface.

### Changed

- Plan fingerprint and continuation-signature derivation now hash directly from
  planner-owned query/access/grouping contracts instead of requiring explain DTO
  projection in production paths.
- Plan-hash call sites now consume planned-query fingerprint authority directly
  (`Query`, planned-query, compiled-query, and session trace surfaces).
- Cursor continuation guard ownership is consolidated under cursor contracts:
  envelope validation, resume-bound derivation, and directional advancement are
  now exposed through cursor-owned APIs consumed by index scan execution.
- Cursor-anchor range-bound reason mapping moved to cursor ownership; access
  lowering no longer owns cursor-anchor bound-mapping variants.
- `EXPLAIN` behavior remains stable while production identity paths no longer
  depend on explain-only helpers; targeted explain/session parity suites are
  rerun and green.
- Structural ownership language was aligned across docs/tests so plan-hash
  semantics explicitly reference planner contracts.

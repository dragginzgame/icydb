# IcyDB 0.183 Changelog

## 0.183.0

### 📝 Summary

- Adds branch-aware composite-prefix SQL routing for page queries shaped like
  `fixed_prefix = X AND branch_field IN (...) ORDER BY primary_key ASC LIMIT N`
  against indexes shaped like `(fixed_prefix, branch_field, primary_key)`.
- Targets generated-list style reads such as
  `collection_id = X AND stage IN ('Draft', 'Review') ORDER BY id ASC LIMIT N`
  on `(collection_id, stage, id)`.
- Keeps this first slice narrow: one fixed equality prefix, one small exact
  branch set, and ascending primary-key suffix ordering.

### 🔧 Changed

- The planner can admit the target query shape as an explicit
  `IndexBranchSet` route instead of falling back to a broad ordered scan,
  unordered multi-lookup, full scan, or materialized sort.
- Branch routes open one index prefix stream per branch and merge branch heads
  by decoded primary key, with defensive duplicate suppression.
- Access-proven predicates are stripped from residual filters, so the fixed
  prefix and branch predicates are not rechecked as row predicates once the
  route proves them.
- Covered/key-only projections can stay index-backed with zero row-store
  hydration, while non-covered projections hydrate only returned rows plus
  page lookahead.
- Default page-shaped SQL execution does not run an exact count path; count
  work remains separate from page work.

### 📊 Audit

- For the focused `LIMIT 3` target fixture, covered `SELECT id` stays at
  `data_store.get = 0` and reads only the bounded branch index window.
- The non-covered `SELECT id, title` fixture dropped from hydrating all 12
  matching branch rows to hydrating the returned page rows plus lookahead.
- The SQL perf audit includes a token-shaped surrogate for the production
  query shape and verifies `IndexBranchSet`, no materialized sort, bounded
  index-entry reads, bounded row hydration, and no grouped/count work on the
  default page path.

### 🧪 Testing

- Adds focused planner coverage for branch-route admission, large-`IN`
  fallback, order preservation, and residual predicate stripping.
- Adds executor coverage for globally primary-key-ordered branch merge output,
  duplicate suppression, `limit + 1` lookahead, covered projection behavior,
  non-covered hydration bounds, diagnostics, and plan/cache identity.
- Restores diagnostics-enabled SQL attribution test coverage needed by the
  branch-set proof suite.

### 🧭 Migration Notes

- No schema migration is required. Existing composite indexes with the target
  shape can be selected by the planner.
- Prefix cardinality metadata, arbitrary large `IN` lists, non-primary ordering
  terms, and cursor-format redesign are deferred.
- The SQL covering branch path is bounded by per-branch page/lookahead work in
  this release; making it share the scalar lazy ordered stream abstraction is
  the next cleanup target.

# 0.133.x

## 0.133.0

`0.133.0` starts the diagnostics and identity audit line with two boundary
cleanups.

`db::diagnostics` now exposes the full storage-report DTO surface through the
`db` module root. `StorageReport` already returned `DataStoreSnapshot`,
`IndexStoreSnapshot`, and `EntitySnapshot` from public accessors; those row
snapshot DTOs are now re-exported beside `StorageReport`, `IntegrityReport`,
`IntegrityStoreSnapshot`, and `IntegrityTotals`. The diagnostics constructors
used only by report assembly were narrowed to crate-internal visibility, so the
public surface stays focused on reading reports rather than externally
constructing report internals.

The identity audit moves generated entity/index name validation onto the shared
`EntityName` and `IndexName` contracts. Entity names and index field segments
now reject the reserved `|` delimiter through the identity owner, index
identities reject empty field lists and empty field segments, and fixed-size
max sentinels use the maximum ASCII byte instead of a lowercase-letter
sentinel. The derive path now calls the same identity constructors used by the
runtime instead of repeating length, ASCII, and generated-name checks locally.

What changed:
- re-exported `DataStoreSnapshot`, `IndexStoreSnapshot`, and `EntitySnapshot`
  from `db`
- narrowed diagnostics DTO constructors that are only used by diagnostics
  assembly and tests
- centralized entity and generated-index identity validation on
  `EntityName` / `IndexName`
- rejected delimiter-bearing entity names and index field segments before they
  can alias generated index identities
- rejected generated indexes with no fields or empty field segments at the
  identity boundary
- made `IndexId` fields private and exposed the entity-tag component through a
  named accessor
- removed stale schema validation error variants that duplicated identity-owned
  validation errors

Retained behavior:
- diagnostics report contents, ordering, accessors, and Candid field names are
  unchanged
- integrity scan behavior is unchanged
- persisted identity byte layout remains fixed-size and unchanged
- generated index names still use the same flat `entity|field|...` text form

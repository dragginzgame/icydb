# 0.119 Changelog

## 0.119.0

- `0.119.0` starts the structural-simplification line by deleting leftover expression-flow scaffolding that no longer carries distinct logic after `0.118`. The first subtractive pass removes the test-only lowering shim at [/home/adam/projects/icydb/crates/icydb-core/src/db/sql/lowering/predicate/compile.rs](/home/adam/projects/icydb/crates/icydb-core/src/db/sql/lowering/predicate/compile.rs), which used to exist only to forward normalized planner boolean expressions into `compile_normalized_bool_expr_to_predicate(...)`. That shim is gone, and the surviving coverage now lives directly in [/home/adam/projects/icydb/crates/icydb-core/src/db/sql/lowering/predicate/mod.rs](/home/adam/projects/icydb/crates/icydb-core/src/db/sql/lowering/predicate/mod.rs), which is the real lowering boundary.

- The same subtractive pass also removes the remaining test-only wrapper entrypoints in [/home/adam/projects/icydb/crates/icydb-core/src/db/predicate/bool_expr.rs](/home/adam/projects/icydb/crates/icydb-core/src/db/predicate/bool_expr.rs). That file no longer exposes local wrapper functions just to call:
  - `canonicalize_runtime_predicate_via_bool_expr(...)`
  - `compile_normalized_bool_expr_to_predicate(...)`

- The subtractive pass now goes one step further and removes the old predicate-side bridge test module entirely:
  - [/home/adam/projects/icydb/crates/icydb-core/src/db/predicate/bool_expr.rs](/home/adam/projects/icydb/crates/icydb-core/src/db/predicate/bool_expr.rs) is deleted
  - [/home/adam/projects/icydb/crates/icydb-core/src/db/predicate/mod.rs](/home/adam/projects/icydb/crates/icydb-core/src/db/predicate/mod.rs) no longer wires a test-only `bool_expr` module
  - the surviving bridge/compiler regression coverage now lives in [/home/adam/projects/icydb/crates/icydb-core/src/db/query/plan/tests/predicate_bridge.rs](/home/adam/projects/icydb/crates/icydb-core/src/db/query/plan/tests/predicate_bridge.rs), which is the planner owner boundary

- The same subtractive line also trims smaller remnants that no longer carried distinct behavior:
  - [/home/adam/projects/icydb/crates/icydb-core/src/db/sql/lowering/predicate/mod.rs](/home/adam/projects/icydb/crates/icydb-core/src/db/sql/lowering/predicate/mod.rs) no longer keeps a local `derive_where_predicate_subset(...)` forwarding helper just to call planner-owned `derive_normalized_bool_expr_predicate_subset(...)`
  - [/home/adam/projects/icydb/crates/icydb-core/src/db/executor/prepared_execution_plan.rs](/home/adam/projects/icydb/crates/icydb-core/src/db/executor/prepared_execution_plan.rs) no longer exposes the dead shared-plan `authority()` accessor that was only serving the old prepared-template lane
  - [/home/adam/projects/icydb/crates/icydb-core/src/db/access/path.rs](/home/adam/projects/icydb/crates/icydb-core/src/db/access/path.rs) and [/home/adam/projects/icydb/crates/icydb-core/src/db/access/plan.rs](/home/adam/projects/icydb/crates/icydb-core/src/db/access/plan.rs) no longer keep runtime-value rebinding helpers that existed only to support the abandoned prepared-template fallback path

- `0.119.0` now also performs the first genuinely destructive prepared-session cleanup in this line. The explicit session-owned prepared API has been removed:
  - [/home/adam/projects/icydb/crates/icydb-core/src/db/session/sql/parameter.rs](/home/adam/projects/icydb/crates/icydb-core/src/db/session/sql/parameter.rs) is deleted
  - [/home/adam/projects/icydb/crates/icydb-core/src/db/session/tests/sql_parameterized.rs](/home/adam/projects/icydb/crates/icydb-core/src/db/session/tests/sql_parameterized.rs) is deleted because it was the only caller of that API
  - [/home/adam/projects/icydb/crates/icydb-core/src/db/session/sql/mod.rs](/home/adam/projects/icydb/crates/icydb-core/src/db/session/sql/mod.rs) and [/home/adam/projects/icydb/crates/icydb-core/src/db/session/tests/mod.rs](/home/adam/projects/icydb/crates/icydb-core/src/db/session/tests/mod.rs) no longer wire that abandoned lane

- That prepared-session deletion also removes the old template and contract scaffolding that only existed to feed it:
  - [/home/adam/projects/icydb/crates/icydb-core/src/db/sql/lowering/prepare.rs](/home/adam/projects/icydb/crates/icydb-core/src/db/sql/lowering/prepare.rs) now keeps only the live normalize-and-lower path and drops the statement-level prepared parameter contract, literal rebinding, range-slot, and fallback-family walkers
  - [/home/adam/projects/icydb/crates/icydb-core/src/db/sql/lowering/expr.rs](/home/adam/projects/icydb/crates/icydb-core/src/db/sql/lowering/expr.rs) no longer keeps parser-owned prepared template-shape classifiers and parameter/literal scanners that were only consumed by that dead lane
  - [/home/adam/projects/icydb/crates/icydb-core/src/db/predicate/model.rs](/home/adam/projects/icydb/crates/icydb-core/src/db/predicate/model.rs), [/home/adam/projects/icydb/crates/icydb-core/src/db/query/plan/expr/ast.rs](/home/adam/projects/icydb/crates/icydb-core/src/db/query/plan/expr/ast.rs), and [/home/adam/projects/icydb/crates/icydb-core/src/db/query/builder/aggregate.rs](/home/adam/projects/icydb/crates/icydb-core/src/db/query/builder/aggregate.rs) no longer carry predicate, grouped-expression, or aggregate template rebinding structures that were only needed for the deleted prepared-session path
  - [/home/adam/projects/icydb/crates/icydb-core/src/db/query/plan/expr/type_inference/mod.rs](/home/adam/projects/icydb/crates/icydb-core/src/db/query/plan/expr/type_inference/mod.rs) no longer keeps dead coarse-family projection helpers in the non-test build just to support that removed fallback contract collector

- That matters because `0.119` is supposed to make the physical flow match the logical model:
  - planner owns boolean-expression branching and predicate recompilation
  - predicate no longer needs even a test-only module that implies a second bridge ownership boundary
  - session no longer carries a parallel prepared query API that is absent from the shipped runtime path
  - lowering and planner no longer keep template-only helper trees just to preserve an abandoned prepared binding subsystem

- This is subtractive only:
  - no new semantics
  - no new admitted SQL
  - no prepared-surface widening
  - no planner rewrite

- Validation for `0.119.0` covered:
  - `cargo clippy -p icydb-core --all-targets -- -D warnings`
  - `cargo check -p icydb-core`

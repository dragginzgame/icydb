# 0.148.x

## 0.148.4

`0.148.4` moves accepted-row normalization for typed materialization and commit
before-images into the data layer. The executor still selects when a row must
be normalized, but the actual raw-row-to-dense-canonical-row sequence is now
owned by the persisted-row boundary instead of being repeated in terminal,
delete, and save code. Commit preflight now freezes the same accepted row
contract before decoding commit-marker rows and validating unique-index
collisions, and strong-relation delete validation reads candidate source rows
through the accepted source-row contract. These paths no longer fall back to
generated-only slot-count validation for old append-only rows.

### Data-owned row normalization

- Added a shared `canonical_row_from_raw_row_with_structural_contract` helper
  that opens one raw row through a selected structural row contract, validates
  declared slots, and emits one dense canonical row.
- Added a shared typed decode helper that first tries generated current-layout
  decode, then normalizes through the structural contract before retrying typed
  materialization.
- Routed typed response materialization through the data-owned decode helper.
- Routed delete commit before-images through the shared canonicalization helper.
- Routed typed and structural save before-image normalization through the same
  helper.

### Commit preflight and unique indexes

- Built commit-preflight structural slot readers from the accepted row contract
  after marker path/fingerprint validation.
- Passed the accepted row contract into index mutation planning.
- Decoded stored rows used to prove unique-index collisions with the accepted
  row contract instead of rebuilding a generated-only row contract.

### Relation validation

- Routed delete-side strong-relation source-row checks through accepted row
  contracts.
- Removed the stale reverse-index raw-row fallback that rebuilt a generated
  structural row contract after commit preflight had already decoded source
  rows into structural slot readers.

## 0.148.3

`0.148.3` carries accepted-schema baseline normalization into typed save
before-images and tightens public non-SQL plus compiled-SQL coverage for the
append-only nullable transition. Typed single-row and batch saves now validate
and normalize old shorter before-images with the accepted row contract before
commit preparation, matching the structural mutation path.

### Typed save baseline normalization

- Threaded the accepted row decode contract from the session save guard into
  `SaveExecutor`.
- Threaded the same accepted row contract through the pre-checked SQL and
  structural save helper so every save executor created after a schema guard
  carries the contract.
- Validated typed save baselines through the accepted row contract when present.
- Normalized typed save commit-marker before-images to dense current-layout
  rows before index/relation preflight consumes them.

### Public execution surfaces

- Added compiled SQL SELECT coverage over an old shorter row after an accepted
  nullable field is appended.
- Added compiled SQL UPDATE coverage that rewrites an old shorter row to the
  current dense layout.
- Added compiled SQL DELETE and DELETE RETURNING coverage over old shorter
  rows.
- Added direct `structural_patch` + `mutate_structural(Update)` coverage over
  an old shorter row after an accepted nullable field is appended.
- Verified the structural update writes back a dense current-layout row that
  typed decode can read and preserves the appended nullable field as `None`.
- Added direct structural update coverage that explicitly sets the newly
  appended nullable field on an old shorter row.
- Added typed `update(entity)`, `replace(entity)`, atomic update batch, and
  non-atomic replace batch coverage over old shorter rows.
- Added typed `insert(entity)` conflict coverage against an existing old
  shorter row so duplicate-key checks use accepted-schema identity validation.
- Added count-only fluent delete coverage over an old shorter row, separate
  from row-returning delete coverage.

## 0.148.2

`0.148.2` carries accepted-schema row normalization through the typed response
surfaces that still crossed generated decoders directly. Fluent load, cursor
page load, ranking row terminals, and row-returning fluent delete now all
normalize old shorter rows through the accepted row layout before materializing
typed entities.

### Typed response boundaries

- Added one shared typed row decode helper that first tries the generated
  current-layout decoder, then falls back to accepted row-layout normalization
  for old append-only nullable rows.
- Routed plain typed load responses and cursor page responses through the
  prepared plan's accepted row layout.
- Routed `take`, top-k, and bottom-k row materialization through the same
  accepted-layout typed response helper.
- Routed fluent delete row-returning packaging through the same helper while
  preserving rollback row bytes for commit preparation.
- Preserved aggregate projection terminal null handling: optional-field `NULL`
  values remain rejected consistently across `values_by`, distinct projections,
  and count-distinct projection paths.

### Coverage

- Added fluent load coverage for old shorter rows after an accepted nullable
  additive transition.
- Added fluent `take`, cursor-paged load, and top-k row terminal coverage for
  the same transition.
- Added fluent delete row-returning coverage that verifies the returned typed
  entity gets `None` for the appended nullable field and the old row is removed.

## 0.148.1

`0.148.1` extends the append-only nullable transition from read-only SQL
selection into SQL UPDATE and DELETE. Old shorter rows can now be selected for
mutation, patched or removed, preflighted, and written back or committed as
dense current-layout row images with the new nullable field materialized as
`NULL`.

### Structural update staging

- Threaded accepted row-decode contracts into structural mutation staging.
- Added accepted-contract sparse patch replay for old shorter baseline rows.
- Materialized commit-marker before images as dense current-layout rows before
  index/relation preflight consumes them.
- Kept normalized after-image writeback dense for all structural save modes.

### SQL UPDATE target selection

- Changed SQL UPDATE target selection to project primary keys through the SQL
  structural projection path instead of materializing full typed entities.
- Kept UPDATE selector predicates, ordering, limits, and offsets on the same
  accepted-schema planning surface as SELECT.
- Preserved generated-compatible write encoding for the final current-layout
  row image.

### SQL DELETE commit preparation

- Preserved accepted row-layout authority when cached shared plans are cloned
  back into typed prepared-plan shells.
- Moved count-only deletes onto structural row decode so old shorter rows do
  not require typed entity materialization before deletion.
- Normalized delete commit-marker before images to dense current-layout rows
  before relation/index preflight consumes them.
- Kept DELETE RETURNING on the structural projection path while sharing the
  same accepted-schema delete commit preparation.

### Coverage

- Added SQL UPDATE coverage that starts from an old shorter row, mutates it
  after a nullable additive transition, verifies typed current-layout decode,
  and confirms the appended nullable field reads back as `NULL`.
- Added SQL DELETE and DELETE RETURNING coverage for old shorter rows after a
  nullable additive transition.
- Re-ran the existing SQL SELECT old-row additive transition coverage.

## 0.148.0

`0.148.0` starts the schema-evolution runtime cutover by enabling the first
non-exact accepted transition: append-only nullable fields with no database
default. Startup reconciliation now accepts that specific schema drift, writes
the generated snapshot as the accepted live schema, and records a dedicated
schema-transition metric bucket for accepted append-only nullable fields.

### Schema transition policy

- Added `SchemaTransitionPlanKind::AppendOnlyNullableFields` for explicit
  policy ownership instead of treating field-count drift as a generic mismatch.
- Kept required appended fields rejected unless a future persisted database
  default policy exists.
- Kept removals, reorders, type changes, primary-key changes, nested leaf
  changes, and unsafe additive changes rejected.
- Preserved exact-match reconciliation for unchanged schemas.

### Row decode

- Split physical row slot count from accepted logical slot count in structural
  row decode.
- Allowed accepted row contracts to read older shorter rows only through
  `AcceptedFieldAbsencePolicy`.
- Materialized missing appended nullable fields as `Value::Null`.
- Kept generated-only row contracts exact-slot-count strict.
- Updated dense, sparse indexed, one-slot required, and lazy
  `StructuralSlotReader` paths to share the same missing-slot behavior.
- Kept missing required fields fail-closed.

### Coverage

- Added transition-policy tests for accepted append-only nullable fields and
  rejected required appended fields.
- Added reconciliation coverage that verifies the accepted snapshot is replaced
  with the generated schema after a safe additive transition.
- Added row-decode parity coverage for cached, dense, sparse, compact, and
  required-slot reads over old shorter rows.
- Added SQL SELECT coverage that reads an old shorter row after accepting a
  nullable additive schema transition and returns `NULL` for the appended
  field.

# 0.157.x

## 0.157.2

### 🔧 Changed

- Added metadata-only startup reconciliation for generated index name hard cuts.
  Stored accepted snapshots with old generated index names can publish the new
  generated names when ordinal, store, key, uniqueness, and predicate contracts
  are unchanged.

### 🛡️ Guardrails

- The rename reconciliation does not rebuild physical index entries and does
  not accept store/key/uniqueness/predicate drift as a rename.

### 🧪 Validation

- Focused transition and reconciliation tests cover accepting a name-only
  generated index change and publishing the generated accepted snapshot.

## 0.157.1

### ✨ Added

- Added SQL `DROP INDEX <name> ON <entity>` support for non-unique
  DDL-published field-path indexes.
- Routed `DROP INDEX` through accepted-catalog binding, schema-owned mutation
  admission, physical index-key cleanup, and accepted-after schema publication.
- Routed `DROP INDEX` from the CLI SQL shell to the generated DDL update
  endpoint instead of the readonly query endpoint.

### 🛡️ Guardrails

- Hard-cut `SHOW INDEXES` to the SQL-style `SHOW INDEXES FROM <entity>` shape,
  with `IN` accepted as the MySQL synonym, instead of keeping IcyDB's older
  `SHOW INDEXES <entity>` shortcut or adding an `ON` variant.
- Hard-cut generated secondary index names from delimiter-style
  `Entity|field` values to SQL-friendly identifiers:
  `idx_<entity>__<field>` for non-unique indexes and
  `uniq_<entity>__<field>` for unique indexes.
- `DROP INDEX` rejects generated/model-declared indexes; only accepted indexes
  added through SQL DDL are droppable in this slice.
- Kept generated-index authority and schema-store publication outside the SQL
  DDL frontend boundary.

### 🧪 Validation

- Focused `icydb-core` DDL tests cover parsing, binding rejection for generated
  indexes, accepted schema publication, physical index cleanup, and query-plan
  invalidation guardrails.

## 0.157.0

`0.157.0` opens the DDL continuation line after the 0.156 config work.

### 🔧 Changed

- Re-centered the active minor-line target on SQL DDL and accepted-catalog
  mutation work.
- Left explicit room for focused developer improvements when they support the
  DDL path or simplify the public IcyDB API without changing runtime behavior.
- Recorded the initial design boundary for 0.157 DDL continuation work in
  [docs/design/0.157-ddl-continuation/0.157-design.md](../design/0.157-ddl-continuation/0.157-design.md).
- Reused the shared CLI table formatter for `icydb config show` and clarified
  config command help text around optional ICP environment comparison.

### 📚 Documentation

- Updated README and INSTALLING guidance for `icydb.toml`, `config init`,
  `config show`, `config check`, generated endpoint switches, and local
  troubleshooting.

### 🧭 Migration Notes

- No runtime or persistence migration is introduced by this initial 0.157
  changelog/design slice.
- Future DDL slices should continue to treat accepted schema snapshots as the
  runtime authority. SQL DDL remains a frontend over catalog-native mutation
  semantics, not the source of truth.

### 🧪 Validation

- Documentation-only slice.

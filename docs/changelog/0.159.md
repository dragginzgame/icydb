# 0.159.x

## 0.159.6

### 🔧 Changed

- Promoted `RawDataStoreKey` to the live primary-row BTree key type and
  removed the separate `RawDataKey` struct. Data-store, commit-marker,
  relation, executor, and recovery internals now use the store-key taxonomy
  name directly.
- Moved persisted raw-byte construction onto `RawDataStoreKey` so corruption
  tests and stable-memory decode paths use explicit persisted-byte entrypoints
  instead of the old data-module tuple wrapper.
- Promoted `RawIndexStoreKey` to the live secondary-index BTree key type and
  removed the separate codec-local `RawIndexKey` wrapper. Index scans, cursor
  anchors, DDL staging overlays, relation checks, commit markers, and recovery
  now use the index store-key taxonomy name directly.
- Promoted `IndexEntryValue` to the live secondary-index BTree value type and
  removed the separate `RawIndexEntry` wrapper. The index key owns row
  identity; the value carries only the storage-owned presence witness.
- Split `icydb-config-build` into focused parser, resolver, model, error, and
  build-script emission modules. The build-script API now uses
  `emit_config_for_build_script()` and no longer writes the stale
  `OUT_DIR/icydb_config.rs` constants file. Raw string/file parser helpers are
  no longer exported as public crate API.

### 🧪 Validation

- Re-ran data-key, raw-data-store-key, index-key, raw-index-store-key,
  index-entry-value, full-scan, commit-marker rejection, cursor, and
  reverse-index tests to cover the live key rename, compact entity/index-prefix
  bounds, and commit/recovery users of primary-row and secondary-index store
  keys and values.
- Re-ran `icydb-config-build`, `icydb-cli`, and all canister package checks to
  cover the build-script API cleanup.

## 0.159.5

### 🔧 Changed

- Removed the duplicate `DataKeyRawRange` wrapper. Entity-local data scans now
  construct prefix bounds from `RawDataStoreKeyRange::entity_prefix(...)`
  directly, keeping the compact store-key taxonomy as the single owner of
  data-store range framing.
- Kept `RawDataKey` conversion at the BTree boundary only, where stable-memory
  storage still needs the concrete persisted key type, and named that
  conversion as `RawDataKey::from_store_range_bound(...)` instead of using an
  unchecked byte constructor at call sites.
- Centralized BTree range-bound construction behind `RawDataKey` helpers so
  data-store scans no longer manually pair taxonomy byte ranges with
  `std::ops::Bound`.

### 🧪 Validation

- Re-ran data-key and full-scan tests to cover compact entity-prefix bounds and
  primary full-scan routing.

## 0.159.4

### 🔧 Changed

- Promoted the compact store-key taxonomy from proof-only/future wording to
  the active live contract. `DataKey` encoding and decoding now route through
  the `DataStoreKey` taxonomy wrapper, and `IndexStoreKey` now carries the
  same user/system key-kind discriminator as the live `IndexKey` codec.
- Routed ordinary live `IndexKey` row-key encoding through `IndexStoreKey`
  while preserving the raw segment writer for prefix/range sentinel bounds
  that intentionally are not valid `EncodedPrimaryKey` payloads.
- Broadened `EncodedIndexComponent` to represent canonical secondary-index
  component bytes generally, instead of validating every component as a
  primary-key overlap payload.

### 🧪 Validation

- Added guards proving taxonomy `RawIndexStoreKey` bytes match the live
  `IndexKey` codec for both user and system key families, and that malformed
  empty index component / primary-key suffix segments fail closed.

## 0.159.3

### 🔧 Changed

- Collapsed the decoded `IndexEntry` model to one key-owned row identity.
  `IndexEntryValue` still stores only the one-byte presence witness, but
  validation, reverse-relation checks, recovery tests, and expression-index
  scan tests now consume the decoded storage key directly instead of routing
  through stale membership-set iterators.

### 🧪 Validation

- Kept the key-owned index-entry compatibility guard and updated recovery and
  staging tests to prove decoded entries expose one row identity.

## 0.159.2

### 🔧 Changed

- Cleaned up row-check diagnostics after the key-owned index-entry hard cut.
  Executor metrics now report key-owned index entries and decoded row
  identities, and no longer expose the dead single-key/multi-key membership
  split from the removed index-entry value format.

### 🧪 Validation

- Kept the existing row-check metrics surface focused on secondary scan
  traversal, row-identity decode, and authoritative row-presence probes.

## 0.159.1

### 🔧 Changed

- Replaced generated canister stable-memory plumbing with `ic-memory` `0.5.1`
  static range/key registration. Generated actors now emit
  `ic_memory_range!`, use `ic_memory_key!` for store openings, declare the
  commit marker as static allocation metadata, and bootstrap through
  `bootstrap_default_memory_manager()`.
- Removed the temporary IcyDB-owned allocation bridge now that `ic-memory`
  owns the generic multi-crate declaration registry and validated
  `MemoryManager` open path.
- Updated the Canic-facing dependencies to `canic-cdk` / `canic-testkit`
  `0.40.2`; the workspace still has no direct `canic` crate dependency.
- Removed IcyDB's duplicate commit-slot declaration lookup. Commit-marker
  memory now relies on `ic-memory`'s validated runtime open path for the
  stable-key/MemoryManager ID binding, while IcyDB keeps only the process-local
  cache consistency check for its configured commit slot.
- Removed the stale multi-membership decode branch from secondary index scans.
  `RawIndexStoreKey` is now the only row-identity source for live index scans,
  while `IndexEntryValue` validates the one-byte existence witness. Structural
  preflight readers now decode compact key-owned membership directly,
  reverse-index mutation preparation writes/deletes concrete source/target
  edges directly, and streaming access contracts no longer carry unused
  uniqueness plumbing.

### 🧪 Validation

- Added focused commit-memory allocation tests and rebuilt generated canister
  wiring through the `ic-memory` static macro path.
- Added a guard proving `IndexEntryValue` membership is derived from the
  `RawIndexStoreKey` even when a test constructor receives a different key.
- Adjusted commit-memory cache tests so they validate cached allocation
  behavior without poisoning the process-global test allocation used by the
  broader recovery suite.

## 0.159.0

### 🔧 Changed

- Added DB-internal compact key taxonomy types for the 0.159 storage-key
  redesign:
  - `PrimaryKeyValue`
  - `EncodedPrimaryKey`
  - `EncodedIndexComponent`
  - `DataStoreKey`
  - `RawDataStoreKey`
  - `IndexStoreKey`
  - `RawIndexStoreKey`
  - `IndexEntryValue`
- Added compact canonical primary-key encoding for `Nat`, `Int`,
  `Timestamp`, `Ulid`, `Principal`, `Subaccount`, `Account`, and `Unit`.
- Compact primary keys now have the live data-store proof shape
  `logical_kind_tag + canonical_payload`.
- Hard-cut live primary data-store keys from fixed
  `EntityTag + StorageKey(64)` bytes to variable-width
  `EntityTag + EncodedPrimaryKey` bytes.
- Hard-cut live secondary index key primary-key suffixes from fixed
  `StorageKey(64)` bytes to compact `EncodedPrimaryKey` bytes.
- Hard-cut live index entry values to key-owned membership: `RawIndexStoreKey`
  now owns row identity, while `IndexEntryValue` stores only the
  storage-owned presence/existence witness.
- Reverse indexes now store one concrete key-owned entry per source/target
  relation edge instead of grouping multiple source keys inside one target
  value.
- Added validation/decode boundaries for raw wrappers:
  `RawDataStoreKey` validates `EntityTag + EncodedPrimaryKey`, and
  `RawIndexStoreKey` validates key kind, index identity, component count,
  length-prefixed compact components, and compact primary-key suffix.
- Full entity scans now use raw entity-prefix lower/upper bounds over
  variable-width `RawDataStoreKey` bytes instead of synthetic minimum/maximum
  primary-key sentinels.
- Commit marker payload sizing and decode now accept bounded variable-width
  compact row keys.
- Timestamp compact payloads use signed-order-preserving biased bytes instead
  of raw signed big-endian bytes.
- Principal compact payloads preserve current length-first logical ordering
  with `len + bytes`.
- Documented the entity primary-key namespace invariant: one accepted entity
  primary-key namespace has exactly one logical primary-key kind.

### 🧪 Validation

- Added compact primary-key roundtrip, malformed tag, malformed length,
  principal max/invalid length, subaccount exact length, account validation,
  unit singleton, ordering equivalence, negative timestamp ordering,
  principal length-first ordering, and primary/index payload equivalence tests.
- Added raw data/index store key roundtrip and malformed-shape
  rejection tests.
- Added entity-prefix range tests proving ordinary and `u64::MAX`
  entity tags scan by prefix without primary-key sentinel keys.
- Added a guard proving the live data-store wire shape matches the compact key
  taxonomy.
- Added index-key tests proving primary-key suffixes use compact bytes while
  primary-key equivalence checks still decode the referenced row identity.
- Added index-entry tests proving membership decodes from the raw index key,
  malformed witnesses fail closed, and values no longer contain duplicated
  primary-key bytes.
- Updated reverse-index recovery tests around the key-owned entry shape.

### ⏭️ Deferred

- Cursor and recovery naming cleanup around the compact key vocabulary.
- Mechanical naming cleanup after the storage-format cut.

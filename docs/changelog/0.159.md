# 0.159.x

## 0.159.0

### 🔧 Changed

- Added DB-internal compact key taxonomy types for the 0.159 storage-key
  redesign:
  - `PrimaryKeyValue`
  - `EncodedPrimaryKey`
  - `EncodedIndexComponent`
  - `DataStoreKey`
  - `RawDataStoreKey`
  - `IndexStoreKey`
  - `RawIndexStoreKey`
  - `IndexEntryValue`
- Added compact canonical primary-key encoding for `Nat`, `Int`,
  `Timestamp`, `Ulid`, `Principal`, `Subaccount`, `Account`, and `Unit`.
- Compact primary keys now have the live data-store proof shape
  `logical_kind_tag + canonical_payload`.
- Hard-cut live primary data-store keys from fixed
  `EntityTag + StorageKey(64)` bytes to variable-width
  `EntityTag + EncodedPrimaryKey` bytes.
- Hard-cut live secondary index key primary-key suffixes from fixed
  `StorageKey(64)` bytes to compact `EncodedPrimaryKey` bytes.
- Hard-cut live index entry values to key-owned membership: `RawIndexKey`
  now owns row identity, while `RawIndexEntry` stores only the
  storage-owned presence/existence witness.
- Reverse indexes now store one concrete key-owned entry per source/target
  relation edge instead of grouping multiple source keys inside one target
  value.
- Added validation/decode boundaries for raw wrappers:
  `RawDataStoreKey` validates `EntityTag + EncodedPrimaryKey`, and
  `RawIndexStoreKey` validates key kind, index identity, component count,
  length-prefixed compact components, and compact primary-key suffix.
- Full entity scans now use raw entity-prefix lower/upper bounds over
  variable-width `RawDataKey` bytes instead of synthetic minimum/maximum
  primary-key sentinels.
- Commit marker payload sizing and decode now accept bounded variable-width
  compact row keys.
- Timestamp compact payloads use signed-order-preserving biased bytes instead
  of raw signed big-endian bytes.
- Principal compact payloads preserve current length-first logical ordering
  with `len + bytes`.
- Documented the entity primary-key namespace invariant: one accepted entity
  primary-key namespace has exactly one logical primary-key kind.

### 🧪 Validation

- Added compact primary-key roundtrip, malformed tag, malformed length,
  principal max/invalid length, subaccount exact length, account validation,
  unit singleton, ordering equivalence, negative timestamp ordering,
  principal length-first ordering, and primary/index payload equivalence tests.
- Added raw data/index store key roundtrip and malformed-shape
  rejection tests.
- Added entity-prefix range tests proving ordinary and `u64::MAX`
  entity tags scan by prefix without primary-key sentinel keys.
- Added a guard proving the live data-store wire shape matches the compact key
  taxonomy.
- Added index-key tests proving primary-key suffixes use compact bytes while
  primary-key equivalence checks still decode the referenced row identity.
- Added index-entry tests proving membership decodes from the raw index key,
  malformed witnesses fail closed, and values no longer contain duplicated
  primary-key bytes.
- Updated reverse-index recovery tests around the key-owned entry shape.

### ⏭️ Deferred

- Cursor and recovery naming cleanup around the compact key vocabulary.
- Mechanical naming cleanup after the storage-format cut.

# 0.153.x

## 0.153.4

`0.153.4` adds the first in-memory staged field-path index-store buffer and
discard diagnostics. The schema mutation runner path can now bind validated
rebuild entries to the accepted persisted store identity without mutating
physical `IndexStore` state or making rebuilt data runtime-visible.

### Staged Store Buffer

- Added a staged field-path index-store buffer that is constructed only from
  validated staged rebuild output.
- Bound buffered entries to the accepted index store path carried by
  `SchemaFieldPathIndexRebuildTarget`.
- Kept buffered store visibility fixed to `StagedOnly`; the patch does not
  mutate physical index stores, invalidate runtime state, or publish snapshots.
- Reused typed runner diagnostics so buffered staged state reports scanned
  rows, skipped rows, and staged index keys while remaining non-publishable.
- Added typed discard diagnostics for in-memory staged buffers so failed or
  canceled staged work has an explicit cleanup report before physical rollback
  exists.
- Added focused coverage proving valid staged rebuild output writes into the
  in-memory store buffer with accepted store identity, no publication, and a
  typed discard report.

### Validation

- `cargo fmt --all`
- `cargo test -p icydb-core schema::mutation --features sql -- --nocapture`
- `cargo check -p icydb-core --tests --features sql`
- `cargo clippy -p icydb-core --all-targets --features sql -- -D warnings`
- `git diff --check`

## 0.153.3

`0.153.3` adds the first in-memory staged field-path rebuild output. The
schema mutation runner path can now derive sorted raw index entries from
accepted rebuild targets and authoritative row slots while keeping rebuilt
state unpublished and invisible to runtime planning.

### Field-Path Rebuild Staging

- Added a schema-owned row input contract for field-path rebuild staging.
- Added in-memory staged field-path rebuild output that records raw index
  entries, source-row count, and skipped non-indexable row count.
- Added staged rebuild validation reports and fail-closed validation errors for
  published visibility, row-count drift, and unsorted or duplicate entries.
- Added runner diagnostics for validated staged field-path rebuilds, including
  completed phases, scanned rows, skipped rows, and staged index keys written.
- Added a runner-report publication readiness guard so validated staged physical
  work remains non-publishable until invalidation and snapshot publication
  phases are present.
- Kept staged output fixed at `StagedOnly` visibility; the patch does not write
  to index stores, publish snapshots, or widen runtime visibility.
- Added focused coverage proving staged field-path rebuild output is
  deterministic, skips non-indexable rows, remains unpublished, reports typed
  runner diagnostics, and rejects mutated staged state.

## 0.153.2

`0.153.2` starts the field-path rebuild execution path by adding an
accepted-target index-key materializer. Future physical runners can now derive
field-path secondary-index keys from accepted mutation targets and canonical row
slots without reopening generated index metadata or planner-visible
`SchemaInfo`.

### Field-Path Rebuilds

- Added an internal index-key constructor that consumes
  `SchemaFieldPathIndexRebuildTarget` directly.
- Rebuild key materialization now uses accepted row-layout slots from the
  mutation target, including nested field-path traversal.
- Kept the primitive staged behind the runner boundary; it does not publish
  schema changes or write index stores yet.
- Added focused coverage proving accepted rebuild targets materialize keys from

## 0.153.1

`0.153.1` adds runtime schema identity contracts for schema mutation
publication. Future runners can now derive stable runtime epochs from accepted
persisted snapshots and keep staged physical work invisible until publication
advances runtime identity.

### Runtime Identity

- Added a schema mutation runtime epoch contract derived from accepted persisted
  snapshots, including entity path, schema version, and a snapshot fingerprint.
- Added a schema mutation publication identity contract that binds the
  before/after runtime epochs to staged or published store visibility.
- Kept staged publication identity pinned to the before epoch, so validated but
  unpublished rebuild work cannot advance planner/cache/runtime identity.
- Added focused mutation tests proving runtime epochs are stable for identical
  accepted snapshots, change when accepted snapshots change, and stay invisible
  until the publication identity is marked published.

## 0.153.0

`0.153.0` starts the schema rebuild runner implementation by making runner
outcomes and diagnostics explicit without changing runtime publication behavior.
The new contracts sit on top of the `0.152` mutation execution preflight
boundary and keep physical rebuild execution deferred.

### Runner Contracts

- Added internal runner phase names for preflight, staged stores, physical
  build, validation, runtime invalidation, snapshot publication, and diagnostics.
- Added staged-store visibility contracts so future rebuild output can be named
  as staged-only before publication.
- Added structured runner rejection kinds and rejection reports for missing
  capabilities and unsupported requirements, with room for validation, rollback,
  and publication failures.
- Added positive runner reports for preflight outcomes, including step count,
  required capabilities, completed phases, and store visibility.
- Added a runner outcome API that classifies plans as no physical work, ready
  for staged physical work, or rejected while keeping `PhysicalWorkReady`
  separate from publication.
- Added a checked runner input contract that binds accepted-before snapshots,
  accepted-after snapshots, and already-derived execution plans together while
  rejecting cross-entity or primary-key-mismatched snapshot pairs.
- Added a no-op runner adapter that consumes checked runner inputs, reports
  metadata-only inputs as no physical work, and rejects physical-work inputs
  through the structured missing-capability outcome until real staged execution
  exists.
- Added focused mutation tests proving metadata-only plans report no physical
  work, supported index rebuild plans report staged physical readiness, and
  missing capabilities or unsupported rewrite requirements reject at preflight.
- Added focused mutation tests proving runner input construction accepts
  same-entity accepted snapshot pairs and fails closed for entity path, entity
  name, and primary-key identity mismatches.
- Added focused mutation tests proving the no-op runner accepts metadata-only
  inputs and fails closed for non-unique field-path index rebuild inputs.

# IcyDB 0.50

This document aggregates detailed release notes for the `0.50.x` minor line.

## 0.50.1 - 2026-03-11 - Slice E Batch 1 (Mechanical Cleanup)

### Summary

`0.50.1` starts Slice E executor simplification by consolidating repeated
aggregate terminal helper paths in a mechanical-only batch.

### Changed

- Consolidated repeated aggregate output-kind decoding in
  `crates/icydb-core/src/db/executor/aggregate/terminals.rs` into shared
  helpers for COUNT/EXISTS and id-returning terminals (MIN/MAX/FIRST/LAST).
- Consolidated shared existing-row reducer setup for index-backed COUNT and
  EXISTS fast paths into one helper path.
- Consolidated FIRST/LAST projection-terminal wrapper mechanics and covering
  index-bounds component scan dispatch in
  `crates/icydb-core/src/db/executor/aggregate/projection/mod.rs`.
- Consolidated runtime payload-window and consistency row-read helper paths in
  `crates/icydb-core/src/db/executor/runtime_context/mod.rs`.
- Removed unused post-access `Deref` pass-through and collapsed non-test phase
  suppressor bindings in
  `crates/icydb-core/src/db/executor/pipeline/operators/post_access/mod.rs`.
- Removed a single-use prepared-row delta metrics wrapper and inlined
  equivalent wiring in
  `crates/icydb-core/src/db/executor/mutation/commit_window.rs`.
- Consolidated repeated continuation/fetch node-property helper paths in
  `crates/icydb-core/src/db/executor/explain/descriptor.rs`.
- Consolidated aggregate empty-window and zero-output helper wiring in
  `crates/icydb-core/src/db/executor/aggregate/mod.rs`.
- Simplified runtime-context consistency read helper layering in
  `crates/icydb-core/src/db/executor/runtime_context/mod.rs`.
- Consolidated duplicated verbose EXPLAIN route fetch-line formatting in
  `crates/icydb-core/src/db/executor/explain/descriptor.rs`.
- Runtime behavior and planner/executor contracts are unchanged.

## 0.50.0 - 2026-03-11 - Line Open (Design + Preparation)

### Summary

`0.50.0` opens the `0.50` line with design and preparation artifacts for the
executor simplification track. This patch is intentionally documentation and
planning only.

### Changed

- Added the `0.50` design line under
  `docs/design/0.50-runtime-capability-expansion/`.
- Added line-level design/status docs that define:
  - core invariants (`plan`, `continuation`, `explain`) that must remain stable
  - slice sequencing for the line (`0`, `A`, `B`, `C`, `D`, `E`)
  - explicit non-goals and success criteria for executor simplification work
- Added Slice 0 preflight artifacts for ownership and baseline tracking:
  - execution notes
  - executor ownership matrix
  - structural baseline metrics
- Added starter slice docs for follow-up work planning (`A` through `E`).
- Runtime/API/storage behavior is unchanged in `0.50.0`; this patch records
  planning and release-line preparation only.

# 0.24 Changelog

## 0.24.7 - 2026-02-22 - Field Aggregate Hardening

### Summary

`0.24.7` continues pre-`0.25` hardening for field-based aggregate execution.

### Changed

- Added a shared aggregate-spec boundary so field-target aggregate requests are validated in one place before execution.
- Unsupported field-target aggregate shapes now fail fast with explicit `Unsupported` errors.
- Locked deterministic tie-break behavior for upcoming field extrema (`field` value first, then primary key ascending).
- Added route-level ineligibility diagnostics for field-extrema fast paths.

### Validation

- Expanded regressions so unsupported field-target aggregate requests consume zero scan budget.

---

## 0.24.6 - 2026-02-22 - Route Authority Hardening

### Summary

`0.24.6` centralizes route decisions and tightens guardrails to reduce planner/executor drift.

### Changed

- Consolidated duplicated routing and eligibility checks across load and aggregate paths.
- Simplified fast-path guard logic around spec arity and alignment.
- Hardened invariant and error-classification boundaries for route-owned decisions.

### Validation

- Added targeted coverage to keep route behavior and aggregate fold-mode decisions stable.

## 0.24.5 - 2026-02-22 - Secondary Min/Max Probe

### Summary

`0.24.5` adds a bounded secondary-index probe for eligible `min()`/`max()` shapes.

### Changed

- Eligible secondary-index extrema routes now probe with `offset + 1` keys before falling back.
- Added safe fallback for stale-leading `MissingOk` boundaries: if bounded probe returns no extrema, execution retries unbounded.
- Kept scan accounting explicit when probe and fallback both execute.
- Consolidated shared guard and hint routing helpers to reduce duplicated fast-path logic.

### Validation

- Added regressions that lock probe-budget behavior for `Strict` and clean `MissingOk` paths.
- Added stale-leading and guard regressions to verify fallback parity and arity/hint invariants.

## 0.24.4 - 2026-02-22 - RouteCapabilities Consolidation

### Validation

- Added capability-focused route regressions for representative descending and distinct+offset shapes.

## 0.24.1 - 2026-02-22 - FIRST and LAST

### Summary

`0.24.1` adds `first()` and `last()` aggregate terminals for load queries.

### Added

- Added `first()` and `last()` to return the first or last matching id in response order.

```rust
let first_id = session.load::<MyEntity>().order_by("id").first()?;
let last_id = session.load::<MyEntity>().order_by_desc("id").last()?;
```

### Changed

- Kept semantics aligned with query ordering, pagination, and consistency behavior.
- Added bounded scan hints for `first()` and limited-window `last()`.
- Cursor tokens are now rejected for non-paged terminal execution; continuation is accepted only through `.page().execute()` paths.

### Validation

- Added parity coverage for `first()`/`last()` against materialized behavior.
- Added scan-budget regressions for `first()` (`offset + 1`) and limited-window `last()` (`offset + limit`).
- Added regressions for unbounded `last()` full-stream behavior and non-paged cursor rejection.

## 0.24.0 - 2026-02-21 - Composite Aggregate Direct Path

### Summary

`0.24.0` introduces direct aggregate execution for composite query shapes while preserving existing results.

### Changed

- Added direct aggregate handling for combined plans (including unions and intersections).
- Kept aggregate semantics aligned with canonical fallback behavior, including count behavior.
- Unified route priority so load and aggregate execution follow one precedence model.

### Cleanup

- Moved aggregate fold internals into a dedicated executor module.
- Reused shared guard/binding helpers so load and aggregate paths enforce the same checks.
- Consolidated index-range eligibility mapping behind a shared helper.

### Validation

- Added regressions proving direct-path parity with fallback aggregate outputs.
- Added focused route-priority and aggregate guard coverage.

# IcyDB 0.169 Changelog

## 0.169.1

### 🔧 Changed

- Storage snapshots now derive each data, index, and schema row's storage mode
  from the actual runtime store backend.
- Heap-backed store roles report `storage: heap` with no `memory_id` or
  `stable_key`.
- Stable stores with intentionally absent allocation identities still report
  `storage: stable`, keeping storage mode separate from allocation metadata.

### 🧪 Testing

- Adds diagnostics coverage for a heap-backed store registered with absent
  allocation identity.
- Extends storage-mode rendering coverage to include the `heap` label.

## 0.169.0

### ➕ Added

- Admits explicit `storage(heap())` store declarations as the first non-stable
  store backend.
- Adds volatile heap-backed `DataStore`, `IndexStore`, and `SchemaStore`
  runtime wrappers using ordered in-memory maps.
- Generates heap store wiring through `init_heap()` constructors and explicit
  absent allocation identity.

### 🔧 Changed

- `storage(heap())` is now the canonical heap storage form instead of reserved
  syntax.
- Heap stores reject stable-only memory ID arguments instead of silently
  assigning them heap meaning.
- Stable stores keep their existing generated stable wiring and allocation
  identity path.

### 🧪 Testing

- Adds compile-pass coverage for heap store declarations.
- Adds heap traversal tests for data, index, and schema stores covering order,
  range bounds, and early-stop behavior.
- Adds generated wiring tests proving stable stores still use stable memory
  keys while heap stores use heap initializers and absent allocation identity.

### 🧭 Migration Notes

- Stable store declarations are unchanged.
- Heap stores are volatile and do not carry stable-memory IDs, stable keys, or
  durable allocation identity.

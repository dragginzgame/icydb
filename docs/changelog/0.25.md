# IcyDB 0.25

This document aggregates detailed release notes for the `0.25.x` minor line.

---

## 0.25.2 - 2026-02-23 - Field Aggregate Additions

### 📝 Summary

`0.25.2` adds three field aggregate terminals: `median_by`, `count_distinct_by`, and `min_max_by`.

### ➕ Added

- Added `median_by("field")` returning the id at the median position in deterministic field order (`field asc`, then primary key asc). Even-window policy is lower median in this release.
- Added `count_distinct_by("field")` counting unique field values in the effective result window. This terminal supports non-orderable field values because it uses value equality, not value ordering.
- Added `min_max_by("field")` returning `Option<(Id<E>, Id<E>)>` in one call, with deterministic primary-key tie breaks.

```rust
let median_id = session.load::<User>().order_by("id").median_by("rank")?;
let distinct_count = session.load::<User>().order_by("id").count_distinct_by("rank")?;
let min_max_ids = session.load::<User>().order_by("id").min_max_by("rank")?;
```

### 🔧 Changed

- New terminals execute through canonical effective-window scan behavior without adding fast-path eligibility.
- Query-level `DISTINCT` remains ineligible for field-extrema fast paths; for `count_distinct_by`, query `DISTINCT` only affects window row membership.

### 🧪 Testing

- Expanded field-terminal parity coverage across DISTINCT/windowed ASC and DESC query shapes.
- Added focused regressions for lower-median policy, non-orderable distinct counting, unknown-field fail-before-scan behavior, and `min_max_by` invariants.

## 0.25.1 - 2026-02-23 - Aggregate Stabilization

### 📝 Summary

`0.25.1` stabilizes `0.25.0` field aggregates without expanding the public aggregate surface.

### 🔧 Changed

- Hardened field-terminal regression contracts without changing aggregate API coverage.
- Locked fallback-vs-fast-path scan behavior for eligible and ineligible field-extrema shapes.

### 📚 Documentation

- Synchronized release docs across status, roadmap, and aggregate user references.

### 🧪 Testing

- Added negative-lock coverage for field-extrema eligibility so ineligible shapes do not silently drift into probe behavior.
- Added deterministic `nth_by` boundary matrix coverage for empty/out-of-range windows.
- Added field-terminal error-classification coverage across unknown/non-orderable/non-numeric/stale-leading cases.

## 0.25.0 - 2026-02-22 - Field Aggregate Expansion

### 📝 Summary

`0.25.0` starts the aggregate expansion line with new field-based aggregate terminals.

### ➕ Added

- Added `min_by("field")` and `max_by("field")` to return ids selected by deterministic field ordering.
- Added `nth_by("field", n)` to return the id at zero-based position `n` in deterministic order.
- Added `sum_by("field")` and `avg_by("field")` for numeric fields, returning decimal values and `None` for empty windows.

```rust
let min_id = session.load::<User>().order_by("id").min_by("rank")?;
let max_id = session.load::<User>().order_by("id").max_by("rank")?;
let second_id = session.load::<User>().order_by("id").nth_by("rank", 1)?;
```

### 🔧 Changed

- Eligible index-leading shapes execute `min_by`/`max_by` through a route-gated streaming path; ineligible shapes use canonical fallback.
- Tie handling is deterministic (`field` first, then primary key ascending).
- Unknown/non-orderable field targets fail fast with explicit `Unsupported` errors before scan work.
- `nth_by` returns `None` when `n` is outside the effective result window.

### 🧪 Testing

- Added DISTINCT/windowed parity coverage for field terminals in ASC and DESC query shapes.
- Added stale-leading secondary-index consistency coverage for field extrema in `MissingOk` and `Strict` consistency modes.
- Added matrix guard coverage that locks expected field-terminal aggregate surface for rank-target cases.

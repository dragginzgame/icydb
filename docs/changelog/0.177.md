# IcyDB 0.177 Changelog

## 0.177.1

### 📝 Summary

- Tightens the explicit schema-version admission line with clearer rejection
  facts, hot-path source-boundary guards, and storage-passive publication
  proof.
- Keeps query planning passive: normal session, query, and executor paths
  continue to consume accepted identity instead of computing candidate
  admission fingerprints.

### 🔧 Changed

- Extends schema-version admission rejection details to report stored and
  candidate schema versions, fingerprint-method versions, and fingerprints.
- Reports the expected next version on version-gap rejection while preserving
  the existing primary diagnostic classes for missing bumps, empty bumps, gaps,
  rollbacks, and fingerprint-method mismatches.
- Cleans up the reconciliation comment around the 0.177 gate so it reflects
  the now-active matrix enforcement before mutation compatibility.
- Adds a storage-publication guard that keeps persisted schema snapshots keyed
  by their declared version instead of a storage-assigned next version.

### 🧪 Testing

- Adds admission-matrix assertions that rejection diagnostics include the
  compared identity facts.
- Adds a source-boundary guard proving candidate admission fingerprinting,
  identity comparison, and schema-version admission policy stay rooted in
  schema reconciliation rather than session/query/executor hot paths.
- Adds a source-boundary guard proving schema storage publication does not
  synthesize or auto-increment schema versions.

## 0.177.0

### 📝 Summary

- Starts the explicit schema-version contract line with an identity/header
  alignment, generated-declaration, admission-fingerprint input, and
  version/fingerprint matrix enforcement slices.
- Keeps public API, persisted format, schema admission policy, query semantics,
  SQL behavior, and writes unchanged.

### 🔧 Changed

- Makes shared query-plan cache identity compare accepted schema fingerprints
  together with their fingerprint-method version.
- Makes SQL compiled-command cache identity compare accepted schema
  fingerprints together with their fingerprint-method version.
- Makes per-compiled SELECT prepared-plan reuse compare accepted schema
  fingerprints together with their fingerprint-method version.
- Adds explicit `schema_version = N` parsing to generated entity declarations,
  rejecting omission, zero, and non-integer values at the macro boundary.
- Carries the declared generated schema version through `EntityModel`,
  schema-node metadata, and `CompiledSchemaProposal` initial snapshot
  construction without adding query-path candidate fingerprint work or
  admission-policy checks.
- Adds a schema-owned admission identity tuple over stored and candidate
  snapshots so reconciliation can compute version, fingerprint-method version,
  and accepted-shape fingerprint before mutation compatibility classification.
- Computes the admission shape fingerprint from a normalized accepted snapshot
  that removes declared `schema_version` while preserving accepted shape,
  field, index, relation, and layout contract facts.
- Keeps generated index display-name changes out of the admission shape
  fingerprint so existing metadata-only generated index rename behavior stays
  compatible with the version contract.
- Enforces the 0.177 admission matrix before mutation compatibility: unchanged
  version with changed shape, empty bumps, version gaps, rollbacks, and
  fingerprint-method mismatches now reject at the schema-version boundary;
  exactly `N + 1` with changed shape proceeds to existing compatibility checks.
- Extends test-only entity fixtures with explicit schema-version overrides so
  transition tests can distinguish stored accepted version `1` from generated
  candidate version `2`.

### ⚠️ Breaking

- Generated entities must now declare `schema_version = 1` or another positive
  integer in the `#[entity(...)]` attribute.

### 🧪 Testing

- Adds fail-closed cache-key guards proving raw schema fingerprint bytes do not
  alias across fingerprint-method versions.
- Extends source-boundary guards so cache identity must carry the accepted
  schema fingerprint method beside raw fingerprint bytes.
- Adds macro compile-fail coverage for omitted, zero, and non-integer
  `schema_version` declarations.
- Adds proposal coverage proving declared generated versions reach initial
  row-layout and accepted-snapshot construction.
- Adds admission-fingerprint coverage proving declared `schema_version` is
  adjacent identity, not an input to the accepted-shape fingerprint, and that
  accepted field/index shape changes still alter the fingerprint.
- Adds matrix coverage for same identity, single-version changed-shape
  admission, missing bump, empty bump, gap, rollback, and fingerprint-method
  mismatch cases.
- Updates runtime boundary tests so unsupported same-version drift now proves
  rejection at the schema-version gate before select planning, row decode, or
  write/delete staging.

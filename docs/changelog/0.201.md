# IcyDB 0.201 Changelog

## 0.201.0 - 2026-07-13

### 📝 Summary

- Makes visitor traversal the single owner of child recursion for `Option` and
  `Vec`, so automatic and custom sanitize and validation hooks run exactly once
  for each visited value.
- Preserves the existing transparent `Box` contract, where the wrapper forwards
  one hook call to its boxed value because traversal does not independently
  visit that child.

### 🩹 Fixed

- Removes duplicate child sanitization that could apply the same mutation twice
  when an authored model contained values inside `Option` or `Vec`.
- Removes duplicate child validation and keeps reported issues attached to the
  traversal-owned path, including indexed paths for vector elements.
- Aligns the container implementations with the documented visitor contract:
  hooks inspect their current node while the traversal driver owns recursion.

### 🧪 Testing

- Adds focused coverage proving that automatic and custom sanitize and validate
  hooks each run once through nested `Option<Vec<_>>` values.
- Covers indexed vector issue paths and the retained one-call forwarding
  behavior for `Box`.
- Existing generated sanitize and validation visitor suites pass, and focused
  all-feature Clippy validation passes with warnings denied.

### 📊 Audit

- Production code removes eight recursive container hook bodies. The touched
  Rust file has 124 insertions and 56 deletions, a net increase of 68 lines;
  the added regression coverage accounts for the increase while the runtime
  implementation becomes simpler.
- `ten_entity_fluent_rows` remains 2,366,627 raw wasm bytes and 2,204,580
  canonically shrunk bytes, both unchanged from `0.200.3`. Raw non-gzipped wasm
  remains the primary size measure.
- No instruction-count change is claimed. The corrected hooks affect authored
  model sanitization and validation rather than query execution strategy.

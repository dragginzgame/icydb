# IcyDB 0.201 Changelog

## 0.201.2 - 2026-07-13

### 📝 Summary

- Moves visitor traversal, runtime-value conversion, collection handling, and
  entity-key contracts out of the central trait implementation hub and into
  their owning domains.
- Encodes entity placement, singleton identity, key eligibility, and
  fixed-width key projection as explicit type and error contracts.
- Removes duplicate typed read adapters, compacts the pre-1.0 diagnostic
  registry, and gives invalid continuation cursors one stable diagnostic.

### ⚠️ Breaking

- Direct `icydb_core::traits` imports for visitor contracts move to
  `icydb_core::visitor`; application imports through `icydb::traits` remain
  supported.
- Direct `icydb_core::traits` imports for runtime conversion and collection
  contracts move to `icydb_core::value`. Runtime decoding no longer supports
  borrowed `&str`; callers must decode an owned `String`.
- Direct `icydb_core::traits` imports for entity-key contracts move to
  `icydb_core::db`, and `PrimaryKeyCodec` is hard-renamed to the one-way
  `PrimaryKeyEncode` contract without an alias.
- Custom primary keys must implement `KeyValueCodec` explicitly. General
  runtime value conversion no longer grants key eligibility to collections or
  other non-key types.
- `EntityPlacement` now proves that its store belongs to its declared canister,
  and `SingletonEntity` is derived only from `()` or `Unit` keys. Manual
  implementations that relied on looser relationships must adopt the new
  bounds.
- `EntityKeyBytes::write_bytes`, `Id::project()`, and `Id::subaccount()` return
  typed errors instead of relying on fixed-buffer assertions. Callers must
  handle the result.
- Manual `PersistedRow` implementations must also satisfy `EntityValue` so the
  public `Entity` facade and persisted runtime use one concrete value proof.
- Removes never-emitted diagnostic codes E77 and E78. Former numeric codes
  E79-E204 shift down to E77-E202; consumers that persist or match raw numeric
  codes must refresh their mappings.

### 🔧 Changed

- `Visitable`, sanitize/validate traits, visitor contexts, and primitive
  visitor implementations now live under the visitor owner. Generated visitor
  implementations use the public visitor path directly.
- Runtime value metadata, encode/decode contracts, enum-context admission,
  collection markers, and shared list/set/map helpers now live under the value
  owner. Generated runtime conversion continues through the hidden macro
  facade.
- Entity-key identity, canonical byte projection, scalar relation proof,
  runtime key conversion, and primary-key encode/decode now live beside the
  canonical key taxonomy.
- Contextual and context-free set/map decoding share the same canonicality and
  duplicate-rejection rules, including collisions introduced by decoded enum
  keys or items.
- `PersistedRow` carries the concrete entity-value proof, and the closed
  `ValueProjectionExpr` helper family is explicitly sealed.

### 🩹 Fixed

- Invalid external continuation cursors now map to E6 with cursor origin from
  planner and executor conversion paths instead of falling through to generic
  unsupported diagnostics.
- Fixed-width entity-key encoding reports wrong buffer sizes and oversized key
  values without a production assertion or panic.

### 🧹 Cleanup

- Collapses duplicate typed and structural primary/index readers into the two
  maintained structural ports, removing adapter-only implementations and
  ineffective sealing markers.
- Removes generated runtime-value strategy helpers and central trait reexports
  made obsolete by owner-local modules. Generated code keeps one hidden wiring
  facade while application traits keep one deliberate public facade.
- Reduces the central core trait module from 1,655 to 426 lines; retained
  definitions moved to owners rather than being replaced by compatibility
  aliases.

### 🧭 Migration Notes

- Replace direct core visitor imports with `icydb_core::visitor` and direct
  core runtime/collection imports with `icydb_core::value`.
- Replace direct core key imports with `icydb_core::db` and rename custom
  `PrimaryKeyCodec` implementations and bounds to `PrimaryKeyEncode`.
- Keep normal application imports through `icydb::traits`; those names remain
  the supported application facade.
- Decode text into `String`, handle the new key/projection results, and refresh
  any stored numeric diagnostic-code lookup tables.
- This is a pre-1.0 hard cut. No old-path aliases, trait wrappers, fallback
  decoders, or diagnostic-number compatibility maps are provided.

### 🧪 Testing

- Focused key-taxonomy and data-key suites pass for all maintained scalar and
  composite key families, malformed key shapes, and typed/structural parity.
- Derive, generated-schema, macro UI, and core UI suites pass with the new
  owner paths and generated composite-key implementation.
- Package-scoped all-target checks and Clippy pass with warnings denied. The
  user-owned full push suite remains the final release gate.

### 📊 Audit

- The source/test change spans about 180 files with approximately 2,568
  insertions and 2,488 deletions, a net increase of about 80 lines. Most file
  churn is import and generated-facade rewiring; the implementation shape is
  simpler because each retained contract has one owner.
- `ten_entity_fluent_rows` measures 2,366,712 raw wasm bytes and 2,204,661
  canonically shrunk bytes. Compared with `0.201.1`, that is +85 raw bytes and
  +81 shrunk bytes, both below 0.004%; deterministic gzip is secondary at
  768,887 raw and 733,332 shrunk bytes.
- No database, row, index, primary-key, cursor, or runtime-value wire format
  changes. No data migration, rewrite, or instruction-count improvement is
  claimed.

### 📚 Documentation

- Records Slices D through G and the visitor-, runtime-value-, and key-owned
  portions of Slice H in the trait-hardening design. Persistence, entity,
  numeric, and remaining generated-only owner moves remain future 0.201 work.

## 0.201.1 - 2026-07-13

### 📝 Summary

- Removes the retired generated field-codec and test-only slot-writer paths,
  leaving accepted field contracts as the single authority for row persistence.
- Keeps generated row code focused on materializing already accepted runtime
  values from `SlotReader`; generated models do not choose persisted bytes.

### ⚠️ Breaking

- Removes `PersistedFieldSlotCodec`, `PersistedFieldMetaCodec`, and `SlotWriter`
  together with their helper and hidden re-export chains. This is a pre-1.0
  hard cut with no aliases or forwarding wrappers.
- Direct implementations or imports of those retired contracts must move to
  accepted field contracts or the maintained scalar, structural, and by-kind
  codec APIs.
- Normal derive-generated applications require no migration because current
  generated code did not consume the removed contracts.

### 🔧 Changed

- Routes typed model-proposal test row creation through the same accepted field
  contracts used by maintained persistence paths.
- Makes generated persisted-row documentation and compile-time assertions
  describe runtime materialization rather than obsolete field-owned storage.
- Retains `PersistedScalar`, `PersistedStructuredFieldCodec`, and
  `PersistedByKindCodec` as separate codecs with distinct responsibilities.

### 🗑️ Removed

- Deletes the retired slot and metadata codec modules and the complete-image
  writer that existed only for test staging.
- Deletes fixture implementations and compile-fail tests whose only purpose was
  preserving the removed slot-codec surface.
- Deletes runtime-value strategy helpers and storage aliases that became
  unreachable after the metadata codec was removed.

### 🧪 Testing

- Accepted-row persistence and mutation-save suites pass, including malformed
  payload rejection, typed/structural parity, defaults, enums, and nullability.
- Generated schema and both compile-time UI suites pass, and focused
  all-feature Clippy validation passes with warnings denied.

### 📊 Audit

- The combined cleanup changes 25 Rust files with 80 insertions and 1,306
  deletions, a net reduction of 1,226 lines. The implementation is simpler:
  one accepted-contract path replaces two unused field-owned adapter layers.
- `ten_entity_fluent_rows` remains 2,366,627 raw wasm bytes and 2,204,580
  canonically shrunk bytes, both unchanged from `0.201.0`. The removed adapters
  were not linked into the measured runtime artifact.
- No database, row, index, cursor, or value-wire format changes. No migration,
  data rewrite, or instruction-count improvement is claimed.

### 📚 Documentation

- Marks trait-hardening Slices B and C complete and records accepted field
  contracts as the persistence authority.

## 0.201.0 - 2026-07-13

### 📝 Summary

- Makes visitor traversal the single owner of child recursion for `Option` and
  `Vec`, so automatic and custom sanitize and validation hooks run exactly once
  for each visited value.
- Preserves the existing transparent `Box` contract, where the wrapper forwards
  one hook call to its boxed value because traversal does not independently
  visit that child.

### 🩹 Fixed

- Removes duplicate child sanitization that could apply the same mutation twice
  when an authored model contained values inside `Option` or `Vec`.
- Removes duplicate child validation and keeps reported issues attached to the
  traversal-owned path, including indexed paths for vector elements.
- Aligns the container implementations with the documented visitor contract:
  hooks inspect their current node while the traversal driver owns recursion.

### 🧪 Testing

- Adds focused coverage proving that automatic and custom sanitize and validate
  hooks each run once through nested `Option<Vec<_>>` values.
- Covers indexed vector issue paths and the retained one-call forwarding
  behavior for `Box`.
- Existing generated sanitize and validation visitor suites pass, and focused
  all-feature Clippy validation passes with warnings denied.

### 📊 Audit

- Production code removes eight recursive container hook bodies. The touched
  Rust file has 124 insertions and 56 deletions, a net increase of 68 lines;
  the added regression coverage accounts for the increase while the runtime
  implementation becomes simpler.
- `ten_entity_fluent_rows` remains 2,366,627 raw wasm bytes and 2,204,580
  canonically shrunk bytes, both unchanged from `0.200.3`. Raw non-gzipped wasm
  remains the primary size measure.
- No instruction-count change is claimed. The corrected hooks affect authored
  model sanitization and validation rather than query execution strategy.

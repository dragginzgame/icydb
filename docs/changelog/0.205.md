# IcyDB 0.205 Changelog

## 0.205.2 - 2026-07-19

### 📝 Summary

- Corrects the new grouped scale contract so ordered execution is claimed only
  when the query supplies the accepted index range proof required by the
  current planner.
- Hard-cuts deterministic SELECT generation to version 2 with bounded
  aggregate-filter, membership, range, numeric-function, and cross-clause
  composition coverage. Production SQL behavior is unchanged.

### 🔧 Changed

- Gives ordered age/name scale controls full-domain accepted-index ranges and
  explicit canonical ordering; equivalent hash controls add a residual
  self-comparison over the same logical rows.
- Preserves observed/finalized group counts and peak live group/aggregate-state
  counters in typed grouped-state drift errors.
- Adds generated `COUNT(...) FILTER (WHERE ...)`, `BETWEEN` / `NOT BETWEEN`,
  `IN` / `NOT IN`, numeric `ABS`, computed alias ordering, and bounded scalar
  and grouped clause composition.
- Validates, renders, serializes, replays, and shrinks every new generated AST
  shape under the existing depth, membership, evaluation, and artifact bounds.
- Attributes numeric-function, aggregate-filter, membership, range, alias-order,
  scalar-composition, and grouped-composition evidence from the actual generated
  structure instead of proxy features.
- Adds accepted `select.scalar_composition` and `select.grouped_composition`
  contract cells and updates the required SQLite-reference scenarios, Tier C
  catalog identity, and coverage-manifest revision.

### 🩹 Fixed

- Stops inferring ordered grouped physical state from `ORDER BY` or grouping
  alone when access planning selected no compatible ordered index stream.
- Corrects numeric-function coverage so arithmetic expressions no longer claim
  function evidence unless a numeric function is actually generated.

### 🗑️ Removed

- Removes SELECT generator version 1 as a current replay identity. Version 2 is
  the sole accepted seeded format; no compatibility decoder or dual generation
  path is retained.

### 🧪 Testing

- Adds an exact grouped secondary-range `COUNT(*)` route proof at the page-size
  boundary and keeps ordered/hash scale state validation fail-closed.
- Refreshes deterministic seed, corpus, selection, Tier C catalog, and coverage
  manifest golden identities for the version-2 generator contract.
- Focused SQL performance harness validation passes with 80 tests and 14
  intentionally ignored live/scheduled tests. Full repository and workspace
  suites remain user-owned push validation.

### 📊 Audit

- No production or canister implementation changed, so runtime complexity and
  expected raw Wasm size remain neutral. A fresh workflow subject and comparable
  performance cohort are still required after publication because evidence is
  bound to the exact source revision.
- SQL-generator source grows to represent previously absent bounded AST shapes;
  it does not add a second generator, replay lane, or compatibility format.

### 📚 Status

- SQL Performance Evidence run `29688890917` completed all eight P1 shards but
  rejected every scale shard at its first incorrectly declared ordered-state
  sentinel. It is diagnostic evidence only and is not a calibration cohort
  member.
- After 0.205.2 lands, closeout restarts with a fresh ordinal one, ordinals two
  and three reusing that exact subject, and the strict three-run cohort review.

## 0.205.0 - 2026-07-19

### 📝 Summary

- Replaces the ordered grouped path's full group table with one active group
  that is finalized at each canonical key transition.
- Keeps `HashMaterialized` as the sole fallback for unproven ordering,
  aggregate-ranked output, residual filtering, and grouped field-target
  `DISTINCT`.
- Preserves public query, SQL, cursor, schema, persisted-format, and response
  contracts.

### 🔧 Changed

- Makes the final route-selected ordered-streaming decision own physical
  secondary-index leaf preservation, so execution cannot reorder a proven
  contiguous group stream into primary-key order.
- Applies aggregate finalization, `HAVING`, cursor filtering, offset, projection,
  limit, and lookahead incrementally as ordered groups close.
- Allows bounded canonical pages to stop scanning after the qualifying
  lookahead group proves page finality.
- Moves dedicated ordered `COUNT(*)` onto the shared group-transition owner
  while retaining its direct key extraction and counter specialization.
- Reports executor-owned rows scanned, groups observed and finalized, peak live
  group/aggregate/distinct state, and early-stop facts through SQL and fluent
  diagnostics.
- Extends the maintained SQL performance profile with exact ordered/hash pairs
  at 16, 256, and 2,048 rows and gives grouped aggregate evidence one neutral
  route-family classification.

### 🩹 Fixed

- Prevents secondary-index candidate canonicalization from violating ordered
  group contiguity when primary-key order differs from group-key order.
- Refreshes the canonical fixture-profile identity after the intentional 0.205
  grouped scale-profile and fixture-distribution change.
- Makes the read-admission documentation invariant validate the correct relative
  Foundations link instead of requiring repository-root text in a nested
  document.

### 🗑️ Removed

- Deletes the obsolete ordered-materialized execution mode, EXPLAIN label,
  metrics spelling, and performance-harness classification without aliases or
  compatibility translation.
- Removes full-table ordered state retention and the dedicated count path's
  ordered all-group bucket allocation.

### 🧪 Testing

- Focused owner tests prove transition finalization, released live-state
  accounting, bounded lookahead stop, and fail-closed rejection of
  non-monotonic group keys.
- Route and accepted-schema SQL tests prove ordered/hash family parity,
  direction handling, `HAVING`, filters, pagination, cursors, explicit hash
  fallback, and adversarial secondary-index ordering.
- SQL and fluent attribution tests prove that diagnostics project the physical
  execution facts rather than reconstructing route intent.
- Focused PocketIC evidence returns identical ordered/hash results. At 16, 256,
  and 2,048 fixture rows, the ordered route scans 3, 33, and 257 rows while the
  hash controls scan 16, 256, and 2,048 rows; ordered peak live aggregate state
  remains one.
- Full repository and workspace suites remain user-owned push validation.

### 📊 Audit

- The maintained 0.205 implementation, test, evidence, and active-documentation
  scope adds approximately 2,400 net lines across about 66 files. Most growth is
  route, semantic-parity, diagnostics, and performance-contract proof rather
  than an additional runtime abstraction.
- Runtime ownership is simpler: one shared ordered transition owner replaces
  generic and count full-table retention, while one explicit hash route owns
  materialized grouping.
- The current measured raw non-gzipped SQL performance Wasm is 3,914,682 bytes,
  18,965 bytes above the accepted 0.204.16 subject. Concurrent dependency-lock
  drift means that observation is not an isolated route-slice attribution.

### 📚 Status

- 0.205.0 lands the ordered-streaming implementation and focused physical
  evidence. It does not claim full 0.205 design closeout.
- Multi-sample comparable correctness/performance evidence and a settled-lock
  raw Wasm measurement remain required before the 0.205 design can close.
- The release also records the completed 0.204 evidence line: three
  deterministic Tier C runs, a reviewed exact-subject Tier D cohort, and a
  passing ordinary performance gate.

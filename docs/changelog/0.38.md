# IcyDB 0.38

This document aggregates detailed release notes for the `0.38.x` minor line.

---

## 0.38.1 - 2026-03-02 -

### 📝 Summary

`0.38.1` completes hardening of fluent field-target terminal dispatch. The transitional planner-slot-first-with-fallback strategy has been closed out: fluent terminals now require planner slot routing, and legacy string fallback paths are confined to executor-internal test coverage only.

### 🔧 Changed

- Fluent field-target terminals no longer fall back to runtime field-name resolution when planner slot resolution is unavailable.
- Unknown aggregate target fields now fail during fluent slot resolution before execution consumes scan budget.
- Legacy string terminal entrypoints were narrowed to executor-internal `#[cfg(test)]` coverage so parity tests can remain without re-opening production fallback drift paths.
- Added structural/signature guards to pin slot-based executor entrypoints and catch dispatch-boundary drift at compile time.
- Expanded regressions to ensure slot routing is used when available and to preserve fail-before-scan behavior for unknown field targets.

---

## 0.38.0 - 2026-03-02 - Pre-Unified Expr Structural Hardening

### 📝 Summary

`0.38.0` is a cleanup release before unified expressions. It reduces duplicate query logic, makes projection behavior more consistent, keeps error categories stable at the public API boundary, and tightens tests so architecture rules are checked structurally.

### 🔧 Changed

- Projection helpers and aggregate builders now route through the same internal projection construction path.
- Executor paths no longer rebuild alternate projection/aggregate shapes; they consume the planned shape.
- Planner remains the semantic rule owner; executor-side checks are focused on invariants.
- Aggregate fingerprinting is now tied to aggregate semantics only, not alias names or explain-only metadata.
- Runtime errors now preserve class and origin details across query/facade conversion paths.
- Regression tests were expanded for projection path parity, planner/executor guard separation, aggregate fingerprint stability, and runtime error class/origin preservation.
- Architectural guard tests were moved away from source-text scans to structural/type-signature checks.
- `TESTING.md` now explicitly forbids `include_str!` for architectural enforcement and string-based semantic error assertions.

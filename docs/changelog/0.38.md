# IcyDB 0.38

## 0.38.0 - 2026-03-02 - Pre-Unified Expr Structural Hardening

### 📝 Summary

`0.38.0` starts the pre-unified-expression hardening pass by converging
projection shape authority, removing duplicated semantic validation, and
stabilizing runtime classification/origin boundaries.

### 🔧 Changed

- Converged projection shape construction on one semantic spine so fluent
  projection terminals and aggregate builders emit the same canonical
  aggregate/projection shape.
- Removed executor-side alternate aggregate/projection construction paths and
  tightened executor behavior to consume planner-owned semantic shape.
- Kept planner as the single semantic validation authority, and reduced
  executor checks to invariant enforcement (`debug_assert!` /
  invariant errors) without semantic rule re-implementation.
- Isolated aggregate fingerprint ownership so aggregate hash identity depends on
  semantic aggregate shape and remains independent from explain-shape metadata
  and aliases.
- Hardened runtime error boundary conversion by preserving runtime class and
  origin through query/facade conversion, including explicit runtime execute
  typing and expanded origin mapping coverage.
- Added regression locks for projection path unification.
- Added regression locks for planner-vs-executor semantic guard separation.
- Added regression locks for aggregate fingerprint stability across
  helper/builder/alias variants.
- Added regression locks for runtime class and origin preservation at facade
  boundaries.
- Replaced string/source-scan architectural guard checks with structural
  enforcement checks (type/signature contracts and semantic parity assertions).
- Consolidated repository test-policy guidance in `TESTING.md`, including
  explicit prohibition on `include_str!` for architectural enforcement and
  string-based semantic error assertions.

# 0.77 — SQL Contract Freeze

## 0.77.0 - 2026-04-12 - Remove SQL Dispatch And Freeze The Narrow SQL Surface

This patch changes the public SQL story materially:

- removes the public typed `execute_sql_dispatch::<E>(...)` facade
- removes generated canister `sql_dispatch::query(...)`
- removes the generated canister `sql_dispatch` module from build output
- keeps typed/fluent APIs as the only public mutation owners
- keeps reduced SQL as a narrow read/introspection surface plus typed
  `query_from_sql(...)` lowering

What changed:

* removed the public typed dispatch envelope and helpers from:
  * `crates/icydb/src/db/session/mod.rs`
  * `crates/icydb/src/db/sql/mod.rs`
* removed the generated canister SQL query shim from:
  * `crates/icydb/src/db/session/generated.rs`
  * `crates/icydb-build/src/db.rs`
* removed stale facade re-exports and perf-attribution exports that only made
  sense while public dispatch still existed
* narrowed the remaining core dispatch helpers to internal-only visibility so
  `icydb-core` no longer exposes them as part of the public crate surface
* rewrote the current SQL docs so they match the new product direction:
  * typed/fluent APIs are the public mutation surface
  * reduced SQL remains a narrow typed query/introspection surface
  * `INSERT`, `UPDATE`, `DELETE`, and `RETURNING` are no longer public SQL
    execution features
* updated SQL lane rejection messages to stop pointing users toward removed
  dispatch APIs and to point them at surviving typed/fluent helpers instead
* softened planner/explain contract tests so they keep proving deterministic
  semantic choice without freezing exact implementation narration:
  * `crates/icydb-core/src/db/session/tests/verbose_route_choice.rs`
  * `crates/icydb-core/src/db/query/intent/tests/verbose.rs`
  * `crates/icydb-core/src/db/query/plan/planner/tests/mod.rs`
  * `crates/icydb-core/src/db/query/explain/tests/descriptor.rs`
  * `crates/icydb-core/src/db/query/explain/tests/aggregate.rs`
* replaced exact chosen-index names, rejected-candidate inventories, render
  snapshots, and broad execution-descriptor bookkeeping assertions with the
  narrower contract that actually matters:
  * chosen access family such as `IndexPrefix(...)` or `IndexRange(...)`
  * stable `order_compatible_preferred` reasoning where order compatibility is
    the semantic tie-break
  * root route family
  * covering versus materialized behavior
  * fallback reason where it changes grouped semantics
  * ordered-route versus materialized-sort behavior
  * `TopNSeek`, `OrderByAccessSatisfied`, `OrderByMaterializedSort`, and
    grouped aggregate node-family presence
  * `existing_row_mode`, `fetch`, and logical `offset` where those fields carry
    real semantic meaning
* moved one reduced-SQL authority-label helper from canonical JSON snapshots to
  typed execution descriptors:
  * `crates/icydb-core/src/db/session/tests/authority_labels.rs`
  * `crates/icydb-core/src/db/session/tests/mod.rs`
  so the reduced-SQL descriptor tests now prove covering/materialized route
  semantics directly and use rendered JSON only for the actual outward
  “removed authority labels stay gone” surface check
* deleted the stale facade/parity/perf files that only existed to exercise the
  removed dispatch surface:
  * `crates/icydb/src/db/session/tests.rs`
  * `canisters/test/sql_parity/src/tests.rs`
  * `canisters/test/sql_parity/src/perf.rs`
  * `canisters/demo/rpg/src/perf.rs`

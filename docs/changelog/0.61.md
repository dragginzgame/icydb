# IcyDB 0.61

This document aggregates detailed release notes for the `0.61.x` minor line.

Patch numbering in active work follows the next release after the workspace
version in `Cargo.toml`. With the workspace currently at `0.61.1`, the active
follow-up work in this line belongs to `0.61.2`.

## 0.61.2 - 2026-03-20 - Shared Container Conversion Helpers And Audit Matrix Reset

### Summary

The third `0.61` patch moves another repeated generated-body family onto one
shared runtime helper seam and resets the wasm audit fixture matrix around
shared helpers under `testing/`.

This slice targets repeated generated container conversion for list, set, map,
and newtype wrapper types. Instead of re-emitting the same collection and map
conversion loops per generated schema type, the derive output now calls shared
helper functions in `traits`.

It also replaces the active wasm audit setup with:

- an empty `minimal` canister that measures the bare database surface with no
  entities
- `one_simple` and `ten_simple` canisters that repeat one thin entity shape
- `one_complex` and `ten_complex` canisters that repeat one richer entity shape
  with embedded records, payload enums, nested collection wrappers, validators,
  sanitizers, and relations

The repeated schema shapes now live behind shared helper macros in
`testing/wasm-fixtures`, so the audit fixtures themselves are thinner and the
measurement matrix is easier to evolve.

### Changed

- Added shared runtime helpers for structured `FieldValue` conversion:
  - `field_value_collection_to_value(...)`
  - `field_value_vec_from_value(...)`
  - `field_value_btree_set_from_value(...)`
  - `field_value_map_collection_to_value(...)`
  - `field_value_btree_map_from_value(...)`
- Added matching shared runtime helpers for generated `From` conversions:
  - `field_value_from_vec_into(...)`
  - `field_value_from_vec_into_btree_set(...)`
  - `field_value_from_vec_into_btree_map(...)`
  - `field_value_into(...)`
- Re-exported those helpers through the `icydb::traits` facade so generated
  code can call them directly without reaching through internal module paths
- Rewrote generated list `FieldValue` impls to use shared runtime helpers:
  - list wrappers now delegate list encode/decode to one shared runtime path
    instead of emitting their own item iteration loop bodies
- Rewrote generated set `FieldValue` impls to use shared runtime helpers:
  - set wrappers now delegate list encode plus duplicate-rejecting set decode
    to one shared runtime path instead of emitting their own loop bodies
- Rewrote generated map `FieldValue` impls to use shared runtime helpers:
  - map wrappers now delegate canonical key sorting, duplicate-key checks, and
    normalized decode to one shared runtime path instead of re-emitting that
    map conversion logic per generated type
- Rewrote generated list/set/map/newtype `From` impls to use shared runtime
  helpers:
  - list and set wrappers now delegate `Vec<I>` conversion to one shared
    runtime path instead of re-emitting `into_iter().map(Into::into).collect()`
    per type
  - map wrappers now delegate pair conversion to one shared runtime path
    instead of re-emitting the key/value conversion body per type
  - newtype wrappers now delegate `Into<T>` conversion to one shared runtime
    path instead of re-emitting the same single-field conversion body per type
- Added one shared wasm-audit fixture helper crate:
  - new crate `testing/wasm-fixtures` owns the repeated simple and complex
    entity-shape macros used by the active wasm audit canisters
  - simple and complex schema fixtures now reuse that shared owner instead of
    carrying the repeated entity-shape definitions in each schema crate
- Rebuilt the active wasm audit schema matrix:
  - `schema/minimal` now defines an empty store-only audit canister with no
    entities
  - new `schema/one_simple` and `schema/ten_simple` crates define the thin
    repeated entity-shape fixtures
  - new `schema/one_complex` and `schema/ten_complex` crates define the richer
    repeated entity-shape fixtures
- Rebuilt the active wasm audit canister matrix:
  - new `canisters/one_simple`, `canisters/one_complex`, `canisters/ten_simple`,
    and `canisters/ten_complex` crates now build those schema fixtures through
    the same generated SQL actor path as the existing audit canisters
  - PocketIC build wiring, DFX canister metadata, and the wasm size/audit
    scripts now recognize `minimal`, `one_simple`, `one_complex`,
    `ten_simple`, and `ten_complex` as the active default audit matrix

## 0.61.1 - 2026-03-20 - Sanitize, Validate, And Schema Compression

### Summary

The second `0.61` patch extends the shared macro-compression seam beyond model
and traversal output.

This patch:

- lifts generated entity and record sanitization onto shared runtime field
  descriptor loops
- lifts generated entity and record validation onto the same runtime seam
- reshapes schema AST emission so repeated field/index/validator/sanitizer
  slices are lowered through local const tables
- remeasures the controlled `minimal` / `twenty` wasm pair to check whether
  those early compression slices are paying off

### Changed

- Lifted entity and record sanitization onto the runtime field-table seam:
  - added `SanitizeFieldDescriptor<T>` plus shared
    `drive_sanitize_fields(...)` helpers in the core visitor module
  - generated entity and record `SanitizeAuto` impls now build one static
    descriptor table and run one shared loop over it instead of emitting inline
    per-field issue-path blocks
- Lifted entity and record validation onto the same runtime seam:
  - added `ValidateFieldDescriptor<T>` plus shared
    `drive_validate_fields(...)` helpers in the core visitor module
  - generated entity and record `ValidateAuto` impls now build one static
    descriptor table and run one shared loop over it instead of emitting inline
    per-field issue-path blocks
- Removed now-dead derive helpers left behind by the loop lift:
  - `FieldWalkBinding` no longer carries the old `self_ref` / `self_mut`
    helpers once entity/record field-walk generation moved fully onto
    receiver-based descriptor helpers
- Compressed generated schema AST emission:
  - `FieldList::schema_part()` now lowers field slices through one local
    `__FIELDS` const table before constructing `schema::node::FieldList`
  - `Item::schema_part()` now lowers validator and sanitizer slices through
    local `__VALIDATORS` / `__SANITIZERS` const tables before constructing
    `schema::node::Item`
  - `Entity::schema_part()` now lowers index slices through one local
    `__INDEXES` const table before constructing `schema::node::Entity`
  - together, these changes reshape generated schema constants away from one
    larger nested literal tree and toward a more table-oriented constant form

### Validation

- `cargo fmt --all`
- `cargo check -p icydb-schema-derive -p icydb-core -p icydb`
- `cargo test -p icydb-schema-tests --locked`
- `cargo clippy --workspace --all-targets -- -D warnings`
- `bash scripts/ci/wasm-size-report.sh`

### Measurement

A fresh `minimal` / `twenty` `wasm-release` measurement was recorded after
these early `0.61` slices.

- `minimal` shrunk wasm: `1,768,823` bytes
- `twenty` shrunk wasm: `3,413,770` bytes
- shrunk wasm delta: `1,644,947` bytes
- added entities: `19`
- estimated shrunk wasm per added entity: `86,576` bytes
- estimated shrunk gz wasm per added entity: `2,841` bytes

The honest read is that these early macro-compression slices did not yet move
wasm size meaningfully. They reduced generated structure and established a
cleaner runtime-loop seam, but the remaining repeated per-entity output is
still large.

### Next Step

- re-scan the remaining generated per-entity emitters now that traversal,
  sanitize, and validate all share one runtime loop seam
- decide whether `FieldValue`, deeper schema/model duplication, or another
  generated metadata family is now the next larger source of per-entity growth

## 0.61.0 - 2026-03-20 - Initial Entity Macro Compression

### Summary

The opening `0.61` patch starts the next wasm-focused macro cleanup after
`0.60` removed generated view DTOs.

This patch does not attempt a full descriptor-runtime rewrite. Instead, it
takes the first smaller structural compression steps that match the current
macro code:

- entity runtime model emission no longer fans out into one generated
  `__MODEL_FIELD_*` constant per field
- generated entity and record traversal now runs through a shared
  field-descriptor loop instead of spelling out per-field `drive` bodies
  inline
- visit/sanitize/validate generation now shares one derive-side field-walk
  lowering path instead of each module rebuilding that field metadata shape on
  its own

### Changed

- Reduced runtime model constant fanout in entity macro output:
  - entity generation now emits one direct `__MODEL_FIELDS` array instead of
    one per-field `__MODEL_FIELD_*` constant plus an array wrapper
  - runtime `FieldModel` lowering now lives behind one derive-side helper in
    the inherent-model path
  - this keeps the emitted entity-model authority the same while shrinking the
    generated constant surface
- Lifted entity and record traversal onto a runtime field-descriptor loop:
  - added `VisitableFieldDescriptor<T>` plus shared
    `drive_visitable_fields(...)` helpers in the core visitor module
  - generated entity and record `Visitable` impls now build one static
    descriptor table and run one shared traversal loop over it
  - field-specific traversal still uses typed generated helper functions per
    field, but the repeated inline `drive` call-site bodies are gone
- Introduced one shared derive-side field-walk lowering layer:
  - added a shared field-walk helper module used by entity/record generation
  - `visitable`, `sanitize`, and `validate` now consume that shared lowering
    path instead of each rebuilding field-name/path/member-access metadata
    independently
  - this is primarily a macro-output and maintainability cleanup, but it also
    establishes the exact seam needed for later runtime-loop work
- Marked the `0.61` design line active:
  - the `0.61` design doc now reflects the post-`0.60` reality and is marked
    `In Progress`

### Validation

- `cargo fmt --all`
- `cargo check -p icydb-schema-derive -p icydb-core -p icydb`
- `cargo test -p icydb-schema-tests --locked`
- `cargo clippy --workspace --all-targets -- -D warnings`

### Measurement

No fresh wasm-size rerun was recorded for this opening patch.

The intent of this first slice was to reduce generated macro surface and land
the runtime-loop seam before measuring again.

### Next Step

- lift sanitize and validate onto the same runtime descriptor seam as
  traversal
- then run a fresh `minimal` / `twenty` measurement to see whether the field
  loop family is actually paying off

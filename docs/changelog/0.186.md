# IcyDB 0.186 Changelog

## 0.186.0

### 📝 Summary

- Starts the shared query filter authority line.
- Promotes the tentative 0.186 design into an active baseline and records the
  current SQL/fluent filter authority map before moving runtime code.
- Adds source guards for the current predicate-subset derivation seams and
  post-access residual-contract creation seam.
- Extends SQL/fluent canonical predicate parity coverage across negated
  membership, `IS NOT NULL`, and negated boolean composition.
- Leaves SQL/fluent query semantics, route choice, cursor format, cache
  identity, EXPLAIN shape, count/cardinality shortcuts, result payloads, and
  persistence unchanged.

### 🧪 Testing

- Added structural source-guard tests for pre-access predicate subset
  derivation and finalized residual filter contract creation.
- Added SQL/fluent convergence cases to the lowering parity matrix.

### 📚 Documentation

- Added the active 0.186 changelog.
- Updated the 0.186 shared-filter status and source map to record the first
  guard-baseline slice.

### 🧭 Migration Notes

- No schema migration is required.
- No SQL syntax, fluent API, result payload, cursor format, persistence format,
  public endpoint behavior, route admission, page-window behavior, cache key,
  EXPLAIN payload, or count/cardinality behavior is intended to change.

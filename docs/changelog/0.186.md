# IcyDB 0.186 Changelog

## 0.186.2

### 📝 Summary

- Makes predicate-only query filters an explicit query-intent semantic
  authority mode instead of representing them with an internal placeholder
  boolean expression.
- Keeps expression-backed filters as the owner of visible filter expression
  identity for logical planning, cache identity, residual filtering, and
  EXPLAIN.
- Removes the EXPLAIN-only fallback that derived residual predicate DTOs from
  residual expressions, so residual predicate diagnostics now come only from
  finalized residual predicate contracts.

### 🧪 Testing

- Added predicate-only authority coverage proving predicate-only filters keep
  predicate access-planning identity and do not expose a logical filter
  expression.
- Rechecked direct COUNT cardinality coverage for predicate-only and
  expression-backed filter shapes.
- Updated EXPLAIN coverage so expression-owned searched-CASE residuals remain
  on the residual expression surface, while predicate-backed residuals expose
  a residual predicate only when planning retained one.
- Tightened the predicate-derivation source guard now that EXPLAIN no longer
  derives predicate facts from expressions.

### 📚 Documentation

- Updated the 0.186 status and source map with the explicit semantic-authority
  baseline and the finalized residual-predicate-only EXPLAIN rule.

### 🧭 Migration Notes

- No schema migration is required.
- No SQL syntax, fluent API, result payload, cursor format, persistence format,
  public endpoint behavior, route choice, cache key, or count/cardinality
  behavior is intended to change.
- EXPLAIN residual predicate diagnostics are stricter: residual predicate
  output is now shown only when the finalized plan carries a residual
  predicate contract, not when an expression merely looks predicate-like.

## 0.186.1

### 📝 Summary

- Names predicate-subset coverage as a pre-access query-intent fact.
- Distinguishes full, partial, and absent coverage over user-visible filter
  semantics without changing the logical-planning projection consumed by
  existing route, residual, count/cardinality, cache, and EXPLAIN code.
- Moves ordinary SQL SELECT and DELETE filters toward expression-owned
  authority while preserving schema-canonicalized predicate mirrors where SQL
  literal coercion is required for strict indexed planning.
- Keeps UPDATE and global aggregate predicate-admission paths explicit and
  fail-closed rather than widening SQL admission.

### 🧪 Testing

- Added source guards for the remaining SQL predicate-admission and
  expression-plus-predicate handoff sites.
- Added direct COUNT cardinality coverage proof for predicate-only,
  fully-covered expression-backed, and residual-expression filter shapes.
- Added cache-identity proof that expression-plus-predicate handoffs remain
  keyed by visible filter semantics rather than predicate mirrors.
- Preserved the SQL covering-index metadata query route after the SELECT
  filter-authority cleanup.

### 📚 Documentation

- Updated the 0.186 status and source map with the coverage fact, SQL
  expression-owned handoff, strict SQL exceptions, count/cardinality, cache,
  and EXPLAIN proof boundaries.

### 🧭 Migration Notes

- No schema migration is required.
- No SQL syntax, fluent API, result payload, cursor format, persistence format,
  public endpoint behavior, cache key, EXPLAIN payload, or count/cardinality
  behavior is intended to change.

## 0.186.0

### 📝 Summary

- Starts the shared query filter authority line.
- Promotes the tentative 0.186 design into an active baseline and records the
  current SQL/fluent filter authority map before moving runtime code.

  post-access residual-contract creation seam.

  membership, `IS NOT NULL`, and negated boolean composition.

  identity, EXPLAIN shape, count/cardinality shortcuts, result payloads, and
  persistence unchanged.

### 🧪 Testing

- Added SQL/fluent convergence cases to the lowering parity matrix.

### 📚 Documentation

# 0.60 Changelog

## 0.60.0 - 2026-03-20 - Generated View Removal

### Summary

The opening `0.60` patch removes the generated view-style DTO system and makes
entity types the direct public read/write surface.

The important behavior change is simple:

- generated `EntityView`, `EntityCreate`, and `EntityUpdate` types are gone
- entity structs now act as the direct DTO surface
- the old internal patch/view layer is gone instead of being left behind as
  dead compatibility machinery

This is a deliberate breaking change. It is an API and macro simplification
slice, not an execution-engine slice.

### Changed

- Removed generated view-style DTO families from schema macro output:
  - entity macros no longer emit separate generated `View`, `Create`, or
    `Update` companion types
  - generated entities now expose their fields directly and derive
    `CandidType`
  - the derive generator no longer carries dead `view/create/update` emission
    scaffolding
- Removed the public facade surface built around those generated DTOs:
  - `icydb` no longer re-exports `Create`, `Update`, `AsView`, or `UpdateView`
    facade traits and aliases
  - entity-facing session/write surfaces now operate on entities directly
    instead of parallel DTO wrappers
  - public error taxonomy no longer advertises patch/update-specific variants
    tied to the removed view/update model
- Removed the internal patch/view layer from `icydb-core`:
  - deleted the old `patch` module family and the internal `traits::view`
    contract layer
  - removed wrapper-type `AsView` impls that only existed to support the old
    DTO conversions
  - deleted stale fixture helpers and macro tests that existed only to exercise
    generated patch/update DTOs
- Cleaned the remaining live naming and documentation fallout:
  - live crate docs, session docs, derive docs, and type comments were updated
    so they no longer describe “view surfaces” or “update views”
  - aggregate/session tests now use entity-oriented response helpers and names
    instead of the removed `.views()` vocabulary
  - the `0.60` design doc is now marked `In Progress`

### Validation

- `cargo check -p icydb-schema-derive -p icydb-core -p icydb`
- `cargo test -p icydb-core --lib --locked`
- `cargo test -p icydb --lib --locked`
- `cargo test -p icydb-schema-tests --locked`
- `cargo clippy --workspace --all-targets -- -D warnings`

### Measurement

No fresh wasm-size rerun was recorded as part of this patch.

The intent of `0.60` is to make that next measurement more meaningful by
removing the generated view/create/update surface entirely before the next
per-entity wasm comparison.

### Next Step

The next useful follow-up in `0.60` is not more patch-layer deletion. The main
remaining work is closeout:

- remeasure wasm size after the DTO removal settles
- decide whether any remaining “view” terminology is still semantically correct
  structural language or stale API wording
- finish the release cut once the line has its post-removal wasm measurement

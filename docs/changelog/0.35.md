# IcyDB 0.35

This document aggregates detailed release notes for the `0.35.x` minor line.

---

## 0.35.0 - 2026-03-01 - Grouped Fluent Surface and Layer Integrity Hardening

### 📝 Summary

0.35.0 finalizes grouped fluent query ergonomics over the grouped runtime introduced in 0.33, and hardens layer ownership across planning, access normalization, and executor routing.

Status tracking:
[docs/status/0.35-group-by-status.md](../status/0.35-group-by-status.md)

### 🔧 Changed - Grouped Fluent Surface

- Added grouped fluent-query entrypoints on load queries (`group_by`, grouped aggregate terminals, grouped hard-limit overrides, and grouped execution with continuation support) across core and facade session APIs.
- Added grouped fluent id-extrema terminals (`group_min`, `group_max`) and covered declaration-order output semantics in grouped execution tests.
- Added grouped fluent terminal behavior coverage for `group_exists`, `group_first`, and `group_last`, including per-group output shape and per-group value assertions.

### 🔧 Changed - Grouped Execution Guardrails

- Added grouped fluent execution coverage for scalar-shape rejection and grouped continuation pagination behavior.
- Allowed grouped pagination planning without explicit `ORDER BY` by relying on canonical grouped-key ordering, while keeping scalar unordered-pagination rejection intact.
- Added explicit intent-level misuse errors for grouped query shapes on scalar fluent paths (`.page()` / `.execute()`), requiring `execute_grouped(...)` at the API boundary.
- Added explicit grouped fluent `group_min_by` / `group_max_by` deferral errors in grouped v1 so field-target extrema rejection is surfaced at intent construction time.

### 🔧 Changed - Layer Ownership and Structure

- Split the `query::plan` hub into focused modules (`model`, `model_builder`, `semantics`, `access_plan`) with `mod.rs` reduced to wiring and re-exports.
- Kept logical-model construction helpers separate from access-coupled builders so model-only plan assembly remains access-agnostic.
- Moved secondary ORDER BY pushdown feasibility ownership into `executor::route`, and removed query-layer physical routing dependency on executor internals.
- Moved logical plan-shape ownership (`ORDER` shape, delete-limit coupling, unordered-pagination policy) into `query::plan::validate`, leaving `query::policy` focused on cursor paging requirements.
- Moved grouped logical-spec compatibility checks (including grouped field-target rejection in v1) into planner validation ownership; executor aggregate contracts now remain runtime-focused.
- Moved access canonicalization/normalization ownership into `db::access`, including key canonicalization and composite plan normalization.
- Moved runtime direction/fold/probe-hint derivation helpers out of query semantics and into executor routing semantics.

### 📊 Audit - Strict Layer Violation Pass

- Added a recurring strict layer audit result snapshot under:
  [docs/audits/recurring/crosscutting/crosscutting-layer-violation-results.md](../audits/recurring/crosscutting/crosscutting-layer-violation-results.md)
- Current strict result for this release slice: no detected upward dependency violations and no detected ownership leaks in categories A-E.

### 🧹 Cleanup

- Consolidated grouped terminal intent wiring behind a single grouped-terminal append helper so grouped aggregate terminal methods do not duplicate intent mutation logic.
- Consolidated grouped fluent field-terminal mappers (`group_by`, `group_min_by`, `group_max_by`) through one query-mapping path to reduce repetitive builder reconstruction code.

# 0.135.x

## 0.135.0

`0.135.0` introduces bounded text as schema metadata. Text fields can now
declare a write-enforced maximum length through the schema macro item contract:

```rust
field(ident = "name", value(item(prim = "Text", max_len = 255)))
```

The runtime model now represents text as `FieldKind::Text { max_len }`, while
`Value::Text`, persisted UTF-8 bytes, index ordering, and query semantics remain
unchanged. The limit is enforced at field storage validation, including nested
list, set, map, and relation-key shapes, and length is counted as Unicode scalar
values rather than UTF-8 bytes.

What changed:
- added `item(prim = "Text", max_len = N)` parsing and validation in
  `icydb-schema-derive`
- rejects `max_len` on non-text item directives and rejects zero-length bounds
- threads `max_len` through schema nodes, generated `EntityModel` field kinds,
  generated index predicate metadata, and schema fixture coverage
- enforces text length before row persistence without changing storage or index
  byte formats, including save-preflight rejection with `Unsupported` executor
  errors for user-provided text that exceeds the declared bound
- includes bounded text metadata in schema description output and commit schema
  fingerprints because it changes write admissibility
- keeps metadata-free `PersistedRow` text inference unbounded with
  `FieldKind::Text { max_len: None }`

Retained behavior:
- no new runtime value variant is introduced
- existing unbounded text fields behave as before
- reads, predicates, ordering, SQL expression evaluation, cursor behavior, and
  index encoding do not branch on `max_len`
- decimal scale handling remains unchanged

Also included:
- query attribution DTOs derive `Default`, and internal facade/audit helpers use
  default-backed construction so future diagnostics-only timing counters can be
  added without forcing every audit harness initializer to change in lockstep

# IcyDB 0.187 Changelog

## 0.187.9

### 📝 Summary

- Continues the 0.187 Clippy-backed hygiene follow-up.
- Removes mechanical lint suppressions by replacing local `as` casts with
  explicit saturating conversions in executor/query count and hash helpers.
- Renames private access-spec cursor fields so the traversal invariant remains
  clear without a field-name suppression.
- Completes `ExplainPlan` debug projection by including the structured access
  decision instead of suppressing the missing-field lint.
- Keeps remaining production suppressions as intentional API, DTO, diagnostic,
  and owner-local planner shape fences.

### 🧪 Testing

- Rechecked the normal Clippy target, including invariants, feature matrix,
  focused SQL and diagnostics Clippy, and workspace all-target Clippy.
- Rechecked focused SQL EXPLAIN lib coverage for the debug-projection surface.
- Rechecked formatting and whitespace diff validation.

### 📚 Documentation

- Updated the 0.187 audit status, findings ledger, and source map with the
  Clippy-backed hygiene follow-up and remaining-suppression classification.

### 🧭 Migration Notes

- No schema migration is required.
- No SQL syntax, fluent API, result payload, cursor format, persistence format,
  public endpoint behavior, route choice, cache identity, or EXPLAIN semantics
  is intended to change.
- The touched runtime code only makes local conversion intent and private
  cursor state clearer; the query explain debug output gains the already-owned
  access-decision field.

  COUNT/EXISTS prefix-cardinality paths share planner proof and terminal

  route, EXPLAIN, and count/cardinality consumers from deriving predicate facts
  directly from frontend-owned SQL or fluent expressions.

  strategy, resolved ORDER, and SQL write invariant traps to fallible,
  optional, or fail-closed runtime behavior.

# IcyDB 0.187 Changelog

## 0.187.5

### 📝 Summary

- Continues the 0.187 recoverable invariant cleanup.
- Makes affine numeric compare flipping return no flipped operator for
  non-compare drift and preserve the original expression instead of trapping.
- Makes scalar COUNT reducer output keep normal count finalization while
  falling back to the reducer-local count if the finalized value shape ever
  drifts.
- Makes grouped projection aggregate scanning propagate its existing traversal
  error instead of using a local invariant expectation.

### 🧪 Testing

- Added structural guards for affine compare flipping, scalar COUNT reducer
  output, and grouped projection aggregate scanning.
- Rechecked grouped planning, affine SQL/query behavior, structural guards,
  `icydb-core` check, and Clippy with diagnostics and SQL EXPLAIN features.

### 📚 Documentation

- Updated the 0.187 audit status, findings ledger, and source map with the
  affine rewrite, COUNT reducer, and grouped projection scan cleanup.

### 🧭 Migration Notes

- No schema migration is required.
- No SQL syntax, fluent API, result payload, cursor format, persistence format,
  public endpoint behavior, route choice, cache identity, or EXPLAIN semantics
  is intended to change.
- The touched paths only change internal drift behavior: impossible affine
  operator drift preserves the original expression, COUNT finalization-shape
  drift returns the reducer-local count, and grouped aggregate scan traversal
  failures are propagated instead of trapping.

## 0.187.0

### 📝 Summary

- Starts the second query-engine audit after the 0.184 query-engine cleanup,
  0.185 branch-aware routing revisit, and 0.186 shared filter authority work.
- Builds the active audit source map and findings ledger for duplicate-looking
  query-engine authority flows.
- Guards the metadata-count boundary so direct SQL COUNT and prepared
  COUNT/EXISTS prefix-cardinality paths share planner proof and terminal
  execution authority.
- Guards SQL write bounds so UPDATE and INSERT staging stay on the shared
  mutation-batch boundary, while DELETE keeps its post-access projection/count
  boundary before commit.
- Guards scalar execution setup so materialized scalar pages, retained-slot
  pages, and aggregate row sinks share the same execution-input spine.
- Keeps shared filter authority in guard mode by preventing downstream cache,
  route, EXPLAIN, and count/cardinality consumers from deriving predicate facts
  directly from frontend-owned SQL or fluent expressions.
- Converts audited query-expression, grouped EXPLAIN/fingerprint, grouped
  strategy, resolved ORDER, and SQL write invariant traps to fallible,
  optional, or fail-closed runtime behavior.

### 🧪 Testing

- Added source guards for metadata-count authority, SQL write bounds, scalar
  execution-input convergence, and downstream filter-authority consumers.
- Validated generated SQL canister parity for query, update, bounded update,
  DDL, numeric, aggregate, grouped aggregate, diagnostics, returning, and
  error paths.
- Rechecked focused query-engine coverage around route, COUNT/EXISTS,
  projection, EXPLAIN, grouping, and SQL write behavior.

### 📚 Documentation

- Added the 0.187 second query-engine audit status, source map, and findings
  ledger.
- Recorded deferred items for cost/selectivity optimization, cursor redesign,
  chunked durable commits, aggregate operator DTO work, streaming DELETE
  collection, and lint-suppression hygiene.

### 🧭 Migration Notes

- No schema migration is required.
- No SQL syntax, fluent API, result payload, cursor format, persistence format,
  public endpoint behavior, route admission, page-window behavior, cache key,
  EXPLAIN payload, or count/cardinality behavior is intended to change.
- The runtime invariant cleanup only changes audited internal drift behavior:
  touched paths now return typed errors, produce no diagnostic projection, or
  choose conservative fallback behavior instead of trapping.

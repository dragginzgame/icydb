# IcyDB 0.46

This document aggregates detailed release notes for the `0.46.x` minor line.

## 0.46.8 - 2026-03-09 - Standards Alignment Continuation

### Summary

`0.46.8` continues standards-alignment with terminal API parity, temporal
return-shape locks, and predicate pushdown/diagnostic stability expansion.

### Changed

- Aligned scalar emptiness/existence terminals across core and facade:
  - `db::query::fluent::load::FluentLoadQuery::not_exists()`
  - `is_empty()` now routes through this alias boundary.
- Added explicit `EXPLAIN` alias parity for NOT EXISTS:
  - `FluentLoadQuery::explain_not_exists()` now aliases
    `FluentLoadQuery::explain_exists()`.
  - regression locks require identical explain descriptors for both.
- Expanded aggregate/session terminal parity coverage:
  - parity assertions now include `exists()`, `not_exists()`, and `is_empty()`
    on the same scalar window.
  - added a scan-budget lock that asserts all three terminals preserve the same
    early-stop contract (`offset + 1` keys scanned on non-empty ordered windows).
- Updated `icydb` facade load/delete terminal wrappers:
  - `is_empty()` and `count()` now delegate directly to core terminal helpers
    instead of materializing through `execute()`.
  - facade scalar load queries now expose `exists()` and `not_exists()`
    convenience terminals.
- Extended temporal boundary coverage:
  - added `Date` semantic-view boundary lock (`Date::as_view()` /
    `Date::from_view(...)`).
  - added `Date` serde wire-shape lock (`"YYYY-MM-DD"`).
  - added one temporal session matrix lock over runtime boundaries:
    `Date`/`Timestamp`/`Duration` remain typed in both `views()` payloads and
    `values_by(...)` scalar projection paths.
- Expanded predicate pushdown contracts and stability:
  - planner now lowers impossible `primary_key IS NULL` directly to
    `AccessPath::ByKeys([])`.
  - `AccessPlan::intersection(...)` treats explicit empty `ByKeys([])` as an
    annihilator; `AccessPlan::union(...)` treats explicit empty `ByKeys([])` as
    an identity when non-empty branches exist.
  - verbose execution explain now emits
    `diagnostic.plan.predicate_pushdown=...` labels (`none`, `applied(...)`,
    `fallback(...)`) with explicit reason paths:
    `fallback(is_null_full_scan)`,
    `fallback(non_strict_compare_coercion)`, and
    `fallback(starts_with_empty_prefix)`.
  - added parity locks requiring equivalent predicate classes to keep reason and
    route-stage diagnostics aligned:
    - `primary_key IS NULL` vs strict `primary_key IN []`
    - canonical-equivalent `IN` sets
    - equal-bounds `BETWEEN`-style predicates vs strict equality
  - non-primary `IS NULL` remains on safe full-scan fallback until nullable
    index pushdown contracts are introduced.
- Updated `0.46` design tracking with these additional locks and behavior notes.

## 0.46.7 - 2026-03-08 - Boundary Naming and Visibility Tightening

### Summary

`0.46.7` tightens executor/internal visibility surfaces, removes one remaining
ambiguous query-facing error type name, and updates the layer-authority metric
to measure duplicated decision ownership rather than repeated accessor tokens.

### Changed

- Updated layer-authority complexity auditing:
  - `scripts/ci/check-layer-authority-invariants.sh` now tracks
    cross-layer predicate duplication using decision-owner function declarations
    (for example `fn ...`) instead of field/accessor token reuse.
  - this removes false-positive duplication drift from passive contract
    propagation and keeps the metric focused on policy re-derivation risk.
  - added explicit complexity-accretion baselines for decision-owner counts:
    - `AccessPath decision owners`
    - `RouteShape decision owners`
    - `Predicate coercion owners`
  - CI now fails when those counts drift above baseline, forcing intentional
    ownership review at the boundary where complexity expands.
- Tightened remaining executor core visibility from `pub(crate)` to
  `pub(in crate::db)` where ownership is db-internal:
  - `ExecutorPlanError`
  - `Context` (and db-internal context helpers)
  - `DeleteExecutor`
  - `ExecutablePlan`
  - aligned caller method visibility in `db/mod.rs` and `db/session.rs` to
    preserve private-interface correctness under `-D warnings`.
- Renamed query intent execution error taxonomy for boundary clarity:
  - `db::query::intent::ExecutionError` -> `QueryExecutionError`
  - updated root re-exports and downstream `icydb` facade mapping accordingly.
- Consolidated route-shape ownership in staged route planning:
  - `RouteIntentStage` now carries `route_shape_kind` as intent-owned state.
  - execution-stage derivation consumes this staged value directly instead of
    re-deriving shape from grouped/aggregate flags.
  - reduced route-shape decision-owner count baseline from `6` to `5`.

### Testing

- `bash scripts/ci/check-layer-authority-invariants.sh`

## 0.46.6 - 2026-03-08 - Planner Stability Authority Consolidation

### Summary

`0.46.6` centralizes query-plan stability helpers under one planner-owned
module and applies explicit `IN` literal canonicalization before access-shape
construction to reduce stability logic drift.

### Changed

- Added a dedicated planner stability helper module:
  - `db/query/plan/stability.rs`
  - owns canonical `IN (...)` literal normalization (`sort + dedup`)
  - owns planner -> access canonical normalization handoff via
    `normalize_planned_access_plan_for_stability(...)`
- Updated planner compare lowering to consume stability-owned `IN` canonicalization:
  - primary-key strict `IN` now builds from canonicalized key sets
  - secondary strict `IN` multi-lookup planning now receives canonicalized
    literal sets before index compatibility filtering
- Updated planner orchestration to route final access-plan normalization through
  the new stability helper instead of directly calling access canonicalization.
- Renamed route-level execution mode enum for boundary clarity:
  - `db::executor::route::ExecutionMode` -> `RouteExecutionMode`
  - updated route planner/runtime consumers and route matrix assertions to use
    the route-owned name and avoid ambiguity with access execution-contract
    `ExecutionMode`
- Added planner stability helper unit coverage:
  - canonical `IN` set normalization is order/duplicate-insensitive
  - planner stability normalization remains idempotent

## 0.46.5 - 2026-03-08 - Numeric Streaming Fold Hardening and Top-N Guard Tightening

### Summary

`0.46.5` extends scalar numeric aggregate streaming coverage for safe shapes,
adds regression locks for canonical deduped key-set windows and duplicate-risk
fan-out fallbacks, and makes Top-N hint derivation enforce deterministic
ordered contracts directly.

### Changed

- Expanded numeric field aggregate streaming fold (`sum_by`, `avg_by`) under
  conservative eligibility:
  - no residual predicate
  - no `DISTINCT`
  - single-path-safe access kinds only
  - ordered paged windows allowed only when `ORDER BY` is exactly primary key
    (ASC or DESC) on PK-safe shapes
- Added one explicit numeric streaming optimization marker:
  - `ExecutionOptimizationCounter::NumericFieldStreamingFoldFastPath`
- Added numeric streaming fold regression coverage for:
  - hit-path activation on unpaged and PK-ordered paged windows
  - hit-path activation on key-direct (`by_id`) windows, including paged
    `ORDER BY id + LIMIT` shape
  - hit-path activation on canonical deduped key-set (`by_ids`) windows
  - fail-closed fallback on non-PK ordered paged windows and predicate-bearing
    shapes
  - fail-closed fallback on index multi-lookup fan-out shapes (explicit
    `IndexMultiLookup` access-shape lock)
  - bounded scan-budget behavior for PK-ordered paged windows (`offset + limit`)
- Tightened route Top-N hint derivation:
  - `top_n_seek_spec` now explicitly re-checks
    `secondary_order_contract_is_deterministic(...)` before deriving hints
  - non-deterministic ordered shapes fail closed directly at the Top-N hint
    boundary even if indirect capability gates change later
- Expanded planner stability regression coverage for canonical `IN` semantics:
  - secondary-index `IN` predicates now lock permutation/duplicate-value
    canonicalization to one identical index multi-lookup access plan
  - primary-key `IN` predicates now lock permutation/duplicate-value
    canonicalization to one identical by-keys access plan
- Updated `0.46` design tracking with the added numeric fail-closed coverage
  and explicit Top-N deterministic guard status.

### Summary

`0.46.4` hardens ordered pushdown eligibility and continues planner/runtime
contract cleanup by moving one remaining window projection policy to explicit
planner accessors.

### Changed

- Hardened ordered pushdown contract gates:
  - route pushdown derivation now uses one shared planner helper,
    `secondary_order_contract_is_deterministic`, before deriving secondary
    order applicability.
  - index-range `LIMIT` pushdown shape eligibility now requires the same
    deterministic order contract.
  - mixed direction and missing-primary-key tie-break ordered shapes now
    fail closed before Top-N or bounded index-range hint derivation.
- Extended route-matrix regression coverage for ordered index-range shapes:
  - added explicit rejection tests for:
    - missing PK tie-break (`ORDER BY rank`)
    - mixed directions (`ORDER BY rank ASC, id DESC`)
  - incompatible ordered shapes now assert both:
    - no index-range limit pushdown spec
    - no Top-N seek spec
- Simplified scalar planner-to-route window projection API:
  - removed `AccessWindowLookaheadPolicy`.
  - `ScalarAccessWindowPlan` now exposes explicit `keep_count()` and
    `fetch_count()` accessors.
  - route continuation projection consumes those explicit projections directly.
- Improved strict `IN ()` lowering behavior in planner compare planning:
  - non-primary-key strict `IN []` now lowers to `AccessPlan::by_keys(Vec::new())`
    instead of a full-scan fallback.
  - added planner regression coverage that locks this constant-empty lowering.
- Updated `0.46` design tracking with:
  - deterministic ordered-contract guard status
  - index-range `LIMIT` eligibility guard status

  - explicit keep/fetch access-window projection status

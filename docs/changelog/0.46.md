# IcyDB 0.46

This document aggregates detailed release notes for the `0.46.x` minor line.

## 0.46.4 - 2026-03-08 - Deterministic Ordered Pushdown Guarding

### Summary

`0.46.4` hardens ordered pushdown eligibility and continues planner/runtime
contract cleanup by moving one remaining window projection policy to explicit
planner accessors.

### Changed

- Hardened ordered pushdown contract gates:
  - route pushdown derivation now uses one shared planner helper,
    `secondary_order_contract_is_deterministic`, before deriving secondary
    order applicability.
  - index-range `LIMIT` pushdown shape eligibility now requires the same
    deterministic order contract.
  - mixed direction and missing-primary-key tie-break ordered shapes now
    fail closed before Top-N or bounded index-range hint derivation.
- Extended route-matrix regression coverage for ordered index-range shapes:
  - added explicit rejection tests for:
    - missing PK tie-break (`ORDER BY rank`)
    - mixed directions (`ORDER BY rank ASC, id DESC`)
  - incompatible ordered shapes now assert both:
    - no index-range limit pushdown spec
    - no Top-N seek spec
- Simplified scalar planner-to-route window projection API:
  - removed `AccessWindowLookaheadPolicy`.
  - `ScalarAccessWindowPlan` now exposes explicit `keep_count()` and
    `fetch_count()` accessors.
  - route continuation projection consumes those explicit projections directly.
- Improved strict `IN ()` lowering behavior in planner compare planning:
  - non-primary-key strict `IN []` now lowers to `AccessPlan::by_keys(Vec::new())`
    instead of a full-scan fallback.
  - added planner regression coverage that locks this constant-empty lowering.
- Updated `0.46` design tracking with:
  - deterministic ordered-contract guard status
  - index-range `LIMIT` eligibility guard status
  - `IN []` constant-empty lowering status
  - explicit keep/fetch access-window projection status

## 0.46.3 - 2026-03-08 - Executor Boundary Hardening and Route Lint Cleanup

### Summary

`0.46.3` continues `0.46` cleanup by tightening executor visibility boundaries,
locking those boundaries with structural tests, and removing remaining route
planner lint regressions.

### Changed

- Tightened executor boundary visibility for key internal contracts:
  - `LoadExecutor` and `SaveExecutor` are now `pub(in crate::db)`.
  - `ExecutorError` and ordered key-stream internals remain scoped to
    `pub(in crate::db::executor)`.
  - `CursorPage` remains `pub(in crate::db)` because session-level routing reads
    page fields directly.
- Expanded load-structure guard coverage:
  - added regression checks that fail if executor-only key/window/error types
    or executor facade types are widened back to `pub(crate)`.
- Completed executor window contract cleanup:
  - removed legacy `needs_extra` toggling from executor window helpers in favor
    of explicit keep/fetch projections.
  - post-access pagination now consumes shared keep-count math.
- Resolved route planner clippy regressions under `-D warnings`:
  - made `index_range_limit_spec_for_execution_mode` a `const fn`.
  - removed `if !... { None } else { ... }` patterns in feasibility derivation.

## 0.46.2 - 2026-03-08 - Window and Covering Contract Consolidation

### Summary

`0.46.2` continues `0.46` contract cleanup by removing duplicate pagination and
covering-eligibility logic across planner, route, explain, and aggregate
projection paths, and by tightening executor-internal visibility boundaries.

### Changed

- Consolidated scalar window ownership under planner contracts:
  - route window derivation now consumes planner-projected
    `ScalarAccessWindowPlan` directly.
  - executor-side duplicate offset/limit keep-window reconstruction was removed.
- Preserved DISTINCT+offset bounded-probe safety after window refactor:
  - bounded aggregate probe-hint suppression now keys to logical DISTINCT
    intent (`scalar_plan().distinct`) so DISTINCT+offset windows remain
    unbounded where required.
- Expanded covering optimization reuse for `count_distinct_by(field)`:
  - `count_distinct_by` now routes through `distinct_values_by` projection
    paths, enabling existing index-covering and constant-covering fast paths
    for eligible shapes.
- Added planner-owned covering eligibility and order contracts:
  - new `query/plan/covering.rs` now owns:
    - index-covering existing-row terminal eligibility
    - covering projection context/order derivation
    - adjacent-distinct eligibility checks for covering projections
  - route terminal selection and EXPLAIN covering-projection reporting now
    consume the shared planner helper instead of duplicating local checks.
  - aggregate covering projection runtime now consumes planner-owned covering
    order/context contracts via a thin adapter.
- Removed executor-local window toggle branching:
  - `executor/window` now exposes explicit keep/fetch projections without the
    legacy `needs_extra` toggle contract.
  - post-access pagination now consumes one shared keep-count helper.
- Tightened executor-only visibility boundaries and locked them with structural
  guards:
  - narrowed ordered key stream traits/types, key comparator, merge/intersect
    stream adapters, and `ExecutorError` to `pub(in crate::db::executor)`.
  - narrowed `LoadExecutor`/`SaveExecutor` to `pub(in crate::db)`.
  - kept `CursorPage` at `pub(in crate::db)` because `db/session` consumes page
    fields directly.
  - added load-structure guard coverage preventing key/window/error executor
    internals from widening back to `pub(crate)`.

## 0.46.1 - 2026-03-08 - Constant-Row Terminal and EXISTS Early-Stop Coverage

### Summary

`0.46.1` adds a lightweight constant scalar terminal and locks one additional
EXISTS short-circuit contract in the aggregate parity matrix.

### Changed

- Added one session-level constant scalar terminal:
  - `DbSession::select_one()` now returns `Value::Int(1)` directly.
  - This path bypasses query planning, routing, and access-layer execution for
    projection-only constant use cases.
- Added regression coverage for the constant terminal execution boundary:
  - `session_select_one_returns_constant_without_execution_metrics` asserts the
    terminal returns `1` and emits no execution metrics.
- Expanded EXISTS early-stop scan-budget coverage:
  - `aggregate_exists_full_scan_window_scans_offset_plus_one` now locks the
    full-scan offset-window contract to `offset + 1` scanned keys.
- Updated `0.46` design tracking to mark `SELECT 1` and EXISTS tracks as
  actively in progress.

## 0.46.0 - 2026-03-08 - Standards-Alignment Kickoff and Early-Exit Planning

### Summary

`0.46.0` starts the standards-alignment line by landing the first
planner/executor early-exit slices and the supporting stability contracts.

The patch keeps query behavior unchanged while reducing unnecessary work for
zero-row and contradiction shapes.

### Changed

- Added planner-level `LIMIT 0` lowering:
  - `db/query/plan/limit_zero.rs`
  - intent planning now emits an empty by-keys execution shape before access
    routing when the effective fetch window is zero.
- Added planner-level constant predicate folding:
  - `db/query/plan/constant_predicate.rs`
  - canonical constant-true predicates are removed from logical planning input
  - canonical constant-false predicates lower to empty execution shapes.
- Added empty-window aggregate short-circuiting in executor aggregate terminals:
  - shared helpers in `db/executor/aggregate/mod.rs`
  - terminal/numeric aggregate execution now returns SQL-correct zero-window
    outputs without entering scan loops when the window is provably empty.
- Extended predicate normalization with conservative contradiction/tightening
  simplification for `AND` compare clauses:
  - conflicting equality/bound combinations now fold to `FALSE`
  - compatible bound combinations tighten to stricter canonical forms.
- Added plan-stability regression coverage for equivalent canonical predicate
  shapes:
  - equivalent forms are locked to identical access-plan shape
  - contradictory forms are locked to the same constant-false/empty shape.
- Added and updated design/status tracking for the `0.46` line:
  - `docs/design/0.46-standards-alignment.md`

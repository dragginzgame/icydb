# 0.0 Changelog

## 0.0.20 - 2025-12-09 - Delete Window Correctness

### Summary

`0.0.20` closes a correctness issue in delete pagination and reduces duplication in load/delete scan handling.

### Changed

- Fixed `DeleteExecutor` so `offset` and `limit` are applied after filtering.
- Stopped scanning once the requested delete window is satisfied, preventing over-delete behavior and unnecessary allocations on ranged/indexed deletes.
- Extracted a shared query-plan scanning/deserialization helper used by load and delete executors to keep behavior consistent.

---

## 0.0.15 - 2025-12-08 - Enum Payload and Boxed Value Support

### Summary

`0.0.15` expands value-model fidelity and schema/runtime compatibility for enum payloads and boxed collection shapes.

### Changed

- Added payload-aware enum handling: `ValueEnum` now carries payloads, and hashing/equality include payload data.
- Updated enum `FieldValue` implementations to preserve payloads.
- Broadened `FieldValue` support to `Box<T>` and `Vec<T>` for nested/boxed schema values.
- Moved the `build!` macro into `icydb-build` and re-exported it from the meta crate to keep runtime crates free of build-script dependencies.

### Validation

- Added design/runtime coverage for enum payload persistence and boxed-value handling.

---

## 0.0.14 - 2025-12-06 - Canic Dependency Split Follow-Up

### Changed

- Removed the direct canic dependency after the `canic-core` and `canic-memory` split.

## 0.0.6 - 2025-11-27 - Early Type and Sanitizer Expansion

### Changed

- Added early finance type support (`Usd`) in `icydb-base`.
- Added `RoundDecimalPlaces` sanitizer with midpoint default strategy.
- Performed dependency trim via `cargo machete`.
- Fixed `Box<T>` indirect/value-view type alignment.
- Moved `VERSION` to the main crate.
- Made `ValueEnum` path optional to support strict and loose enum matching.
- Made enum matching case-insensitive (for example `"common"` matches `Rarity::Common`).

---

## 0.0.1 - 2025-11-26 - IcyDB Reboot

### Summary

`0.0.1` starts the IcyDB reboot with a new crate surface and aligned naming/docs.

### Changed

- Introduced the `icydb` public meta-crate as the rebooted surface.
- Refreshed docs and guides to use `icydb` tags, endpoints, and examples.
- Updated path resolution to prefer `icydb::` for downstream users while keeping internal crates on direct deps to avoid cycles.
- Aligned observability/query endpoint and codegen naming on the `icydb_*` prefix.

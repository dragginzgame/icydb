# 0.58 Changelog

## 0.58.3 - 2026-03-18 - Optional SQL Surface and Wasm SQL Matrix

### Summary

`0.58.3` adds a semver-safe SQL feature boundary to the completed `0.58.x`
line. The SQL session/query/dispatch surface is now controlled by a
default-on `sql` feature, so non-SQL consumers can compile out the public SQL
APIs and the generated canister `sql_dispatch` glue without reopening the
execution/runtime refactor work that `0.58.0` closed.

This patch also fixes the remaining ownership drift behind that feature
boundary. Standalone predicate parsing now lives under `db::predicate`, while
shared reduced-SQL lexer/token/error infrastructure lives under one neutral
`db::reduced_sql` boundary instead of being parked under predicate or SQL.
With that split in place, schema/index validation consumes predicate-owned
semantics, SQL statement parsing consumes the shared reduced-SQL lexical layer,
and the `db::sql` module itself is fully optional when the `sql` feature is
disabled.

### Changed

- Added a default-on `sql` feature to the public crate boundary:
  - `icydb-core`, `icydb`, and the example canisters now forward one shared
    feature decision for SQL-facing APIs.
  - disabling default features removes the SQL session/dispatch/query surface
    from the public API while preserving the non-SQL typed query/runtime path.
- Repaired the parser/layering boundary behind the feature gate:
  - standalone predicate parsing (`parse_sql_predicate`) now lives under
    `db::predicate`, so schema/index predicate validation consumes
    predicate-owned semantics instead of importing parser authority from
    `db::sql`.
  - shared reduced-SQL tokenization, token-cursor primitives, and parse errors
    now live under `db::reduced_sql`, which both predicate parsing and SQL
    statement parsing depend on.
  - SQL-only field-identifier rewrite helpers were moved out of `db::predicate`
    and back under `db::sql::identifier`, so predicate no longer owns
    SQL-shaped helper code just to keep the feature compiling.
- Gated generated canister SQL glue behind the same feature:
  - generated `sql_dispatch` modules are now emitted behind
    `#[cfg(feature = "sql")]`.
  - canisters built without SQL no longer carry the generated query-lane
    dispatch wrappers or the SQL query entrypoints.
- Tightened the feature boundary after the parser-ownership move:
  - `db::sql` is now compiled only when the `sql` feature is enabled.
  - SQL-only query-intent helpers that were previously left alive for
    type-checking now use real `#[cfg(feature = "sql")]` boundaries again.
- Extended wasm size tooling with an optional SQL matrix mode:
  - `scripts/ci/wasm-size-report.sh` now accepts
    `WASM_SQL_VARIANTS=sql-on|sql-off|both`.
  - `scripts/ci/wasm-audit-report.sh` accepts a single SQL variant and carries
    that selection through artifact naming/report text.
  - the default workflow is unchanged; SQL-on remains the normal path unless a
    size audit explicitly asks for SQL-off or both variants.

## 0.58.2 - 2026-03-18 - Final Cleanup Before 0.59

### Summary

`0.58.2` is a cleanup and hardening patch that closes the `0.58.x` line
without reopening the execution redesign work planned for `0.59`. It removes
dead typed executor scaffolding left behind by the structural de-monomorphizing
work, narrows a few lingering release-build surfaces to test-only code, and
replaces one remaining panic-on-poison cache access in save validation with the
same recover-and-continue strategy already used elsewhere in the workspace.

### Changed

- Removed the dead legacy typed load stage-loop subsystem from the executor
  pipeline:
  - deleted the unused typed load dispatch/payload/state/stage files that were
    no longer part of the production erased-surface execution path.
  - kept the production load orchestrator focused on the monomorphic
    erased-surface route that `0.58.0` established.
  - inlined the stage-order guard labels directly in the guard module so the
    deleted stage planner files are no longer required just to preserve the
    test/invariant contract.
- Hardened save validation around schema-cache poisoning:
  - schema-info cache lookup now recovers via `PoisonError::into_inner` instead
    of panicking if the mutex has been poisoned.
  - this keeps save validation aligned with the rest of the `0.58.x` hardening
    work, where cache/test harness poisoning is treated as recoverable cleanup
    debt rather than a reason to abort the process.
- Tightened release-vs-test code boundaries:
  - several cursor, executable-plan, and scalar entrypoint helpers that only
    exist for tests now use `#[cfg(test)]` instead of living in production code
    behind `allow(dead_code)` attributes.
  - one truly dead cursor helper was removed entirely instead of being kept as
    compatibility clutter.
- Removed a few stale no-op leftovers from the executor tree:
  - a dead no-op access-path helper in the orchestrator strategy context
    module.
  - a stale `unused_imports` expectation around grouped route observability.

### Validation

- `cargo check -p icydb-core --lib --locked`
- `cargo test -p icydb-core --lib --locked`
- `cargo clippy -p icydb-core --lib --tests --locked -- -D warnings`

## 0.58.1 - 2026-03-18 - Release Hardening and SQL Dispatch Consistency

### Summary

`0.58.1` is a release-hardening patch for the completed `0.58` line. It keeps
the structural/runtime work from `0.58.0` intact, but fixes small consistency
and developer-experience gaps that were still visible during final cleanup:
query-lane SQL `DELETE` support now matches the documented unified dispatch
contract, generated code emits fewer avoidable lints, and PocketIC integration
tests no longer fail the whole workspace when the local server binary is not
configured.

### Changed

- Tightened unified SQL dispatch behavior around query-lane execution:
  - `execute_sql_dispatch_query_lane_*` now supports `DELETE` statements
    consistently instead of advertising `DELETE` support while rejecting the
    `QueryMode::Delete` branch at runtime.
  - row-producing query-lane SQL now has one coherent result contract:
    `SELECT` and `DELETE` both return `Projection` payloads through the unified
    dispatch surface.
  - `DELETE` rows returned through unified SQL dispatch are projected in
    declared entity-model field order, preserving stable column labels and
    shell-facing row rendering.
  - core/facade tests were updated so the legacy compatibility helpers
    (`sql_projection_columns`, `execute_sql_projection`) now treat row-producing
    `DELETE` SQL the same way as `SELECT`.
- Hardened developer/test ergonomics for SQL canister integration tests:
  - PocketIC SQL integration tests now degrade cleanly when `POCKET_IC_BIN` is
    not configured instead of panicking immediately and then poisoning the test
    mutex for the rest of the file.
  - this restores `cargo test --workspace --locked` as a reliable local and CI
    workflow in environments that do not provide a PocketIC server binary.
- Continued release-polish cleanup in generated and runtime code:
  - generated SQL/canister glue avoids more avoidable lint-triggering wrappers
    and repeated match arms.
  - generated entity tag construction now stays on the safe
    `EntityTag::new(...)` path rather than relying on macro-emitted `unsafe`
    transmute patterns.
  - grouped DISTINCT runtime cleanup removed remaining unnecessary wrapped
    return values and no-op reducer dispatch paths without changing behavior.

### Validation

- `cargo fmt --all`
- `cargo check --workspace --all-targets --locked`
- `cargo test --workspace --locked`
- targeted `cargo clippy` cleanup on the generated SQL/canister and grouped
  aggregate paths

## 0.58.0 - 2026-03-18 - Runtime Size-Reduction Closeout

### Summary

`0.58.0` closes the runtime size-reduction line by completing the profitable
structural de-monomorphization work, removing the remaining high-value generic
orchestration and adapter duplication, and explicitly stopping once the
remaining WASM size was shown to come from typed execution kernels instead of
more safe local refactors.

### Changed

- Completed the structural de-monomorphization line around runtime identity and
  execution boundaries:
  - `EntityTag` is now the authoritative runtime identity across the hot
    execution path.
  - index identity, cursor continuation state, and runtime routing are now
    structurally keyed instead of being implicitly type-owned.
  - grouped/scalar/traversal orchestration was flattened where that work
    materially reduced duplication.
- Completed the structural plan handoff pivot:
  - `AccessPlannedQuery` is now canonicalized as a structural plan over
    `AccessPlan<Value>`.
  - `ContinuationContract` is structural.
  - `ExecutablePlan<E>` now owns a structural plan plus a typed execution
    sidecar at the boundary.
  - executor preparation and execution-input boundaries now consume the
    structural plan rather than treating typed access plans as planner
    authority.
- Reduced grouped runtime duplication where it paid:
  - grouped continuation and grouped DISTINCT execution were pushed onto
    structural row/runtime surfaces.
  - grouped DISTINCT now uses one structural extraction path and one shared
    numeric finalize authority.
  - grouped projection/finalization and row-view access paths were flattened to
    reduce wrapper and helper duplication.
- Reduced write-path duplication where it paid:
  - commit preparation and save batching no longer duplicate reader-shape or
    wrapper-heavy orchestration bodies.
  - reverse-index preparation remains in the write path, but the remaining
    duplication there was measured as legitimate entity-specific work rather
    than removable wrapper fanout.
- Reduced generator/emitted canister glue noise where it paid:
  - generated SQL route helpers now avoid a number of trivial lint-triggering
    wrappers and repeated match arms.
  - generated entity tag construction no longer uses `unsafe` transmute in
    macro output; codegen now calls the safe `EntityTag::new(...)` constructor.
- Locked the line with measurement, not assumptions:
  - profitable local cuts were kept
  - non-paying SQL/generator/grouped-adapter experiments were reverted
  - the line now explicitly ends at the point where remaining size is dominated
    by typed execution bodies

### Outcome

The final `0.58` position is:

- structural de-monomorphization is sufficiently complete
- redundant generic instantiations at the system boundary have been eliminated
- no further high-confidence local size wins remain in this line

The remaining WASM size is now primarily attributable to:

- typed execution kernels
- `E`-specialized execution bodies

This is treated as architectural debt for the next line, not unfinished cleanup
inside `0.58`.

### Validation

- `cargo check --workspace --all-targets --locked`
- `cargo test -p icydb-core --lib --locked`
- `cargo test -p icydb --lib --locked`

- `cargo check -p canister_demo_rpg --locked`

### Release Framing

`0.58` closes the local optimization phase.

`0.59` is the next architectural phase and will target execution-kernel
de-monomorphization rather than more planner/runtime cleanup.

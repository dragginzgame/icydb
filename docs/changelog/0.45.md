# IcyDB 0.45

This document aggregates detailed release notes for the `0.45.x` minor line.

## 0.45.4 - 2026-03-07 - Fingerprint Determinism and Schema Suffix Lock Extension

### 📝 Summary

`0.45.4` closes another cleanup slice by tightening predicate-fingerprint
determinism around coercion semantics and adding schema compile-fail locks for
relation naming suffix enforcement.

### 🔧 Changed

- Extended predicate literal canonicalization at fingerprint/sort-key
  boundaries so identity follows coercion semantics:
  - `NumericWiden` now canonicalizes numeric-coercible literals (for example
    `Int(1)` and `Decimal(1.0)`) to one fingerprint identity.
  - `TextCasefold` now canonicalizes case-only text literal differences,
    including `IN (...)` list cases.
  - strict coercion remains intentionally subtype- and case-sensitive.
- Extended query fingerprint/continuation signature regression matrices to lock:
  - strict implicit-vs-explicit equivalence
  - numeric-widen literal subtype equivalence
  - text-casefold case-only and `IN`-list equivalence
  - strict-vs-text-casefold and strict-vs-collection-element distinctness
- Added schema-level compile-fail UI tests for relation suffix enforcement:
  - one/optional relation fields must end with `_id`
  - many relation fields must end with `_ids`
  - these locks now exist at external schema usage boundary in
    `icydb-schema-tests`, not only derive-unit coverage
- Updated `0.45` status tracking to mark the fingerprint determinism suite
  complete for this release line.

### 🧪 Testing

- `cargo test -p icydb-core db::predicate::encoding::tests -- --nocapture`
- `cargo test -p icydb-core db::query::fingerprint::fingerprint::tests -- --nocapture`
- `cargo test -p icydb-schema-tests --test compile_fail -- --nocapture`
- `make fmt-check`

## 0.45.3 - 2026-03-07 - Planner and Cursor Stability Follow-Through

### 📝 Summary

`0.45.3` continues the cleanup line with tighter planner boundary checks,
stronger cursor replay invariants, and reduced facade DTO coupling.

### 🔧 Changed

- Tightened facade DTO field boundaries while keeping immutable construction:
  - `PagedResponse` and `PagedGroupedResponse` fields are now private with
    constructor/accessor APIs.
  - facade `Error` fields are now private with stable accessor methods.
- Extended serialization contract locks to additional public DTO families:
  - runtime metrics DTOs (`EventReport`, `EntitySummary`)
  - diagnostics snapshot DTOs (`StorageReport`, `EntitySnapshot`)
- Expanded grouped planner validation coverage:
  - grouped cursor-policy/order-prefix boundary locks
  - grouped unsupported-HAVING-operator rejection locks
  - grouped structure/projection compatibility boundary locks
  - explicit delete-with-grouping rejection at planner intent-policy boundary
- Expanded fingerprint determinism coverage by canonicalizing compare `IN (...)`
  literal-list encoding so equivalent list order/duplicate variants produce the
  same fingerprint and continuation signature.
- Expanded cursor stability coverage for PK fast-path vs `by_ids` fallback
  shapes:
  - local-token continuation replay parity
  - encoded-token decode/replay parity
  - cross-shape signature mismatch rejection

## 0.45.1 - 2026-03-07 - Stability and Consistency Follow-Through

### 📝 Summary

`0.45.1` extends the feature-cleanup line with deeper stability locks and
maintainability cleanup across routing, continuation, EXPLAIN snapshots, and
encoding/fingerprint boundaries.

The patch stays focused on internal correctness and change safety rather than
new end-user query features.

### 🔧 Changed

- Finalized canonical executor route-shape consumption for runtime and route
  tests by converging on one `ExecutionRouteShape` boundary instead of
  repeated direct mode-axis reads.
- Continued fast-path telemetry consolidation by moving aggregate-path matrix
  tests onto the shared optimization counter taxonomy.
- Expanded cursor stability coverage with signature-mismatch rejection and
  replay parity matrices across fast-path vs fallback and limit-pushdown vs
  fallback shapes.
- Expanded EXPLAIN stability coverage with grouped and verbose snapshot locks
  so descriptor/runtime output changes are explicit and test-guarded.
- Extended serialization contract locks beyond initial query/error/EXPLAIN
  coverage:
  - added stable wire-shape locks for public core describe DTOs
  - added additional facade query/error variant-label and payload-shape locks
- Split the continuation gravity-well module into focused submodules while
  preserving behavior:
  - `capabilities`
  - `engine`
  - `scalar`
- Split the load terminal gravity-well module into focused submodules while
  preserving behavior:
  - `terminal/mod.rs` (shared helpers + tests)
  - `terminal/bytes.rs` (`bytes` terminals and fast-path folding)
  - `terminal/ranking.rs` (`take` and top/bottom-k ranking terminals)
- Split the load module contract block into a dedicated module while
  preserving behavior:
  - `load/contracts.rs` now owns load cursor/page DTOs, grouped stage
    projection contracts, and load-executor contract type definitions
  - `load/mod.rs` now acts primarily as module wiring + re-export boundary
- Split access execution-contract summary helpers into a dedicated submodule
  while preserving behavior:
  - `access/execution_contract/summary.rs` now owns access-strategy debug
    summary/rendering helpers
  - `access/execution_contract/pushdown.rs` now owns secondary ORDER BY
    pushdown matcher helpers

- Converted access execution-contract module to canonical directory-root
  layout while preserving behavior:
  - `access/execution_contract.rs` moved to
    `access/execution_contract/mod.rs`
  - access boundary tests were updated to the new canonical module path
- Split access execution-contract internals into focused submodules while
  preserving behavior:
  - `access/execution_contract/types.rs`
  - `access/execution_contract/executable.rs`
  - `access/execution_contract/route.rs`
  - `access/execution_contract/strategy.rs`
  - `access/execution_contract/mod.rs` now acts as module boundary wiring +
    re-export surface
- Split aggregate projection covering-component decode helpers into a
  dedicated submodule while preserving behavior:
  - `aggregate/projection.rs` moved to `aggregate/projection/mod.rs`
  - covering order/context/dedupe helpers now live in
    `aggregate/projection/covering.rs`
  - covering decode logic now lives in `aggregate/projection/decode.rs`
- Continued encoding-consistency cleanup by removing duplicated comparator and

  - shared fingerprint hash writers reused across `hash_parts`,
    `projection_hash`, and `aggregate_hash`
  - single index literal encoding authority in index-predicate compilation

## 0.45.0 - 2026-03-07 - Feature Cleanup Foundations

### 📝 Summary

`0.45.0` starts the feature-cleanup line with boundary hardening and
consistency work across query planning, executor routing, schema derive, and
public API contracts.

The release focuses on correctness and maintainability, not new query features:

- tighter visibility and DTO boundary control
- stabilized error and serialization contracts
- compile-time schema relation validation hardening
- expanded planner-policy regression coverage
- initial executor/planner overlap reduction with shared route contracts

### 🔧 Changed

- Reduced direct public field coupling across query/predicate plan DTOs by
  tightening field visibility and exposing explicit accessor methods.
- Normalized execution error boundary naming:
  `QueryExecuteError` was renamed to `ExecutionError` in core query intent
  surfaces, and wrapper/facade mappings were updated accordingly.
- Added serialization-shape stability locks for public query/error DTOs and
  EXPLAIN execution descriptors, including explicit serde representation
  attributes and regression tests.
- Hardened schema derive relation validation:
  - relation fields now require canonical suffixes (`_id` / `_ids`)
  - relation primitive/key-type mismatches are rejected via generated compile-
    time assertions
  - generated relation helper APIs now align with canonical field naming and
    include typed setter/add/remove helpers
- Added derive-layer unit tests for relation suffix enforcement and updated
  schema-test fixtures to canonical relation naming.
- Expanded planner validation coverage with direct regression tests for:
  - cursor order/limit requirements
  - delete limit/order and offset policy constraints
  - grouped distinct policy constraints
  - key-access conflict and unsupported key-access+predicate combinations
- Removed duplicated executor-side access-order semantic checks by centralizing
  route-order satisfaction under shared route helpers:
  - `access_order_satisfied_by_route_contract(...)`
  - `secondary_order_contract_active(...)`
- Added overlap anti-regression tests to keep route capability and post-access
  ordering gates aligned under the shared route contract.
- Kept the workspace warning-clean under Rust `1.94` lint expectations by
  addressing strict clippy constructor/option-evaluation findings in relation
  helper and sort-expression constructors.

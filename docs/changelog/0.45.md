# IcyDB 0.45

This document aggregates detailed release notes for the `0.45.x` minor line.

## 0.45.0 - 2026-03-07 - Feature Cleanup Foundations

### 📝 Summary

`0.45.0` starts the feature-cleanup line with boundary hardening and
consistency work across query planning, executor routing, schema derive, and
public API contracts.

The release focuses on correctness and maintainability, not new query features:

- tighter visibility and DTO boundary control
- stabilized error and serialization contracts
- compile-time schema relation validation hardening
- expanded planner-policy regression coverage
- initial executor/planner overlap reduction with shared route contracts

### 🔧 Changed

- Reduced direct public field coupling across query/predicate plan DTOs by
  tightening field visibility and exposing explicit accessor methods.
- Normalized execution error boundary naming:
  `QueryExecuteError` was renamed to `ExecutionError` in core query intent
  surfaces, and wrapper/facade mappings were updated accordingly.
- Added serialization-shape stability locks for public query/error DTOs and
  EXPLAIN execution descriptors, including explicit serde representation
  attributes and regression tests.
- Hardened schema derive relation validation:
  - relation fields now require canonical suffixes (`_id` / `_ids`)
  - relation primitive/key-type mismatches are rejected via generated compile-
    time assertions
  - generated relation helper APIs now align with canonical field naming and
    include typed setter/add/remove helpers
- Added derive-layer unit tests for relation suffix enforcement and updated
  schema-test fixtures to canonical relation naming.
- Expanded planner validation coverage with direct regression tests for:
  - cursor order/limit requirements
  - delete limit/order and offset policy constraints
  - grouped distinct policy constraints
  - key-access conflict and unsupported key-access+predicate combinations
- Removed duplicated executor-side access-order semantic checks by centralizing
  route-order satisfaction under shared route helpers:
  - `access_order_satisfied_by_route_contract(...)`
  - `secondary_order_contract_active(...)`
- Added overlap anti-regression tests to keep route capability and post-access
  ordering gates aligned under the shared route contract.
- Kept the workspace warning-clean under Rust `1.94` lint expectations by
  addressing strict clippy constructor/option-evaluation findings in relation
  helper and sort-expression constructors.

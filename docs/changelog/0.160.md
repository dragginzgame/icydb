# 0.160.x

## 0.160.0 - 2026-05-21

### ➕ Added

- Added a structured logical explain access-decision DTO projected from the
  planner-owned access-choice snapshot. JSON explain now includes selected
  access kind, selected semantic index name, selected reason, candidates,
  alternatives, rejections, and selected residual-work summary.
- Added composite access detail to logical JSON explain. `IndexPrefix` reports
  bound fields, bound values, and unbound trailing fields; `IndexRange` reports
  equality prefix fields/values, range anchor, bound inclusivity, and trailing
  fields.
- Added structured residual-work fields to logical JSON explain, including
  residual burden class, scalar residual presence, predicate residual presence,
  access-bound constraint count, and residual predicate count.
- Added fail-closed fluent access requirements:

```rust
load::<MovementIntent>()
    .filter(session_id.eq(id))
    .filter(turn_number.eq(turn))
    .filter(status.eq(Open))
    .require_index()
    .require_no_residual_filter();
```

```rust
load::<MovementIntent>()
    .filter(session_id.eq(id))
    .require_index_named("movement_intent_session_turn_status");
```

```rust
load::<MovementIntent>()
    .filter(session_id.eq(id))
    .filter(turn_number.eq(turn))
    .require_access_path(RequiredAccessPath::IndexPrefix);
```

```rust
let explain = load::<MovementIntent>()
    .filter(session_id.eq(id))
    .filter(turn_number.eq(turn))
    .explain()?;

let decision = explain.access_decision();
let json = explain.render_json_canonical();
```

### 🔧 Changed

- Standalone query planning now finalizes the model-only access-choice snapshot
  before returning explain/plan contracts, matching the structured DTO’s
  planner-owned source-of-truth requirement without changing access ranking.
- Text explain now keeps access decisions compact while JSON remains the
  structured tooling surface.

### 🧪 Testing

- Added focused planner/explain tests for structured access-decision JSON,
  compact text rendering, composite prefix/range detail, fail-closed access
  requirements, selected-decision preservation on failure, and internal
  structured planner assertion helpers.

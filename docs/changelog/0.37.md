## 0.37.2 - 2026-03-01 - Resource Follow-Up Closure

### 📝 Summary

`0.37.2` closes the deferred resource-model follow-ups by making grouped
strategy labels explicitly materialized and consolidating non-grouped
materialized DISTINCT helper ownership.

### 🔧 Changed

- Relabeled route-level grouped strategy variants to explicit runtime-truthful
  names:
  - `HashMaterialized`
  - `OrderedMaterialized`
- Kept grouped metrics strategy projection aligned to explicit materialized
  labels and preserved grouped runtime `ExecutionMode::Materialized`.
- Added a shared non-grouped materialized DISTINCT helper boundary:
  - `executor::aggregate::materialized_distinct::insert_materialized_distinct_value`
- Routed scalar materialized DISTINCT terminals through that helper:
  - `count_distinct_by(field)`
  - `distinct_values_by(field)`
- Clarified `RESOURCE_MODEL.md` language for non-grouped materialized DISTINCT
  helper ownership.
- Added audit closure report:
  - `docs/audits/reports/2026-03-01/resource-model-followup-0.37.2.md`

### 🧪 Testing

- `cargo test -p icydb-core db::executor::tests::route::aggregate::route_plan_grouped_wrapper_selects_ordered_group_strategy_for_index_prefix_shape -- --nocapture` (pass)

## 0.37.1 - 2026-03-01 - Builder Hardening + Migration Lock

### 📝 Summary

`0.37.1` closes the aggregate builder hardening pass by locking removed helper
terminals with compile-fail guards, adding builder-path typed validation
coverage, and publishing a direct migration map for the hard cut.

### 🔧 Changed

- Added compile-fail UI coverage that locks grouped helper-terminal removal on
  both query and fluent API surfaces (`group_count` / `group_count_distinct_by`
  no longer compile).
- Added intent-level builder validation regression coverage for:
  - unsupported DISTINCT kinds (`exists().distinct()`)
  - grouped v1 field-target rejection (`max_by("field")` under `GROUP BY`)
  - non-numeric global `sum(field).distinct()` target rejection
- Published migration guidance for the hard cut:
  - `docs/migration/0.37-aggregate-builder-hard-cut.md`
- Synced `0.37` status tracker phase gates so typed validation parity and
  migration/changelog readiness are explicitly closed.

```rust
// 0.37+ replacement for removed group_* terminals
query.group_by("name")?.aggregate(count()).execute_grouped()?;
```

## 0.37.0 - 2026-03-01 - Aggregate API Consolidation Start

### 📝 Summary

`0.37.0` starts the aggregate fluent API consolidation line and carries forward
contract-freeze regression locking for grouped explain/cursor surfaces.

### 🔧 Changed

- Added composable aggregate builder surface for grouped/global aggregate intent:
  - `AggregateExpr`
  - `count()`
  - `count_by(field)`
  - `sum(field)`
  - `exists()`
  - `first()`
  - `last()`
  - `min()`
  - `max()`
  - `min_by(field)`
  - `max_by(field)`
  - `.distinct()`
- Added aggregate entrypoints:
  - `Query::aggregate(AggregateExpr)`
  - `FluentLoadQuery::aggregate(AggregateExpr)`
- Removed grouped combinatorial helper terminals from query/fluent surfaces as
  an intentional pre-`1.0` hard cut:
  - `group_count*`
  - `group_sum_distinct_by`
  - `group_exists`
  - `group_first`
  - `group_last`
  - `group_min*`
  - `group_max*`
- Consolidated aggregate semantic helpers behind `AggregateExpr` so plan
  validation, grouped strategy compatibility checks, route direction/fold
  derivation, and grouped aggregate fingerprint tagging share one authority.
- Added strict grouped builder regression tests that lock equivalence across:
  - grouping shape
  - ordering-sensitive grouped shape
  - DISTINCT flag shape
  - projection/aggregate-order shape
  - plan fingerprint parity
  - continuation signature and token-byte parity
- Added grouped explain projection freeze assertions for ordered grouped + HAVING
  and hash grouped + DISTINCT shapes to lock strategy, grouped fields,
  aggregate projection, HAVING projection, and grouped hard-limit surfaces.
- Added grouped continuation token `v1` DESC wire-vector snapshot coverage to
  lock direction-sensitive grouped cursor payload encoding.
- Kept grouped runtime behavior unchanged; this is regression hardening while
  `0.37` API consolidation work begins.

```rust
let page = session
    .load::<Order>()
    .aggregate(sum("rank").distinct())
    .execute_grouped()?;
```

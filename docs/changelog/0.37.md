# IcyDB 0.37

This document aggregates detailed release notes for the `0.37.x` minor line.

---

## 0.37.5 - 2026-03-01 - Contract Freezing

### 📝 Summary

`0.37.5` freezes core planner/runtime contracts so topology, cursor semantics,
grouped route strategy, and access canonicalization behavior do not drift
silently across future refactors.

### 🔧 Changed

- Sealed execution-topology branch surfaces with explicit enum handling for
  `ExecutionMode`, `GroupedExecutionStrategy`, and `ContinuationMode`, plus a
  grouped materialization invariant (`grouped execution must remain
  materialized`) to prevent silent topology expansion.
- Froze cursor semantics with dedicated regressions for scalar and grouped
  initial-to-continuation behavior, ASC/DESC parity, offset+limit progression,
  boundary/signature mismatch rejection, and stable continuation payload
  encodings.
- Froze grouped route and strategy contracts with explain+route projection
  lock tests that pin grouped strategy, execution mode, group/aggregate/HAVING
  projections, and grouped hard-limit semantics for the same shape.
- Centralized and froze grouped strategy-to-metrics mapping under one helper,
  with stable mapping assertions from route strategy labels to grouped plan
  metrics strategy labels.
- Added explicit global DISTINCT grouped-shape rejection coverage for HAVING
  coexistence to keep that special-case policy boundary fail-closed.
- Locked access canonicalization boundaries with regressions for
  `ByKeys([id]) -> ByKey(id)` collapse, planner-vs-intent canonical-shape
  parity, canonicalization idempotency, and executor source-boundary guards
  that keep direct `AccessPath` variant matching dispatcher-owned.

## 0.37.4 - 2026-03-01 - Cursor Policy + Access Canonicalization Harmonization

### 📝 Summary

`0.37.4` closes a text and invocation-drift follow-up by centralizing cursor
paging policy wording and normalizing access canonicalization invocation
patterns.

### 🔧 Changed

- Centralized canonical cursor paging policy messages and constructors on
  `CursorPlanError` (`cursor_requires_order_message`,
  `cursor_requires_limit_message`, `cursor_requires_non_empty_order_message`),
  and routed cursor-order validation through those constructors.
- Updated plan and intent cursor policy error surfaces to reference
  `CursorPlanError` message accessors, removing duplicated cursor paging strings
  across modules.
- Normalized key-access intent canonicalization to use the shared
  `normalize_access_plan_value(...)` `db::access` entrypoint, so planner and
  intent canonicalization invocations now pass through one public boundary.
- Removed the extra `canonical_by_keys_path` surface and kept canonicalization
  helpers private to `db::access::canonical`, reducing invocation pattern drift
  without changing canonicalization ownership.
- Isolated execution trace contracts into a dedicated executor boundary:
  `db::executor::trace` now owns `ExecutionTrace`,
  `ExecutionAccessPathVariant`, `ExecutionOptimization`, and trace projection
  helpers.
- Kept non-contract trace plumbing intentionally outside the trace contract
  module:
  - load-path lifecycle wiring (trace creation + one final outcome application)
  - response/db payload storage and accessor surfaces for traced paged outputs

## 0.37.3 - 2026-03-01 - Cross-Cutting Audit Closure

### 📝 Summary

`0.37.3` closes the initial cross-cutting audit pass by reducing branching
fan-out across grouped execution, access-shape routing, grouped validation, and
continuation handling boundaries.

### 🔧 Changed

- Decomposed grouped execution into an explicit staged spine:
  `resolve_grouped_route`, `build_grouped_stream`, `execute_group_fold`, and
  `finalize_grouped_output`.
- Centralized executor access-shape dispatch under one router boundary so route,
  load, stream, metrics, and preparation paths query runtime strategy
  capabilities instead of pattern-matching `AccessPath` variants across modules.
- Split grouped plan validation into strict domain gates:
  `validate_group_structure`, `validate_group_policy`, and
  `validate_group_cursor_constraints`.
- Introduced one cursor continuation application spine entrypoint:
  `cursor::spine::apply_continuation`, and routed kernel continuation-boundary
  filtering through that shared boundary.
- Normalized invariant construction callsites across query/executor paths behind
  per-module `invariant(...)` helpers to keep invariant classification and
  wording consistent while reducing constructor boilerplate.
- Removed now-unused index-range shape probing helpers from access plan/path
  types to keep access discriminant ownership concentrated at the dispatcher
  boundary.
- Published audit closure reports for this patch under
  `docs/audits/reports/2026-03-01/`.

### 📝 Summary

strategy labels explicitly materialized and consolidating non-grouped
materialized DISTINCT helper ownership.

  names:
  - `HashMaterialized`

  - `count_distinct_by(field)`
  - `distinct_values_by(field)`
- Clarified `RESOURCE_MODEL.md` language for non-grouped materialized DISTINCT
  helper ownership.

terminals with compile-fail guards, adding builder-path typed validation
coverage, and publishing a direct migration map for the hard cut.

### 🔧 Changed

## 0.37.0 - 2026-03-01 - Aggregate API Consolidation Start

### 📝 Summary

`0.37.0` starts the aggregate fluent API consolidation line and carries forward
contract-freeze regression locking for grouped explain/cursor surfaces.

### 🔧 Changed

- Added composable aggregate builder surface for grouped/global aggregate intent:
  - `AggregateExpr`
  - `count()`
  - `count_by(field)`
  - `sum(field)`
  - `exists()`
  - `first()`
  - `last()`
  - `min()`
  - `max()`
  - `min_by(field)`
  - `max_by(field)`
  - `.distinct()`
- Added aggregate entrypoints:
  - `Query::aggregate(AggregateExpr)`
  - `FluentLoadQuery::aggregate(AggregateExpr)`
- Removed grouped combinatorial helper terminals from query/fluent surfaces as
  an intentional pre-`1.0` hard cut:
  - `group_count*`
  - `group_sum_distinct_by`
  - `group_exists`
  - `group_first`
  - `group_last`
  - `group_min*`
  - `group_max*`
- Consolidated aggregate semantic helpers behind `AggregateExpr` so plan
  validation, grouped strategy compatibility checks, route direction/fold
  derivation, and grouped aggregate fingerprint tagging share one authority.
- Added strict grouped builder regression tests that lock equivalence across:
  - grouping shape
  - ordering-sensitive grouped shape
  - DISTINCT flag shape
  - projection/aggregate-order shape
  - plan fingerprint parity
  - continuation signature and token-byte parity
- Added grouped explain projection freeze assertions for ordered grouped + HAVING
  and hash grouped + DISTINCT shapes to lock strategy, grouped fields,
  aggregate projection, HAVING projection, and grouped hard-limit surfaces.
- Added grouped continuation token `v1` DESC wire-vector snapshot coverage to
  lock direction-sensitive grouped cursor payload encoding.
- Kept grouped runtime behavior unchanged; this is regression hardening while
  `0.37` API consolidation work begins.

```rust
let page = session
    .load::<Order>()
    .aggregate(sum("rank").distinct())
    .execute_grouped()?;
```

# IcyDB 0.203 Changelog

## 0.203.2 - 2026-07-15

### 📝 Summary

- Finishes a second repository-wide excavation after the initial 0.203 hard
  cut. Runtime behavior, generated code, tests, and active documentation now
  use the surviving authority and response contracts directly.
- Removes historical states instead of translating them. Current construction
  succeeds through the canonical owner or fails with a typed cause.
- Carries one schema-owned mutation plan from transition policy into concrete
  runners and reports the selected transition kind without reconstructing a
  coarser diagnostic identity.
- Makes accepted-schema publication the sole commit point for physical schema
  mutations, with exact rollback for rejected index additions, field drops,
  and index drops.

### ⚠️ Breaking

- Generated `db()` and `core_db()` return `Result<_, DatabaseBootstrapError>`.
  Callers must handle or propagate bootstrap failure, normally with `db()?`.
  The original typed `ic-memory` cause is retained; generated code no longer
  caches a string and panics.
- SQL row mutations use `execute_trusted_sql_mutation`; catalog DDL uses
  `execute_admin_sql_ddl`. The former ambient names are removed without
  aliases.
- Removes the facade `MutationResult` family. Typed writes return entities,
  entity batches, affected-row counts, or `RowProjectionOutput` directly; SQL
  continues to return `SqlQueryResult`.
- Removes the hidden fluent raw `.limit(...)` hook and its terminal-specific
  rejection errors. Deliberate prefixes use `partial_window(...)`; scalable
  continuation uses `page(...)` and `next_page(...)`.
- Removed runtime-boundary and intent-error variants compact the pre-1.0
  diagnostic registry. Consumers of raw numbers must regenerate.
- Schema-transition metrics add explicit field-path index, expression-index,
  and metadata-rename counters. Consumers must regenerate the current metrics
  payload instead of treating those transitions as exact matches.
- Raw cursor byte codecs and the duplicate `AdminBatchRequest::with_cursor` /
  `cursor` methods are removed. Pagination tokens remain opaque strings and
  trusted continuation uses `AdminBatchRequest::next`.

### 🔧 Changed

- `EntityAuthority` is internal and production construction requires the
  accepted row-decode and schema contracts in one step. Incomplete generated
  metadata construction is test-only.
- `MutationPlan` is the sole schema plan from transition policy through
  concrete runner input. Publication preflight remains the sole publication
  gate, and unsupported pseudo-states no longer flow through derived rebuild
  or execution DTOs.
- Schema-transition diagnostics map directly from the selected plan kind;
  index additions and metadata-only rename no longer masquerade as exact
  matches.
- Runtime recovery, relation validation, delete commits, integrity rebuilds,
  and schema reconciliation require registered hooks; no hookless alternate
  semantics remain.
- The stable-cell header recognized before the database boot record is named as
  pre-boot state. Its fail-closed overwrite protection is unchanged.
- Generated database accessors are Rust 1.97 Clippy-clean, strong-relation save
  validation derives targets only from registered hooks and accepted primary
  keys, and SQL write-policy reports cannot construct mixed admitted/rejected
  states.
- Field-path and expression-index rebuilds publish accepted metadata only
  through the real schema gate and roll back their physical attempt when that
  gate rejects. Accepted runtime `SchemaInfo` views always carry their selected
  catalog instead of mixing accepted and generated field authority.
- `DROP COLUMN` retains exact row before-images and `DROP INDEX` retains the
  complete remove/remap key set until accepted-schema publication succeeds;
  rejection restores the accepted-before physical state.
- Pre-push validation covers the complete local gate and CI uses Rust 1.97.

### 🗑️ Removed

- Removes dead `MutationResult` and projected-row response facades, neutral
  planner/executor type aliases, obsolete response render aliases, unreachable
  write-policy modes, and schema mutation compatibility wrappers.
- Removes the generic schema mutation execution module, derived rebuild and
  execution plans, runner capability negotiation, and future-only mutation
  states, plus test-only runner/diagnostic projections with no production
  consumer.
- Removes anti-resurrection source guards and stale benchmark labels that named
  deleted builders, policy lanes, fields, and fluent `.limit(...)` spellings.
- Removes obsolete active-document references to ambient SQL write/DDL names,
  historical grouped compatibility framing, and an incorrect cursor wire
  version.
- Removes fake runtime-invalidation/publication sinks, runtime-epoch identity,
  visibility/phase/progress DTOs, and test-only staged write, discard, and
  rollback reports from schema mutation execution.
- Removes raw cursor codec exports, duplicate admin-batch cursor methods, and
  the post-validation accepted-before field retained only for tests.

### 🧭 Migration Notes

- Propagate generated session construction with `db()?` / `core_db()?` and use
  `DatabaseBootstrapError::cause()` when the concrete memory failure matters.
- Rename SQL row-mutation and DDL calls to their trusted/admin names. No
  transitional methods are provided.
- Treat continuation tokens as opaque and construct trusted first/next batches
  with `AdminBatchRequest::new()` and `AdminBatchRequest::next(cursor)`.
- Consume typed write return values directly and replace partial fluent
  `.limit(n)` usage with the endpoint contract that actually applies:
  `partial_window(n)`, `page(n)`, or `next_page(n, cursor)`.

### 🧪 Testing

- Focused all-feature core and facade compilation covers the contracted query,
  schema, recovery, response, diagnostic, and bootstrap changes. Mutation,
  transition, reconciliation, and transition-metric suites cover the final
  schema-plan collapse.
- Generated heap, stable, SQL, demo, and audit canisters compile through the
  fallible bootstrap boundary. The user-owned full push suite remains the final
  release gate.
- Focused publication-race regressions prove exact row and index restoration
  for rejected destructive DDL, while successful dense-row compaction and
  index-ordinal remapping remain unchanged.

### 📊 Audit

- The 0.203.2 slice spans 215 files with 3,874 insertions and 9,591 deletions,
  a net reduction of 5,717 lines. Rust alone is reduced by 6,078 lines.
- The implementation shape is materially simpler: incomplete authorities,
  duplicate response names, hookless semantics, compatibility pseudo-states,
  duplicate schema mutation plans, generic runner capabilities, raw fluent
  limits, and stringly bootstrap failures disappear without replacement
  shims.
- No runtime performance or wasm-size claim is made; rollback retains bounded
  physical before-images during destructive DDL, and no benchmark or raw wasm
  artifact was measured for this cleanup.

## 0.203.1 - 2026-07-14

### 📝 Summary

- Completes the pre-1.0 excavation by collapsing durable formats, catalog
  identities, generated metadata, diagnostics, caches, fixtures, and tests onto
  one current representation.
- Removes compatibility state rather than translating it. Current data either
  decodes and executes directly or fails closed with a typed error.

### ⚠️ Breaking

- Database boot records now use `ICYDBNOW`, raw schema values use `ICYDBCAT`,
  and cursor tokens use `ICYQ` with the sole current wire version `1`.
  Database, row, journal, commit-marker, schema, and cursor formats have no
  decoder for prior bytes.
- Accepted fields require dense IDs `1..N` and physical slots `0..N-1`.
  Accepted indexes require dense one-based ordinals `1..N`; generated index
  metadata now follows the same identity contract.
- Explain response types use their unversioned current names and no longer
  carry a redundant schema-version field. Removed versioned names have no
  aliases.
- Repository audit, test, demo, and SQL-performance fixtures use compact stable
  memory ranges. Existing deployments built from those fixtures must be
  reinstalled rather than upgraded in place.

### 🔧 Changed

- `DROP COLUMN` validates every affected row, materializes accepted nulls or
  defaults for retained fields absent from older row shapes, rewrites surviving
  payloads into dense slot order, remaps primary-key/index/relation field
  references, and only then publishes the accepted schema.
- `DROP INDEX` removes the target physical keys and shifts every surviving
  higher ordinal down before publishing compacted index metadata.
- Enum reconciliation builds a dense current catalog, prunes unused types and
  tail variants, and rejects any transition that would move a surviving live
  identity or change its contract.
- Accepted-schema publication retains only the selected current root, bundle,
  and entity snapshots; heap, durable, journaled, and folded views all prune
  superseded history.
- Predicate, coercion, projection, grouping, and query-fingerprint tags are
  dense and current. Grouping/projection helpers and scalar sentinels no longer
  advertise obsolete version suffixes.

### 🗑️ Removed

- Removes persisted commit-marker row operations, single-row marker variants,
  and the separate row-operation replay module. Recovery publishes the marker's
  current journal batches and folds them through the maintained recovery path.
- Removes obsolete query and SQL cache method versions, the `MethodVersion`
  miss bucket, duplicate coarse plan counters, and redundant explain schema
  metadata.
- Removes the legacy CLI performance payload parser, compatibility-only
  `test_entity!` forms, the parallel test schema-fingerprint implementation,
  and tests whose only purpose was preserving retired formats or aliases.

### 🧭 Migration Notes

- There is no persisted-format migration or compatibility window. Recreate
  pre-0.203.1 databases and reinstall fixture/demo canisters before use.
- Discard outstanding cursor strings and restart pagination after deployment.
- Regenerate code and update imports for the unversioned explain response
  types. Metrics consumers must stop expecting the removed coarse plan and
  cache-method fields.

### 🧪 Testing

- Maintained coverage now freezes only current boot, row, journal, marker,
  schema, enum, and cursor formats while retaining malformed and unknown-future
  fail-closed checks.
- Dense-schema coverage rejects fragmented field and index identities and
  verifies that non-trailing field drops and first-of-many index drops rewrite
  both accepted metadata and physical storage.
- Closeout coverage restores the primary-key identity-mismatch fixture and
  aligns canonical query and schema-recovery snapshots with the current dense
  fingerprint and retained-root formats.
- Focused core, derive, CLI, cache, metrics, explain, schema, recovery, DDL, and
  fixture checks pass. The full push suite remains the user-owned release gate.

### 📊 Audit

- The complete 0.203.1 worktree spans 158 files with approximately 2,656
  insertions and 3,269 deletions, a net reduction of about 613 lines.
- Implementation shape is simpler overall: compatibility parsers, replay
  modules, duplicate fingerprints, historical catalog state, retired slots,
  versioned DTO names, and coarse metrics disappear. Field and index drops add
  intentional linear physical rewrite work at DDL time.
- No runtime performance or wasm-size claim is made for this patch because no
  benchmark or raw wasm artifact measurement was run.

## 0.203.0 - 2026-07-14

### 📝 Summary

- Completes the post-rewrite excavation of IcyDB's query stack so ordinary
  fluent reads, trusted SQL, planning, execution, admission, responses, and
  diagnostics each have one documented authority.
- Deletes obsolete public and internal architecture outright. No alias,
  deprecated wrapper, compatibility dispatch, fallback, or anti-resurrection
  test preserves a retired 0.202-era surface.

### ⚠️ Breaking

- Trusted reduced-SQL reads use `execute_trusted_sql_query` and the matching
  trusted attribution methods. The former ambient read names are removed;
  generated `icydb_query` remains controller-gated before trusted dispatch.
- Removes `PlannedQuery`, `CompiledQuery`, `.planned()`, `.plan()`, the hidden
  raw facade `Query`, and its direct execution/trace wrappers. Fluent explain,
  trace, and plan-hash terminals are the maintained diagnostic surface.
- Removes `ProjectionResponse`, `ProjectionRows`, `SqlProjectionRows`, and
  `SqlQueryRowsOutput`. SQL projections and write-returning results use
  `RowProjectionOutput` directly.
- Removes `SqlScalarAggregateAttribution`; both SQL and fluent diagnostics use
  `ScalarAggregateAttribution`.
- Admission-policy DTOs are no longer application-facing. Eight rejection
  codes with no runtime producer are removed, and the remaining pre-1.0 raw
  diagnostic numbers are compacted.
- The CLI update-policy spelling is `primary-key`; the obsolete `primary_key`
  command-line alias is removed.

### 🔧 Changed

- Planner diagnostics carry typed candidate identity and rejection causes to
  the rendering boundary instead of reconstructing them from labels.
- Grouped diagnostics report the selected hash or ordered execution mode and
  the planner's real fallback reason. They no longer manufacture eligibility,
  rejection, or `materialized_fallback` route state.
- Ordinary fluent reads expose cursor/keyset continuation through `page(...)`
  and `next_page(...)`, not OFFSET. Trusted/admin SQL retains its intentional
  `OFFSET` semantics, backed by the internal planner and executor contracts.
- Read admission is owned by ordinary fluent execution. Explicit trusted
  fluent and trusted SQL methods remain isolated bypasses with caller-owned
  authorization and resource policy.
- EXPLAIN, metrics, errors, public guides, and architecture contracts now use
  the same current ownership and route vocabulary as execution.

### 🗑️ Removed

- Deletes the test-only SQL-as-`PublicRead` execution path, SQL-specific
  response budgeting, and unproduced admin/development admission lanes.
- Deletes no-op scan, materialization, projection, and response-byte policy
  fields together with always-empty diagnostic projections and unreachable
  error variants.
- Deletes generated-model fallback metrics, fictitious grouped-route
  observability state, and compatibility-only compile-fail fixtures.
- Deletes the test-only fluent OFFSET hook; OFFSET behavior is tested only at
  the trusted SQL and internal planner/executor boundaries that own it.

### 🧭 Migration Notes

- Rename trusted SQL read calls to `execute_trusted_sql_query` or the matching
  trusted attribution method. Do not expose these helpers to arbitrary callers.
- Replace removed query-stage and response-wrapper imports with fluent
  diagnostic terminals and `RowProjectionOutput`.
- Use `ScalarAggregateAttribution` wherever the removed SQL-prefixed alias was
  referenced.
- Consumers that persist or compare raw diagnostic numbers must regenerate
  against 0.203.0. Removed unproducible codes and their old numeric positions
  are not preserved before 1.0.
- No database, row, index, schema, journal, commit, or cursor migration is
  required. Current persisted formats and cursor bytes are unchanged.

### 🧪 Testing

- Focused admission, SQL lowering/execution, planner, route, cursor, aggregate,
  response, diagnostics, CLI, generated-code, feature, and architecture checks
  pass with warnings denied where applicable.
- Compatibility-only negative fixtures and tests for the shadow public-SQL
  architecture are removed; maintained current surfaces retain positive,
  rejection, boundary, and regression coverage.
- The user-owned full push suite remains the final release gate.

### 📊 Audit

- The complete 0.203.0 worktree spans 161 files with approximately 2,082
  insertions and 4,849 deletions, a net reduction of about 2,767 lines. Rust
  source decreases by about 2,878 lines across 141 files.
- Implementation shape is simpler: three obsolete owner modules, six
  compatibility-only UI fixture files, duplicate public stages and payload
  names, inert policy facts, and reconstructed route state disappear without
  replacement layers.
- The representative SQL-on `ten_entity_fluent_rows` wasm-release artifact is
  2,096,342 raw non-gzipped bytes, 556 bytes smaller than the 0.202.1 baseline.
  Deterministic gzip is secondary at 737,466 bytes, 67 bytes smaller; the
  canonical shrunk artifact is 355 bytes smaller.
- No runtime performance claim is made because physical query and mutation
  algorithms are unchanged.

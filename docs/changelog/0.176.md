# IcyDB 0.176 Changelog

## 0.176.7

### 📝 Summary

- Skips commit-preflight accepted `SchemaInfo` projection for writes whose
  accepted schema snapshot has no secondary/expression indexes.
- Keeps commit row decoding on accepted row contracts, indexed commit planning
  on expression-aware accepted `SchemaInfo`, public API, persisted format,
  SQL/write semantics, cache identity, and 0.177 schema-version policy
  unchanged.

### 🔧 Changed

- Makes commit-preflight schema info optional inside accepted commit schema
  contracts.
- Derives the forward-index-planning need from accepted persisted snapshot
  indexes before building the expression-aware planner projection.
- Leaves accepted row-contract construction eager because no-index commits
  still need accepted row decoding and canonical row materialization.

### 🧪 Testing

- Updates the SQL UPDATE runtime counter guard to prove no-index write execution
  skips the commit-preflight schema projection.
- Tightens source-boundary guards so indexed commit preflight still builds
  expression-aware accepted schema info before forward-index planning.

## 0.176.6

### 📝 Summary

- Closes the 0.176 accepted catalog cleanup line with source-boundary guards
  for the remaining read-helper and SQL write-selector ownership seams.
- Keeps public API, SQL behavior, writes, persisted schema format, cache
  identity, runtime constructors, and 0.177 schema-version policy unchanged.

### 📊 Audit

- Tightens the fluent/schema-info source guard so the crate-internal helper must
  reuse accepted query catalog context and must not reopen accepted snapshots or
  reconstruct standalone `SchemaInfo`.
- Adds SQL write selector guards proving UPDATE and INSERT SELECT selector
  binding consume accepted authority-carried `SchemaInfo` and do not rebuild the
  same accepted schema projection.

### 🧪 Testing

- Revalidates the accepted schema-info, SQL write selector, and accepted catalog
  source-boundary guards.

## 0.176.5

### 📝 Summary

- Reuses accepted authority-carried `SchemaInfo` for SQL write selector
  planning in UPDATE and INSERT SELECT paths.
- Keeps public SQL behavior, write admission, write save contracts, persisted
  schema format, cache identity, and 0.177 schema-version policy unchanged.

### 🔧 Changed

- Removes standalone accepted `SchemaInfo` reconstruction from SQL INSERT
  SELECT source binding after accepted authority has already been built.
- Changes SQL UPDATE selector binding to consume the accepted schema view
  attached to the selector authority instead of rebuilding it from the same
  accepted snapshot.
- Leaves write save-contract schema projection in place because mutation
  staging still owns row-decode and save-contract safety.

### 🧪 Testing

- Adds a runtime counter guard proving UPDATE selector execution reuses the
  authority-carried schema view while retaining the remaining selector-plan and
  write save-contract schema views.
- Revalidates INSERT SELECT and UPDATE RETURNING behavior matrices.

## 0.176.4

### 📝 Summary

- Routes the crate-internal accepted schema-info helper used by fluent field
  resolution and live index rendering through accepted query catalog context.
- Keeps public API, query semantics, DDL behavior, writes, persisted schema
  format, cache identity, and 0.177 schema-version policy unchanged.

### 🔧 Changed

- Replaces the helper's direct accepted-snapshot ensure plus standalone
  `SchemaInfo` reconstruction with `accepted_schema_catalog_context_for_query`
  and the catalog-owned planner projection.
- Leaves save/write contract construction on its write-boundary accepted schema
  path, where generated-compatible row-layout proofing is still intentional.

### 🧪 Testing

- Adds a runtime counter guard proving the helper reuses a cached query catalog
  snapshot without accepted snapshot decode, generated-compatible write proof
  construction, or all-entity schema metadata.
- Revalidates fluent load behavior around nullable additive schema transitions.

## 0.176.3

### 📝 Summary

- Routes SQL EXPLAIN execution through the accepted catalog context carried by
  immediate SQL compilation, so EXPLAIN binding, accepted authority, and verbose
  route finalization reuse one catalog-owned planner projection.
- Keeps public SQL output, explain rendering, query semantics, cache identity,
  persisted schema format, DDL behavior, writes, and 0.177 schema-version
  policy unchanged.

### 🔧 Changed

- Threads the accepted `SchemaInfo` projection from the EXPLAIN entrypoint into
  plan/json rendering and EXPLAIN EXECUTION rendering instead of reconstructing
  it inside the renderer.
- Adds a context-aware EXPLAIN dispatch path for compiled SQL execution
  contexts, including diagnostics attribution, so immediate compile-and-execute
  calls keep the compile-time accepted catalog.
- Keeps explicit owned compiled EXPLAIN execution functional by loading a fresh
  accepted catalog only when no compile context is available.

### 🧪 Testing

- Adds a runtime counter guard proving public EXPLAIN EXECUTION VERBOSE builds
  one accepted `SchemaInfo` projection across authority construction, binding,
  and verbose route finalization after catalog bootstrap.
- Revalidates the EXPLAIN EXECUTION matrix and shared query-plan reuse behavior.

## 0.176.2

### 📝 Summary

- Moves accepted planner-view reuse into the accepted catalog context so one
  operation can share the same `SchemaInfo` projection across SQL compilation,
  aggregate request construction, accepted authority construction, and shared
  query-plan misses.
- Keeps public API, persisted schema format, query semantics, DDL behavior,
  writes, cache identity, and 0.177 schema-version policy unchanged.

### 🔧 Changed

- Adds a lazy crate-internal `SchemaInfo` projection to
  `AcceptedSchemaCatalogContext`.
- Builds accepted execution authority from the catalog-owned projection instead
  of reconstructing an independent accepted schema view.
- Routes SQL compiled-command cache misses through the catalog-owned projection
  instead of rebuilding `SchemaInfo` directly from the snapshot.

### 🧪 Testing

- Adds a SQL global aggregate guard proving a shared-plan miss builds one
  accepted `SchemaInfo` projection while still constructing accepted execution
  authority and planner-visible indexes once.
- Revalidates the warmed filterless identity-first hit guard and the
  schema-aware filtered shared-plan path.

## 0.176.1

### 📝 Summary

- Moves eligible warmed filterless shared query-plan cache hits onto accepted
  catalog identity/header lookup before full accepted snapshot decode or
  accepted planner-view construction.
- Keeps filtered/schema-aware query normalization, SQL semantics, DDL behavior,
  writes, persisted schema format, and 0.177 schema-version policy unchanged.

### 🔧 Changed

- Adds a crate-internal accepted catalog identity for one selected schema
  snapshot, carrying entity/store identity, accepted schema version, fingerprint
  method version, and the existing accepted schema cache fingerprint.
- Adds one-entity latest catalog identity selection that retains the selected
  raw snapshot bytes so cache misses decode the same snapshot that produced the
  cache identity.
- Computes raw/header fingerprints through the same accepted runtime schema
  fingerprint algorithm used by decoded accepted snapshots.
- Routes only filterless shared query-plan lookups through identity-first cache
  probing; misses still decode the selected snapshot and build accepted
  authority and `SchemaInfo` once.

### 🧪 Testing

- Adds runtime counter guards proving an eligible warmed filterless shared
  query-plan hit builds zero accepted `SchemaInfo`, performs zero accepted
  snapshot decodes, performs zero generated-compatible proof construction, and
  avoids all-entity schema metadata.
- Proves raw/header accepted catalog fingerprints match the existing decoded
  accepted schema fingerprint for representative plain and indexed schemas.
- Keeps filtered query coverage on the existing schema-aware path so predicate
  normalization semantics remain unchanged.

## 0.176.0

### 📝 Summary

- Establishes accepted schema catalog context as the session boundary for
  query, SQL, explain, aggregate, and DDL preparation paths.
- Keeps accepted snapshots as runtime authority while reducing repeated
  snapshot, fingerprint, `SchemaInfo`, and `EntityAuthority` reconstruction.

### 🔧 Changed

- Query cache setup now carries an accepted catalog context containing the
  accepted snapshot and runtime schema fingerprint, so cache lookup and miss
  planning consume the same authority object.
- Shared query-plan lookup can reuse catalog context supplied by callers,
  letting aggregate execution build accepted schema info and shared query plans
  from one accepted catalog load.
- Query execution explain reuses accepted authority carried by the shared
  prepared plan instead of rebuilding authority and schema info after cache
  lookup.
- SQL compiled-command cache context now carries accepted catalog context into
  compile and execution contexts, preserving one accepted snapshot/fingerprint
  pair across the SQL cache boundary.
- SQL DDL preparation now binds through accepted catalog context instead of
  constructing accepted entity authority only to discard it.

### 🧹 Cleanup

- Removes the old session helper that returned a raw accepted snapshot and
  cache fingerprint pair for query cache setup.
- Removes contextless SQL SELECT wrappers that recomputed accepted schema
  fingerprint after authority construction.
- Centralizes accepted authority derivation and accepted schema-info projection
  on the accepted catalog context.
- Centralizes accepted save-contract assembly for public structural writes and
  SQL insert/update writes.

### 📊 Audit

- Reduced the targeted duplicate session-source markers for accepted schema
  snapshot/fingerprint helpers, SQL SELECT fingerprint wrappers, and
  contextless accepted entity authority calls from 11 to 0.
- Preserved direct accepted snapshot loading in write-admission and bootstrap
  paths where generated-compatible write descriptor checks still intentionally
  own mutation safety.

### 🧪 Testing

- Revalidates source boundary guards for accepted schema authority, query
  planning, SQL DDL, SQL write, and journaled hot-path constraints.
- Revalidates shared query-plan cache reuse, SQL compiled-command cache keys,
  SQL surface identity, query explain convergence, global aggregate cache reuse,
  SQL DDL preparation/publication, and SQL insert/update/delete write paths.

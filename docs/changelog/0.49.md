# IcyDB 0.49

This document aggregates detailed release notes for the `0.49.x` minor line.

## 0.49.2 - 2026-03-11 - Execution & Observability Stabilization Follow-Through

### Summary

`0.49.2` completes the remaining executor/explain stabilization slices in the
`0.49` line by hardening EXPLAIN internal ownership, adding executor diagnostics
contracts, and replacing ad-hoc load orchestration branching with a deterministic
stage-descriptor loop. Runtime semantics are unchanged.

### Changed

- Stabilized EXPLAIN internal ownership boundaries:
  - split canonical JSON rendering into
    `db::query::explain::json`
  - split node-family ownership into
    `db::query::explain::nodes::{scan,pipeline,aggregate,terminal}`
  - rewired execution-layer label mapping through node ownership contracts
    without changing EXPLAIN output semantics
- Added executor diagnostics contracts under owner namespace:
  - `db::executor::diagnostics::node` exposes stable node correlation contracts
    (`ExecutionNodeIdentity`, `ExecutionNodeLayer`)
  - `db::executor::diagnostics::counters` exposes additive row-flow counters
    (`rows_scanned`, `rows_filtered`, `rows_aggregated`, `rows_emitted`)
  - executor root wiring now routes diagnostics usage through
    `db::executor::diagnostics`
- Completed pipeline stage-descriptor cleanup:
  - added `db::executor::pipeline::stages` with deterministic stage planning
    and stage labels
  - rewired `execute_load` orchestration in
    `db::executor::pipeline::orchestrator` to execute planned stage descriptors
    via one deterministic stage loop
  - kept structural stage-order guard assertions aligned with planned
    descriptors instead of hard-coded local lists
- Re-ran full workspace release gate after these changes and fixed strict clippy
  findings in new orchestration/diagnostics code paths.

## 0.49.1 - 2026-03-11 - Executor Contract Ownership Cleanup

### Summary

`0.49.1` completes executor contract ownership cleanup by removing the
`db::executor::shared` namespace and re-homing cross-layer helpers under owner
modules. Runtime/query behavior is unchanged; this patch is architecture and
guardrail hardening.

### Changed

- Removed the remaining `db::executor::shared` module tree and re-homed
  contract surfaces by owner:
  - load/paging and grouped-route contract DTOs now live under
    `db::executor::pipeline::contracts`
  - execution input/output/stream contract DTOs now live under
    `db::executor::pipeline::contracts::execution`
  - projection evaluation/materialization helpers now live under
    `db::executor::projection`
  - load-executor construction/context helper methods now live under
    `db::executor::context::load`
- Updated executor module wiring and imports to consume owner-named contract
  surfaces (`pipeline::contracts`, `projection`, `context`) instead of
  `shared::*`.
- Added structural/CI guardrails so contract ownership cannot drift back:
  - `crates/icydb-core/src/db/executor/tests/load_structure.rs` now asserts:
    - `src/db/executor/shared` directory is absent
    - executor source files do not reference `executor::shared`
  - `scripts/ci/check-layer-authority-invariants.sh` now fails if:
    - any executor source references `executor::shared`
    - `crates/icydb-core/src/db/executor/shared` exists
- Updated the 0.49 design doc with an explicit executor namespace rule:
  `db::executor::shared` must not exist; cross-layer helpers must remain
  owner-named.

## 0.49.0 - 2026-03-11 - Executor Architecture Stabilization

### Summary

`0.49.0` finalizes the executor architecture split introduced in `0.48` by
locking layer boundaries (`scan -> pipeline -> aggregate -> terminal`),
stabilizing continuation compatibility contracts, and expanding execution
observability through EXPLAIN and metrics.

No new query-surface language features are introduced in this release.

### Changed

- Stabilized executor layer boundaries and structural guardrails:
  - enforced forbidden dependency directions in
    `scripts/ci/check-layer-authority-invariants.sh`
  - locked legacy `db::executor::load` removal and cross-layer import rules in
    `crates/icydb-core/src/db/executor/tests/load_structure.rs`
- Hardened continuation compatibility fail-closed behavior:
  - grouped continuation replay now explicitly rejects grouped projection-width
    drift (`COUNT` vs `COUNT + MAX`) in
    `crates/icydb-core/src/db/executor/tests/cursor_validation.rs`
  - grouped continuation-window lowering remains executable-plan-owned via
    `grouped_pagination_window(...)`
- Expanded EXPLAIN execution metadata in
  `crates/icydb-core/src/db/query/explain/render.rs`:
  - deterministic `node_id` assignment
  - layer visibility (`scan|pipeline|aggregate|terminal`)
  - additive execution details:
    `execution_mode_detail`, `predicate_pushdown_mode`,
    `fast_path_selected`, `fast_path_reason`
- Added row-flow execution counters to metrics sink/report surfaces:
  - `rows_filtered`
  - `rows_aggregated`
  - `rows_emitted`
  - implemented in `metrics::{sink,state}` and emitted from executor
    finalization paths via `db::executor::plan_metrics`
- Updated docs/contracts to reflect stabilization status and new metrics
  vocabulary:
  - `docs/design/0.49-execution-observability-hardening/0.49-design.md`
  - `docs/contracts/RESOURCE_MODEL.md`
  - `docs/contracts/CURSOR.md`
  - `docs/governance/code-hygiene.md`

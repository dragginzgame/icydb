# 0.158.x

## 0.158.0

### 🔧 Changed

- Started the schema-DDL line after the 0.157 index-DDL closeout.
- SQL now parses `ALTER TABLE ... ADD COLUMN ...` into the DDL frontend model,
  including nullable/default intent.
- `ALTER TABLE ... ADD COLUMN ...` now routes through the DDL surface and binds
  the target accepted entity before failing closed with an explicit unsupported
  schema-DDL diagnostic.
- The CLI SQL shell now routes `ALTER TABLE ...` statements to the generated
  DDL update endpoint instead of the readonly query endpoint.

```sql
ALTER TABLE Character ADD COLUMN nickname text;
ALTER TABLE Character ADD COLUMN score nat DEFAULT 0 NOT NULL;
```

### 🛡️ Guardrails

- `ALTER TABLE ... ADD COLUMN ...` does not publish accepted schema metadata or
  rewrite rows in this patch.
- Field DDL remains fail-closed until accepted field-transition publication,
  missing-slot decode policy, and write encoding are wired end to end.

### 🧪 Validation

- Added parser coverage for additive column DDL intent, nullable flags, and
  default literals.
- Added DDL binding coverage proving additive column DDL validates the target
  entity before returning the unsupported schema-DDL diagnostic.
- Added CLI shell routing coverage for `ALTER TABLE ... ADD COLUMN ...`.

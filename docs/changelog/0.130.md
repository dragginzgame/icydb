# 0.130.x

## 0.130.0

`0.130.0` opens the prepared/lowering contraction line with a narrow
prepared SQL command cleanup, and continues the access contraction with a
secondary-order pushdown DTO collapse, deletion of the old borrowed access
dispatch mirror, and removal of the executable-path wrapper around physical
payloads. It also deletes the private route-intent enum that mirrored the
public route-plan request families, and collapses access-metrics projections
that only existed inside access capabilities. Lowered SQL command ownership
now exposes direct SELECT and DELETE handoff helpers, the remaining
session-side fallback invariants for those paths are constructed through
`QueryError` owner methods, access pushdown state now has one direct
applicability enum instead of nested applicability/eligibility mirrors,
lowering/query-cache code consumes the access-owned path contract directly,
runtime traversal consumes `ExecutionPathPayload` without an extra
`ExecutableAccessPath` shell, route intent staging constructs the existing
stage contract directly, and access capabilities derive metrics directly from
the executable path kind without cached boolean mirrors. This keeps session SQL
compilation, INSERT SELECT execution, route behavior, compiled cache keys,
cursor format, pushdown eligibility, metrics categories, and persisted data
unchanged.

For the access module specifically, this removes 454 raw lines of duplicate
or weak scaffolding and adds back 201 lines of simpler owner-local
replacement code, for a 253-line net reduction in `db/access` itself. That is
roughly a 7% net shrink of the module from the pre-cleanup shape, with the
removed scaffolding concentrated in dispatch mirroring, nested pushdown DTOs,
executable-path wrapping, and cached capability projections.

What changed:
- added `LoweredSqlCommand::into_select_query(...)` so callers consume lowered
  SELECT artifacts through the lowered-command owner instead of destructuring
  the command envelope repeatedly
- added prepared SELECT/DELETE lowering helpers plus a prepared SELECT
  structural-bind helper so session compilation and INSERT SELECT source
  execution consume SQL-lowering-owned handoffs
- made the prepared DELETE helper return the lowered DELETE query together
  with the normalized delete statement, so session compilation no longer
  clones `PreparedSqlStatement` or revalidates the DELETE statement kind after
  lowering
- moved prepared INSERT and UPDATE statement extraction behind the same
  prepared/lowering owner, including the INSERT SELECT source path, so session
  code no longer constructs those fallback invariants locally
- moved compiled DELETE statements from a lowered base-query shape to the same
  reusable structural query artifact used by compiled SELECT statements, so
  DELETE execution no longer repeats lowered-shape binding
- moved compiled SELECT, compiled DELETE, and INSERT SELECT lane-mismatch
  fallback invariants behind `QueryError` constructors
- retargeted session SQL compile and INSERT SELECT execution to those owners
  without changing accepted SQL, output payloads, shared query-plan caching, or
  access lowering behavior
- kept the typed lowered-query binding helper cfg-aligned with its test-only
  callers and boxed compiled SQL EXPLAIN payloads so the prepared SQL compile
  artifact stays within the clippy enum-size budget
- deleted `PushdownSurfaceEligibility`
- deleted `SecondaryOrderPushdownEligibility`
- made `PushdownApplicability` directly represent `NotApplicable`, `Eligible`,
  and `Rejected`
- retargeted explain conversion, executor descriptors, route pushdown tests,
  and explain pushdown tests to consume the single applicability shape
- reduced the touched access pushdown slice by 122 net lines while preserving
  route selection and explain output behavior
- deleted `access/dispatch.rs`
- moved `AccessPathKind` onto `access/path.rs`
- retargeted access lowering to match `AccessPath` directly inside the access
  boundary instead of first constructing `AccessPathDispatch`
- retargeted execution trace and shared query cache-key construction to
  consume `AccessPath::kind()` plus access-owned payload accessors instead of
  the removed borrowed dispatch mirror
- tightened the access boundary guard so `dispatch.rs` is no longer an allowed
  raw `AccessPath::` decision owner
- reduced the touched access-dispatch slice by 40 net lines while preserving
  executable lowering, execution trace variants, and shared cache-key identity
- deleted the `ExecutableAccessPath` wrapper
- moved executable path projections (`kind`, index-prefix details,
  index-range details, and semantic range bounds) onto `ExecutionPathPayload`
- made `ExecutableAccessNode::Path` and `ExecutableAccessPlan::as_path()`
  expose `ExecutionPathPayload` directly
- retargeted cursor validation, physical stream traversal, aggregate/bytes
  terminal fast paths, executable summaries, and access capabilities to consume
  `ExecutionPathPayload` directly
- reduced the touched access/cursor/executor executable-path slice by 186 net
  lines while preserving traversal, cursor, and fast-path behavior
- deleted the private `RouteIntent` enum and its re-export because
  `RoutePlanRequest` already owns the public load/mutation/aggregate/grouped
  route-family selection
- made the route-intent stage builders construct `RouteIntentStage` directly,
  preserving the existing route-shape invariant checks while reducing the
  layer-health route-shape decision-owner count from 3 to 2
- deleted private `AccessScanKind` / `AccessPlanKind` metric projection
  wrappers from `access/capabilities.rs`
- removed cached path-kind boolean fields from `SinglePathAccessCapabilities`
  and derive those capability facts from `AccessPathKind` at the accessor
  boundary
- retargeted the one aggregate numeric shape assertion to
  `ExecutionPathPayload::kind()` now that the capability snapshot no longer
  exposes a redundant kind accessor
- reduced this access-capability cleanup by another 80 net lines while
  preserving metrics categories and route capability behavior
- reduced `db/access` itself by 253 net lines overall in this patch while
  preserving the semantic access tree, executable access tree, physical path
  payload, and capability-projection boundaries

Retained boundaries:
- `PreparedSqlStatement` still owns entity-normalized parsed SQL before
  lowering
- `LoweredSqlCommand` owns generic-free command shape after lowering
- `CompiledSqlCommand` remains the session-owned semantic compile artifact
  cached above shared query-plan execution
- `executor/planning/route/pushdown.rs` still computes secondary-order
  eligibility from access capabilities plus the planner-owned deterministic
  order contract; `access/order_pushdown.rs` owns the DTO and stable labels,
  not route policy derivation
- raw `AccessPath::` branching remains access-boundary-local; external callers
  now use the coarse kind/accessor contract instead of a separate dispatch
  enum
- `ExecutionPathPayload` is now the single physical leaf contract; composite
  execution still flows through `ExecutableAccessPlan` / `ExecutableAccessNode`
- `RouteIntentStage` still owns normalized intent-stage facts; the removed
  enum was only a private mirror between `RoutePlanRequest` and that stage
- `SinglePathAccessCapabilities` still owns executable path capability facts;
  it no longer caches booleans that are pure functions of `AccessPathKind`

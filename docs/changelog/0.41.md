# IcyDB 0.41

This document aggregates detailed release notes for the `0.41.x` minor line.

## 0.41.3 - 2026-03-06 - Pre-EXPLAIN Observability and Route Polish

### 📝 Summary

`0.41.3` packages the pre-EXPLAIN feature slice and its final route-polish
follow-up. This release exposes stable developer-facing execution diagnostics
and introspection surfaces while keeping ordered aggregate routing contracts
explicit and authority boundaries clean.

### 🔧 Changed

- Added a stable scalar execution-descriptor projection for aggregate terminals
  (access strategy, aggregation, ordering source, limit, cursor) and wired
  terminal explain surfaces (`explain_exists`, `explain_min`, `explain_max`,
  `explain_first`, `explain_last`) to consume that shared contract.
- Added explicit terminal ordering-source labels for seek-capable routes
  (`IndexSeekFirst` / `IndexSeekLast`) so pre-EXPLAIN output can distinguish
  index-edge seek routing from generic access-order and materialized fallback
  execution.
- Added response-level execution metrics surfaces
  (`rows_scanned`, `rows_materialized`, `index_only`,
  `execution_time_micros`) as ANALYZE-ready observability primitives.
- Added plan-hash and trace diagnostics surfaces
  (`plan_hash_hex()`, `DbSession::trace_query(...)`, fluent `.trace()`) so
  developers can inspect planner/executor decisions without running full explain
  tooling.
- Added schema introspection helpers
  (`show_indexes::<Entity>()`, `describe_entity::<Entity>()`) for low-friction
  index/entity inspection alongside trace output.
- Finalized ordered aggregate route-hint derivation so bounded physical fetch
  hints are projected for all eligible scalar aggregate terminals, while
  extrema-specific seek authority remains explicit via `AggregateSeekSpec`.
- Kept executor runtime authority guards aligned with the no-raw-`AccessPath`
  policy in runtime modules during aggregate explain wiring.

## 0.41.2 - 2026-03-06 - Ordered Path and Pushdown Follow-Up

### 📝 Summary

`0.41.2` follows the aggregate-path work with planner/executor polish for
ordered scalar execution. This slice improves ordered `LIMIT` behavior, extends
covering projection for id/value outputs, and tightens planner simplifications
and pushdown coverage.

### 🔧 Changed

- Extended covering index-projection pushdown to
  `values_by_with_ids(field)` for eligible scalar index-backed shapes so
  id/value projection output can bypass row materialization while preserving
  strict missing-row policy behavior and effective-window ordering.
- Added a safe ORDER BY sort-skip gate for unique secondary index-prefix
  shapes: when planner pushdown policy confirms index-order compatibility, the
  post-access pipeline now treats access traversal order as authoritative and
  skips redundant in-memory ordering.
- Extended load route budget-safety/order-satisfaction gates so unique secondary
  ORDER BY-compatible shapes can receive bounded scan-budget hints for paged
  windows (including `ORDER BY ... DESC LIMIT 1` first-page shapes).
- Added an unpaged scalar runtime seek hint for streaming-safe
  `ORDER BY ... LIMIT 1` load execution (`execute()`): when no continuation is
  requested and no residual predicate/distinct phase is present, runtime now
  limits both physical fetch and load scan-budget to one row.
- Added planner-side simplification for exact `ByKey` load shapes that removes
  redundant `LIMIT 1` pagination metadata (offset `0`) and strips duplicate
  `id == literal` predicates when access already resolves to the same `ByKey`
  literal.
- Extended planner compare pushdown so strict text `StartsWith` predicates on
  indexed leading fields compile to index-range access shapes
  (`[prefix, next_prefix)`), and added explicit planner matrix coverage for
  secondary-index `IN` pushdown and `BETWEEN`-equivalent range lowering.
- Added predicate normalization follow-up coverage that locks child-dedup
  canonicalization for `AND`/`OR` and keeps cheaper predicate clauses ordered
  ahead of expensive text-contains evaluation in normalized trees.

## 0.41.1 - 2026-03-05 - Aggregate Fast-Path Follow-Up

### 📝 Summary

`0.41.1` follows `0.41.0` with aggregate-path performance improvements on
index-backed scalar shapes. This slice adds index-only `COUNT(*)`/`EXISTS`
execution, bounded field-extrema probing for safe `MIN(field)` index-leading
cases, and tighter covering `distinct_values_by(field)` dedupe behavior for
eligible index-ordered shapes while preserving existing stale-key and strict
missing-row semantics.

### 🔧 Changed

- Added an index-backed `COUNT(*)` fast path for eligible scalar index-prefix
  and index-range shapes that executes through key-stream reduction without row
  materialization/deserialization.
- Added a primary-key point-lookup `COUNT(*)` fast path (`ByKey`) that folds
  through existing-row checks, so `COUNT WHERE pk = ?` resolves as a parity-safe
  `0/1` result without row materialization.
- Kept strict/ignore missing-row behavior unchanged in that path by running
  count folds under existing-row consistency checks, and added strict stale-key
  regression coverage for secondary-index count execution.
- Added an index-backed `EXISTS` fast path for eligible unordered scalar
  index-prefix/index-range shapes so existence probes short-circuit on the
  first existing row without row materialization/deserialization.
- Added stale-key parity coverage that asserts covering `EXISTS` and covering
  `COUNT` outputs match forced materialized aggregate execution and canonical
  row-materialized baseline execution.
- Added strict stale-key regression coverage for fast-path-eligible covering
  `EXISTS` shapes, and test-only fast-path-hit markers that assert eligible
  unordered shapes take the covering branch while ordered fallback shapes do not.
- Added bounded one-key probe hints for field-target `MIN(field)` on index-leading
  shapes, plus unbounded retry fallback when ignore-mode probe windows can
  under-fetch due to stale leading keys.
- Kept field-target `MAX(field)` bounded probes conservative under the current
  primary-key-ascending tie-break contract (no one-key probe) to preserve
  correctness when multiple rows share the same maximal field value.
- Added an adjacent-key covering `distinct_values_by(field)` optimization for
  index-ordered shapes where the target field is the first unbound index
  component, while preserving canonical first-observed fallback dedupe for
  non-contiguous covering shapes.

## 0.41.0 - 2026-03-05 - Minor Features Kickoff

### 📝 Summary

`0.41.0` opens the `0.41.x` line for small feature and runtime cleanup work.
This first slice improves scalar `LIMIT` window plumbing by centralizing the
route keep/fetch arithmetic used by continuation projection and by moving load
early-stop scan-budget hint gating behind a strategy-owned contract.

### 🔧 Changed

- Added one shared executor window helper for deriving `(keep_count, fetch_count)`
  from `(offset, limit)` in one call.
- Routed scalar continuation route-window projection through that shared helper
  instead of computing keep/fetch with duplicated window calls.
- Added `AccessStrategy::load_window_early_stop_hint(...)` and rewired route
  load scan-budget hint derivation to consume that strategy contract.
- Removed continuation-owned load scan-budget gate helper from route contracts;
  route feasibility now parity-asserts against the access-strategy contract.
- Added a regression test that locks parity between the new helper and the
  canonical page-window projections.
- Added no-cursor `LIMIT` pagination matrix coverage across `ByKey`, `ByKeys`,
  `KeyRange`, and `FullScan` primary-access shapes to lock result correctness
  and bounded-scan behavior where applicable.
- Added a covering-index constant-projection fast path for scalar field-target
  terminals: `values_by(field)`, `distinct_values_by(field)`,
  `first_value_by(field)`, and `last_value_by(field)` now bypass row
  materialization when the target field is fixed by index-prefix equality
  bindings (`IndexPrefix` / `IndexRange` prefix).
- Scoped that fast path to `MissingRowPolicy::Ignore` and added strict-policy
  regression coverage to keep missing-row corruption surfacing behavior intact
  under strict row-consistency execution.
- Added aggregate projection matrix coverage that locks parity for covering
  constant projections against effective-window baseline execution output.
- Added a second covering-index projection path for non-constant field targets
  by decoding index component payloads directly for eligible index-backed scalar
  shapes, while preserving strict missing-row corruption behavior and effective-
  window parity for `values_by`, `distinct_values_by`, `first_value_by`, and
  `last_value_by`.
- Added aggregate projection matrix coverage for the non-constant covering path,
  including strict-missing-row parity checks when primary rows are absent.

# 0.150.x

## 0.150.0

`0.150.0` starts accepted index authority with the field-path half of the
index contract. Accepted schema snapshots now have a persisted index-contract
surface, generated field-path indexes project into accepted field IDs and row
slots during schema proposal construction, and the persisted schema codec now
hard-cuts to a shape that carries those index contracts. Accepted `SchemaInfo`
also reads top-level index membership and field-path index metadata from the
accepted snapshot instead of generated `EntityModel` index metadata. Expression
indexes are intentionally not accepted yet; they still need a stable canonical
expression contract before runtime can use them as accepted authority.

### Accepted Field-Path Index Contracts

- Added `PersistedIndexSnapshot`, `PersistedIndexKeySnapshot`, and
  `PersistedIndexFieldPathSnapshot` to accepted schema snapshots.
- Added `PersistedSchemaSnapshot::new_with_indexes(...)` and retained the
  field-only constructor for owner-local field-focused fixtures.
- Projected generated field-path indexes from schema proposals into accepted
  top-level `FieldId`, accepted row slot, full field path, persisted field
  kind, nullability, uniqueness, ordinal, store path, and predicate SQL display
  metadata.
- Skipped generated expression index proposals in this first field-path slice
  so expression contracts cannot become accepted runtime authority before their
  stable representation is designed.
- Hard-cut the persisted schema snapshot codec to version `3` and round-tripped
  accepted field-path index contracts through Candid.
- Added accepted snapshot integrity validation for duplicate index identities,
  empty index keys, and index field slots that diverge from the accepted row
  layout.
- Routed accepted `SchemaInfo` field index-membership flags through persisted
  index contracts instead of generated `EntityModel::indexes`.
- Added an internal `SchemaInfo` field-path index metadata surface backed by
  accepted index contracts, including ordinal, name, store, uniqueness, field
  ID, accepted row slot, field path, persisted kind, nullability, and predicate
  SQL display metadata.
- Kept generated `SchemaInfo` views on generated index metadata for
  proposal/model-only callers.
- Extended write-boundary guards so accepted `SchemaInfo` construction cannot
  regain generated index membership lookup or lose its persisted index metadata
  surface.

# IcyDB 0.197 Changelog

## 0.197.12

### 📝 Summary

- Completes the 0.197 focused exact-key manifest and records a current
  PocketIC-backed artifact for all 33 required scenarios.
- Adds saved before/after artifact helpers that emit focused delta JSON and
  Markdown, then immediately apply the manifest gate so generated deltas cannot
  omit required route, admission, counter, result-signature, or explanation
  fields.
- Distinguishes valid non-application cases as `NotApplied`, keeping unique
  secondary equality, partial composite primary-key filters, and
  expression-wrapped primary-key comparisons out of validation-failure
  accounting unless they are otherwise invalid.
- Fixes signed numeric fluent exact-primary-key predicates so string-backed
  `FilterExpr::eq("id", value)` over signed primary keys reaches `ByKey` /
  `ByKeys` instead of being rejected as an unbounded scan.
- Tightens exact primary-key `AND` planning so contradictory or intersecting
  key clauses resolve to `Empty` / narrowed exact-key access before broader
  secondary candidates can win.
- Clarifies the closeout boundary for SQL placeholders, read-admission
  exact-key filters, and fast-path proof ownership.

### 🧪 Testing

- Adds a non-ignored manifest completeness test that checks required scenario
  keys, required delta fields, expected result labels, and closeout-failure
  labels.
- Adds an ignored saved-delta gate that validates captured focused delta JSON
  exactly covers the manifest and carries required route, admission,
  instruction, result, and explanation fields.
- Adds an ignored focused-current PocketIC capture path that writes the current
  `sql_perf_197_pk_canonicalization_after.json` and Markdown summary.
- Adds regression coverage for signed fluent exact-key predicates, exact-key
  `Empty` route proof, and public-read exact-key input caps.

The saved before/after delta helper can be run with:

```text
ICYDB_197_PK_FOCUSED_BEFORE_JSON=<path-to-before-json> \
ICYDB_197_PK_FOCUSED_AFTER_JSON=<path-to-after-json> \
ICYDB_197_PK_FOCUSED_DELTA_OUT=<path-to-delta-json> \
cargo test -p icydb-testing-integration --test pk_canonicalization_focused_artifact \
  pk_canonicalization_focused_delta_writes_from_saved_before_after_artifacts -- --ignored
```

### 📊 Performance

- No runtime performance win is claimed.
- The current PocketIC capture records 33 scenarios, 21 rows with instruction
  counters, 20 admitted counter-measured rows, 5 contract/not-measured rows, 24
  aggregate `data_store.get` calls, and 13,622,028 aggregate total
  instructions.
- The focused delta helper reads saved artifacts only; it does not produce a
  before baseline or run the full deterministic SQL matrix by itself.
- Fresh focused before/after artifacts and the full deterministic SQL matrix
  are still required before final 0.197 performance closeout or any broad
  speedup claim.
- Wasm-size delta was not measured.

### 📊 Complexity Delta

- Production complexity increases modestly through signed numeric exact-key
  access support and tighter exact-key set-intersection / contradiction
  planning.
- Audit/test harness complexity increases materially through the focused
  manifest gate, saved-artifact delta generator, PocketIC current capture, and
  Markdown/JSON artifact emission.
- Documentation complexity increases slightly because the implementation
  ledger now distinguishes current behavior evidence from missing before/delta
  performance closeout evidence.

### 🧪 Validation

- `cargo fmt --all`
- Focused exact-key regression tests
- Focused public-read cap tests
- Focused artifact manifest test
- Focused artifact target clippy
- Targeted 0.197 clippy over `icydb-core`, the SQL perf audit canister, and

- Ignored PocketIC current-capture test

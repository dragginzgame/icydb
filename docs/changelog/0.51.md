# 0.51 Changelog

## 0.51.2 - 2026-03-12 - Slice H Access Descriptor Collapse

### Summary

`0.51.2` continues Slice H semantic-authority cleanup by removing the
`AccessExecutionDescriptor` passthrough layer and routing execution through one
runtime access contract.

### Changed

- Introduced `ExecutableAccess` as the canonical runtime access request in
  `db::executor::stream::access`, carrying:
  - executable access plan
  - stream bindings
  - physical fetch hint and optional index predicate execution
- Removed `AccessExecutionDescriptor` and its constructor wrappers.
- Replaced routed key-stream request payloads from descriptor-based variants to
  `RoutedKeyStreamRequest::ExecutableAccess(...)`.
- Updated route/executor stream resolution boundaries so context traversal now
  consumes `ordered_key_stream_from_runtime_access(...)`.
- Updated scalar fallback, fast-stream handlers, aggregate streaming folds, and
  bytes streaming terminals to construct `ExecutableAccess` directly from
  planner access plans (`access.resolve_strategy().into_executable()`), with no
  descriptor indirection.

### Validation

- `cargo fmt --all`
- `cargo check -p icydb-core --locked`
- `cargo test -p icydb-core db::executor::scan::fast_stream::tests:: --locked`
- `cargo test -p icydb-core db::executor::tests::route:: --locked`
- `cargo test -p icydb-core db::executor::tests::pagination::index_range_matrix:: --locked`
- `rg -n "AccessExecutionDescriptor" crates/icydb-core/src` (no matches)

### Line Status

- Slice H is in progress; this patch completes the execution-descriptor
  collapse step.

## 0.51.1 - 2026-03-12 - Slice F Completion (A-F Locked)

### Summary

`0.51.1` advances the `0.51` stabilization track through Slice F and locks the
remaining planner/executor numeric-core contracts needed before storage
invariant freeze work.

### Changed

- Completed stabilization coverage through Slice F:
  deterministic plan/explain snapshots (A), ordering contracts (B),
  continuation envelope contracts (C), projection determinism (D), aggregate
  strategy contracts with grouped `AVG` support (E), and numeric core
  unification (F).
- Consolidated numeric arithmetic authority under shared decimal helpers and
  routed scalar/grouped aggregate numeric reducers through that single surface.
- Consolidated numeric comparison authority for planner range bounds and
  aggregate field comparisons through shared numeric-first comparator helpers.
- Locked grouped ordering comparators and cursor-facing canonical ordering to
  shared numeric-or-strict compare authority with deterministic canonical
  fallback.
- Added index-ordering comparator parity guards so semantic component ordering
  and encoded-byte ordering remain aligned under the same numeric/strict
  comparison contract.

## 0.51.0 - 2026-03-12 - Engine Contract Stabilization Line Open

### Summary

`0.51.0` opens the `0.51` line by freezing the first engine-stabilization
contracts and adding deterministic regression surfaces for planner/executor
shape identity.

### Changed

- Added the `0.51` design baseline with explicit contract slices for planner,
  executor, continuation, projection, aggregate strategy, numeric core, and
  storage invariants.
- Added canonical executable-plan snapshot serialization for
  `query -> executable plan -> explain`, with deterministic field ordering and
  map-iteration-independent output.
- Added Slice A snapshot baselines for four core execution shapes:
  by-key, secondary-index ordered, index-range, and grouped aggregate.
- Tightened snapshot identity metadata to include compact index summaries and
  planner-relevant execution fields (`index`, `bound_type`,
  `ordering_direction`, `projection_coverage_flag`).
- Added Slice B deterministic-ordering contract coverage in planner semantics
  tests for:
  - rejecting `LIMIT` without `ORDER BY`
  - requiring both `ORDER BY` and `LIMIT` for continuation paging
  - appending primary-key tie-breaks for non-unique ordering keys
  - grouped canonical ordering when grouped pagination has no explicit order
- Added Slice C continuation-envelope regressions for fluent pagination
  surfaces, including:
  - replay determinism (`identical query + identical cursor`) for scalar and
    grouped continuation paths
  - strict scalar cursor advancement/suffix progression checks
  - cross-shape fail-closed replay checks for scalar semantically equivalent
    shapes with shape-local continuation signatures
  - fail-closed unsupported cursor token version checks at scalar and grouped
    fluent API boundaries

# IcyDB 0.51

This document aggregates detailed release notes for the `0.51.x` minor line.

## 0.51.0 - 2026-03-12 - Engine Contract Stabilization Line Open

### Summary

`0.51.0` opens the `0.51` line by freezing the first engine-stabilization
contracts and adding deterministic regression surfaces for planner/executor
shape identity.

### Changed

- Added the `0.51` design baseline with explicit contract slices for planner,
  executor, continuation, projection, aggregate strategy, numeric core, and
  storage invariants.
- Added canonical executable-plan snapshot serialization for
  `query -> executable plan -> explain`, with deterministic field ordering and
  map-iteration-independent output.
- Added Slice A snapshot baselines for four core execution shapes:
  by-key, secondary-index ordered, index-range, and grouped aggregate.
- Tightened snapshot identity metadata to include compact index summaries and
  planner-relevant execution fields (`index`, `bound_type`,
  `ordering_direction`, `projection_coverage_flag`).
- Added Slice B deterministic-ordering contract coverage in planner semantics
  tests for:
  - rejecting `LIMIT` without `ORDER BY`
  - requiring both `ORDER BY` and `LIMIT` for continuation paging
  - appending primary-key tie-breaks for non-unique ordering keys
  - grouped canonical ordering when grouped pagination has no explicit order
- Added Slice C continuation-envelope regressions for fluent pagination
  surfaces, including:
  - replay determinism (`identical query + identical cursor`) for scalar and
    grouped continuation paths
  - strict scalar cursor advancement/suffix progression checks
  - cross-shape fail-closed replay checks for scalar semantically equivalent
    shapes with shape-local continuation signatures
  - fail-closed unsupported cursor token version checks at scalar and grouped
    fluent API boundaries

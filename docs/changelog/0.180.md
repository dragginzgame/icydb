# IcyDB 0.180 Changelog

## 0.180.4

### 📝 Summary

- Hard-cuts public canister error output to numeric-only `Error` records:
  `code: nat16`, `class: nat8`, and `origin: nat8`.
- Removes rich diagnostic enum/detail trees from public Candid output while
  keeping Rust-side diagnostic reconstruction available for CLI and host tools.
- Renumbers public diagnostic leaf codes into one compact sequential
  pre-1.0 range.

### ⚠️ Breaking

- Public Candid error records no longer include `detail`, and the remaining
  `code`, `class`, and `origin` fields are numeric wire values.
- Callers that previously matched Candid diagnostic variants should compare
  numeric `code` values or use host-side rendering through IcyDB tooling.

### 🔧 Changed

- Collapses broad diagnostic codes plus structured diagnostic details into one
  public numeric leaf code before errors cross the canister boundary.
- Keeps rich diagnostic details internal to core/Rust APIs so the CLI can still
  render useful human-readable messages without exposing the full diagnostic
  graph in every canister interface.
- Removes the diagnostic-code crate's Candid/serde wire feature and optional
  wire dependencies from the production facade path.
- Makes public error-code raw values sequential from `1..=124`, with test
  coverage guarding that compact layout before 1.0.

### 📊 Audit

- Refreshes the wasm-footprint audit after the wire-shape hard cut.
- The audited canisters are now smaller than the 0.179.5 baseline by about
  5.9-7.0 KiB raw `.wasm` each, and about 19-22 KiB raw `.wasm` smaller than
  the pre-hard-cut 0.180.3 audit snapshot.
- Regenerated audit fixture Candid files no longer contain the rich diagnostic
  enum/type names.

### 🧪 Testing

- Adds facade coverage for the numeric-only `Error` Candid shape and detail
  reconstruction through `Error::diagnostic()`.
- Updates SQL canister integration assertions to compare numeric leaf error
  codes instead of Candid diagnostic detail variants.
- Validates the slice with focused crate tests, SQL canister test compilation,
  workspace clippy, repository invariants, and the wasm-footprint audit.

## 0.180.3

### 📝 Summary

- Hardens CI and developer bootstrap downloads for actionlint and rustup.
- Adds a CI invariant that protects mutation/publication atomicity from future
  async or re-entrant interleaving points.

### 🧹 Cleanup

- Makes the actionlint installer reuse an already-installed pinned binary
  before attempting a network download.
- Downloads actionlint release archives to a temporary file with retries before
  validating and extracting them, avoiding direct `curl | tar` behavior on
  partial HTTP failures.
- Downloads the rustup bootstrap script to a temporary file with retries before
  executing it, avoiding direct `curl | sh` behavior on partial HTTP failures.
- Adds a mutation atomicity invariant to the existing `check-invariants` path,
  rejecting async functions, await points, spawned work, timers, and canister
  calls in guarded mutation/publication/generated mutation paths.

### 🧪 Testing

- Validates the new atomicity guard through the normal invariant suite.
- Adds a local cached-binary smoke check for the actionlint installer so the
  pinned-version reuse path is covered without requiring network access.

## 0.180.2

### 📝 Summary

- Adds compact runtime-boundary diagnostic details for generated SQL/schema
  endpoint rejections and public response-shape errors.
- Keeps canister-facing error payloads code/detail-first while leaving
  human-readable boundary text in the CLI renderer.
- Trims unused internal Candid derives without changing public SQL catalog or
  input/output value DTO wire shapes.

### 🔧 Changed

- Adds a compact runtime-boundary detail family for generated SQL controller
  gates, generated schema controller gates, SQL entity dispatch failures,
  targetless SQL DDL, public query response shape mismatches, mutation result
  shape mismatches, and explicit row-projection field mismatches.
- Routes generated SQL/schema endpoint helpers and public response-shape helper
  errors through the compact runtime-boundary detail instead of returning only
  broad runtime unsupported diagnostics.
- Extends CLI diagnostic rendering so host tools display useful text for the
  new runtime-boundary details without linking that prose into canister-facing
  crates.

### 🧹 Cleanup

- Removes an unused Candid derive from the catalog count helper while keeping
  SQL catalog result DTOs wire-compatible.
- Removes unused Candid derives from the internal runtime value union and enum
  payload helper; public input/output value DTOs keep their wire derives.

### 🧪 Testing

- Adds facade and CLI renderer coverage for compact runtime-boundary detail
  preservation.
- Tightens SQL canister integration assertions so interface-origin generated
  SQL DDL rejects preserve compact runtime-boundary detail across Candid.

## 0.180.1

### 📝 Summary

- Hard-cuts the public `icydb::Error` payload from text-first
  `kind`/`origin`/`message` fields to compact `code`/`class`/`origin`/`detail`
  fields.
- Routes public conversions from core query and runtime errors through
  owner-provided compact diagnostic mappings instead of reconstructing broad
  public kinds from rendered messages.
- Adds CLI-owned text rendering for compact diagnostic payloads so host tools
  display useful messages without linking text rendering into canister-facing
  crates.
- Updates generated endpoint and response helpers to construct compact public
  errors without formatting public error prose.

### ⚠️ Breaking

- Public Candid error records no longer include `message` or `kind`.
- Callers should inspect `code`, `class`, `origin`, and `detail`, then render
  text in host/CLI tooling when human output is needed.

### 🔧 Changed

- Adds core-owned compact mappings for structured error details, schema DDL
  admission reasons, SQL unsupported-feature labels, store details, and query
  error categories.
- Keeps `Error::diagnostic()` as the round-trip public diagnostic accessor for
  callers that want the compact payload as one value.
- Routes decoded shell and observability endpoint errors through the CLI
  renderer, with recovery hints still applied after rendering.

### 🧪 Testing

- Updates facade Candid-shape tests to require the compact wire fields and to
  reject the removed text-first fields.
- Adds focused core and query tests for compact detail preservation across DDL
  admission, SQL unsupported-feature, response-cardinality, and runtime error
  conversions.
- Adds focused CLI renderer tests for schema DDL admission, unsupported SQL
  feature, and detail-free runtime diagnostics.

## 0.180.0

### 📝 Summary

- Starts the compact diagnostics line with a dedicated diagnostic-code crate
  that can be shared by canisters, the public facade, and future CLI rendering.
- Adds a public `Error::diagnostic()` bridge so callers can begin asserting
  compact diagnostic identity before the later production wire hard cut.
- Keeps the public error wire payload unchanged for this first slice.

### ➕ Added

- Adds compact diagnostic codes for query, schema DDL admission, store, and
  runtime failure families.
- Adds compact detail enums for query kind, runtime kind, unsupported SQL
  feature identity, and schema DDL admission reason identity.
- Adds optional wire derives behind the diagnostic-code crate's `wire` feature
  so production and local/debug surfaces can choose the appropriate payload.

### 🧭 Migration Notes

- This slice is additive. Existing public `Error` Candid records still expose
  `kind`, `origin`, and `message`.
- Callers can start using `Error::diagnostic()` and
  `Error::diagnostic_code()` as the migration path toward code-first error
  handling.

# IcyDB 0.173 Changelog

## 0.173.5

### 🔧 Changed

- Closes the journaled protocol-foundation proof matrix without admitting
  public `storage(journaled(...))` runtime storage.
- Records exact closeout evidence for reserved journaled syntax, runtime
  non-admission, embedded commit-marker journal publication, journal-sequence
  ordering, fold watermark semantics, schema replay, index materialization,
  stable behavior, heap volatility, codec preservation, and 0.174 admission
  gates.
- Keeps 0.174 responsible for public derive admission, generated journaled
  wiring, runtime store construction, recovery, fold, diagnostics, relation
  behavior, and final source-audit closeout.

### 🧪 Testing

- Adds a source/design guard proving the 0.173 closeout keeps journaled runtime
  admission absent and leaves the public admission gate in 0.174.
- Records closeout validation across formatting, journaled derive/UI rejection,
  write-boundary guards, focused commit/recovery/codec/capability/registry/
  diagnostics filters, workspace check, core clippy, source audit, and diff
  hygiene.

## 0.173.4

### 🔧 Changed

- Defines future journaled index truth: data and schema are logical durable
  truth, while the index BTree is deterministic materialized state derived from
  recovered rows and accepted schema.
- Defines schema replay ordering: canonical stable schema snapshots load first,
  committed journal `SchemaPut` records apply in journal order, and row records
  resolve only through the reconstructed accepted schema timeline.
- Records fail-closed behavior for row/schema fingerprint mismatches,
  out-of-order schema references, index divergence, and unavailable accepted
  schema authority.

### 🧪 Testing

- Adds a source/design guard proving schema replay and index truth are explicit
  and do not rely on generated-model fallback or independent index journal
  truth.
- Closes proof rows P9 and P10 only; full protocol-foundation closeout remains
  the 0.173.5 responsibility.

## 0.173.3

### 🔧 Changed

- Defines the fold watermark as the durable replay boundary for future
  journaled stores.
- Records the planned watermark shape as
  `highest_folded_journal_sequence + fold_epoch`.
- Defines replay and restart behavior for folded batches, missing post-watermark
  batches, persisted fold intent, partial stable-BTree apply, watermark
  persistence, and tail cleanup.
- Clarifies that tail cleanup is space reclamation, not durability authority.

### 🧪 Testing

- Adds a source/design guard proving the fold watermark and replay-boundary
  model is explicit before runtime admission.

## 0.173.2

### 🔧 Changed

- Chooses monotonic `journal_sequence` as the durable journal-tail ordering
  authority for the future journaled store.
- Defines the conceptual journal-tail shape as
  `journal_sequence -> encoded JournalBatch`, with `batch_id` reserved for
  idempotence and `commit_marker_id` reserved for visibility binding.
- Records fail-closed handling for duplicate sequence values, duplicate batch
  IDs, sequence gaps, out-of-order batches, marker mismatches, and folded
  batches still visible past the fold watermark.

### 🧪 Testing

- Adds a source/design guard proving the journal batch envelope and tail
  ordering model are no longer left as open choices.

## 0.173.1

### 🔧 Changed

- Chooses embedded bounded journal batch payload in the existing commit marker
  as the 0.173 journal publication strategy.
- Records recovery behavior for missing, partial, duplicate, mismatched,
  already-present, and bytes-present-but-not-committed journal batches.
- Keeps the strategy design-only: 0.173 still does not admit public
  `storage(journaled(...))`, generated journaled wiring, runtime journaled
  handles, or journal codecs reachable from runtime stores.

### 🧪 Testing

- Adds a source/design guard proving the 0.173 strategy is explicitly chosen
  and no longer left as a candidate list.

## 0.173.0

### ➕ Added

- Starts the journaled protocol-foundation line without admitting public
  `storage(journaled(...))` runtime storage.
- Adds the active 0.173 design for resolving journaled commit-marker
  publication, journal batch identity, tail ordering, fold watermark, schema
  replay, and index-materialization rules before runtime admission.
- Adds the 0.174 handoff design for future journaled cached-stable runtime
  admission after the 0.173 protocol foundation closes.

### 🔧 Changed

- Splits the previous journaled runtime-admission draft into:
  - `0.173`: protocol foundation, no public runtime admission;
  - `0.174`: future runtime admission after protocol closeout.
- Records the 0.173 boundary inventory across schema/derive, generated wiring,
  registry/capabilities, store wrappers, commit/recovery, diagnostics,
  relation, and source-audit seams.
- Keeps journaled storage parser-reserved: accepted schemas, generated wiring,
  runtime store handles, diagnostics rows, relation branches, and store
  constructors remain stable/heap only.

### 🧪 Testing

- No runtime validation is required for this design-only setup slice.
- The design records the closeout validation and source-audit commands required
  before 0.173 can be marked closed.

# IcyDB 0.26

This document aggregates detailed release notes for the `0.26.x` minor line.

---

## 0.26.2 - 2026-02-23 - Runtime Slot Hardening Completion

### 📝 Summary

`0.26.2` closes the compiled field-projection hardening line by removing remaining runtime field-name lookups in hot query paths.

### 🔧 Changed

- Predicate filtering now compiles predicates into slot-resolved runtime form once per query, then evaluates rows through `get_value_by_index` only.
- Removed remaining shared `field_index` runtime lookups and consolidated slot resolution behind shared model helpers.
- Ordering, cursor-boundary comparison, and predicate filtering now consistently operate on pre-resolved field slots in execution loops.

---

## 0.26.1 - 2026-02-23 - Field Projection Guardrails

### 📝 Summary

`0.26.1` hardens the `0.26.0` slot-based projection rollout with stronger invariants and CI guardrails.

### 🔧 Changed

- `IndexKey::new` now classifies missing declared index fields as explicit invariant violations instead of skipping index-key materialization.
- CI enforces a string-free runtime field-projection contract with dedicated invariant checks for runtime modules.
- Ordering and predicate runtime paths now resolve field names to schema slots once per phase and evaluate rows through slot reads.

### 🧪 Testing

- Added commit/index regression coverage that locks missing-index-field behavior to explicit invariant-violation classification.
- Added cross-crate field-projection invariant coverage asserting `EntityModel.fields` order matches `get_value_by_index` slot output.

---

## 0.26.0 - 2026-02-23 - Compiled Field Projection

### 📝 Summary

`0.26.0` moves runtime field projection from name lookups to slot/index access.

### ⚠️ Breaking

- Removed `FieldProjection::get_value(&self, field: &str)`.
- `FieldProjection` now requires only `get_value_by_index(&self, index: usize) -> Option<Value>`.

### 🔧 Changed

- Field-target aggregate terminals now resolve target fields once into stable field slots and read values by index during execution.
- Field aggregate reducers (`min_by`, `max_by`, `nth_by`, `median_by`, `sum_by`, `avg_by`, `count_distinct_by`, `min_max_by`) now use index projection in both materialized and streaming paths.
- Additional runtime paths now use index projection, including index-key materialization, ordering/cursor field access, and predicate row reads.

### 🧭 Migration Notes

- If you implement `FieldProjection` manually, `get_value_by_index` must follow declared schema field order.

```rust
impl FieldProjection for MyEntity {
    fn get_value_by_index(&self, index: usize) -> Option<Value> {
        match index {
            0 => Some(self.id.to_value()),
            1 => Some(self.name.to_value()),
            _ => None,
        }
    }
}
```

### 🧪 Testing

- Added slot-resolution regression coverage in aggregate field helpers to lock schema-order index mapping.
- Kept aggregate field parity coverage passing under slot-based projection.

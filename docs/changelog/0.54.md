# IcyDB 0.54

This document aggregates detailed release notes for the `0.54.x` minor line.

## 0.54.0 - 2026-03-13 - Filtered Index Metadata Foundation

### Summary

`0.54.0` starts the filtered/conditional-index line by introducing optional
index-predicate metadata across schema and runtime index models.

This patch is a metadata foundation only: planner implication gates and
mutation-path conditional index maintenance remain in follow-up slices.

### Changed

- Extended schema index metadata to carry optional predicate text:
  - `schema::node::Index` now supports
    `predicate: Option<&'static str>`
  - added `Index::new_with_predicate(...)` and `Index::predicate()`
  - display output now includes `WHERE ...` when predicate metadata is present.
- Extended runtime index metadata to carry optional predicate text:
  - `model::index::IndexModel` now supports
    `predicate: Option<&'static str>`
  - added `IndexModel::new_with_predicate(...)` and `IndexModel::predicate()`
  - display output now includes `WHERE ...` when predicate metadata is present.
- Extended schema-derive index codegen to emit optional predicate metadata into
  both schema-node and runtime index models.
- Added module-local tests that lock predicate/non-predicate accessor and
  display shape behavior in both schema and runtime index models.
- Added `0.54` design status tracking artifact:
  - `/docs/design/0.54-conditional-indexes/0.54-status.md`

### Validation

- `cargo test -p icydb-schema --locked node::index::tests:: -- --nocapture`
- `cargo check -p icydb-schema --locked`

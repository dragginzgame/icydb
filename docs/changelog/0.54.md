# 0.54 Changelog

## 0.54.1 - 2026-03-14 - SQL Dispatch + Canister Query Surface

### Summary

`0.54.1` makes canister-local SQL integration simpler and more deterministic by
generating one actor-owned SQL dispatch surface, improving SQL endpoint output
shapes, and tightening SQL-surface diagnostics for common developer errors.

### Changed

- Added generated actor-local SQL dispatch helpers under `sql_dispatch`:
  - entity resolution: `from_statement_sql(...)`, `from_entity_name(...)`
  - route helpers: `entities()`, `projection_rows(...)`, `explain(...)`
  - convenience execution surfaces: `query(...)`, `query_rows(...)`
- Added one structured SQL endpoint payload contract:
  - `icydb::db::sql::SqlQueryRowsOutput` (`entity`, `columns`, `rows`,
    `row_count`)
  - conversion helper `SqlQueryRowsOutput::from_projection(...)`
- Added one canister-facing SQL integration path in docs:
  - README canister section now shows `sql_entities`, `query`, and `query_rows`
    endpoint wiring with generated `sql_dispatch`.
- Tightened SQL dispatch behavior to be deterministic and entity-keyed:
  - route resolution is based on parsed SQL entity name
  - unsupported entity errors are immediate and include supported entity list
  - no fallback trial chain / last-error accumulation behavior.
- Improved EXPLAIN failure legibility while preserving execution parity:
  - unordered pagination remains rejected as
    `QueryErrorKind::UnorderedPagination`
  - SQL-surface message now includes wrapped SQL and stable-order fix hint.
- Tightened reduced SQL parser diagnostics for clause-order mistakes:
  - `SELECT ... LIMIT/OFFSET ... ORDER BY ...` now returns actionable syntax
    guidance
  - `DELETE ... LIMIT ... ORDER BY ...` now returns actionable syntax guidance
  - parser tests moved to a split module (`parser/mod.rs` + `parser/tests.rs`)
    to keep parser ownership and test coverage easier to maintain.
- Hardened generated actor clippy cleanliness:
  - generated `SqlEntityRoute` impl uses `Self::...` variant references
  - route lookup no longer emits chained `if` patterns that trigger
    `possible_missing_else`
  - generated `primary_key_field` surface is `const`
  - generated `icydb_snapshot()` surface no longer emits
    `needless_question_mark`.

### Validation

- `cargo fmt --all`
- `cargo test -p icydb-core db::sql::parser::tests -- --nocapture`
- `cargo clippy -p canister_demo_rpg --tests -- -D warnings`
- `cargo test -p canister_demo_rpg -- --nocapture`
- `cargo test -p icydb-testing-integration --test sql_canister -- --nocapture`

## 0.54.0 - 2026-03-13 - Filtered / Conditional Index Runtime

### Summary

`0.54.0` delivers the full filtered/conditional-index runtime slice set
(A-E): metadata model extension, schema-time predicate validation boundaries,
mutation-path conditional index maintenance, planner implication gating, and
recovery parity/matrix locks.

### Changed

- Added optional index predicate metadata across schema/runtime index models and
  schema-derive codegen:
  - `schema::node::Index` and `model::index::IndexModel` now support
    `predicate: Option<&'static str>`
  - added `new_with_predicate(...)` + `predicate()` accessors
  - index display shapes now include `WHERE ...` when metadata is present.
- Added schema-time predicate validation boundaries for index metadata:
  - new parser boundary `db::sql::parser::parse_sql_predicate(...)`
  - schema validation now parses + validates index predicates against entity
    schema contracts
  - explicit error taxonomy:
    - `ValidateError::InvalidIndexPredicateSyntax`
    - `ValidateError::InvalidIndexPredicateSchema`
- Added mutation-path conditional index maintenance:
  - index mutation planning now treats predicate evaluation as membership gating
    for old/new index key synthesis
  - commit-prepare delta classification now uses the same membership gating to
    keep marker/forward semantics aligned
  - conditional unique checks now apply only for predicate-true rows.
- Added planner implication gating for filtered-index eligibility:
  - index candidates are filtered before route-shape selection unless
    `query_predicate ⇒ index_predicate`
  - deterministic implication checks cover strict compare/bound forms
    (fail-closed for unsupported forms).
- Added end-to-end conditional-index matrix and recovery parity coverage:
  - live apply vs replay snapshot parity for full membership transition matrix
  - conditional-unique conflict classification parity between live update and
    recovery replay
  - startup recovery rebuild coverage that drops stale/predicate-false entries
    and restores predicate-true membership from authoritative rows.
- Updated the 0.54 status tracker to completed state (`A-E`):
  - `/docs/design/0.54-conditional-indexes/0.54-status.md`

# 0.149.x

## 0.149.17

`0.149.17` removes executor-authority generated schema reopening from runtime
plan validation. Executor-bound plans now have to carry planner-frozen static
shape, which session planning already builds from accepted schema metadata, while
executor validation preserves schema-free access invariants such as inverted
key-range rejection.

### Executor validation authority

- Removed `EntityAuthority::schema_info()` so executor validation cannot reopen
  `SchemaInfo::cached_for_entity_model(self.model)` at runtime.
- Changed executor plan validation to fail closed when a plan reaches the
  executor without finalized static planning shape, instead of recomputing
  access validation from generated schema metadata.
- Kept defensive runtime access validation for schema-free structural
  invariants, including inverted primary-key and index ranges, without
  reopening generated `SchemaInfo`.
- Extended write-boundary guards to require executor validation to depend on
  planner-frozen static shape, schema-free access invariants, and to reject
  `EntityAuthority::schema_info()`.

Accepted structural writes now encode updated slots and dense after-images
through the accepted field contract selected by the persisted schema snapshot,
while generated-only callers keep the existing generated field-model
compatibility encoders.

### Structural patch serialization

- Split the mixed sparse patch serializer into generated-only and
  accepted-contract helpers.
- Made accepted sparse patch serialization reject missing accepted slot
  contracts instead of falling through to generated field metadata.
- Kept generated-only patch serialization on the generated field-model encoder
  for compatibility callers that have not selected an accepted schema snapshot.
- Renamed the complete accepted after-image serializer helper so insert/replace
  default filling is visibly accepted-contract-only.
- Split dense runtime-value row emission into accepted-contract and
  generated-contract helpers.
- Routed accepted serialized patch replay through the accepted dense row
  emitter so old-row overlays have no generated field fallback at final row
  emission.
- Split mutation write-bound validation into accepted-contract and
  generated-contract helpers so structural patch and persisted-row invariant
  checks no longer branch between schema authorities inside their slot loops.

  field has a database default.

  fields remain rejected.

## 0.149.2

`0.149.2` adds the first schema macro authoring surface for database defaults.
Supported literal `default = ...` values now also become encoded database
defaults, `db_default = ...` can override that database value, and function or
path defaults such as `Ulid::generate` remain Rust construction-only.
Accepted structural insert/replace staging also now fills omitted defaulted
fields from accepted schema defaults before typed materialization, and schema
description rows now report encoded database-default metadata.

### Macro-authored database defaults

- Added a `db_default = ...` field attribute for explicitly overriding the
  database default separately from the Rust construction default.
- Made supported literal `default = ...` values emit database defaults by
  default, while function/path defaults remain outside database policy.
- Encoded supported primitive default literals directly into canonical
  persisted slot payload bytes during derive expansion.
- Started the supported surface narrowly with required primitive fields:
  `Bool`, bounded or unbounded `Text`, fixed-width signed integers,
  fixed-width unsigned integers, `Float32`, and `Float64`.
- Added fail-closed validation for unsupported default targets, optional or
  repeated fields, text length overflow, incompatible literal kinds, and
  out-of-range numeric defaults.
- Extended schema AST field metadata so explicit `db_default` values are
  visible in schema descriptions separately from Rust defaults.
- Added macro-test coverage proving explicit database defaults reach
  `FieldDatabaseDefault::EncodedSlotPayload(...)`, supported literal
  construction defaults also emit encoded default payloads, and function/path
  defaults still produce `FieldDatabaseDefault::None`.
- Tightened additive transition policy so explicit persisted default payloads
  must decode through the accepted field kind and must not decode as `NULL`.
- Routed accepted structural insert/replace serialization through a complete
  after-image serializer that fills omitted nullable/defaulted slots from the
  accepted row contract while preserving sparse overlay semantics for updates.
- Added database-default metadata to generated and accepted schema description
  rows as `default=slot_payload(bytes=N)`, keeping the display byte-oriented
  instead of decoding schema defaults into runtime values.

## 0.149.1

`0.149.1` adds the first persisted database-default contract. Defaults are
stored as already-encoded slot payload bytes in schema metadata, carried through
the schema snapshot codec, and projected into accepted row decode contracts so
missing appended slots can materialize from schema-owned bytes.

### Encoded defaults

- Added `SchemaFieldDefault::SlotPayload(...)` as the persisted-schema default
  representation. This keeps defaults tied to field codec bytes instead of
  storing runtime `Value` data in schema metadata.
- Added `FieldDatabaseDefault::EncodedSlotPayload(...)` as the generated model
  bridge for future explicitly authored database defaults. The derive still
  emits `None`, so Rust construction defaults remain outside database policy.
- Extended accepted row decode contracts to carry default payloads and added
  `DefaultIfMissing` as the explicit missing-slot policy for defaulted fields.
- Made structural row contracts decode missing defaulted slots through the
  accepted field decode contract.
- Allowed append-only generated fields with an explicit persisted default
  payload to use the same additive transition plan as nullable append-only
  fields.
- Added tests for schema snapshot default round-tripping, additive defaulted
  transition admission, and old-row default materialization.
- Added malformed default-payload coverage so bad schema-owned default bytes
  fail at old-row materialization instead of being treated as trusted values.

## 0.149.0

`0.149.0` starts the schema-defaults and accepted-write-encoding line by moving
structural runtime-value encoding onto accepted field contracts. Structural
patch writes that already carry an accepted row contract can now encode through
saved `PersistedFieldKind` metadata instead of adapting back through generated
field models.

### Accepted write encoding

- Added an accepted structural-field encoder that consumes
  `PersistedFieldKind` and shares the same low-level binary primitives as the
  generated-compatible structural encoder.
- Added an accepted runtime-value field-contract encoder for slot payloads
  selected by `AcceptedFieldDecodeContract`, including value-storage,
  scalar-slot, and structural fallback lanes.
- Routed accepted structural patch serialization through the accepted field
  encoder when a row contract is attached, while keeping generated-only patch
  serialization on the existing generated-compatible path.
- Preserved authored decimal normalization for accepted field contracts, so
  runtime values are normalized to the saved decimal scale before persisted
  bytes are emitted.
- Kept `Value` as a runtime boundary type only. Persisted rows remain
  slot-framed bytes owned by field codecs, not dynamic `Value` payloads.
- Added accepted-vs-generated encoder parity coverage for simple, recursive,
  and relation-list structural field shapes.
- Added direct accepted structural patch coverage for decimal scale
  normalization.

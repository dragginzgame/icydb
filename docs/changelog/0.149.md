# 0.149.x

## 0.149.1

`0.149.1` adds the first persisted database-default contract. Defaults are
stored as already-encoded slot payload bytes in schema metadata, carried through
the schema snapshot codec, and projected into accepted row decode contracts so
missing appended slots can materialize from schema-owned bytes.

### Encoded defaults

- Added `SchemaFieldDefault::SlotPayload(...)` as the persisted-schema default
  representation. This keeps defaults tied to field codec bytes instead of
  storing runtime `Value` data in schema metadata.
- Added `FieldDatabaseDefault::EncodedSlotPayload(...)` as the generated model
  bridge for future explicitly authored database defaults. The derive still
  emits `None`, so Rust construction defaults remain outside database policy.
- Extended accepted row decode contracts to carry default payloads and added
  `DefaultIfMissing` as the explicit missing-slot policy for defaulted fields.
- Made structural row contracts decode missing defaulted slots through the
  accepted field decode contract.
- Allowed append-only generated fields with an explicit persisted default
  payload to use the same additive transition plan as nullable append-only
  fields.
- Added tests for schema snapshot default round-tripping, additive defaulted
  transition admission, and old-row default materialization.
- Added malformed default-payload coverage so bad schema-owned default bytes
  fail at old-row materialization instead of being treated as trusted values.

## 0.149.0

`0.149.0` starts the schema-defaults and accepted-write-encoding line by moving
structural runtime-value encoding onto accepted field contracts. Structural
patch writes that already carry an accepted row contract can now encode through
saved `PersistedFieldKind` metadata instead of adapting back through generated
field models.

### Accepted write encoding

- Added an accepted structural-field encoder that consumes
  `PersistedFieldKind` and shares the same low-level binary primitives as the
  generated-compatible structural encoder.
- Added an accepted runtime-value field-contract encoder for slot payloads
  selected by `AcceptedFieldDecodeContract`, including value-storage,
  scalar-slot, and structural fallback lanes.
- Routed accepted structural patch serialization through the accepted field
  encoder when a row contract is attached, while keeping generated-only patch
  serialization on the existing generated-compatible path.
- Preserved authored decimal normalization for accepted field contracts, so
  runtime values are normalized to the saved decimal scale before persisted
  bytes are emitted.
- Kept `Value` as a runtime boundary type only. Persisted rows remain
  slot-framed bytes owned by field codecs, not dynamic `Value` payloads.
- Added accepted-vs-generated encoder parity coverage for simple, recursive,
  and relation-list structural field shapes.
- Added direct accepted structural patch coverage for decimal scale
  normalization.

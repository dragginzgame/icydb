# 0.149.x

## 0.149.11

`0.149.11` makes accepted schema contracts mandatory on save and typed response
paths, then removes the generated structural-patch write fallbacks that were
kept only for the transition. Session write boundaries now pass accepted
row-decode contracts directly into `SaveExecutor`, structural mutations
serialize and replay through accepted field contracts only, typed and structural
after-image emission encodes final row bytes from accepted field contracts,
typed response materialization consumes the selected `RowLayout` contract
directly, and persisted-row tests build write fixtures via accepted schema
contracts instead of generated-model patch helpers.

### Save executor authority

- Made `SaveExecutor` store an `AcceptedRowDecodeContract` directly instead of
  `Option<AcceptedRowDecodeContract>`.
- Replaced production save construction with
  `SaveExecutor::new_with_accepted_contract(...)`.
- Kept the generated-model save constructor test-only, where it now constructs
  an accepted test contract from the generated schema proposal.
- Routed typed save before-image validation and structural mutation staging
  through accepted contracts without generated fallback branches.
- Routed typed and normalized structural after-image emission through
  `canonical_row_from_entity_with_accepted_contract(...)`, making accepted
  field contracts the final persisted row encoder for production saves.
- Routed structural insert/replace entity materialization through accepted row
  contracts as well, leaving the generated serialized-patch materializer as a
  test-only parity fixture.

### Structural patch write boundary

- Removed generated-model structural patch serialization and replay helpers.
- Removed raw-row patch replay helpers that accepted only `EntityModel`.
- Kept accepted-contract sparse update replay and complete insert/replace
  serialization as the production structural write paths.
- Updated write-boundary guard tests so they now require accepted-contract
  structural update builders instead of lane-specific generated/accepted
  builders.

### Persisted-row cleanup

- Removed dead generated-row canonicalization helpers exposed by the save
  cutover.
- Made the old generated complete-row writer and dense serialized-patch bridge
  test-only so parity fixtures can still compare against it without leaving it
  in production write paths.
- Removed obsolete generated structural patch replay tests or moved them onto
  accepted-schema fixture contracts.
- Added `decode_raw_row_for_entity_key_with_contract(...)` as the explicit
  typed materialization boundary for accepted row contracts.
- Routed typed response decoding through the selected `RowLayout` contract
  instead of trying generated decode first and accepted normalization second.
- Made `RawRow::try_decode<E>()` and generated-model
  `StructuralSlotReader::from_raw_row(...)` construction test-only.
- Removed no-longer-used direct accessors such as `RowLayout::primary_key_slot`,
  `RawRow::apply_serialized_structural_patch(...)`, and
  `SerializedStructuralPatch::is_empty()`.

- Relaxed prepared SQL normalization so expected entity names do not have to be

## 0.149.2

`0.149.2` adds the first schema macro authoring surface for database defaults.
Supported literal `default = ...` values now also become encoded database
defaults, `db_default = ...` can override that database value, and function or
path defaults such as `Ulid::generate` remain Rust construction-only.
Accepted structural insert/replace staging also now fills omitted defaulted
fields from accepted schema defaults before typed materialization, and schema
description rows now report encoded database-default metadata.

### Macro-authored database defaults

- Added a `db_default = ...` field attribute for explicitly overriding the
  database default separately from the Rust construction default.
- Made supported literal `default = ...` values emit database defaults by
  default, while function/path defaults remain outside database policy.
- Encoded supported primitive default literals directly into canonical
  persisted slot payload bytes during derive expansion.
- Started the supported surface narrowly with required primitive fields:
  `Bool`, bounded or unbounded `Text`, fixed-width signed integers,
  fixed-width unsigned integers, `Float32`, and `Float64`.
- Added fail-closed validation for unsupported default targets, optional or
  repeated fields, text length overflow, incompatible literal kinds, and
  out-of-range numeric defaults.
- Extended schema AST field metadata so explicit `db_default` values are
  visible in schema descriptions separately from Rust defaults.
- Added macro-test coverage proving explicit database defaults reach
  `FieldDatabaseDefault::EncodedSlotPayload(...)`, supported literal
  construction defaults also emit encoded default payloads, and function/path
  defaults still produce `FieldDatabaseDefault::None`.
- Tightened additive transition policy so explicit persisted default payloads
  must decode through the accepted field kind and must not decode as `NULL`.
- Routed accepted structural insert/replace serialization through a complete
  after-image serializer that fills omitted nullable/defaulted slots from the
  accepted row contract while preserving sparse overlay semantics for updates.
- Added database-default metadata to generated and accepted schema description
  rows as `default=slot_payload(bytes=N)`, keeping the display byte-oriented
  instead of decoding schema defaults into runtime values.

structural runtime-value encoding onto accepted field contracts. Structural

field models.

  `PersistedFieldKind` and shares the same low-level binary primitives as the
  generated-compatible structural encoder.

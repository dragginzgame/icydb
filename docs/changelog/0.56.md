# IcyDB 0.56

This document aggregates detailed release notes for the `0.56.x` minor line.

## 0.56.0 - 2026-03-15 - Reduced SQL Introspection + Entity Listing

### Summary

`0.56.0` adds first-class reduced-SQL introspection lanes
(`DESCRIBE <entity>` and `SHOW INDEXES <entity>`), keeps ownership staged
(`parser -> lowering -> session`), and ships helper/canister integration updates
so shell workflows can describe entities, list indexes, and enumerate supported
entities.

### Changed

- Added parser-owned SQL introspection statement support:
  - new `SqlStatement::Describe(SqlDescribeStatement)` statement lane.
  - new `SqlStatement::ShowIndexes(SqlShowIndexesStatement)` statement lane.
  - new reduced parser path for `DESCRIBE <entity>`.
  - new reduced parser path for `SHOW INDEXES <entity>`.
  - fail-closed unsupported-feature handling for invalid forms, including
    `EXPLAIN DESCRIBE ...` and unsupported `SHOW` variants.
- Extended lowered SQL command ownership with one introspection variant:
  - added `SqlCommand::DescribeEntity`.
  - added `SqlCommand::ShowIndexesEntity`.
  - DESCRIBE lowering validates entity identity using the same trailing-segment
    matching rule used by other SQL surfaces.
  - SHOW INDEXES lowering validates entity identity using the same
    trailing-segment matching rule.
  - both introspection lanes stay outside `Query<E>` lowering and planner
    row-shape execution.
- Extended SQL route classification authority:
  - added `SqlStatementRoute::Describe { entity }`.
  - added `SqlStatementRoute::ShowIndexes { entity }`.
  - `sql_statement_route(...)` now classifies query/explain/describe/show-indexes
    lanes.
- Added dedicated core/facade SQL introspection APIs:
  - core:
    `DbSession::describe_sql::<E>(&str) -> Result<EntitySchemaDescription, QueryError>`.
  - core: `DbSession::show_indexes_sql::<E>(&str) -> Result<Vec<String>, QueryError>`.
  - facade:
    `DbSession::describe_sql::<E>(&str) -> Result<EntitySchemaDescription, Error>`.
  - facade: `DbSession::show_indexes_sql::<E>(&str) -> Result<Vec<String>, Error>`.
  - lane boundaries are explicit and fail-closed:
    - `query_from_sql(...)` rejects DESCRIBE and SHOW INDEXES.
    - `explain_sql(...)` rejects DESCRIBE and SHOW INDEXES.
    - `describe_sql(...)` rejects non-DESCRIBE statements.
    - `show_indexes_sql(...)` rejects non-`SHOW INDEXES` statements.
- Updated generated canister `sql_dispatch` helper behavior for `0.56`:
  - `sql_dispatch::query(...)` now recognizes helper-level `SHOW ENTITIES`
    directly and returns deterministic shell lines:
    `surface=entities`, then `entity=<Name>`.
  - `sql_dispatch::query(...)` now also renders:
    - `DESCRIBE <entity>` as shell-friendly schema lines.
    - `SHOW INDEXES <entity>` as shell-friendly index-listing lines.
  - added generated helper surfaces:
    - `sql_dispatch::describe_schema(...)` (typed describe payload).
    - `sql_dispatch::describe(...)` (shell describe lines).
    - `sql_dispatch::show_indexes(...)` (shell index lines).
  - `sql_dispatch::query_rows(...)` and `sql_dispatch::projection_rows(...)`
    reject DESCRIBE and SHOW INDEXES with deterministic unsupported-lane errors.
- Added explicit canister-facing DESCRIBE endpoints:
  - `canisters/quickstart` and `canisters/minimal` now expose `describe(sql)` as
    a separate query method.
  - both canisters now delegate DESCRIBE rendering to generated
    `sql_dispatch::describe(...)` helpers.
- Updated SQL shell workflow and docs:
  - `scripts/dev/sql.sh` now auto-selects `--method describe` for DESCRIBE input
    (unless caller explicitly overrides method).
  - README SQL Quickstart now includes DESCRIBE, `SHOW ENTITIES`, and
    `SHOW INDEXES` examples.
  - SQL subset contract docs now include DESCRIBE + SHOW INDEXES lanes and
    generated-helper boundaries for `0.56`.
- Added non-regression coverage:
  - parser/lowering/session route tests for DESCRIBE + SHOW INDEXES
    classification and lowering.
  - canister integration tests for helper-level `SHOW ENTITIES` behavior and
    generated query-lane DESCRIBE + SHOW INDEXES behavior while keeping
    projection endpoints fail-closed for introspection lanes.

## 0.56.1 - 2026-03-16 - Describe/Show Contract Hardening

### Summary

`0.56.1` hardens the reduced-SQL introspection lanes by freezing shell-render
output contracts for `DESCRIBE` and `SHOW INDEXES`, removing version-coupled
generated-helper error wording, and expanding canister integration coverage for
normalized introspection SQL inputs.

### Changed

- Added output-contract vector tests in `icydb` SQL rendering helpers:
  - `render_describe_lines_output_contract_vector_is_stable`
  - `render_show_indexes_lines_output_contract_vector_is_stable`
- Updated generated helper unsupported-lane wording to remove release-line
  coupling:
  - changed `query_rows ... not available ... for 0.56` to endpoint-scoped
    generic wording.
- Expanded PocketIC canister integration coverage for introspection SQL
  normalization:
  - mixed-case keyword acceptance (`dEsCrIbE`, `sHoW InDeXeS`)
  - schema-qualified entity acceptance (`public.Character`)
  - optional trailing semicolon acceptance (`...;`)
  - coverage added for both explicit `describe` endpoint and generated `query`
    rendering lane.

### Validation

- `cargo fmt --all`
- `cargo test -p icydb db::sql::tests -- --nocapture`
- `POCKET_IC_BIN="$(bash scripts/ci/ensure-pocket-ic-bin.sh)" cargo test -p icydb-testing-integration --test sql_canister -- --nocapture`

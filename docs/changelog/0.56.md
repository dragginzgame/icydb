# IcyDB 0.56

This document aggregates detailed release notes for the `0.56.x` minor line.

## 0.56.5 - 2026-03-16 - SHOW COLUMNS + Grouped HAVING NULL + Expression Prefix Eligibility

### Summary

`0.56.5` bundles three bounded expansions on top of `0.56.4`:
`SHOW COLUMNS <entity>` introspection lane support, grouped SQL `HAVING`
`IS NULL` / `IS NOT NULL` support, and expression-index casefold
starts-with eligibility for existing `LOWER(field)` index families.

### Changed

- Added parser-owned `SHOW COLUMNS` statement support:
  - new `SqlStatement::ShowColumns(SqlShowColumnsStatement)` lane.
  - new parse path for `SHOW COLUMNS <entity>`.
  - stable fail-closed labels for unsupported `SHOW` families now include
    `SHOW COLUMNS`.
  - stable fail-closed label for modifier misuse:
    `SHOW COLUMNS modifiers`.
- Extended lowering with one dedicated command lane:
  - new `SqlCommand::ShowColumnsEntity`.
  - entity matching uses the same trailing-segment policy as other SQL
    entity-scoped lanes.
- Extended core/facade session surfaces:
  - route metadata now includes `SqlStatementRoute::ShowColumns { entity }`.
  - new typed helper:
    `show_columns::<E>() -> Vec<EntityFieldDescription>`.
  - new SQL entrypoint:
    `show_columns_sql::<E>(&str) -> Result<Vec<EntityFieldDescription>, ...>`.
  - centralized lane gating now includes deterministic wrong-lane rejection
    messages for `SHOW COLUMNS`.
- Extended unified query envelope and rendering:
  - `SqlQueryResult` now includes
    `ShowColumns { entity, columns: Vec<EntityFieldDescription> }`.
  - added deterministic shell rendering helper
    `render_show_columns_lines(...)`.
  - updated `scripts/dev/sql.sh` to render `ShowColumns` payloads.
- Extended generated canister SQL dispatch:
  - `sql_dispatch::query(...)` now routes `SqlStatementRoute::ShowColumns`.
  - generated route adapters now execute `show_columns_sql::<E>(...)` and
    return `SqlQueryResult::ShowColumns`.
- Expanded coverage:
  - parser/lowering tests for `SHOW COLUMNS` parse/lower/entity-mismatch
    behavior.
  - core/facade route + wrong-lane matrix tests include `SHOW COLUMNS`.
  - sql-render contract tests include `render_show_columns_lines(...)`.
  - PocketIC SQL canister integration now covers
    `SHOW COLUMNS` (direct + normalized forms).
- Extended grouped SQL `HAVING` with one bounded null-family expansion:
  - parser now accepts `HAVING <symbol> IS NULL` and
    `HAVING <symbol> IS NOT NULL` in grouped queries.
  - supported symbols remain unchanged:
    - grouped key fields.
    - already-projected aggregate terminals.
  - grouped boolean boundaries stay unchanged and fail-closed:
    - grouped `HAVING` remains `AND`-only.
    - grouped `OR` / `NOT` remain unsupported.
  - added parser/lowering/runtime/facade parity coverage for this family,
    including deterministic rejection for adjacent unsupported forms
    (`HAVING ... IS TRUE`).
- Extended expression-index eligibility for one bounded casefold prefix family:
  - new planner/shared helper:
    `starts_with_lookup_value_for_key_item(...)`.
  - planner now accepts `CompareOp::StartsWith` with
    `CoercionId::TextCasefold` for supported expression key items.
  - existing `LOWER(field)` expression indexes now lower casefold starts-with
    predicates to `IndexRange`.
  - unsupported expression kinds remain fail-closed.
  - expression-key starts-with lowering uses one safe lower-bounded envelope
    (`>= lowered_prefix`, unbounded upper) under current key framing, with
    residual predicate filtering preserving exact starts-with semantics.
  - added planner/access-choice/runtime parity tests for the new shape.
- Updated SQL subset contract docs to list grouped `HAVING`
  `IS NULL` / `IS NOT NULL` support.

## 0.56.0 - 2026-03-15 - Reduced SQL Introspection + Entity Listing

### Summary

`0.56.0` adds first-class reduced-SQL introspection lanes
(`DESCRIBE <entity>` and `SHOW INDEXES <entity>`), keeps ownership staged
(`parser -> lowering -> session`), and ships helper/canister integration updates
so shell workflows can describe entities, list indexes, and enumerate supported
entities.

### Changed

- Added parser-owned SQL introspection statement support:
  - new `SqlStatement::Describe(SqlDescribeStatement)` statement lane.
  - new `SqlStatement::ShowIndexes(SqlShowIndexesStatement)` statement lane.
  - new reduced parser path for `DESCRIBE <entity>`.
  - new reduced parser path for `SHOW INDEXES <entity>`.
  - fail-closed unsupported-feature handling for invalid forms, including
    `EXPLAIN DESCRIBE ...` and unsupported `SHOW` variants.
- Extended lowered SQL command ownership with one introspection variant:
  - added `SqlCommand::DescribeEntity`.
  - added `SqlCommand::ShowIndexesEntity`.
  - DESCRIBE lowering validates entity identity using the same trailing-segment
    matching rule used by other SQL surfaces.
  - SHOW INDEXES lowering validates entity identity using the same
    trailing-segment matching rule.
  - both introspection lanes stay outside `Query<E>` lowering and planner
    row-shape execution.
- Extended SQL route classification authority:
  - added `SqlStatementRoute::Describe { entity }`.
  - added `SqlStatementRoute::ShowIndexes { entity }`.
  - `sql_statement_route(...)` now classifies query/explain/describe/show-indexes
    lanes.
- Added dedicated core/facade SQL introspection APIs:
  - core:
    `DbSession::describe_sql::<E>(&str) -> Result<EntitySchemaDescription, QueryError>`.
  - core: `DbSession::show_indexes_sql::<E>(&str) -> Result<Vec<String>, QueryError>`.
  - facade:
    `DbSession::describe_sql::<E>(&str) -> Result<EntitySchemaDescription, Error>`.
  - facade: `DbSession::show_indexes_sql::<E>(&str) -> Result<Vec<String>, Error>`.
  - lane boundaries are explicit and fail-closed:
    - `query_from_sql(...)` rejects DESCRIBE and SHOW INDEXES.
    - `explain_sql(...)` rejects DESCRIBE and SHOW INDEXES.
    - `describe_sql(...)` rejects non-DESCRIBE statements.
    - `show_indexes_sql(...)` rejects non-`SHOW INDEXES` statements.
- Updated generated canister `sql_dispatch` helper behavior for `0.56`:
  - `sql_dispatch::query(...)` now recognizes helper-level `SHOW ENTITIES`
    directly and returns deterministic shell lines:
    `surface=entities`, then `entity=<Name>`.
  - `sql_dispatch::query(...)` now also renders:
    - `DESCRIBE <entity>` as shell-friendly schema lines.
    - `SHOW INDEXES <entity>` as shell-friendly index-listing lines.
  - added generated helper surfaces:
    - `sql_dispatch::describe_schema(...)` (typed describe payload).
    - `sql_dispatch::describe(...)` (shell describe lines).
    - `sql_dispatch::show_indexes(...)` (shell index lines).
  - `sql_dispatch::query_rows(...)` and `sql_dispatch::projection_rows(...)`
    reject DESCRIBE and SHOW INDEXES with deterministic unsupported-lane errors.
- Added explicit canister-facing DESCRIBE endpoints:
  - `canisters/quickstart` and `canisters/minimal` now expose `describe(sql)` as
    a separate query method.
  - both canisters now delegate DESCRIBE rendering to generated
    `sql_dispatch::describe(...)` helpers.
- Updated SQL shell workflow and docs:
  - `scripts/dev/sql.sh` now auto-selects `--method describe` for DESCRIBE input
    (unless caller explicitly overrides method).
  - README SQL Quickstart now includes DESCRIBE, `SHOW ENTITIES`, and
    `SHOW INDEXES` examples.
  - SQL subset contract docs now include DESCRIBE + SHOW INDEXES lanes and
    generated-helper boundaries for `0.56`.
- Added non-regression coverage:
  - parser/lowering/session route tests for DESCRIBE + SHOW INDEXES
    classification and lowering.
  - canister integration tests for helper-level `SHOW ENTITIES` behavior and
    generated query-lane DESCRIBE + SHOW INDEXES behavior while keeping
    projection endpoints fail-closed for introspection lanes.

### Validation

## 0.56.2 - 2026-03-16 - Unified Query Envelope Hard Cut

### Summary

`0.56.2` hard-cuts generated canister SQL execution to one query entrypoint
(`sql_dispatch::query(...)`) that returns one typed `SqlQueryResult` envelope
across projection/explain/introspection/entity-listing surfaces, while also
eliminating downstream unused-`sql` warnings from empty-entity generated route
stubs.

### Changed

- Added one unified SQL endpoint envelope in `icydb::db::sql`:
  - `SqlQueryResult::Projection(SqlQueryRowsOutput)`
  - `SqlQueryResult::Explain { entity, explain }`
  - `SqlQueryResult::Describe(EntitySchemaDescription)`
  - `SqlQueryResult::ShowIndexes { entity, indexes }`
  - `SqlQueryResult::ShowEntities { entities }`
- Added deterministic render helpers for the unified payload:
  - `SqlQueryResult::render_lines()`
  - `SqlQueryResult::render_text()`
  - `render_show_entities_lines(...)`
- Hard-cut generated `sql_dispatch` public surface to one query helper:
  - `sql_dispatch::query(...) -> Result<SqlQueryResult, Error>`
  - removed split helper entrypoints:
    - `query_rows(...)`
    - `projection_rows(...)`
    - `explain(...)`
    - `describe_schema(...)`
    - `describe(...)`
    - `show_indexes(...)`
- Updated canister SQL endpoint surfaces to match the hard cut:
  - `canisters/quickstart` query endpoint now returns `SqlQueryResult`.
  - `canisters/minimal` query endpoint now returns `SqlQueryResult`.
  - generated actor-surface tests now assert split helper removals.
- Updated PocketIC SQL integration tests to decode/query-assert by
  `SqlQueryResult` variants for projection, explain, describe, show-indexes,
  and show-entities.
- Updated `scripts/dev/sql.sh` to always call canister `query(...)` and render
  each `SqlQueryResult` variant to readable shell lines.
- Hardened generated `SqlEntityRoute` executor stubs for empty-entity canisters
  by marking SQL inputs as used in:
  - `execute_projection_rows(...)`
  - `execute_explain(...)`
  - `execute_describe_schema(...)`
  - `execute_show_indexes(...)`
  This removes downstream `unused variable: sql` warnings in generated
  `actor.rs`.

### Validation

- `cargo test -p icydb db::sql::tests -- --nocapture`
- `cargo test -p canister_sql_test -p canister_minimal --lib -- --nocapture`

## 0.56.3 - 2026-03-16 - SHOW ENTITIES Route Unification + Grouped HAVING SQL

### Summary

`0.56.3` moves `SHOW ENTITIES` from helper-only SQL string matching into the
same reduced SQL parser/lowering/session route authority as other statement
lanes, and enables constrained grouped SQL `HAVING` lowering on top of existing
planner/runtime grouped-HAVING contracts.

### Changed

- Promoted `SHOW ENTITIES` to one first-class reduced SQL statement lane:
  - parser now emits `SqlStatement::ShowEntities(...)`.
  - lowering now emits `SqlCommand::ShowEntities`.
  - route classification now emits `SqlStatementRoute::ShowEntities`.
- Added dedicated session/facade execution surfaces for this lane:
  - core: `DbSession::show_entities_sql(&str) -> Result<Vec<String>, QueryError>`.
  - facade: `DbSession::show_entities_sql(&str) -> Result<Vec<String>, Error>`.
  - typed helper: `DbSession::show_entities() -> Vec<String>` now returns
    runtime-registered entity names from runtime hooks.
- Updated generated `sql_dispatch` to route `SHOW ENTITIES` through parsed
  statement metadata instead of ad-hoc SQL text inspection:
  - removed helper string matcher path.
  - `query(...)` now branches `SHOW ENTITIES` via `sql_statement_route(...)`.
  - `SHOW ENTITIES` payload now comes from `db().show_entities_sql(...)`,
    keeping session authority as the source of truth.
- Added reduced parser support for grouped `HAVING` clause lists:
  - added parser contracts for HAVING symbols and clauses.
  - supported symbols are grouped key fields and aggregate terminals already
    present in grouped projection.
  - clause lists are conjunctive (`AND`) only in this patch.
  - unsupported HAVING boolean forms (`OR`, `NOT`) remain fail-closed.
- Added SQL lowering support that maps grouped HAVING to existing query intent
  APIs:
  - group-field HAVING lowers through `having_group(...)`.
  - aggregate HAVING lowers through `having_aggregate(...)` using projection
    aggregate index resolution.
  - scalar/global aggregate lanes reject HAVING as unsupported in this slice.
- Expanded SQL parser/lowering/session test coverage:
  - parser roundtrip for grouped `HAVING` shapes.
  - lowering parity between grouped SQL HAVING and fluent grouped HAVING
    intent.
  - grouped SQL runtime matrix now includes one `HAVING COUNT(*) > ...` case.
  - route and lane contract checks for `SHOW ENTITIES`.
- Kept the line clippy-clean by:
  - removing the redundant `SHOW ENTITIES` parser helper wrapper.
  - removing one stale `#[expect(clippy::too_many_lines)]` annotation from the
    PocketIC SQL integration suite.
- Updated SQL subset contract docs to reflect grouped `HAVING` support scope
  and fail-closed boundaries.

## 0.56.4 - 2026-03-16 - SQL Lane Control-Flow Consolidation

### Summary

`0.56.4` consolidates duplicated reduced-SQL lane control flow in core session
and generated `sql_dispatch` routing, and folds duplicated wrong-lane SQL tests
into matrix suites, while preserving API shape, canonical authority boundaries,
and fail-closed unsupported-lane behavior.

### Changed

- Consolidated core session SQL lane gating in
  `crates/icydb-core/src/db/session/sql.rs`:
  - introduced one internal lane-kind authority (`SqlLaneKind`) for lowered
    command and route gating.
  - introduced one surface-kind authority (`SqlSurface`) and one centralized
    unsupported-lane message map used by:
    - `query_from_sql(...)`
    - `explain_sql(...)`
    - `describe_sql(...)`
    - `show_indexes_sql(...)`
    - `show_entities_sql(...)`
  - centralized default lowered-command compilation setup through one
    `compile_sql_command_ignore(...)` helper to remove repeated
    `MissingRowPolicy::Ignore` wiring.
- Simplified generated `sql_dispatch` route branching in
  `crates/icydb-build/src/db.rs`:
  - replaced route `if statement.is_*()` chaining with one `match` over
    `SqlStatementRoute`.
  - introduced one shared explain result adapter helper
    (`explain_result_for_statement(...)`).
  - removed redundant helper-level lane re-checks in
    `describe_schema_for_statement(...)`,
    `show_indexes_result_for_statement(...)`, and
    `show_entities_result_for_statement(...)` because lane selection now
    happens once at the `query(...)` route match.
  - removed now-unused generated helper-level unsupported-statement constructors
    that previously mirrored those redundant checks.
- Consolidated repeated lane-rejection tests into matrix suites:
  - core `DbSession` SQL tests in `crates/icydb-core/src/db/session/tests.rs`
    now matrix-drive wrong-lane rejection coverage for:
    - `query_from_sql` rejecting `EXPLAIN` / `DESCRIBE` /
      `SHOW INDEXES` / `SHOW ENTITIES`.
    - `describe_sql` rejecting non-describe lanes.
    - `show_indexes_sql` rejecting non-show-indexes lanes.
    - `show_entities_sql` rejecting non-show-entities lanes.
    - `explain_sql` rejecting non-explain introspection lanes.
  - facade SQL tests in `crates/icydb/src/db/session/mod.rs` now matrix-drive:
    - `query_from_sql` rejecting non-query lanes.
    - `explain_sql` rejecting non-explain introspection lanes.
    - introspection surfaces (`describe_sql`, `show_indexes_sql`,
      `show_entities_sql`) rejecting wrong lanes in one parity matrix.
  - added one shared unsupported-execution assertion helper in core SQL tests
    to remove repeated rejection assertion boilerplate.
- Kept public behavior unchanged:
  - no changes to public core/facade SQL method signatures.
  - no changes to `SqlQueryResult` variant shape.
  - no changes to route/entity fail-closed behavior for unsupported entities
    and unsupported statement lanes.

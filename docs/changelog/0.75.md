# 0.75 — Crosscutting Audit Refresh

## 0.75.0

`0.75.0` starts the next audit-driven cleanup line by rerunning the recurring complexity-accretion audit and then cutting down the biggest branch-heavy DB hotspots through structural decomposition instead of behavior rewrites. The goal in this first patch is not new planner policy or new runtime shortcuts; it is to turn several large mixed-owner files back into real module boundaries so later `0.75` work lands against narrower surfaces.

What changed:

* reran the recurring complexity-accretion audit and used that report as the selection rule for the first `0.75` cleanup slice, which identified `db::executor::pipeline::operators::terminal::runtime`, `db::reduced_sql`, `db::query::plan::access_choice::evaluator`, `db::sql::lowering`, `db::query::plan::planner::range`, `db::executor::explain::descriptor::shared`, `db::index::scan`, and `db::data::persisted_row` as the clearest structural hotspots

What this means architecturally:

* `0.75.0` is a boundary-cleaning patch, not a semantics patch
* the main `0.75` complexity win here is not fewer total code paths; it is that the highest-pressure DB owners are no longer carrying mixed lexical, planning, explain, scan, persisted-row, and runtime orchestration responsibilities in single files
* the reduced-SQL, lowering, planner-range, explain-shared, index-scan, and persisted-row roots are now real subsystem boundaries again, which makes later authority, DRY, and velocity audits easier to act on without first re-cutting the file layout
* terminal runtime remains a hotspot even after the split, but it is now at least decomposed into a real directory boundary with the SQL-heavy implementation kept under an explicit child instead of continuing to accrete in a single root file

What did not change:

* no planner admission widening
* no SQL syntax or semantics change
* no grouped policy change
* no covering-read policy change
* no persisted-row format change
* no new route-specific fast path
* no new perf claim; this patch is structural first

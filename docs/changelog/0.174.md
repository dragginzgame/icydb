# IcyDB 0.174 Changelog

## 0.174.4

### ➕ Added

- Adds journaled recovery replay from canonical stable BTrees plus committed
  marker-bound journal tail batches.
- Adds recovery repair for missing journal-tail publication using embedded
  commit-marker journal batches.

### 🔧 Changed

- Rebuilds journaled live data and schema projections during recovery without
  folding row/schema mutations into canonical stable BTrees.
- Extends startup index rebuild to treat journaled indexes as derived
  materialized state from recovered rows.

### 🧪 Testing

- Adds focused session and source-guard evidence for journal-tail recovery,
  marker-bound publication repair, live index rebuild, and no pre-fold
  canonical BTree mutation.

## 0.174.3

### ➕ Added

- Adds journaled data, index, and schema store wrappers with live Rust BTree
  projections over canonical stable BTree bases.
- Adds journaled public write/read/index coverage proving live rows and
  materialized indexes are visible before recovery and fold land.

### 🔧 Changed

- Routes journaled write commits through marker-bound journal-tail appends
  before updating live projections.
- Keeps canonical data/index/schema stable BTrees untouched during normal
  journaled writes; fold remains the only future path into the canonical base.
- Keeps journaled recovery fail-closed until the recovery slice lands.

### 🧪 Testing

- Adds focused session and source-guard evidence for live journaled write/read,
  indexed SQL routing, journal-tail append, and no-normal-write fold behavior.

## 0.174.2

### ➕ Added

- Adds the bounded/fallible journal batch codec, `JournalBatch` envelope, and
  logical row/schema journal records for the journaled cached-stable tail.
- Adds stable BTreeMap-backed journal-tail storage keyed by monotonic
  `journal_sequence`, with ordered read validation for gaps, duplicate batch
  IDs, and physical-key/encoded-sequence mismatches.

### 🔧 Changed

- Hard-cuts the internal commit-marker payload to embed marker-bound journal
  batch payloads under the 0.173 publication strategy.
- Threads generated journaled wiring through a `JournalTailStore` and
  `register_journaled_store` without adding journaled data/index/schema
  runtime wrappers, recovery replay, or fold behavior.

### 🧪 Testing

- Adds codec, journal-tail, commit-marker, registry, generator, and
  source-guard evidence for commit-marker-bound journal batch storage.

## 0.174.1

### ➕ Added

- Admits the full four-ID `storage(journaled(...))` form at the schema,
  derive, and generated-build wiring boundary.
- Adds journaled cached-stable schema metadata, runtime capability projection,
  four-role allocation identity metadata, and generated journal-tail memory
  declaration.

### 🔧 Changed

- Keeps stable and heap generated wiring unchanged while adding journaled
  canonical data/index/schema stable-store wiring.
- Keeps journaled write/recovery behavior fail-closed until the journal
  runtime store, codec, recovery, and fold slices land.

### 🧪 Testing

- Adds schema, derive, registry, generator, compile-pass, compile-fail, and
  source-guard evidence for the schema/build admission boundary.

## 0.174.0

### ➕ Added

- Starts the journaled cached-stable runtime-admission line with an
  admission-readiness check instead of public runtime admission.

### 🔧 Changed

- Records the 0.173 protocol decisions as binding for 0.174 implementation:
  embedded bounded journal batch payload, monotonic `journal_sequence`, fold
  watermark replay boundary, accepted-schema replay, and derived index
  materialization.
- Keeps public `storage(journaled(...))` reserved until generated wiring,
  runtime construction, recovery, fold, diagnostics, relation behavior, and
  source audits all pass in one branch.
- Leaves runtime behavior unchanged for the setup slice.

### 🧪 Testing

- Adds a source/design guard proving 0.174.0 starts from the closed 0.173
  foundation while keeping journaled runtime admission gated.

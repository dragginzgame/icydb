# IcyDB 0.165 Changelog

## 0.165.7

### 🔧 Changed

- Renamed cursor-token decode handoff values from generic `*Parts` vocabulary
  to decoded payload/component vocabulary.

  ```text
  ScalarTokenParts -> DecodedScalarTokenPayload
  GroupedTokenParts -> DecodedGroupedTokenPayload
  GroupedContinuationToken::into_parts() -> into_components()
  ```

- Extended the continuation/cursor/window naming note with the role proof and
  old-vocabulary scan terms for the cursor-token decode payload hard cut.

### 🧪 Testing

- Validated the slice with formatting, core check, focused cursor-token checks,
  old-vocabulary scans, core clippy, and diff whitespace checks.

### 🧪 Testing

## 0.165.3

### 🔧 Changed

- Renamed SQL aggregate terminal table helpers from `resolve_*` to `intern_*`
  where they return an existing terminal index or insert a new terminal.

  ```text
  resolve_or_insert_global_aggregate_terminal -> intern_global_aggregate_terminal
  resolve_having_global_aggregate_terminal_index -> intern_having_global_aggregate_terminal_index
  QueryResponse / ProjectionResponse -> kept public response facade vocabulary
  SqlQueryResult / SqlQueryRowsOutput / SqlGroupedRowsOutput -> kept public SQL endpoint vocabulary
  accepted old names under crates/** -> no live-code hits
  ```

- Added a helper-verb family note that keeps true `resolve_*`, `derive_*`,
  `prepare_*`, and `canonicalize_*` helpers while rejecting `resolve_*` for
  mutating terminal-table interning.
- Added the public-surface naming family note with explicit keep decisions for
  public response/result, SQL endpoint, diagnostics, and EXPLAIN DTO names.
- Added the residual vocabulary sweep note for accepted 0.165 renames and
  classified remaining old-name hits as release notes, family notes, archived
  design/changelog history, or retained audit artifacts rather than live code.
- Marked the 0.165 naming audit and completed family notes as complete after
  the residual sweep and broad workspace validation.

### 🧪 Testing

- Validated the slice with focused SQL lowering checks, formatting, workspace
  check, workspace clippy, concrete old-name scans, and diff whitespace checks.

## 0.165.2

### 🔧 Changed

- Removed the duplicate SQL scalar aggregate descriptor-shape alias and kept
  the execution-facing `PreparedSqlScalarAggregatePlanFragment` as the single
  name for that prepared aggregate strategy fragment.

  ```text
  PreparedSqlScalarAggregateDescriptorShape -> PreparedSqlScalarAggregatePlanFragment
  PreparedSqlScalarAggregateStrategy::descriptor_shape() -> plan_fragment()
  prepared_descriptor_shape() -> prepared_plan_fragment()
  ```

- Renamed the SQL aggregate terminal dedup wrapper from broad semantics
  vocabulary to semantic-key vocabulary so equality authority is separate from
  prepared aggregate behavior.

  ```text
  AggregateTerminalSemantics -> AggregateTerminalSemanticKey
  ```

- Added a SQL lowering/query shape family note that keeps true structural
  `*Shape` names while rejecting descriptor-shape wording for execution-facing
  aggregate fragments.
- Added an identity/semantics family note that keeps `AggregateIdentity`,
  `AggregateSemanticKey`, and `PreparedAggregateSemantics` in their current
  roles while removing a misleading `*Semantics` wrapper name.

### 🧪 Testing

- Validated the slice with focused SQL lowering checks, formatting, core
  clippy, and diff whitespace checks.

## 0.165.1

### 🔧 Changed

- Renamed covering projection and index-covering metadata concepts to
  `CoveringProjectionFacts` and `IndexCoveringAccessFacts` so reusable derived
  planner inputs no longer look like owner-local contexts or broad metadata.

  ```text
  CoveringProjectionContext -> CoveringProjectionFacts
  covering_index_projection_context* -> covering_index_projection_facts*
  CoveringAccessMetadata -> IndexCoveringAccessFacts
  ```

- Renamed the prepared-plan resident payload from
  `PreparedExecutionPlanCoreShared` to `PreparedExecutionPlanResidents` and
  renamed its extraction helper to `into_residents()`.

  ```text
  PreparedExecutionPlanCoreShared -> PreparedExecutionPlanResidents
  PreparedExecutionPlanCore::into_shared() -> into_residents()
  prepared-plan shared field wording -> residents
  ```

- Renamed validated cursor-state types from `PlannedCursor` /
  `GroupedPlannedCursor` to `ValidatedCursor` / `ValidatedGroupedCursor`, and
  moved the cursor module from `planned` to `validated`.

  ```text
  PlannedCursor -> ValidatedCursor
  GroupedPlannedCursor -> ValidatedGroupedCursor
  cursor::planned -> cursor::validated
  validate_planned_cursor(...) -> validate_cursor_token(...)
  validate_planned_cursor_state(...) -> validate_cursor_state(...)
  ```

- Added family notes for covering reads, prepared execution, and
  continuation/cursor/window naming, including role proofs, kept-name
  decisions, deferred candidates, and residual vocabulary scan terms.

### 🧪 Testing

- Validated the slice with focused covering tests, prepared-execution name
  checks, core check, core clippy, formatting, and diff whitespace checks.

## 0.165.0

### 🧭 Migration Notes

- 0.165 is a pre-1.0 naming hard cut. Accepted internal names are renamed
  directly instead of keeping aliases, compatibility shims, or duplicate
  vocabulary.

### 🔧 Changed

- Added the 0.165 naming audit and role-alignment design policy, including
  role-family rules for `*Contract`, `*Decision`, `*Facts`, `*Context`,
  `*Identity`, and `*Semantics`.
- Added active naming architecture guidance so future cleanup can distinguish
  conventional database terms from IcyDB-specific role names.
- Renamed ordered-load route terminology from contract vocabulary to selected
  route-mode vocabulary.

  ```text
  LoadOrderRouteContract -> LoadOrderRouteMode
  access_order_satisfied_by_route_contract -> access_order_satisfied_by_route_mode
  ```

- Renamed grouped execution mode derivation input vocabulary from projection to
  context.

  ```text
  GroupedExecutionModeProjection -> GroupedExecutionModeContext
  ```

- Renamed route capability bundle terminology to route capability facts.

  ```text
  RouteCapabilities -> RouteCapabilityFacts
  route_capabilities -> route_capability_facts
  ```

### 🧪 Testing

- Validated the route/access selection naming slice with focused route tests,
  workspace checks, core clippy, formatting, and diff whitespace checks.

# IcyDB 0.165 Changelog

## 0.165.1

### 🔧 Changed

- Renamed covering projection and index-covering metadata concepts to
  `CoveringProjectionFacts` and `IndexCoveringAccessFacts` so reusable derived
  planner inputs no longer look like owner-local contexts or broad metadata.

  ```text
  CoveringProjectionContext -> CoveringProjectionFacts
  covering_index_projection_context* -> covering_index_projection_facts*
  CoveringAccessMetadata -> IndexCoveringAccessFacts
  ```

- Renamed the prepared-plan resident payload from
  `PreparedExecutionPlanCoreShared` to `PreparedExecutionPlanResidents` and
  renamed its extraction helper to `into_residents()`.

  ```text
  PreparedExecutionPlanCoreShared -> PreparedExecutionPlanResidents
  PreparedExecutionPlanCore::into_shared() -> into_residents()
  prepared-plan shared field wording -> residents
  ```

- Renamed validated cursor-state types from `PlannedCursor` /
  `GroupedPlannedCursor` to `ValidatedCursor` / `ValidatedGroupedCursor`, and
  moved the cursor module from `planned` to `validated`.

  ```text
  PlannedCursor -> ValidatedCursor
  GroupedPlannedCursor -> ValidatedGroupedCursor
  cursor::planned -> cursor::validated
  validate_planned_cursor(...) -> validate_cursor_token(...)
  validate_planned_cursor_state(...) -> validate_cursor_state(...)
  ```

- Added family notes for covering reads, prepared execution, and
  continuation/cursor/window naming, including role proofs, kept-name
  decisions, deferred candidates, and residual vocabulary scan terms.

### 🧪 Testing

- Validated the slice with focused covering tests, prepared-execution name
  checks, core check, core clippy, formatting, and diff whitespace checks.

## 0.165.0

### 🧭 Migration Notes

- 0.165 is a pre-1.0 naming hard cut. Accepted internal names are renamed
  directly instead of keeping aliases, compatibility shims, or duplicate
  vocabulary.

### 🔧 Changed

- Added the 0.165 naming audit and role-alignment design policy, including
  role-family rules for `*Contract`, `*Decision`, `*Facts`, `*Context`,
  `*Identity`, and `*Semantics`.
- Added active naming architecture guidance so future cleanup can distinguish
  conventional database terms from IcyDB-specific role names.
- Renamed ordered-load route terminology from contract vocabulary to selected
  route-mode vocabulary.

  ```text
  LoadOrderRouteContract -> LoadOrderRouteMode
  access_order_satisfied_by_route_contract -> access_order_satisfied_by_route_mode
  ```

- Renamed grouped execution mode derivation input vocabulary from projection to
  context.

  ```text
  GroupedExecutionModeProjection -> GroupedExecutionModeContext
  ```

- Renamed route capability bundle terminology to route capability facts.

  ```text
  RouteCapabilities -> RouteCapabilityFacts
  route_capabilities -> route_capability_facts
  ```

### 🧪 Testing

- Validated the route/access selection naming slice with focused route tests,
  workspace checks, core clippy, formatting, and diff whitespace checks.

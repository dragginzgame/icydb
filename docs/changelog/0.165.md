# IcyDB 0.165 Changelog

## 0.165.13

### 🔧 Changed

- Renamed grouped runtime payload, continuation window, and test construction
  helpers away from generic parts vocabulary where the helpers expose grouped
  row values, grouped response rows/cursors, pagination window fields, or
  test-only grouped inputs.

  ```text
  RuntimeGroupedRow::into_parts() -> into_group_key_and_aggregate_values()
  StructuralGroupedProjectionResult::into_parts() -> into_rows_and_cursor()
  GroupedContinuationWindow::into_parts() -> into_pagination_window_fields()
  ```

- Renamed private execution handoff helpers away from generic parts vocabulary
  where the helpers expose stream-resolution fields, structural cursor rows,
  grouped fold inputs, scan anchors, cache inputs, or delete projection rows.

  ```text
  ResolvedExecutionKeyStream::into_parts() -> into_stream_resolution_fields()
  StructuralCursorPage::into_parts() -> into_data_rows_and_cursor()
  GroupedStreamStage::parts_mut() -> fold_inputs_mut()
  ```

- Added a grouped runtime payload naming note with role proof and
  old-vocabulary scan terms for the new hard cut.
- Added an execution handoff residual naming note with role proof, kept public
  response DTO decisions, and old-vocabulary scan terms.

### 🧪 Testing

- Validated the slice with formatting, core check, focused grouped,
  projection, continuation, and aggregate checks, old-vocabulary scans, core
  clippy, and diff whitespace checks.

  terminal inputs, and the grouped aggregate test constructor now names test
  inputs.

  and residual sweep notes with role proof and old-vocabulary scan terms for
  the aggregate helper hard cut.

## 0.165.0

### 🧭 Migration Notes

- 0.165 is a pre-1.0 naming hard cut. Accepted internal names are renamed
  directly instead of keeping aliases, compatibility shims, or duplicate
  vocabulary.

### 🔧 Changed

- Added the 0.165 naming audit and role-alignment design policy, including
  role-family rules for `*Contract`, `*Decision`, `*Facts`, `*Context`,
  `*Identity`, and `*Semantics`.
- Added active naming architecture guidance so future cleanup can distinguish
  conventional database terms from IcyDB-specific role names.
- Renamed ordered-load route terminology from contract vocabulary to selected
  route-mode vocabulary.

  ```text
  LoadOrderRouteContract -> LoadOrderRouteMode
  access_order_satisfied_by_route_contract -> access_order_satisfied_by_route_mode
  ```

- Renamed grouped execution mode derivation input vocabulary from projection to
  context.

  ```text
  GroupedExecutionModeProjection -> GroupedExecutionModeContext
  ```

- Renamed route capability bundle terminology to route capability facts.

  ```text
  RouteCapabilities -> RouteCapabilityFacts
  route_capabilities -> route_capability_facts
  ```

### 🧪 Testing

- Validated the route/access selection naming slice with focused route tests,
  workspace checks, core clippy, formatting, and diff whitespace checks.

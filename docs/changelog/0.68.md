# 0.68 Changelog

## 0.68.1 - 2026-04-04 - Cursorless SQL Projection Lane Cleanup

### Summary

`0.68.1` keeps the shipped SQL surface unchanged and does not yet land the
first-class covering-read runtime from the `0.68` design. Instead, it cleans
up the existing cursorless structural SQL projection lane so that it stops
paying more of the generic paged-load machinery than that surface actually
needs. The goals in this patch are:

- make the cursorless retained-slot SQL projection lane explicit instead of
  rediscovering that shape late inside the shared scalar page materializer
- stop carrying full row payloads and outward cursor assembly through the
  immediate SQL projection path when that lane never exposes them
- reduce widened select attribution on the kept projection-heavy shapes without
  reopening the reverted late-bound covering-read runtime experiments

### Changed

- added an explicit cursor-emission contract to the scalar execution inputs in:
  - `crates/icydb-core/src/db/executor/pipeline/contracts/execution/inputs.rs`
  - `crates/icydb-core/src/db/executor/pipeline/contracts/execution/mod.rs`
  - `crates/icydb-core/src/db/executor/pipeline/contracts/mod.rs`
  - `CursorEmissionMode` now makes the no-cursor SQL projection lane an owned
    contract instead of an implicit property rediscovered during final page
    assembly
- added an explicit row-payload contract for the same scalar lane in:
  - `crates/icydb-core/src/db/executor/terminal/page.rs`
  - `crates/icydb-core/src/db/executor/pipeline/operators/post_access/contracts/row.rs`
  - `crates/icydb-core/src/db/executor/order.rs`
  - `KernelRowPayloadMode` now distinguishes:
    - full-row structural payloads for cursor/entity lanes
    - slot-only structural payloads for cursorless retained-slot SQL
      projection lanes
  - `KernelRow` now carries `Option<DataRow>` explicitly, so the slot-only lane
    can stay fail-closed instead of relying on accidental full-row assumptions
- reduced cursorless SQL row-collector overhead in:
  - `crates/icydb-core/src/db/executor/pipeline/operators/terminal/runtime.rs`
  - `crates/icydb-core/src/db/executor/pipeline/entrypoints/scalar/mod.rs`
  - `crates/icydb-core/src/db/executor/kernel/mod.rs`
  - the short path now uses one explicit `RowCollectorStreamRequest` envelope
    instead of widening helper signatures
  - cursorless SQL staging now stops after `offset + limit` when that cap is
    valid for the retained-slot lane
  - cursorless SQL page shaping now skips the generic page-window pass when the
    staged rows are already final
  - the constant-covering slot-row short path now uses that same redundant
    page-window skip instead of reshaping rows twice
- continued the related borrowed/cached structural read cleanup in:
  - `crates/icydb-core/src/db/data/persisted_row/mod.rs`
  - `crates/icydb-core/src/db/predicate/runtime.rs`
  - `crates/icydb-core/src/db/executor/mutation/save_validation.rs`
  - `crates/icydb-core/src/db/index/key/build.rs`
  - `crates/icydb-core/src/db/cursor/{boundary.rs,mod.rs}`
  - `crates/icydb-core/src/db/executor/projection/materialize/structural.rs`
  - read-only structural consumers now reuse cached slot values and borrowed
    ordering reads more consistently, which keeps the current slot-only SQL
    lane from cloning `Value`s just to compare, validate, or sort them again
- tightened one general secondary-index scan boundary in:
  - `crates/icydb-core/src/db/index/scan.rs`
  - common membership scans no longer decode raw index keys unless an
    index-only predicate actually needs decoded key components
- added regression coverage in:
  - `crates/icydb-core/src/db/session/tests/mod.rs`
  - cursorless secondary-order covering-shaped SQL projection queries now lock:
    - direct `ORDER BY name ASC LIMIT 1` parity against the entity lane
    - the matching `LIMIT/OFFSET` page-window parity on the same ordered shape

## 0.68.0 - 2026-04-04 - Planner-Owned Covering Read Groundwork

### Summary

`0.68.0` starts the `0.68` covering-read line with one narrow internal slice.
It still does not widen the shipped SQL or fluent query surface, and it does
not keep a new runtime fast path yet. This patch only lands the planner-owned
contract and the shared projection-shape helper that later runtime work will
consume.

The goals in this patch are:

- land one planner-owned scalar covering-read contract before broader executor
  routes start consuming it
- keep phase-1 covering eligibility intentionally narrow:
  - direct projected index components
  - direct projected primary key output
  - direct projected constants satisfied by bound index-prefix equality values
- remove duplicated "direct field under optional alias wrappers" logic from the
  executor now that planner and executor both need the same rule

### Changed

- added the first planner-owned covering-read contract in:
  - `crates/icydb-core/src/db/query/plan/covering/mod.rs`
  - new phase-1 covering-read contract types:
    - `CoveringReadFieldSource`
    - `CoveringReadField`
    - `CoveringReadPlan`
  - new planner helper:
    - `covering_read_plan(...)`
  - covering access metadata is now derived once and shared across:
    - covering projection order derivation
    - constant projection recovery from prefix-bound equality values
    - the new per-field covering-read source derivation
- shared the direct-field projection rule in:
  - `crates/icydb-core/src/db/query/plan/expr/projection.rs`
  - `crates/icydb-core/src/db/executor/projection/materialize/structural.rs`
  - planner and executor now both use the same helper when a projection must
    prove that one output stays on one direct field leaf under optional alias
    wrappers
  - this removes the executor-local duplicate helper and keeps later covering
    route work anchored to one projection-shape authority
- no new runtime covering-read consumer ships in this patch
  - the first runtime experiment was intentionally backed out because it was
    mixed on the widened PocketIC select cohort
  - `IndexComponent { .. }`, `PrimaryKey`, and `Constant(...)` sources are
    planner-owned contract data only in `0.68.0`
  - later `0.68` work can now target the runtime side against one explicit
    planner boundary instead of rediscovering covering eligibility ad hoc
- added planner-local regression coverage in:
  - `crates/icydb-core/src/db/query/plan/covering/tests.rs`
  - accepted phase-1 shapes now include:
    - direct index component projection
    - direct primary-key projection
    - direct prefix-bound constant projection
  - rejected phase-1 shapes now include:
    - computed/non-field expressions
    - non-coverable row-field projections
    - residual-predicate shapes without strict predicate compatibility

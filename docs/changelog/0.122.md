## 0.122.2

- `0.122.2` is the other Codex window’s session-SQL compression pass. It removes more write-lane wrappers, duplicative query-vs-update compile shells, explain-only plan-building hops, and a wider round of single-caller parser/lowering/session helper shells without changing SQL behavior.

- The first follow-on compression cluster lands in [/home/adam/projects/icydb/crates/icydb-core/src/db/session/sql/execute/write.rs](/home/adam/projects/icydb/crates/icydb-core/src/db/session/sql/execute/write.rs):
  - collapsed the insert/update generated-field guard pair into one helper:
    - `reject_explicit_sql_write_to_generated_field(..., statement_kind)`
  - deleted the write `RETURNING` row ladder:
    - `sql_write_statement_row(...)`
    - `sql_write_statement_projection(...)`
  - `sql_write_statement_result(...)` now rebuilds full entity rows directly before handing them to the real `RETURNING` shaper
  - deleted the typed delete wrappers:
    - `execute_typed_sql_delete_projection(...)`
    - `execute_typed_sql_delete_count(...)`
    - `execute_typed_sql_delete_returning(...)`
  - `execute_sql_delete_statement(...)` now branches directly between:
    - the direct delete-count terminal

- That shortens the main write-lane chains from:
  - `INSERT/UPDATE -> sql_write_statement_result -> sql_write_statement_projection -> sql_write_statement_row -> sql_returning_statement_projection`
  - to:
  - `INSERT/UPDATE -> sql_write_statement_result -> sql_returning_statement_projection`

- And for delete:
  - `DELETE -> execute_sql_delete_statement -> execute_typed_sql_delete_count`
  - becomes:
  - `DELETE -> execute_sql_delete_statement -> execute_delete_count`
  - `DELETE RETURNING -> execute_sql_delete_statement -> execute_typed_sql_delete_returning -> execute_typed_sql_delete_projection -> sql_returning_statement_projection`
  - becomes:
  - `DELETE RETURNING -> execute_sql_delete_statement -> structural projection terminal -> sql_returning_statement_projection`

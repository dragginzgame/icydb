# 0.122 Changelog

## 0.122.0

- `0.122.0` starts the residual-scaffolding collapse in the session SQL compile/execute bridge.

- The first compression cluster is `/db/session/sql/execute/*`, with one proof question:
  - can the session SQL compile/execute surface reach the same prepared-statement, lowered-select, and grouped-select owners in fewer hops without changing behavior?

- The compile-side pass lands in [/home/adam/projects/icydb/crates/icydb-core/src/db/session/sql/execute/route.rs](/home/adam/projects/icydb/crates/icydb-core/src/db/session/sql/execute/route.rs):
  - deleted the module-level `prepare_sql_statement_for_authority(...)` shell
  - deleted the module-level `validate_sql_metadata_entity_for_authority(...)` shell
  - `compile_sql_statement_for_authority(...)` now keeps those repeated pieces as local compile-lane closures instead of routing through extra module-level wrappers before reaching:
    - `prepare_sql_statement(...)`
    - direct metadata entity validation
  - this keeps the same statement-family behavior, but shortens the routine compile follow-through for:
    - `SELECT`
    - `DELETE`
    - `INSERT`
    - `UPDATE`
    - `EXPLAIN`
    - `DESCRIBE`
    - `SHOW INDEXES`
    - `SHOW COLUMNS`

- The execute-side pass lands in the lowered/grouped bridge:
  - [/home/adam/projects/icydb/crates/icydb-core/src/db/session/sql/execute/lowered.rs](/home/adam/projects/icydb/crates/icydb-core/src/db/session/sql/execute/lowered.rs)
  - [/home/adam/projects/icydb/crates/icydb-core/src/db/session/sql/execute/mod.rs](/home/adam/projects/icydb/crates/icydb-core/src/db/session/sql/execute/mod.rs)
  - [/home/adam/projects/icydb/crates/icydb-core/src/db/session/sql/execute/write.rs](/home/adam/projects/icydb/crates/icydb-core/src/db/session/sql/execute/write.rs)

- Two single-call execute shells are gone:
  - `execute_lowered_sql_projection_core(...)`
    - used once by `INSERT ... SELECT` source execution
    - the write path now binds lowered `SELECT` directly through `bind_lowered_sql_select_query_structural(...)` and then calls `execute_structural_sql_projection_without_sql_cache(...)`
  - `execute_structural_sql_grouped_statement_select_core(...)`
    - used once by the normal compiled grouped `SELECT` execution path
    - the grouped branch in `execute_compiled_sql_with_cache_attribution(...)` now resolves the prepared plan and executes grouped rows directly without another forwarding helper in the middle

- The same cluster exposes one more lowering-to-session bounce that is now gone:
  - [/home/adam/projects/icydb/crates/icydb-core/src/db/session/sql/execute/lowered.rs](/home/adam/projects/icydb/crates/icydb-core/src/db/session/sql/execute/lowered.rs) no longer keeps `structural_query_from_lowered_select(...)`
  - the two remaining call sites now bind lowered `SELECT` shapes directly through the canonical lowering owner:
    - [/home/adam/projects/icydb/crates/icydb-core/src/db/session/sql/execute/route.rs](/home/adam/projects/icydb/crates/icydb-core/src/db/session/sql/execute/route.rs)
    - [/home/adam/projects/icydb/crates/icydb-core/src/db/session/sql/execute/write.rs](/home/adam/projects/icydb/crates/icydb-core/src/db/session/sql/execute/write.rs)
  - that shortens one more chain from:
    - session compile or write path -> session lowered wrapper -> lowering binder
  - to:
    - session compile or write path -> lowering binder

- That means the main chain contractions in this pass are:
  - compile:
    - before: `compile_sql_statement_for_authority -> prepare_sql_statement_for_authority -> prepare_sql_statement`
    - after: `compile_sql_statement_for_authority -> prepare_sql_statement`
  - metadata compile:
    - before: `compile_sql_statement_for_authority -> validate_sql_metadata_entity_for_authority`
    - after: `compile_sql_statement_for_authority -> direct metadata entity check`
  - `INSERT ... SELECT` source execution:
    - before: `execute_sql_insert_select_source_rows -> execute_lowered_sql_projection_core -> execute_structural_sql_projection_without_sql_cache`
    - after: `execute_sql_insert_select_source_rows -> bind_lowered_sql_select_query_structural -> execute_structural_sql_projection_without_sql_cache`
  - grouped compiled `SELECT` execution:
    - before: `execute_compiled_sql_with_cache_attribution -> execute_structural_sql_grouped_statement_select_core -> execute_grouped_sql_statement_from_prepared_plan_with`
    - after: `execute_compiled_sql_with_cache_attribution -> execute_grouped_sql_statement_from_prepared_plan_with`

- Functions deleted:
  - `prepare_sql_statement_for_authority(...)`
  - `validate_sql_metadata_entity_for_authority(...)`
  - `execute_lowered_sql_projection_core(...)`
  - `execute_structural_sql_grouped_statement_select_core(...)`
  - `structural_query_from_lowered_select(...)`

- This pass is intentionally compressive rather than semantic:
  - no new SQL surface
  - no changed planner ownership
  - no changed grouped or write behavior
  - less session-layer forwarding ceremony around the same real owners

- Validation covered:
  - `cargo fmt --all`
  - `cargo test -p icydb-core execute_sql_statement_inner -- --nocapture`
  - `cargo test -p icydb-core execute_sql_statement_insert_select_matrix_accepts_supported_source_shapes -- --nocapture`
  - `cargo test -p icydb-core sql_compile_cache_covers_query_surface_read_explain_and_metadata_families -- --nocapture`
  - `cargo clippy -p icydb-core --all-targets -- -D warnings`

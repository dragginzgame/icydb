# 0.144.x

## 0.144.5

`0.144.5` continues the structural ownership audit by reducing executor
visibility to the narrowest surfaces currently required by production code and
tests, while tightening structural value-storage decode ownership around
persisted custom fields.

### Executor visibility cleanup

- Made executor page-window sizing helpers private to `executor::window`; only
  the exported window-state and kernel cursor helpers remain visible to other
  executor modules.
- Kept `CompiledPath` available through the projection boundary while making
  its segment storage private; callers continue to use the existing accessor.
- Narrowed execution profiling and grouped route-mode internals from crate-wide
  visibility to executor-only visibility.
- Narrowed secondary-order pushdown helper methods to executor-only visibility
  and made the core secondary-order pushdown matcher private.
- Removed stale secondary-order pushdown rejection variants that had no
  construction sites after visibility was tightened.
- Narrowed page-materialization test helpers from crate-wide visibility to the
  owning page module.

### Value-storage codec cleanup

- Fixed structured `Option<Ulid>` decode by keeping null probing inside the
  value-storage lane, where local tags such as the ULID tag remain valid
  non-null payloads.
- Added a generated persisted-row regression for an asset-selection shaped
  record with `asset_ids: Vec<Ulid>` and `default_asset_id: Option<Ulid>`.
- Renamed the value-storage null probe and borrowed-view constructors so their
  lane and trust boundary are explicit at call sites.
- Routed runtime `Value` local-tag decode through the same typed value-storage
  decoders used by persisted structured fields.
- Removed unused tuple splitters, retained unsafe walkers, text-key map lookup
  duplication, and simple `Value` wrapper decode helpers from the value-storage
  owner.

### Validation

- Re-ran `cargo fmt --all`.
- Re-ran `cargo check -p icydb-core`.
- Re-ran `cargo test -p icydb-core executor -- --nocapture`.
- Re-ran `cargo test -p icydb-core value_storage -- --nocapture`.
- Re-ran `cargo test -p icydb-schema-tests structured_field_value -- --nocapture`.
- Re-ran `git diff --check`.

## 0.144.2

`0.144.2` completes the scalar expression consolidation left open after the
initial compiled-expression merge and closes the non-SQL session ownership
audit. Runtime expression execution now has one engine:
`CompiledExpr::evaluate` over a context-specific value reader, while session
execution consumes prepared-plan shape and grouped runtime consumes a single
planner-selected route instead of re-deriving query intent.

### Runtime expression consolidation

- Replaced runtime scalar projection evaluation with compiled-expression reader
  adapters, so row projections, retained-slot projections, ordering
  expressions, runtime filters, grouped output, HAVING, and aggregate terminal
  expressions all evaluate through `CompiledExpr`.
- Changed scalar projection materialization and static planning artifacts to
  store `CompiledExpr` after the plan boundary instead of carrying
  `ScalarProjectionExpr` into executor loops.
- Updated scalar aggregate terminal expression caches to intern compiled
  expressions at cache insertion time, preserving deduplication while avoiding
  per-row compilation.
- Removed the production scalar operator evaluator and projection eval operator
  module; numeric, comparison, boolean, CASE, and function-call semantics now
  stay centralized behind the compiled-expression evaluator.

### Reader safety and field paths

- Extended the value-reader boundary so reader-owned failures preserve their
  original error class and origin instead of degrading to missing-slot NULL-like
  behavior.
- Kept invalid context use fail-closed: aggregate reads in row contexts, slot
  reads in grouped-only contexts, and out-of-bounds grouped reads remain
  deterministic evaluation errors.
- Preserved nested field-path behavior across the unified engine: scalar
  projection missing paths materialize `NULL`, while filter paths reject
  missing subfields before `IS NULL` admission can treat absence as a value.
- Kept nested value-storage decoding in executor readers, leaving
  `compiled_expr.rs` free of executor row types, storage imports, grouping
  mechanics, and projection orchestration.

### Session orchestration ownership

- Updated the shared non-SQL prepared execution seam so scalar/grouped/delete
  dispatch reads from `PreparedExecutionPlan` shape rather than the original
  `Query<E>` intent after planning has already completed.
- Split fluent ranked terminal session helpers into explicit top/bottom
  methods for rows, values, and values-with-ids, removing the remaining
  direction-flag branch from the non-SQL fluent session adapter.
- Audited response and write helpers as orchestration-only: cursor payloads
  stay cursor-owned, grouped rows stay executor-owned until response
  finalization, and structural writes still route through `SaveExecutor`
  without interpreting storage or codec bytes.

### Grouped route contract sealing

- Added `GroupedExecutionRoute` as the planner-owned grouped execution route
  artifact that combines global DISTINCT, Top-K selection, and dedicated
  grouped `COUNT(*)` fold admission into one closed contract.
- Removed executor-side grouped fold route recombination: grouped runtime now
  matches the planner-selected route instead of inspecting grouped DISTINCT,
  Top-K, and fold-path signals independently.
- Removed the separate Top-K flag and fold-path field from `GroupedRouteStage`;
  cursor finalization and grouped fold dispatch now read from the same grouped
  route artifact.

### Validation

- Verified the consolidation with targeted compiled-expression, projection,
  HAVING, scalar aggregate terminal, and aggregate numeric test filters.
- Re-ran `cargo check -p icydb-core`,
  `cargo clippy -p icydb-core --all-targets -- -D warnings`, the layer
  authority invariant script, and whitespace checks.
- Audited executor runtime paths for remaining `ScalarProjectionExpr`
  evaluation; production references are gone, with only test compatibility
  helpers still compiling scalar planner expressions into `CompiledExpr`.
- Re-ran the non-SQL session ownership validation with `cargo fmt --all
  --check`, `cargo check -p icydb-core --tests --features sql`,
  `cargo clippy -p icydb-core --all-targets --features sql -- -D warnings`,
  `cargo test -p icydb-core --features sql -- --nocapture`, and
  `git diff --check`.
- Verified the grouped route contract with
  `cargo test -p icydb-core grouped_executor_handoff --features sql --
  --nocapture` and `cargo check -p icydb-core --tests --features sql`.

## 0.144.1

`0.144.1` tightens the SQL lowering and expression-evaluation boundaries after
the structural audit found one remaining executor-shaped aggregate handoff and
parallel grouped expression evaluators.

### SQL lowering ownership

- Removed executor DTO construction from SQL aggregate lowering.
- `PreparedSqlScalarAggregateStrategy` now exposes executor-neutral aggregate
  plan fragments, and `db/session/sql` performs the final
  `StructuralAggregateTerminal` adaptation at the session-to-executor boundary.
- Renamed the aggregate fragment surface away from runtime wording:
  `PreparedSqlScalarAggregateRuntimeDescriptor` is now
  `PreparedSqlScalarAggregatePlanFragment`, and `runtime_descriptor()` is now
  `plan_fragment()`.
- Verified production `db/sql/lowering` has no direct references to `DbSession`,
  executor types, runtime adapters, prepared plans, or structural aggregate
  terminals.

### Compiled expression ownership

- Replaced the separate grouped input and grouped projection expression trees
  with one `CompiledExpr` IR used by row evaluation, grouped aggregate input and
  filter evaluation, grouped projection, and HAVING.
- Added `CompiledExprValueReader` as the only runtime value-access boundary;
  row readers expose slots, while grouped readers expose group keys and
  aggregate outputs through the same evaluator contract.
- Moved planner-facing expression compilation into a dedicated compile module
  so `compiled_expr.rs` contains only the IR, reader trait, evaluator, and
  expression error taxonomy.
- Removed the duplicate scalar operator evaluator modules and routed scalar
  projection operators through the unified compiled expression semantics.
- Added regression coverage for invalid reader access, out-of-bounds grouped
  reads, CASE condition failures, and specialized CASE admission parity so
  unsupported context use fails as a diagnostic instead of falling through as
  SQL `NULL`.

## 0.144.0

`0.144.0` opens the structural ownership audit. The release starts with the
query/executor boundary because recent planning and terminal cleanups made that
the highest-risk place for drift.

### Structural audit docs

- Added the `0.144` structural ownership audit design and status documents.
- Split the audit lens by subsystem:
  - execution modules own behavior and runtime orchestration boundaries;
  - value/storage modules own representation and encoding boundaries;
  - schema/build modules own generated structure and contracts.
- Recorded the current audit order: `db/query`, `db/session`, `db/executor`,
  value/storage, schema crates, and build/derive tooling.
- Added explicit decision rules for moving, splitting, merging, deleting, or
  deferring modules during the audit.

### Query ownership

- Clarified `query::builder::aggregate` as the owner for aggregate expression
  builders and fluent aggregate terminal descriptors.
- Added query-owned EXPLAIN projection artifacts:
  - `query::explain::predicate` derives residual predicate explain DTOs from
    query expression trees;
  - `query::explain::projection` derives stable projection field labels for
    route and projection diagnostics.
- Made those explain artifacts the intended boundary for executor diagnostics:
  executor code should request richer query explain artifacts instead of
  importing `Expr` and matching expression trees locally.

### Executor ownership

- Removed executor-local predicate/projection expression interpretation from
  EXPLAIN descriptor assembly.
- Removed a no-op grouped Top-K finalization match that enumerated every `Expr`
  variant only to clone the same order expression.
- Deleted the executor route `semantics` forwarding module. Aggregate meaning
  helpers now resolve directly through the query-owned `AggregateKind` methods.
- Removed duplicate grouped execution-mode recomputation from route
  observability. Observability now projects the stored route artifact instead
  of recombining `GroupedPlanStrategy` with route capability inputs.

### Strategy integrity audit

- Confirmed the main execution strategy artifacts are produced by `query::plan`:
  - `DistinctExecutionStrategy`
  - `GroupedPlanStrategy`
  - `GroupedDistinctExecutionStrategy`
  - `GroupedFoldPath`
- Confirmed production executor code consumes those artifacts as route/runtime
  inputs rather than constructing them from aggregate, grouping, or `DISTINCT`
  semantics.

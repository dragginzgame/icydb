# IcyDB 0.196 Changelog

## 0.196.10

### 📝 Summary

- Reduces SQL boolean predicate compile work without changing public API,
  cursor tokens, persisted row/index/schema formats, stable storage layout, or
  execution semantics.
- Changes the runtime predicate compiler from eager TRUE/FALSE pair
  construction to demand-driven truth-branch construction.
- Preserves SQL three-valued membership behavior: `IN` / `NOT IN` lists with
  `NULL` still admit only SQL-TRUE rows, and `NOT` still requests the inverse
  branch explicitly.
- Keeps the optimization compile-side only; route facts, result signatures,
  cursor signatures, store counters, and index-entry counts are expected to
  remain unchanged.

### 🔧 Changed

- Boolean predicate compilation now asks for only the branch it needs:
  top-level access predicates request the TRUE branch, while unary `NOT`
  recursively requests the FALSE branch from its child.
- `AND`, `OR`, searched `CASE`, compare leaves, null tests, field predicates,
  text predicates, collection membership, and compact SQL membership all share
  the same `BoolTruth` demand-driven path.
- Compact SQL membership no longer builds the opposite membership predicate
  when the caller only needs the TRUE set.
- `NOT IN (... NULL ...)` still fails closed to `False` for the FALSE branch,
  preserving the existing SQL UNKNOWN behavior.

## 0.196.9

### 📝 Summary

- Tightens the next ordered-read execution slice without changing public API,
  cursor tokens, persisted row/index/schema formats, stable storage layout, or
  cache identity.
- Skips post-access reorder for pure covering reads when the selected stream
  has already proven the requested projection order.
- Short-circuits single-primary-key covering row assembly and stops
  ignore-mode aggregate covering windows once the effective page limit has
  been emitted.
- Applies explicit `NOT IN` / `!=` branch exclusions before enforcing the
  branch-set cap, so syntactically over-cap `IN (...)` predicates can still use
  the bounded branch route when exclusions reduce the admitted branch set under
  the cap.
- Adds deterministic matrix coverage for the pruned-over-cap branch predicate
  shape so future full-matrix deltas cannot miss it.

### 🔧 Changed

- Proven-order pure covering projections now avoid a redundant post-access
  reorder step when the primary-key or index-order stream already satisfies
  the requested order contract.
- Single-primary-key covering projections use a narrow assembly fast path
  instead of building the generic covering row representation.
- Two-branch covering merge windows read smaller chunks when the page fetch
  hint is large enough to make the old fair chunk over-read visible.
- Aggregate covering windows using ignore-mode row consistency stop once the
  effective page limit has been emitted.
- Branch-set planning prunes explicit exclusions before the cap check while
  preserving the existing hard cap for the surviving branch list.
- Raw over-cap branch lists without exclusions still use the safe fallback
  path.

## 0.196.8

### 📝 Summary

- Fixes resumed branch-set SQL cursor pages so secondary-prefix fast streams
  preserve the prepared continuation boundary instead of rebuilding a fresh
  no-cursor binding.
- Keeps the slice hard-cut and format-neutral: no persisted format, cursor
  token format, public API, cache identity, public read-admission contract, or
  stable storage format changed.
- This is a correctness fix, not a performance optimisation claim.

### 🩹 Fixed

- Secondary fast-stream route requests now carry the full
  `AccessStreamBindings` produced by route planning.
- The secondary-index fast-stream executor reuses those bindings when invoking
  the structural fast-stream request instead of reconstructing bindings from
  prefix specs and direction.
- Aggregate secondary fast paths now construct an explicit initial
  `AccessStreamBindings` value for their no-cursor path, keeping aggregate and
  scalar routing contracts aligned.
- Cursor pages no longer apply a bounded physical fetch hint unless the access
  continuation can be honored by a primary-key boundary or an index-scan anchor.
  This prevents a cursor-admitted route from silently degrading into an
  unsafe no-boundary physical scan.

  before/after performance evidence instead of focused-only proof.

  artifacts so optimized, unchanged, failed, and regressed scenarios are visible

  recovery or journal semantics change.

### 🔧 Changed

- Selected secondary index-range bounds now count as access-proven predicates
  during residual stripping. Queries such as an age-range filter ordered by the
  same age index no longer retain redundant post-access residual work when the
  chosen range already proves the lower and upper bounds.

  residual filters, and composite ranges strip only the fixed prefix equality
  plus the selected range slot bounds.
- Public read admission can now admit exact primary-key `by_id(...)` and
  bounded `by_ids(...)` reads from the selected key-count upper bound, without
  requiring a redundant `LIMIT`.
- Oversized `by_ids(...)` sets still fail the public returned-row policy.

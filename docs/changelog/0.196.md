# IcyDB 0.196 Changelog

## 0.196.2

### 📝 Summary

- Promotes bounded ordered-read limit-stop proof into verbose EXPLAIN route
  diagnostics.
- Keeps the slice diagnostic-only: no query execution semantics, cursor
  encoding, cache identity, persisted format, recovery behavior, public API, or
  public read-admission policy changed.

### 🔧 Changed

- Verbose route diagnostics now include `diag.r.limit_stop_after`.
- Directly streaming bounded ordered reads report the planned limit, lookahead,
  and fetch window, for example
  `possible(limit=3,lookahead=1,fetch=6)`.
- Routes that cannot prove final limit-stop behavior report a stable disabled
  reason such as `requires_materialized_sort`,
  `residual_filter_blocks_direct_streaming`, `continuation_applied`,
  `no_limit`, or `no_bounded_fetch`.

### 🧪 Testing

- Updated verbose EXPLAIN snapshots for top-N, materialized-order,
  index-prefilter, index-range-residual, and fallback route shapes.
- Added a focused assertion that top-N verbose EXPLAIN carries the
  `limit_stop_after` proof beside the existing fetch hint.

### 📊 Performance

- No runtime performance delta was measured for this diagnostics-only slice.
- Full SQL matrix and wasm-size measurement were not rerun.

### 📚 Documentation

- Updated the 0.196 design and implementation-result artifacts to distinguish
  matrix-level structured `limit_stop_after` attribution from the verbose
  EXPLAIN route diagnostic.

### 📊 Complexity Delta

- Code footprint for this slice was small: 8 tracked files changed at the last
  snapshot, with about +97 / -2 tracked lines.
- Runtime execution complexity is unchanged. The implementation only projects
  existing route facts into verbose diagnostics.
- Wasm-size delta was not measured.

- `cargo test -p icydb-core --all-features db::query::intent::tests::explain -- --nocapture`

## 0.196.1

### 📝 Summary

- Hardens the 0.196 SQLite-audit evidence layer without changing query
  semantics, cursor encoding, cache identity, persisted format, recovery
  behavior, or public read admission.
- Makes materialized ordering and bounded ordered-read proof attribution harder
  to cherry-pick in future matrix closeouts.

### 🔧 Changed

- Materialized `ORDER BY` execution nodes now expose `order_by_idx_hint`, a
  compact normalized shape for the requested deterministic ordering.
- Verbose route diagnostics now include `diag.r.order_by_idx_hint` for
  non-streaming ordered load routes.
- SQL perf matrix samples and delta rows now carry `order_by_idx_hint`.
- Matrix samples and before/after deltas now report `limit_stop_after` as a
  structured proof object with returned limit, lookahead, observed match/index
  counts, and disabled reason.
- The matrix classifier now separates Blob `ORDER BY bucket, id` routes from
  pushable Blob `ORDER BY bucket, label, id` routes because the declared ordered
  metadata index is `(bucket, label, id)`.

### 🧪 Testing

- Added EXPLAIN coverage for materialized-order index-shape hints.
- Updated verbose explain snapshots for materialized order diagnostics.
- Added SQL perf matrix tests for order-hint extraction, Markdown delta output,
  Blob index suffix-gap classification, pushable Blob `(bucket, label, id)`
  classification, and structured `limit_stop_after` attribution.

### 📊 Performance

- A fresh order-hint deterministic matrix refresh passed before the
  classifier/`limit_stop_after` schema hardening:
  - 1,756 union scenarios;
  - 1,675 common successful scenarios;
  - 81 common failures;
  - 0 new failures;
  - 0 resolved failures;
  - 0 result-signature changes;
  - 0 cursor-signature changes;
  - 1,663 order-hint transitions.
- The classifier and `limit_stop_after` schema hardening were validated with
  focused integration tests and clippy. The full matrix was not rerun after
  those report-schema-only changes.

### 📚 Documentation

- Updated the 0.196 design and implementation-result artifacts to describe
  order-shape attribution, Blob index suffix-gap classification, structured
  limit-stop proof attribution, and scratch-artifact retention.

### 📊 Complexity Delta

- Code footprint for this diagnostics slice is report/test heavy: 11 tracked
  files changed at the last snapshot, with about +556 / -33 tracked lines.
- Runtime complexity is unchanged for query execution semantics; the production
  change is additional diagnostics attached to explain output.
- Matrix complexity increased intentionally so future optimizer claims must
  show route shape, limit-stop proof, and disabled reasons.
- Wasm-size delta was not measured. No wasm/generated output was changed.

## 0.196.0

### 📝 Summary

- Closes the SQLite-informed ordered-read pushdown slice with adversarial
  before/after performance evidence instead of focused-only proof.
- Adds full deterministic SQL matrix route classification and union-key delta
  artifacts so optimized, unchanged, failed, and regressed scenarios are visible
  across the whole generated matrix.
- Keeps the production runtime slice narrow: no persisted-format change, no
  cursor-token protocol change, no public API compatibility shim, and no
  recovery or journal semantics change.

### 🔧 Changed

- Selected secondary index-range bounds now count as access-proven predicates
  during residual stripping. Queries such as an age-range filter ordered by the
  same age index no longer retain redundant post-access residual work when the
  chosen range already proves the lower and upper bounds.
- The range-bound proof stays conservative. Stricter sibling predicates remain
  residual filters, and composite ranges strip only the fixed prefix equality
  plus the selected range slot bounds.
- Public read admission can now admit exact primary-key `by_id(...)` and
  bounded `by_ids(...)` reads from the selected key-count upper bound, without
  requiring a redundant `LIMIT`.
- Oversized `by_ids(...)` sets still fail the public returned-row policy.

### 🩹 Fixed

- Fixes the `E188` public read-admission failure for exact `by_id(...)`
  lookup paths that were already route-proven as primary-key access but lacked
  a literal `LIMIT`.

### 🧪 Testing

- Added planner tests for access-proven index-range residual stripping,
  stricter residual predicates, and composite prefix-plus-range stripping.
- Added session read-admission tests proving primary-key `by_id(...)` lookup
  is admitted without a redundant `LIMIT`, while broader unbounded public reads
  remain rejected.
- Added adversarial cursor continuation tests for primary-key and secondary
  ordered paths, including deletion, insertion before and after cursor
  boundaries, duplicate ordered values, primary-key tie-breaks, and indexed
  field updates.
- Added diagnostics coverage for bounded secondary-order and equality-prefix
  ordered-suffix routes.
- Extended the SQL perf matrix harness with route family/outcome labels,
  stable result signatures, cursor signatures, canister wasm profile recording,
  and a saved-report delta helper with closeout gates.

### 📊 Performance

- Full deterministic matrix closeout passed:
  - 1,756 union scenarios;
  - 1,675 common successful scenarios;
  - 699 improved scenarios;
  - 976 regressed scenarios;
  - 0 new failures;
  - 0 resolved failures;
  - 0 result-signature changes;
  - 0 cursor-signature changes;
  - 10 route-fact changes from eligible to pushed.
- The largest intended wins came from bucket-range and age-range ordered reads
  where redundant residual bounds were removed from the selected access path.
- Representative improvements:
  - `blob.select.lengths.bucket_range.bucket_label_asc.limit3`: -979,543 total
    instructions, `data_store.get` -6, index ranges -2, index entries -11;
  - `blob.select.lengths.bucket_range.bucket_label_asc.limit1`: -826,953 total
    instructions, `data_store.get` -3, index ranges -3, index entries -14;
  - `user.select.text_expr.age_range.age_desc.limit1`: -509,495 total
    instructions, `data_store.get` -2, index ranges -2, index entries -7;
  - `user.select.numeric_expr.age_range.age_asc.limit1`: -505,639 total
    instructions, `data_store.get` -2, index ranges -2, index entries -7.
- Route aggregate deltas included:
  - `secondary_order` / `pushed`: 255 scenarios, -6,488,741 total
    instructions;
  - `secondary_order` / `eligible_but_not_pushed`: 116 scenarios,
    -3,806,227 total instructions;
  - `primary_order` / `pushed`: 397 scenarios, -1,158,849 total
    instructions;
  - `equality_prefix_ordered_suffix` / `pushed`: 1 scenario, -192,393 total
    instructions.
- The largest regressions were storage-mirror materialized-order scenarios and
  stayed within the configured closeout gate. No broad semantic or failure-key
  regression was accepted.

### 📚 Documentation

- Added the 0.196 design directory with the SQLite-comparison audit, design
  plan, machine-readable findings, implementation results, and compact
  machine-readable closeout summary.
- Updated the read-admission contract to document exact selected primary-key
  access as a valid returned-row bound.
- Added the optimizer canonicalization design as an adjacent artifact. Where it
  overlaps the SQLite-comparison audit execution line, it can merge into 0.196
  rather than standing as a marker that the remaining audit findings moved out
  of scope.

### 📊 Complexity Delta

- Code footprint for the 0.196 implementation slice was mostly evidence and
  tests: 11 tracked files changed before this changelog update, with about
  +2,845 / -79 tracked lines, plus the 0.196 design result files.
- Runtime complexity increased only narrowly: range residual stripping now
  carries selected range-bound clauses, and admission can derive a returned-row
  bound from exact selected primary-key access.
- Matrix and test complexity increased intentionally to make future optimizer
  claims harder to cherry-pick.
- Wasm-size delta was not measured for this slice. The full matrix recorded the
  same `debug` canister wasm profile before and after.

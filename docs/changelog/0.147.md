# 0.147.x

## 0.147.3

`0.147.3` extends the generated-compatible accepted row-layout guard to typed
save execution and SQL write staging. Writes still use generated entity and
structural patch encoding, but they now prove accepted schema compatibility
through the same descriptor boundary before generated write codecs can run.

### SQL write schema guard

- Added a session helper that returns the accepted schema snapshot only after
  deriving generated-compatible execution authority from
  `AcceptedRowLayoutRuntimeDescriptor`.
- Routed SQL `INSERT` and SQL `UPDATE` through that helper before literal
  normalization, selector binding, patch construction, and save execution.
- Moved SQL write column inference, required-field checks, generated-field
  synthesis, and literal normalization onto the accepted runtime descriptor.
  Generated `FieldModel` metadata is now consulted only after the accepted
  field has been selected, for generated-only behavior that is still owned by
  the generated codec layer.
- Moved SQL structural patch slot selection onto accepted descriptor fields;
  patch staging now converts the accepted slot directly after the session has
  proven the descriptor is generated-compatible with the current codec bridge.
- Moved SQL write `RETURNING` all-column labels and field counts onto accepted
  descriptor fields for `INSERT`, `UPDATE`, and `DELETE`, so statement result
  shaping follows the accepted row contract before generated value extraction.
- Moved accepted-schema `SHOW COLUMNS` field enumeration onto the persisted
  snapshot itself. The generated model remains involved in full `DESCRIBE`
  only for index and relation metadata that is not persisted yet.
- Reused the `DESCRIBE` field-table renderer for `SHOW COLUMNS`, so the SQL
  shell now has one human-facing column format: full `DESCRIBE` includes
  fields, indexes, and relations, while `SHOW COLUMNS` renders only the shared
  fields section.
- Made `DESCRIBE` / `SHOW COLUMNS` spell unconstrained text and blob fields as
  `text(unbounded)` and `blob(unbounded)`, matching the accepted schema
  contract instead of hiding unboundedness behind plain type labels.
- Moved accepted-schema `SchemaInfo` field enumeration onto persisted snapshot
  fields. Generated field metadata now remains only as a temporary field-kind
  bridge for planner/predicate code that has not been converted to persisted
  field kinds.
- Routed typed save wrappers through the same generated-compatible schema guard
  before constructing `SaveExecutor`, while keeping an already-checked internal
  save path for SQL statements that need the guard before mutation staging.
- Routed direct typed-save schema guard failures into the existing `Save`
  executor error metrics, so guard failures that happen before the normal save
  span still increment the established error-class counters.
- Kept mutation behavior unchanged: unsupported schema layout or payload drift
  still fails before generated write codecs can stage rows, and additive schema
  transitions remain unsupported until write target slots are descriptor-owned.
- Changed `scripts/dev/sql.sh --init` to use reinstall mode when the target
  canister already exists, so local pre-1.0 hard-cut schema snapshot formats do
  not survive what the helper presents as a clean fixture initialization.

### Bounded text and blob fields

- Extended `FieldKind::Blob` and persisted schema snapshots with `max_len`, so
  blob length constraints survive generated model metadata, accepted schema
  storage, schema descriptions, and schema compatibility checks.
- Allowed `item(prim = "Blob", max_len = N)` beside the existing
  `item(prim = "Text", max_len = N)` directive while keeping `max_len` rejected
  for primitives without length semantics.
- Required text and blob schema fields to choose either `max_len = N` or
  `unbounded`, so unconstrained strings and bytes cannot be introduced by
  omission.
- Documented the accepted unbounded spelling as `item(prim = "Text",
  unbounded)` and `item(prim = "Blob", unbounded)`; `unbounded = true` is also
  accepted by the macro parser for explicit call sites that prefer name-value
  directives.
- Enforced text limits by Unicode scalar count and blob limits by byte count
  through the same recursive field validation path used before row bytes are
  emitted.

## 0.147.2

`0.147.2` starts routing shared prepared row-layout construction through the
accepted runtime descriptor while preserving the current exact-match behavior.
The bridge accepts only generated-compatible layouts because field payload
decode still depends on generated `FieldModel` entries. It also fixes
schema-derived `item(is = "...")` persisted rows so the generated write/read
path follows the field type's `FieldTypeMeta` storage contract instead of
forcing the legacy direct-structured payload lane. Inline schema primitive
fields, including decimal fields with `scale`, now use the generated
`FieldModel` boundary so their schema annotation remains the storage authority.

### Schema-derived item field persistence

- Changed schema-derived persisted-row generation for `item(is = "...")`
  fields to require `PersistedFieldMetaCodec` and encode/decode required,
  optional, and repeated fields through the item type's field metadata.
- Added repeated field-meta persisted-row helpers that model `many,
  item(is = T)` as `FieldKind::List(&T::KIND)` while preserving `T`'s own
  storage-decode contract, so decimal/newtype lists are not forced into
  value-storage through `Vec<T>` metadata.
- Added macro regression coverage proving required, optional, and repeated
  decimal `item(is = "ClampDecimal")` entity fields emit metadata-owned
  persisted slot bytes.
- Renamed the active structured persisted-row helper surface from
  `*_custom_*_slot_payload` to `*_structured_*_slot_payload` and renamed the
  schema-derive `item(is = "...")` helpers accordingly, so current code names
  describe the active structured and metadata-owned storage lanes directly.
- Removed low-level `PersistedRow` field storage hints. Fields that are not
  handled by explicit primitive/container slot-codec impls now require a
  type-owned `PersistedFieldSlotCodec`; decimal scale must come from a
  schema/newtype contract instead of `#[icydb(scale = ...)]`, and structured
  storage must come from the named structured helper surface instead of
  `#[icydb(value)]`.
- Reduced the low-level `PersistedRow` derive to a slot bridge: it now only
  rejects removed field hints, indexes fields, handles top-level `Option<T>`
  missing-slot behavior, emits field-local `PersistedFieldSlotCodec`
  assertions, and calls the slot-codec trait for encode/decode.
- Reduced schema-derived persisted-row emission to schema cardinality plus
  explicit owners: inline primitive/relation fields use the generated
  `FieldModel` runtime slot adapter, while `item(is = "...")` fields use
  type-owned metadata codecs. This removes Rust type-name inference for scalar,
  `Vec<u8>`, and structural storage lanes without forcing raw `Decimal` to
  guess a field scale.
- Moved the remaining primitive, collection, and test-only schema/newtype
  storage routing behind `PersistedFieldSlotCodec` impls so the derive no
  longer imports or infers `FieldKind`.
- Made `Value` runtime-only by removing its field metadata, slot-codec, and
  structured persisted-field codec impls; typed wrappers now own persistence
  for enum/profile fixtures that still project through runtime `Value`.
- Documented the persisted-row `Value` boundary as a schema-validated runtime
  adapter and added a source guard that fails CI if production code reintroduces
  persisted-field contract impls for `Value` or `Vec<Value>`.
- Added compile-fail coverage that rejects removed `#[icydb(...)]` persisted-row
  field hints at the derive boundary.
- Added compile-fail coverage proving `Value` no longer satisfies either
  `PersistedFieldSlotCodec` or `PersistedStructuredFieldCodec`.

### Generated-compatible accepted row layout

- Added a `RowLayout` constructor that freezes field count and primary-key slot
  from `AcceptedRowLayoutRuntimeDescriptor` after validating that every accepted
  field slot still matches the generated field index.
- Extended generated-compatible validation to require matching persisted kind,
  storage-decode contract, and leaf codec, so fields that keep the same name and
  slot but change payload format are rejected before generated row decode can
  misread fresh data.
- Added an `EntityAuthority` handoff that replaces the generated fallback row
  layout with the accepted descriptor-derived row layout at the session shared
  plan cache boundary.
- Centralized the accepted-schema snapshot plus descriptor-derived authority
  pairing in `DbSession`, so session call sites cannot independently combine
  accepted schema identity with generated-only row-layout authority.
- Kept slot reorders and additive/remapped layouts rejected at this bridge so
  the current generated-field decoder cannot silently decode payloads through
  the wrong field metadata.
- Added row-decode tests for exact accepted layout conversion, slot-reorder
  rejection, and payload-contract drift rejection at the generated-compatible
  bridge.

## 0.147.1

`0.147.1` continues the live-schema authority boundary. It makes schema
transition policy decisions directly observable in runtime metrics and adds the
accepted row-layout runtime descriptor that row decode/write paths will consume
when accepted schema becomes the runtime layout authority.

### Schema transition policy metrics

- Added `SchemaTransitionOutcome` as the stable metrics taxonomy for transition
  policy decisions.
- Added global `EventOps` and per-entity `EntitySummary` counters for
  transition checks, accepted exact-match plans, and each rejected transition
  category.
- Routed existing-snapshot reconciliation through the transition metrics
  recorder before returning either an accepted exact-match reconciliation result
  or an unsupported schema-evolution error.
- Kept first-create reconciliation out of transition counters because no
  stored accepted snapshot exists yet.
- Added guard coverage proving schema reconciliation totals, schema transition
  totals, and SQL compile rejection totals equal their low-cardinality bucket
  sums globally and per entity.

### Accepted row-layout runtime descriptor

- Added `AcceptedRowLayoutRuntimeDescriptor` as the schema-owned artifact that
  projects accepted schema snapshots into explicit runtime facts.
- Added `AcceptedRowLayoutRuntimeField` so durable field ID, physical slot,
  persisted kind, nested leaf metadata, storage decode contract, leaf codec,
  database default metadata, and missing-slot policy are bundled once by schema
  code instead of re-derived from generated model fields or raw persisted
  snapshots.
- Added descriptor lookups by physical slot, durable field ID, and persisted
  field name so future decode/write call sites can consume one schema-owned
  runtime surface.
- Added `AcceptedFieldAbsencePolicy` to keep nullable/default missing-slot
  behavior explicit and separate from Rust struct defaults.
- Validated descriptor construction whenever reconciliation returns an accepted
  schema snapshot, while keeping transition behavior exact-match-only until
  decode/write call sites consume the descriptor.
- Added coverage that generated-extra row-layout facts are still rejected with
  concise field metadata, so additive fields do not become accepted just
  because the runtime descriptor boundary now exists.

## 0.147.0

`0.147.0` is the accepted-schema authority closeout release. It keeps the
0.146 architecture endpoint intact while making local schema-change testing and
schema-drift failures easier to use: dev-shell upgrades can preserve stable
memory, startup reconciliation runs before row replay/index rebuild when
recovery is needed, and row-layout transition errors now summarize the first
changed field/slot fact with field metadata instead of dumping raw layout
structures. Schema reconciliation now also accepts through an explicit
`SchemaTransitionPlan`, so future supported transitions have a plan artifact
to extend instead of overloading the old exact-match branch.

### Schema-change upgrade testing

- Added `scripts/dev/sql.sh --upgrade` so local SQL canisters can be rebuilt
  and upgraded without fixture reset or stable-memory wipe.
- Documented the local rejection flow in `sql.sh --help`: initialize once,
  change the generated schema, upgrade, then query or describe the entity.

### Startup rejection ordering

- Moved schema reconciliation ahead of startup row replay/index rebuild when
  recovery work is required.
- Kept post-recovery reconciliation so successful recovery still validates
  accepted schema metadata before operation-specific work proceeds.
- Added a regression test proving schema drift is reported before incompatible
  stored rows can fail row decode during rebuild.

### Transition plan boundary

- Replaced the bare `SchemaTransitionDecision::ExactMatch` accept branch with
  `SchemaTransitionDecision::Accepted(SchemaTransitionPlan)`.
- Added `SchemaTransitionPlanKind::ExactMatch` as the first accepted plan kind,
  preserving current exact-match behavior while creating the closed artifact
  future additive transitions will extend.
- Routed reconciliation outcome selection through the accepted plan kind
  instead of matching a one-off exact-match decision variant.

### Schema drift diagnostics

- Replaced raw `SchemaRowLayout` debug dumps in row-layout transition
  rejections with concise schema-owned summaries.
- Row-layout mismatch errors now report stored/generated layout versions,
  stored/generated field counts, and the field ID, slot, name, and kind for
  the first changed, stored-extra, or generated-extra layout fact.

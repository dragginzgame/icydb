field decode contract it actually consumes, and splits generated-compatible
runtime value encoding behind a field-model helper. Production row readers,
direct decode helpers, patch-backed readers, and canonical slot readers now
call the contract-first decode helper, while internal write/canonicalization
paths use the resolved-field encode helper instead of re-entering the exported
model/slot adapter.

### Runtime decode boundary

- Added the internal `decode_runtime_value_from_field_contract(...)` entrypoint
  for persisted-row callers that already resolved a `StructuralFieldDecodeContract`.
- Added the internal `encode_runtime_value_for_field_model(...)` helper for

  decode helpers, and serialized patch readers to use the contract-first name.
- Updated canonical row payload normalization and structural patch serialization

- Kept `decode_slot_into_runtime_value(model, slot, bytes)` and
  `encode_runtime_value_into_slot(model, slot, value)` exported as
  generated/runtime compatibility wrappers; they now only resolve the field
  contract/model and delegate.
- Kept decode behavior, scalar fast paths, validation, and malformed-payload
  error taxonomy unchanged.

### Status

- Runtime value materialization now has a named contract-first internal
  boundary, and runtime value encoding has a named resolved-field boundary.
- Remaining model/slot decode/encode access is compatibility surface for
  generated runtime helpers and low-level persisted-row tests.

## 0.147.9

`0.147.9` retires the standalone generated-model structural patch setter and
moves sparse structural write omission checks up to the accepted-schema session
boundary. Field-name structural patch construction now has one active owner:
`DbSession::structural_patch(...)`, which resolves fields through the accepted
schema descriptor before producing the slot-addressed core patch. Sparse
insert/replace patches also consult the descriptor's absence policy before
generated derive materialization runs, so missing required fields are rejected
from database schema metadata rather than discovered through Rust construction
or generated decode defaults. Session writes also prove the accepted schema is
still generated-compatible before mutation staging starts, which keeps
unsupported accepted-schema transitions from reaching row write/decode
boundaries.

### Structural patch construction

- Removed `StructuralPatch::set_field(model, field_name, value)` from the core
  patch container. Core patches now only record already validated field-slot
  assignments.
- Removed the public facade `StructuralPatch::set_field(...)` convenience.
  Public callers should build field-bearing patches through
  `DbSession::structural_patch(...)`; `StructuralPatch::new()` remains available
  for explicit empty-patch mutation cases.
- Migrated executor structural mutation coverage off the generated-model
  setter and onto the session-owned accepted-schema patch constructor.
- Kept persisted-row tests on slot-index patch construction because those tests
  intentionally validate the low-level slot serialization boundary.

### Sparse write absence policy

- Added a session-owned sparse structural patch absence-policy gate for

## 0.147.0

`0.147.0` is the accepted-schema authority closeout release. It keeps the
0.146 architecture endpoint intact while making local schema-change testing and
schema-drift failures easier to use: dev-shell upgrades can preserve stable
memory, startup reconciliation runs before row replay/index rebuild when
recovery is needed, and row-layout transition errors now summarize the first
changed field/slot fact with field metadata instead of dumping raw layout
structures. Schema reconciliation now also accepts through an explicit
`SchemaTransitionPlan`, so future supported transitions have a plan artifact
to extend instead of overloading the old exact-match branch.

### Schema-change upgrade testing

- Added `scripts/dev/sql.sh --upgrade` so local SQL canisters can be rebuilt
  and upgraded without fixture reset or stable-memory wipe.
- Documented the local rejection flow in `sql.sh --help`: initialize once,
  change the generated schema, upgrade, then query or describe the entity.

### Startup rejection ordering

- Moved schema reconciliation ahead of startup row replay/index rebuild when
  recovery work is required.
- Kept post-recovery reconciliation so successful recovery still validates
  accepted schema metadata before operation-specific work proceeds.
- Added a regression test proving schema drift is reported before incompatible
  stored rows can fail row decode during rebuild.

### Transition plan boundary

- Replaced the bare `SchemaTransitionDecision::ExactMatch` accept branch with
  `SchemaTransitionDecision::Accepted(SchemaTransitionPlan)`.
- Added `SchemaTransitionPlanKind::ExactMatch` as the first accepted plan kind,
  preserving current exact-match behavior while creating the closed artifact
  future additive transitions will extend.
- Routed reconciliation outcome selection through the accepted plan kind
  instead of matching a one-off exact-match decision variant.

### Schema drift diagnostics

- Replaced raw `SchemaRowLayout` debug dumps in row-layout transition
  rejections with concise schema-owned summaries.
- Row-layout mismatch errors now report stored/generated layout versions,
  stored/generated field counts, and the field ID, slot, name, and kind for
  the first changed, stored-extra, or generated-extra layout fact.

## 0.65.4 - 2026-03-27 - SQL Prefix LIKE Expansion and Executor Test Revival

### Summary

`0.65.4` keeps the canonical-row contract intact and combines one user-visible
SQL text-predicate expansion with a large executor-test revival checkpoint.

This patch makes strict prefix `LIKE` work on plain fields and on both
`LOWER(...)` and `UPPER(...)`, then uses the revived executor/query/diagnostics
test roots to flush out and fix a few real drifted behaviors that had fallen
out of the active harness.

- accept strict prefix `LIKE 'prefix%'` on plain text fields
- accept the same strict prefix form through both `LOWER(field)` and
  `UPPER(field)` without changing broader wildcard support
- revive more dormant executor, query-intent, and diagnostics tests as live
  owner-local coverage
- fix singleton `Unit` scalar-row encoding and diagnostics storage-report
  identity collapse exposed by that revived coverage

## 0.65.3 - 2026-03-26 - Explain and Schema Wasm Trim Pass

### Summary

`0.65.3` keeps the post-`0.65.2` canonical-row model unchanged and focuses on
raw canister wasm reduction.

This patch trims deterministic but non-essential ordered-container machinery
from the query explain, schema describe/info, planner order validation, and
predicate coercion metadata surfaces without changing row invariants, query
semantics, or explain output shape.

- replace retained `BTreeMap`-style helpers on small deterministic metadata
  paths with lighter owner-local containers
- keep EXPLAIN JSON/text field order and verbose diagnostic output stable
- cut shrunk `wasm-release` size by `7,483` bytes on both `minimal` and
  `one_simple`

### Changed

- trimmed EXPLAIN execution metadata container overhead in:
  - `crates/icydb-core/src/db/query/explain/execution.rs`

  - `crates/icydb-core/src/db/query/explain/tests.rs`
  - explain node properties now use a small sorted `ExplainPropertyMap`
    instead of retaining tree-map machinery while preserving deterministic

- trimmed schema metadata container overhead in:
  - `crates/icydb-core/src/db/schema/info.rs`
  - `crates/icydb-core/src/db/schema/describe.rs`
  - schema info field tables now use compact sorted vectors with binary search
    instead of `BTreeMap`/`BTreeSet` helpers on the read-only metadata path

## 0.65.2 - 2026-03-26 - Canonical Row Cleanup and Audit Pass

### Summary

`0.65.2` keeps the `0.65.1` canonical-row contract intact and focuses on
cleanup plus final write-boundary hardening around the new invariant.

The runtime no longer carries a production row writer that can fabricate
missing-slot shapes, `RawRow` is no longer a production persistence bypass, the
main `0.65` hotspot files are simpler to maintain, and the current tree now has
a fresh audit/report baseline for DRY, layer, structure, and complexity
follow-up work.

- split malformed-row fabrication away from the production slot writer
- lock production row-store writes to `CanonicalRow` instead of `RawRow`
- route save/replay after-images back through the canonical row emitter and add
  source/trybuild guards around that boundary
- reduce owner-local branching in planner, predicate runtime, and EXPLAIN
  descriptor hotspots
- publish fresh audit snapshots for the post-`0.65.1` tree

## 0.65.1 - 2026-03-26 - Structural Read Boundary Hardening

### Summary

`0.65.1` closes the remaining read-side integrity gap left after the `0.65.0`
storage-format hard-cut.

The canonical row contract is now enforced at the shared structural read
boundary itself, not only in write, replay, and consumer-local decode paths.
Any malformed declared slot now fails when the row is first opened through
`StructuralSlotReader`, even if the immediate caller only intended to read some
other field.

- make structural row construction fully decode every declared slot before any
  consumer can observe the row
- remove the last canonical scalar predicate branch that still carried optional
  slot semantics
- add regressions for malformed unprojected scalar payloads and structural slot
  span corruption

### Changed

- hardened the shared structural read boundary in:
  - `crates/icydb-core/src/db/data/persisted_row.rs`
  - `StructuralSlotReader::from_raw_row(...)` now validates row shape and fully
    decodes every declared slot before returning the reader
  - malformed but unreferenced fields can no longer stay latent behind
    projection-only, relation-only, or index-only read paths
- tightened canonical scalar predicate execution in:
  - `crates/icydb-core/src/db/predicate/runtime.rs`
  - scalar `IS NULL` / `IS NOT NULL` evaluation now uses
    `required_scalar(...)` directly instead of preserving an optional
    `get_scalar(...)` branch on the canonical lane
- expanded invariant coverage in:
  - `crates/icydb-core/src/db/executor/projection/tests.rs`
  - `crates/icydb-core/src/db/data/persisted_row.rs`
  - structural reads now have explicit regression coverage for:
    - malformed unprojected scalar slots
    - slot-span overflow / out-of-bounds corruption

## 0.65.0 - 2026-03-26 - Canonical Row Invariant Hard-Cut

### Summary

`0.65.0` makes canonical persisted rows a hard runtime contract.

After this release, every stored row must carry exactly one declared slot per
model field, every slot payload must be present and decodable, and scalar slots
must use the canonical `0xFF`-prefixed scalar envelope instead of older raw
CBOR-shaped scalar payloads.

This is a breaking storage-format cut, not a compatibility release. The runtime
now rejects malformed or tolerant legacy row shapes during commit validation,
commit replay, startup rebuild, query decode, projection, predicate execution,
index-key construction, and reverse-relation maintenance instead of attempting
to normalize or silently tolerate them.

- remove the remaining absent-slot tolerance from the canonical structural row
  lane
- make commit, replay, and read paths all enforce the same row-shape contract
- keep mixed or intentionally generic slot-reader APIs only where optional-row
  domains still actually exist

### Breaking

- persisted rows are now invalid if `slot_count != model.fields().len()`
- persisted rows are now invalid if any declared slot payload is missing or
  zero-length
- scalar slots are now invalid if they are stored as raw CBOR instead of the
  canonical scalar envelope
- there is no runtime repair or compatibility normalization path for older
  malformed rows

### Changed

- hardened structural row decode in:
  - `crates/icydb-core/src/db/data/structural_row.rs`
  - structural row decode now fails closed on slot-count drift and zero-length
    slot payloads
- hardened serialized patch replay in:
  - `crates/icydb-core/src/db/data/row.rs`
  - `crates/icydb-core/src/db/data/persisted_row.rs`
  - serialized structural patches now require concrete payload bytes for every
    emitted slot
  - the old “clear slot” replay shape is gone from the canonical patch path
- hardened commit recovery and validation in:
  - `crates/icydb-core/src/db/commit/prepare.rs`
  - commit recovery now validates marker authority before model-driven row
    decode
  - commit-marker row validation now decodes every declared slot through the
    canonical required field contract
- hardened structural read/decode in:
  - `crates/icydb-core/src/db/executor/terminal/row_decode.rs`
  - structural terminal row decode now reads scalar and non-scalar payloads
    through `required_scalar(...)` and `required_bytes(...)`
  - raw-CBOR scalar slot payloads are rejected instead of being accepted as an
    older equivalent encoding
- pushed canonical required-slot semantics through structural query execution:
  - `crates/icydb-core/src/db/predicate/runtime.rs`
  - `crates/icydb-core/src/db/executor/projection/eval/mod.rs`
  - `crates/icydb-core/src/db/executor/projection/eval/scalar.rs`
  - `crates/icydb-core/src/db/executor/projection/materialize/mod.rs`
  - `crates/icydb-core/src/db/executor/projection/materialize/structural.rs`
  - structural predicates, scalar expression evaluation, and SQL projection
    materialization now fail closed on missing declared slots instead of
    flattening those failures into generic “missing field” results
- pushed the same invariant through structural index and relation paths:
  - `crates/icydb-core/src/db/index/plan/mod.rs`
  - `crates/icydb-core/src/db/index/key/build.rs`
  - `crates/icydb-core/src/db/relation/reverse_index.rs`
  - canonical index-key construction and singular strong-relation decode now
    use required slot reads instead of optional fallback branches
- cleaned up now-dead compatibility helpers and stale missing-slot branches in:
  - `crates/icydb-core/src/db/data/mod.rs`
  - `crates/icydb-core/src/error/mod.rs`
  - `crates/icydb-core/src/db/data/persisted_row.rs`

### Migration Notes

- if existing stored rows relied on missing declared slots, tolerant slot
  counts, or raw-CBOR scalar payloads, they must be rewritten out of band
  before use under `0.65.0`
- acceptable recovery paths are:
  - explicit migration tooling
  - manual state rewrite
  - state reset
- there is no in-runtime “upgrade and normalize old rows on read” behavior in
  this line

- `cargo clippy -p icydb-core --all-targets -- -D warnings`
- `cargo test -p icydb-core -- --nocapture`
- `cargo test -p icydb --lib -- --nocapture`

### Notes

- this release is intentionally storage-strict; the simplification is part of
  the product change, not just an internal cleanup
- the remaining optional-slot behavior in `icydb-core` is limited to generic or
  mixed-row seams that are not on the canonical structural row path

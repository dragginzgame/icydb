# IcyDB 0.167 Changelog

## 0.167.0

### ➕ Added

- Added an explicit per-store storage model with `StoreStorage` and
  `StoreStableMemoryConfig`.

  ```rust
  StoreStorage::Stable(StoreStableMemoryConfig::new(data, index, schema))
  ```

- Added the nested derive syntax for the only implemented 0.167 storage mode:

  ```rust
  #[store(
      ident = "SESSION_STORE",
      store_name = "sessions",
      canister = "AppCanister",
      storage(stable(
          data_memory_id = 20,
          index_memory_id = 21,
          schema_memory_id = 22,
      )),
  )]
  ```

- Added compile-fail coverage for missing store storage, old flat memory IDs,
  reserved heap syntax, heap syntax with stable IDs, and unknown storage modes.

### ⚠️ Breaking

- Store declarations must now place stable memory IDs inside
  `storage(stable(...))`.

  ```rust
  // removed
  #[store(data_memory_id = 20, index_memory_id = 21, schema_memory_id = 22)]

  // required
  #[store(storage(stable(
      data_memory_id = 20,
      index_memory_id = 21,
      schema_memory_id = 22,
  )))]
  ```

- The old flat store memory-ID syntax is rejected rather than aliased or
  shimmed. This is a source-level macro hard cut only.

### 🔧 Changed

- Moved store validation behind the explicit stable-storage branch while
  preserving the existing stable memory range, reserved ID, stable-key, and
  data/index/schema distinctness checks.

- Moved generated store registry wiring behind an explicit stable-storage
  branch. Stable stores still emit `ic_memory_key!` for data, index, and schema
  roles and still register all three stable allocation identities.

- Reserved `storage(heap())` for a future volatile store line without adding a
  runtime heap store variant or generated heap wiring in 0.167.

### 🧪 Testing

- Proved the storage-boundary matrix for schema model ownership, derive macro
  admission/rejection, stable generated wiring, unchanged allocation identity,
  and no generated heap path.

### 📚 Documentation

- Updated the 0.167 storage-boundary design to reflect the implemented hard
  cut, proof matrix status, and deferred heap-store work.

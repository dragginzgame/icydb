# IcyDB 0.202 Changelog

## 0.202.0 - 2026-07-13

### 📝 Summary

- Makes Rust `Default` an explicit domain-model decision instead of an
  implicit capability of every generated schema type.
- Removes the invalid enum sentinel that existed to satisfy the former blanket
  default policy.
- Keeps database defaults under accepted-schema authority and closes omission,
  materialization, and DDL-owned-field gaps found during the separation audit.

### ⚠️ Breaking

- Generated entities, records, newtypes, enums, and tuples implement Rust
  `Default` only when their schema declares `traits(add(Default))` and the type
  has a valid construction strategy.
- Removes `variant(unspecified)`, the generated `Unspecified` enum variant, and
  the validation path that treated that placeholder as an invalid value.
- Generated enum schema descriptions no longer expose the Rust-only
  `EnumVariant::default()` or `EnumVariant::unspecified()` metadata accessors.
- ICRC-3 generic-value `Unit` no longer decodes through an unspecified enum
  placeholder; values outside the declared enum domain fail with a typed
  decode error.
- This is a pre-1.0 hard cut. No aliases, fallback spellings, placeholder
  variants, dual trait policy, or legacy metadata reconstruction remain.

### 🔧 Changed

- Lists, sets, and maps retain their intrinsic empty Rust default at their
  owning generated nodes. Generated create inputs retain `Default` as the
  canonical representation of no authored fields.
- Explicit entity and record defaults require construction values for every
  field. Newtype defaults require a construction expression, enum defaults
  require one selected variant, and tuple or payload-enum defaults carry the
  required transitive trait bounds.
- Impossible `Default` requests fail during macro expansion at the owning type
  instead of being silently suppressed or surfacing later at an unrelated call
  site.
- Rust construction defaults and accepted database defaults are independent:
  `field(default = ...)` continues to propose database metadata without
  implicitly granting `impl Default` to its containing domain type.

### 🩹 Fixed

- SQL inserts, typed creates, and structural insert or replace writes now use
  one accepted-schema omission policy for defaulted, nullable, generated, and
  managed fields.
- Missing structural slots reuse bundle-validated accepted default payloads
  directly, including catalog-backed unit-enum defaults, instead of attempting
  a lossy decode and re-encode through a non-enum runtime value.
- SQL writes preserve explicit `NULL` and DDL-owned field values through typed
  preflight and final row emission. Only an omitted field invokes its accepted
  default or nullable absence policy.
- Real `SET DEFAULT` and `DROP DEFAULT` schema changes reject when an index key
  or filtered-index predicate depends on the field and would require a physical
  rebuild; exact no-op mutations remain legal.

### 🗑️ Removed

- Deletes the shared default-on trait policy and the enum-specific invalid
  sentinel machinery rather than retaining exceptions around either rule.
- Removes the obsolete accepted patch-backed entity materializer after insert,
  replace, and update validation converged on one canonical structural
  after-image.

### 🧭 Migration Notes

- Add `traits(add(Default))` only to domain types that have an intentional,
  valid Rust construction default. Otherwise construct the value explicitly.
- Remove `variant(unspecified)` and handle invalid external values as typed
  construction or decode errors rather than as an in-domain placeholder.
- Stop reading generated enum default or unspecified flags as schema metadata;
  accepted database-default metadata remains the runtime authority.
- No database recreation, row rewrite, index rebuild, cursor conversion, or
  persisted-format migration is required for 0.202.0.

### 🧪 Testing

- Focused derive, generated-schema, macro behavior, macro UI, ICRC-3 codec,
  accepted enum-default, missing-slot, sparse mutation, SQL write, and
  structural persistence suites pass.
- Package-scoped all-target, all-feature Clippy passes with warnings denied for
  the four affected packages. Formatting and patch hygiene checks pass.
- The user-owned full push suite remains the final release gate.

### 📊 Audit

- The slice spans 67 implementation, documentation, and focused-test files
  with approximately 1,310 insertions and 560 deletions. About 315 inserted
  lines are standalone behavior tests and diagnostic snapshots.
- The implementation shape is simpler at the generated trait boundary and
  neutral overall: one implicit trait rule, one invalid sentinel feature, two
  schema-only flags, three duplicate omission decisions, and one default
  decode/re-encode path are gone. One index-dependency guard is added to keep
  physical entries consistent.
- `ten_entity_fluent_rows` measures 2,366,601 raw wasm bytes and 2,204,554
  canonically shrunk bytes, reductions of 111 and 107 bytes from 0.201.4.
  Deterministic gzip is secondary at 768,620 raw and 733,155 shrunk bytes,
  reductions of 209 and 166 bytes.
- Runtime performance is expected to remain neutral. The slice removes a
  default decode/re-encode step and does not add a new storage representation,
  so a separate performance probe is not applicable.

### 📚 Documentation

- Records the explicit Rust-default policy, the accepted database-default
  authority boundary, the enum hard cut, closeout validation, complexity, and
  measured wasm impact in the normative 0.202 design.

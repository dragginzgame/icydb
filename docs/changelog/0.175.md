# IcyDB 0.175 Changelog

## 0.175.0

### ➕ Added

- Starts the journaled store audit, hardening, and DRY cleanup line with an
  active 0.175 design covering ordered-overlay traversal, snapshot boundaries,
  hot-path guards, recovery/fold edge evidence, and closeout criteria.
- Adds a shared ordered-overlay traversal primitive for journaled canonical
  stable maps, volatile live maps, and tombstones so data, index, and schema
  store wrappers can preserve the same streaming read-view semantics.
- Adds accepted-schema-backed entity metadata to catalog surfaces so
  `SHOW ENTITIES` can report storage mode, column count, index count, relation
  count, and `sv` schema version without widening default output with verbose
  paths.
- Adds SQL `SHOW MEMORY` catalog inspection for stable-memory tags, memory IDs,
  and owning stores.

### 🔧 Changed

- Routes journaled data and index traversal through streaming overlay paths
  that honor visitor early-stop before materializing a full merged view.
- Applies the same streaming overlay boundary to journaled schema snapshot
  reads that are on SQL query hot paths.
- Carries accepted schema, schema fingerprint, and accepted authority through
  cold SQL compile-to-execute paths so query calls do not rely on persisted
  query-cache writes to avoid schema reconstruction.
- Keeps fold-only journaled index materialization isolated behind a
  purpose-named helper.

### 🧪 Testing

- Adds source-audit coverage for journaled traversal materialization,
  fold-only snapshot helpers, diagnostics authority, and generated-model
  fallback boundaries.
- Adds regression coverage for stable and journaled primary-order SQL
  projections, mixed canonical/live/tombstone overlay traversal, schema-store
  latest-snapshot traversal, and compact catalog column rendering.

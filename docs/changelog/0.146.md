# 0.146.x

## 0.146.3

`0.146.3` keeps moving schema metadata consumers behind the accepted schema
snapshot boundary and expands the runtime metrics report so operational
snapshots expose the counters the engine already maintains. `DESCRIBE` /
`SHOW COLUMNS` no longer rebuild their own field-name slot map from raw
snapshot fields, accepted metadata now drives the entity header plus top-level
field kind/queryability facts, and `icydb_metrics()` now reports saved-row,
per-entity, detailed plan-route, and window-filter details that were previously
collapsed or hidden.

### Accepted schema lookup boundary

- Added accepted-schema field lookup helpers for persisted field names and
  routed schema description formatting through them.
- Removed the local `BTreeMap` staging from schema description formatting.
- Updated accepted-schema `DESCRIBE` / `SHOW COLUMNS` formatting so top-level
  field kind and queryability facts come from `PersistedFieldKind`.
- Updated accepted-schema `DESCRIBE` headers so entity path/name and
  primary-key label come from the accepted snapshot where available.
- Added a `PersistedFieldKind` to planner `FieldType` projection and used it
  for accepted metadata queryability, preparing the later live-schema SQL
  admission switch.
- Added strict SQL literal canonicalization for `PersistedFieldKind` and routed
  SQL INSERT/UPDATE literal validation through the accepted schema snapshot.
- Kept output shape unchanged: top-level fields still report their current
  accepted slot and nested fields still report no top-level slot.
- Kept nested structured-field rows generated-model based because persisted
  snapshots do not yet carry nested leaf descriptions.
- Added a guard proving accepted top-level metadata wins when formatting field
  descriptions.

### Metrics observability

- Added aggregate and per-entity `rows_saved` accounting so save spans now
  report affected rows instead of only instruction totals.
- Expanded per-entity metrics summaries to expose already-collected save,
  row-flow, index, reverse-index, relation-validation, unique-violation, and
  non-atomic partial-commit counters.
- Split plan metrics into exact top-level route counters for by-key, by-keys,
  key-range, index-prefix, index-multi-lookup, index-range, explicit full-scan,
  union, and intersection paths while preserving the older coarse
  key/index/range/full-scan counters for existing consumers.
- Added metrics window metadata: `window_end_ms`, `window_duration_ms`,
  `window_filter_matched`, `requested_window_start_ms`, and
  `active_window_start_ms`, so empty future-window reports explain why they are
  empty.
- Sorted per-entity summaries by total visible activity so write-heavy and
  maintenance-heavy entities are not hidden below read-heavy entities.

## 0.146.2

`0.146.2` tightens the schema-authority handoff after the initial schema store
went live. Reconciled schema snapshots now carry an explicit accepted wrapper,
database defaults have their own runtime-model contract, and persisted field
kinds have a schema-owned SQL capability projection ready for the later live
schema admission switch.

### Accepted schema authority

- Added `AcceptedSchemaSnapshot` as the typed handoff for snapshots that have
  passed reconciliation.
- Updated SQL `DESCRIBE` and `SHOW COLUMNS` persisted-schema helpers to accept
  the accepted wrapper instead of arbitrary decoded `PersistedSchemaSnapshot`
  payloads.
- Preserved the current initial-version-only reconciliation policy.

### Database defaults

- Added explicit runtime-model database-default metadata.
- Kept the initial database-default state `None`, preserving the rule that Rust
  `Default` does not authorize database slot synthesis.
- Routed persisted schema proposal construction through the runtime-model
  database-default contract instead of hard-coding it at the proposal boundary.

### SQL capability projection

- Added a schema-owned SQL capability projection over `PersistedFieldKind`.
- The projection records selectable, comparable, orderable, groupable, and
  aggregate-input facts for accepted persisted schema fields.
- Kept this as groundwork only: SQL planning still uses generated model
  metadata until live schema authority is fully switched.
- Added coverage for blob, numeric, collection, structured, and relation-key
  capability edges.

### Describe compatibility

- Kept the `EntityFieldDescription` Candid `slot` field as plain `u16` for
  CLI/canister compatibility.
- Preserved Rust `Option<u16>` accessor semantics by mapping a reserved no-slot
  value for nested fields.
- Added a Candid shape guard so `slot` cannot drift back to an optional wire
  field unnoticed.

## 0.146.1

`0.146.1` turns the reserved schema memory into a concrete raw schema metadata
map and starts persisting generated initial schema snapshots at the recovery
boundary. `SchemaStore` now owns a stable BTreeMap keyed by entity tag and
schema version, the raw snapshot payload has a typed persisted-schema codec,
and startup reconciliation writes or verifies each registered entity's initial
snapshot before row/index operations proceed. The current `DESCRIBE` /
`SHOW COLUMNS` surfaces also expose generated physical row slots while live
row-layout authority is still pending.

### Schema store map boundary

- Added `RawSchemaKey` as the durable key for one entity schema version.
- Added `RawSchemaSnapshot` as the raw encoded value boundary for future
  persisted schema snapshots.
- Updated `SchemaStore` from a bare stable-memory holder into a thin
  BTreeMap-backed persistence wrapper with insert, lookup, presence, length,
  and test-clear operations.
- Kept raw schema keys and encoded payloads inside the schema store boundary;
  reconciliation now uses typed snapshot insert/load entrypoints.
- Added schema-store tests for key encoding, raw payload round-tripping, and
  raw snapshot persistence.

### Startup reconciliation

- Added a schema-owned reconciliation pass over registered runtime hooks.
- First contact writes the generated initial persisted schema snapshot for each
  entity into its store's `SchemaStore`.
- Later contacts decode the stored snapshot and require an exact match with the
  compiled proposal until explicit schema-evolution rules exist.
- Rejected snapshots now report the first detected schema difference, such as
  entity identity, primary-key field ID, row layout, field count, field kind,
  nullability, default, storage decode, or leaf codec drift.
- Wired reconciliation through the commit recovery gate, guarded once per store
  registry so normal read/write entrypoints do not repeatedly scan schema
  metadata.
- Added focused tests for first-write, idempotent reread, and changed-snapshot
  rejection.

### Persisted snapshot codec

- Added a schema-owned Candid wire codec for `PersistedSchemaSnapshot` without
  deriving wire traits on the internal schema DTOs.
- Encoded the codec version, schema version, entity identity, primary-key field
  ID, row layout, field metadata, database defaults, storage decode policy,
  leaf codec, and recursive persisted field kind graph.
- Added typed round-trip coverage through `RawSchemaSnapshot` for nested
  `Map<Text, List<Uint>>` metadata.

### Describe slot visibility

- Added slot visibility to `EntityFieldDescription`.
- Top-level generated fields now report their current physical row slot.
- Nested structured-record leaves report no slot because they are not separate
  top-level persisted row slots.
- SQL `DESCRIBE` and `SHOW COLUMNS` now use the accepted initial persisted
  schema snapshot for top-level slot values, while the infallible Rust metadata
  helpers remain generated-model based.
- Updated shell rendering so `DESCRIBE` tables and `SHOW COLUMNS` lines include
  the slot value.

### Naming cleanup

- Renamed clear index-store fields, parameters, and locals that were still
  called `store` so row, index, and schema store references stay visually
  distinct during the 0.146 triplet rollout.

## 0.146.0

`0.146.0` starts the schema identity and layout authority work by introducing
schema-owned identity artifacts before adding any durable `__icydb_schema`
payload. Generated `EntityModel` metadata now projects into a compiled schema
proposal, that proposal can build the first persisted-schema snapshot shape for
a new database, and generated stores now reserve a third stable memory for
future schema metadata alongside row and index storage.

### Schema identity foundation

- Added `FieldId` as the durable logical field identity, separate from generated
  field order and executor slot indexes.
- Added schema-owned `SchemaVersion`, `SchemaFieldSlot`, and `SchemaRowLayout`
  types so future reconciliation can map durable field IDs to physical row
  slots without using loose integer pairs.
- Added `CompiledSchemaProposal` and `CompiledFieldProposal` as the bridge from
  generated `EntityModel` / `FieldModel` metadata into schema-identity terms.
- Routed the existing commit-schema fingerprint through the compiled proposal
  without changing the current fingerprint contract.

### Persisted snapshot shape

- Added `PersistedSchemaSnapshot` and `PersistedFieldSnapshot` as the owned
  shape intended for the future `__icydb_schema` payload.
- Added owned `PersistedFieldKind` conversion from generated `FieldKind` so
  live schema snapshots do not depend on generated static metadata for nested,
  relation, enum, collection, or structured field shapes.
- Added `SchemaFieldDefault` with only an explicit `None` state for now, keeping
  database defaults separate from Rust struct `Default` behavior.
- Added schema proposal tests covering initial field IDs, row layout,
  persisted snapshot construction, and storage contracts.

### Store memory boundary

- Added `SchemaStore` as the runtime owner for the per-store schema stable
  memory. It is intentionally only a memory boundary in this slice; startup
  reconciliation and the concrete `__icydb_schema` map remain separate work.
- Extended `StoreHandle` and `StoreRegistry` so generated stores bind row,
  index, and schema stores as one triplet.
- Extended the derive/build path so every `#[store(...)]` declaration now
  supplies `schema_memory_id`, and generated canister wiring emits the matching
  `SchemaStore` static.
- Updated schema validation so `schema_memory_id` is range-checked,
  reserved-ID checked, unique within its store, and unique across all memories
  registered to the same canister.

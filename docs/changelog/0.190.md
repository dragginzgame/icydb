# IcyDB 0.190 Changelog

## 0.190.6

### 📝 Summary

- Tightens focused SQL and SQL EXPLAIN test-helper feature gates after the
  0.190.5 no-default cleanup.
- Keeps model-only EXPLAIN descriptor helpers and execution-trace assertion
  accessors out of no-default test builds.
- Fixes a projection-mask cfg drift exposed by SQL-only focused test builds.

### 🔧 Changed

- Gated model-only SQL EXPLAIN descriptor assembly and the session
  authority-label fixture helper behind `sql-explain`, matching the SQL syntax
  and descriptor surface they exercise.
- Gated execution-trace stats accessors and access-requirement display helpers
  behind SQL-enabled tests where those assertion helpers are used.
- Kept the no-SQL aggregate reducer signatures shared with SQL builds while
  documenting the no-SQL `unnecessary_wraps` lint expectation for test builds
  as well as production builds.
- Aligned the prepared projection contract's projected-slot mask storage with
  the SQL test and diagnostics paths that read it.

### 🧪 Testing

- Rechecked formatting.
- Rechecked the supported local Clippy gate.
- Rechecked focused SQL execution-convergence tests.
- Rechecked focused SQL EXPLAIN authority-label tests.

### 🧭 Migration Notes

- No public API, SQL/fluent behavior, generated endpoint behavior, persisted
  format, response shape, cursor format, or cache identity changes are intended.
- This release continues the no-default helper-surface cleanup without adding
  no-default all-targets Clippy to the supported release matrix.

## 0.190.2

### 📝 Summary

- Extends Work Track C with an upgrade-style guarded recovery reentry proof.
- Starts Work Track D with a deterministic malformed-byte corpus across DB-owned
  persisted decode boundaries.
- Adds a first Work Track F large-index recovery characterization floor and
  records 0.191 productization evidence notes.

### 🔧 Changed

- Adds a test-only recovery helper that clears volatile recovery-domain and
  schema-readiness bookkeeping for one database domain without changing stable
  marker, journal, row, schema, or index bytes.
- Adds a checked-in evidence note for large-index recovery characterization,
  checksum decisions, backup/import threat modeling, production recovery-size
  policy, and operator durability documentation.

### 🧪 Testing

- Added an upgrade-style marker-cleared recovery test that interrupts after
  marker clear, clears volatile readiness bookkeeping, then proves guarded
  recovery restores `Ready` from stable row/index/journal state.
- Added a malformed persisted-format corpus for row envelopes, persisted schema
  snapshot bytes, raw index-key/index-entry bytes, structural value-storage
  envelopes, and by-kind structural fields.
- Added a secondary-index rebuild characterization over 256 authoritative rows
  plus stale index-only state, proving startup recovery rebuilds the row-derived
  index set and clears the marker.
- Rechecked focused recovery and corpus tests, the full commit recovery module,
  db-level tests, formatting, diff hygiene, and all-features all-targets
  `icydb-core` Clippy.

### 🧭 Migration Notes

- No public API, SQL/fluent behavior, generated endpoint behavior, persisted
  format, response shape, cursor format, or cache identity changes are intended.
- The new recovery helper and malformed corpus are test-only. The large-index
  evidence is a host regression floor, not a production IC recovery-size bound.

## 0.190.0

### 📝 Summary

- Starts the 0.190 IC reliability proof expansion.
- Extends the 0.189 host failpoint harness into journal-tail folding,
  secondary-index rebuild, journaled-index fold, and marker-cleared readiness
  windows.
- Adds explicit intermediate recovery oracles for durable phases between marker
  visibility and final recovered readiness.
- Hardens marker-absent guarded recovery so interrupted recovery domains still
  complete volatile readiness restoration.

### 🔧 Changed

- Adds test-only failpoints before journal-tail batch folding and after fold
  watermark persistence.
- Adds test-only failpoints after secondary-index rebuild clear and after
  journaled index materialized-view fold.
- Names intermediate recovery authority for journal-tail fold readiness,
  persisted fold watermark state, secondary-index rebuild clear, and journaled
  index materialization fold.
- Keeps durable row-op replay failpoints outside the active claim because the
  current durable format recovers through marker-bound journal batches.
- Tracks interrupted recovery domains so marker-absent guarded reentry can
  finish readiness restoration without treating ordinary `Building` indexes as
  recovery-owned state.

### 🧪 Testing

- Added guarded retry coverage for structured returned errors and host unwind
  interruptions at the new journal-tail and derived-index failpoint sites.
- Added oracle assertions for marker visibility, journal-tail visibility,
  fold-watermark state, raw row/index snapshots, and index readiness.
- Added marker-cleared/readiness-not-restored coverage proving guarded reentry
  restores `Ready` visibility after interruption.
- Rechecked `icydb-core` all-features library tests and all-targets Clippy.

### 🧭 Migration Notes

- No public API, SQL/fluent behavior, generated endpoint behavior, persisted
  format, response shape, cursor format, or cache identity changes are intended.
- The recovery-domain readiness guard is volatile runtime state only; it does
  not change marker, journal, row, schema, or index persisted bytes.

# IcyDB 0.168 Changelog

## 0.168.0

### 📊 Audit

- Started the runtime store-backend boundary line with a concrete source
  inventory of direct data/index/schema traversal call sites.

- Recorded the initial residual scans proving stable `btreemap::Iter` type
  names are already confined to store modules, while higher layers still depend
  on store traversal APIs such as:

  ```rust
  DataStore::entries()
  DataStore::range(...)
  DataStore::range_for_entity(...)
  IndexStore::entries()
  ```

### 📚 Documentation

- Promoted the 0.168 design to the active runtime store backend-boundary plan:
  stable remains the only implemented backend, `storage(heap())` remains
  reserved and rejected, and removed traversal APIs are hard-cut with no
  aliases, shims, or backwards-compatible fallback routes.
